# 15. 三数之和

## 题目信息

- 题号：LeetCode 15
- 标题：3Sum
- 难度：中等（Medium）
- 模式：排序 + 双指针

## 一句话思路

先排序，固定第一个数 `nums[i]`，再在右侧用双指针找两个数，使三数之和等于 `0`。

## 为什么想到这个模式

题目要求找三元组 `[a, b, c]`，满足：

```text
a + b + c = 0
```

直接三重循环是 O(n^3)。如果先固定一个数 `a = nums[i]`，问题就变成：

```text
在 i 右侧找 b + c = -a
```

这是典型的 Two Sum 有序数组变体。排序后，`left` 和 `right` 的和太小就右移 `left`，太大就左移 `right`，每次移动都有明确方向。

选择这个模式的信号：

- 要找多个数的组合。
- 不要求保留原始下标。
- 需要去重。
- 排序后可以根据和的大小单调移动指针。

## 状态定义 / 数据结构

- `i`：固定的第一个数。
- `left`：第二个数候选，从 `i + 1` 开始。
- `right`：第三个数候选，从数组末尾开始。
- `result`：不重复三元组集合。

排序会改变原数组顺序，但题目只要求返回数值三元组，不要求下标。

## 核心不变量

在固定 `i` 后，双指针始终只在区间 `(i, n-1]` 内搜索，保证每个三元组的下标顺序是：

```text
i < left < right
```

排序带来的单调性是不变量的基础：

- 如果 `sum < 0`，当前和太小，只能移动 `left` 才可能增大。
- 如果 `sum > 0`，当前和太大，只能移动 `right` 才可能减小。
- 如果 `sum == 0`，记录答案后，必须跳过左右两侧重复值，避免重复三元组。

去重不变量：

- 同一个 `i` 值只处理第一次。
- 同一个答案中的 `left/right` 值记录后只保留第一次组合。

## 手推全过程

输入：

```text
[-1, 0, 1, 2, -1, -4]
```

排序后：

```text
[-4, -1, -1, 0, 1, 2]
```

| i | nums[i] | left/right | sum | 决策 |
|---|---------|------------|-----|------|
| 0 | -4 | -1 + 2 | -3 | 太小，left++ |
| 0 | -4 | -1 + 2 | -3 | 太小，left++ |
| 0 | -4 | 0 + 2 | -2 | 太小，left++ |
| 0 | -4 | 1 + 2 | -1 | 太小，left++，结束 |
| 1 | -1 | -1 + 2 | 0 | 记录 `[-1,-1,2]`，左右去重 |
| 1 | -1 | 0 + 1 | 0 | 记录 `[-1,0,1]`，左右去重 |
| 2 | -1 | - | - | `nums[2] == nums[1]`，跳过 |
| 3 | 0 | 1 + 2 | 3 | 太大，right--，结束 |

答案：

```text
[[-1, -1, 2], [-1, 0, 1]]
```

## Go 代码实现

```go
func threeSum(nums []int) [][]int {
    sort.Ints(nums)
    result := [][]int{}
    n := len(nums)

    for i := 0; i < n-2; i++ {
        if nums[i] > 0 {
            break
        }
        if i > 0 && nums[i] == nums[i-1] {
            continue
        }

        left, right := i+1, n-1
        for left < right {
            sum := nums[i] + nums[left] + nums[right]
            if sum == 0 {
                result = append(result, []int{nums[i], nums[left], nums[right]})
                left++
                right--
                for left < right && nums[left] == nums[left-1] {
                    left++
                }
                for left < right && nums[right] == nums[right+1] {
                    right--
                }
            } else if sum < 0 {
                left++
            } else {
                right--
            }
        }
    }
    return result
}
```

## 错误写法与反例

### 错误 1：固定值不去重

```go
for i := 0; i < n-2; i++ {
    // 缺少 nums[i] == nums[i-1] 的跳过
}
```

反例：`[-1, -1, 0, 1]`

如果两个 `-1` 都作为固定值处理，会得到重复答案 `[-1,0,1]`。

### 错误 2：找到答案后只移动一边

```go
if sum == 0 {
    result = append(result, []int{nums[i], nums[left], nums[right]})
    left++
}
```

反例：`[-2, 0, 0, 2, 2]`

只移动一边会让另一边重复值继续参与匹配，容易产生重复答案。正确做法是左右都移动，并跳过重复值。

### 错误 3：去重时比较错方向

```go
for left < right && nums[left] == nums[left+1] {
    left++
}
```

记录答案后已经执行了 `left++`，此时应该和前一个位置比较：`nums[left] == nums[left-1]`。比较 `left+1` 可能跳过逻辑不完整，也容易在边界附近写出越界。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n^2) |
| 空间 | O(1)，不计返回结果；排序栈空间取决于语言实现 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为固定一个数后，剩余两数之和可以在有序数组中用双指针单调收敛。

2. 这个算法维护的不变量是什么？
   固定 `i` 后，只在右侧区间搜索 `left < right`；根据 `sum` 与 `0` 的关系移动指针，且每个数值组合只记录一次。

3. 每一步为什么不会漏答案？
   排序后，`sum < 0` 时只有增大 `left` 才可能变大，`sum > 0` 时只有减小 `right` 才可能变小，被排除的组合不可能满足目标。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是去重：`i`、`left`、`right` 三处都要处理。漏掉任意一处都会产生重复三元组。
