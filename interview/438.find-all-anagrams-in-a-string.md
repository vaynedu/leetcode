# 438. 找到所有字母异位词（Find All Anagrams in a String）

## 题目信息

- 题号：LeetCode 438
- 标题：Find All Anagrams in a String
- 难度：中等（Medium）
- 模式：滑动窗口 / 固定长度窗口

## 一句话思路

维护长度等于 `len(p)` 的窗口，窗口字符计数和 `p` 完全一致时，记录当前左边界。

## 为什么想到这个模式

题目要在字符串 `s` 中找所有 `p` 的异位词起点。异位词的核心条件是：

```text
窗口长度 = len(p)
窗口字符计数 = p 的字符计数
```

因为必须是连续子串，而且窗口长度固定，所以这是固定长度滑动窗口。和 567 的区别是：567 找到一个就返回，438 要记录所有满足条件的位置。

选择这个模式的信号：

- 连续子串。
- 子串长度固定。
- 判断字符频次是否相等。
- 需要输出所有起点。

## 状态定义 / 数据结构

- `need`：`p` 中每个字符需要出现的次数。
- `window`：当前窗口中相关字符出现的次数。
- `valid`：有多少种字符已经满足 `window[c] == need[c]`。
- `L, R`：窗口边界，代码中窗口是 `[L, R)`。
- `result`：所有满足条件的窗口左边界。

当 `valid == len(need)` 且窗口长度等于 `len(p)` 时，当前窗口是一个异位词。

## 核心不变量

每次检查答案前，窗口长度都恰好是 `len(p)`。

`valid` 的含义始终是“已经完全匹配的字符种类数”：

- 加入字符 `c` 后，如果 `window[c] == need[c]`，说明 `c` 这一类刚好满足，`valid++`。
- 移除字符 `d` 前，如果 `window[d] == need[d]`，移除后会不满足，所以先 `valid--`。

这个不变量避免每次都比较两个 map，把窗口判断降到 O(1)。

## 手推全过程

以 `s = "cbaebabacd", p = "abc"` 为例：

```text
need = {a:1, b:1, c:1}
```

| 窗口 | 计数状态 | valid | 结果 |
|------|----------|-------|------|
| `"cba"`，L=0 | a/b/c 都满足 | 3 | 记录 0 |
| `"bae"`，L=1 | 缺 c，多 e | 2 | 不记录 |
| `"aeb"`，L=2 | 缺 c，多 e | 2 | 不记录 |
| `"eba"`，L=3 | 缺 c，多 e | 2 | 不记录 |
| `"bab"`，L=4 | b 超量，缺 c | 2 | 不记录 |
| `"aba"`，L=5 | a 超量，缺 c | 2 | 不记录 |
| `"bac"`，L=6 | a/b/c 都满足 | 3 | 记录 6 |
| `"acd"`，L=7 | 缺 b，多 d | 2 | 不记录 |

答案是 `[0, 6]`。

## Go 代码实现

```go
func findAnagrams(s string, p string) []int {
    need := make(map[rune]int)
    window := make(map[rune]int)

    for _, c := range p {
        need[c]++
    }

    L, R := 0, 0
    valid := 0
    result := []int{}

    for R < len(s) {
        c := rune(s[R])
        if _, ok := need[c]; ok {
            window[c]++
            if window[c] == need[c] {
                valid++
            }
        }
        R++

        if R-L == len(p) {
            if valid == len(need) {
                result = append(result, L)
            }
            d := rune(s[L])
            if _, ok := need[d]; ok {
                if window[d] == need[d] {
                    valid--
                }
                window[d]--
            }
            L++
        }
    }

    return result
}
```

## 错误写法与反例

### 错误 1：窗口长度没到 `len(p)` 就检查

```go
if valid == len(need) {
    result = append(result, L)
}
```

反例：`s = "abx", p = "ab"`。

如果不限制窗口长度，窗口 `"abx"` 里 `a/b` 都满足，也可能误判。必须只检查长度等于 `len(p)` 的窗口。

### 错误 2：记录右边界而不是左边界

```go
result = append(result, R)
```

题目要求返回异位词起始位置。反例 `"cba"` 对 `"abc"`，正确答案是 `0`，不是 `3`。

### 错误 3：移除字符后再判断 `valid--`

```go
window[d]--
if window[d] == need[d] {
    valid--
}
```

如果移除前刚好满足，移除后会变成少一个，此时应该减少 `valid`。所以必须在 `window[d]--` 前判断。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(k)，k 是字符种类数；小写字母场景可视为 O(1) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要找连续子串，且子串长度固定为 `len(p)`，窗口左右边界每次整体向右滑动。

2. 这个算法维护的不变量是什么？
   检查答案时窗口长度恒等于 `len(p)`；`valid` 表示当前窗口中完全满足频次要求的字符种类数。

3. 每一步为什么不会漏答案？
   每个长度为 `len(p)` 的窗口都会被滑动窗口恰好检查一次，字符频次相等时立即记录左边界。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是 `valid--` 的时机。移除字符前如果刚好满足，移除后就不满足，判断顺序反了会导致误判。
