# 239. 滑动窗口最大值（Sliding Window Maximum）

## 题目信息
- 题号：LeetCode 239
- 标题：Sliding Window Maximum
- 难度：困难（Hard）

---

## 一句话思路
**单调 deque**：维护递减队列，队首存当前窗口最大值的下标，右扩左缩时保持单调性。

---

## 核心思路

### 朴素解法
每个窗口遍历找最大 → O(N×K)

### 单调 deque 优化（O(N)）
- deque 存**下标**，保持**单调递减**
- 队首永远是当前窗口最大值的下标

### 四步操作
```
1. 新元素入队前：把比它小的从队尾踢掉（永无出头之日）
2. 新元素下标加入队尾
3. 踢掉已经滑出窗口的队首（队首下标 < L）
4. 队首就是当前窗口最大值
```

---

## 关键点/考点
```
时间：O(N)，每个元素最多入队出队各一次
空间：O(K)

追问：
- 为什么存下标不存值？→ 方便判断是否滑出窗口
- 为什么从队尾删？→ 队尾可能有很多小元素
- 为什么单调递减？→ 队首永远是最大的
```

---

## 易错点
```
1. deque 存的是下标，不是值
2. 比较的是 nums[下标] 的大小
3. 队首下标 < L 时要移除（已滑出窗口）
4. 新元素入队前先从队尾删除所有更小的
```

---

## 记忆口诀
```
单调递减 deque，存的是下标不是值。
新来元素比它小，队尾全踢。
窗口右移左跟随，队首出界就移除。
队首永远最大值，每个窗口 O(1)。
```

---

## Go 代码实现

```go
func maxSlidingWindow(nums []int, k int) []int {
    if len(nums) == 0 || k == 0 {
        return []int{}
    }

    var deque []int // 存下标，保持单调递减
    result := []int{}

    for R := 0; R < len(nums); R++ {
        // 1. 新元素入队前：把比它小的从队尾踢掉
        for len(deque) > 0 && nums[deque[len(deque)-1]] < nums[R] {
            deque = deque[:len(deque)-1]
        }
        // 2. 新元素下标加入队尾
        deque = append(deque, R)

        // 3. 踢掉已经滑出窗口的队首
        if len(deque) > 0 && deque[0] <= R-k {
            deque = deque[1:]
        }

        // 4. 窗口形成后（至少 k 个元素），记录最大值
        if R >= k-1 {
            result = append(result, nums[deque[0]])
        }
    }

    return result
}
```

## 测试用例

### 正常情况
- `nums=[1,3,-1,-3,5,3,6,7], k=3` → `[3,3,5,5,6,7]`
- `nums=[1], k=1` → `[1]`
- `nums=[1,3,-1,-3,5,3,6,7], k=1` → `[1,3,-1,-3,5,3,6,7]`

### 边界情况
- `nums=[], k=0` → `[]`
- `nums=[1,2,3,4,5], k=5` → `[5]`
- `nums=[5,4,3,2,1], k=3` → `[5,4,3]`

### 极端情况
- 全部相同：`[1,1,1,1,1], k=3` → `[1,1,1]`
- 递减：`[9,8,7,6,5], k=3` → `[9,8,7]`
- 递增：`[1,2,3,4,5], k=3` → `[3,4,5]`
