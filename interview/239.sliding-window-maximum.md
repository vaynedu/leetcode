# 239. 滑动窗口最大值（Sliding Window Maximum）

## 题目信息

- 题号：LeetCode 239
- 难度：困难
- 模式：滑动窗口 / 单调队列
- 可视化：`doc/algorithm-visualizations/239.sliding-window-maximum.html`

## 一句话思路

用单调递减 deque 保存候选下标，队首永远是当前窗口最大值。

## 为什么想到这个模式

题目要求每个固定长度窗口的最大值。固定窗口可以滑动，但如果每次重新扫描窗口就是 O(nk)。

最大值只需要维护“可能成为最大值的候选”。当新元素比队尾元素大时，队尾元素在当前窗口和未来窗口中都不可能再成为最大值，可以删除。这就是单调队列。

## 状态定义 / 数据结构

- `deque`：保存下标，且对应值从队首到队尾单调递减。
- `right`：当前加入窗口的下标。
- 当前窗口左边界：`left = right - k + 1`。
- `res`：每个窗口的最大值。

## 核心不变量

- deque 中的下标都在当前窗口内。
- deque 中的值单调递减：`nums[deque[0]] >= nums[deque[1]] >= ...`。
- 队首 `deque[0]` 始终是当前窗口最大值下标。

## 手推全过程

以 `nums=[1,3,-1,-3,5,3,6,7], k=3` 为例：

| right | 加入 | deque 对应值 | 窗口形成后答案 |
|-------|------|--------------|----------------|
| 0 | 1 | `[1]` | - |
| 1 | 3 | `[3]`，1 被踢掉 | - |
| 2 | -1 | `[3,-1]` | 3 |
| 3 | -3 | `[3,-1,-3]` | 3 |
| 4 | 5 | `[5]`，前面更小全踢掉 | 5 |
| 5 | 3 | `[5,3]` | 5 |
| 6 | 6 | `[6]` | 6 |
| 7 | 7 | `[7]` | 7 |

结果：`[3,3,5,5,6,7]`。

## 错误写法与反例

错误写法：deque 存值而不是下标。

```go
deque = append(deque, nums[right])
```

反例：`nums=[1,3,1,2,0,5], k=3`

如果只存值，很难判断队首的 `3` 是不是已经滑出窗口。必须存下标，才能用 `deque[0] <= right-k` 判断过期。

另一个易错点：入队前只删除一个队尾。

```go
if len(deque) > 0 && nums[deque[len(deque)-1]] < nums[right] {
    deque = deque[:len(deque)-1]
}
```

必须用 `for` 删除所有更小候选。

## Go 代码实现

```go
func maxSlidingWindow(nums []int, k int) []int {
    deque := []int{}
    res := []int{}

    for right := 0; right < len(nums); right++ {
        for len(deque) > 0 && nums[deque[len(deque)-1]] < nums[right] {
            deque = deque[:len(deque)-1]
        }
        deque = append(deque, right)

        if deque[0] <= right-k {
            deque = deque[1:]
        }

        if right >= k-1 {
            res = append(res, nums[deque[0]])
        }
    }
    return res
}
```

## 复杂度

| 指标 | 值 |
|------|-----|
| 时间 | O(n)，每个下标最多入队出队一次 |
| 空间 | O(k)，deque 最多保存一个窗口的候选 |

## 四问复盘

1. **为什么属于滑动窗口？** 固定长度窗口向右滑动，每步输出当前窗口最大值。
2. **维护的不变量是什么？** deque 下标在窗口内，且对应值单调递减。
3. **为什么不会漏答案？** 被新元素踢掉的旧元素更小且更早过期，未来不可能成为最大值。
4. **最容易错的边界？** deque 要存下标；过期条件是 `deque[0] <= right-k`。
