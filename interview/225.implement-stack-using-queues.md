# 225. 用队列实现栈

## 题目信息

- 题号：LeetCode 225
- 标题：Implement Stack using Queues
- 难度：简单（Easy）
- 模式：队列 / 栈 / 数据结构模拟

## 一句话思路

用一个队列保存元素；每次 `Push` 新元素后，把它前面的旧元素依次移动到队尾，让队首始终是栈顶。

## 为什么想到这个模式

队列是先进先出，栈是后进先出。要用队列模拟栈，需要让“最后加入的元素”最先出队。

单队列方案把成本放在 `Push`：

```text
先把新元素入队
再把旧元素全部轮转到新元素后面
```

这样队首就是最新入栈元素，`Pop` 和 `Top` 都能 O(1) 操作队首。

## 状态定义 / 数据结构

- `queue`：内部队列。
- `queue[0]`：当前栈顶。
- `queue[len(queue)-1]`：当前栈底附近元素。

Push 旋转：

```text
原队列：[top ... bottom]
append x 后：[top ... bottom, x]
旋转旧元素后：[x, top ... bottom]
```

## 核心不变量

任意时刻：

- `queue` 的顺序等于栈从顶到底的顺序。
- `queue[0]` 始终是最近 push 且尚未 pop 的元素。
- `Pop()` 删除 `queue[0]`，等价于弹出栈顶。

每次 `Push` 后必须旋转 `len(queue)-1` 次，旧元素全部移到新元素后面。

## 手推全过程

操作：

```text
Push(1)
Push(2)
Push(3)
```

状态：

```text
Push(1): [1]
Push(2): append 后 [1,2]，旋转 1 次 -> [2,1]
Push(3): append 后 [2,1,3]，旋转 2 次 -> [3,2,1]
```

之后：

```text
Pop() = 3
Pop() = 2
Pop() = 1
```

符合栈的 LIFO。

## Go 代码实现

```go
type MyStack struct {
    queue []int
}

func StackConstructor() MyStack {
    return MyStack{}
}

func (this *MyStack) Push(x int) {
    this.queue = append(this.queue, x)
    for i := 0; i < len(this.queue)-1; i++ {
        this.queue = append(this.queue, this.queue[0])
        this.queue = this.queue[1:]
    }
}

func (this *MyStack) Pop() int {
    if len(this.queue) == 0 {
        return 0
    }
    val := this.queue[0]
    this.queue = this.queue[1:]
    return val
}

func (this *MyStack) Top() int {
    if len(this.queue) == 0 {
        return 0
    }
    return this.queue[0]
}

func (this *MyStack) Empty() bool {
    return len(this.queue) == 0
}
```

## 错误写法与反例

### 错误 1：Push 后不旋转

如果只做普通入队，队首会是最早加入的元素，行为就是队列，不是栈。

反例：

```text
Push(1), Push(2), Pop()
```

正确结果是 `2`，不旋转会返回 `1`。

### 错误 2：旋转次数写成 `len(queue)`

新元素也会被转走，队首不再是新元素。正确次数是旧元素数量，也就是 `len(queue)-1`。

### 错误 3：Top 删除元素

`Top` 只能查看栈顶，不能改变结构；删除应由 `Pop` 完成。

## 复杂度

| 操作 | 时间复杂度 |
|------|------------|
| Push | O(n) |
| Pop | O(1) |
| Top | O(1) |
| Empty | O(1) |

空间复杂度：O(n)。

## 四问复盘

1. 这题为什么属于这个模式？
   因为要用 FIFO 队列模拟 LIFO 栈，需要通过旋转队列改变出队顺序。

2. 这个算法维护的不变量是什么？
   `queue[0]` 始终是栈顶，整个队列顺序等于栈从顶到底的顺序。

3. 每一步为什么不会漏答案？
   每次 Push 都把新元素旋到队首，Pop 总是删除队首，因此弹出顺序严格符合后进先出。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是旋转次数，少转会让旧元素挡在前面，多转会把新元素转到后面，都会破坏栈顶。
