# 785. 判断二分图

## 题目信息

- 题号：LeetCode 785
- 标题：Is Graph Bipartite?
- 难度：中等（Medium）
- 模式：图 / BFS 染色

## 一句话思路

给每个连通分量做 BFS 染色：当前节点染 1，邻居染 -1；若发现相邻节点同色，则不是二分图。

## 为什么想到这个模式

二分图要求所有边都连接两个不同集合的节点。等价于能否给图染两种颜色，使每条边两端颜色不同。

图可能不连通，所以要从每个未染色节点启动 BFS/DFS。

## 状态定义 / 数据结构

- `color[i] = 0`：未染色。
- `color[i] = 1` 或 `-1`：两种颜色。
- `queue`：当前连通分量的 BFS 队列。

## 核心不变量

一旦节点被染色，它的所有邻居必须染成相反颜色。

遍历边 `(u, v)` 时：

- `v` 未染色：染成 `-color[u]`。
- `v` 已染色且 `color[v] == color[u]`：出现冲突，返回 false。

## 手推全过程

图 `[[1,3],[0,2],[1,3],[0,2]]`：

```text
0 -- 1
|    |
3 -- 2
```

| 节点 | 颜色 | 动作 |
|------|------|------|
| 0 | 1 | 起点 |
| 1 | -1 | 邻居反色 |
| 3 | -1 | 邻居反色 |
| 2 | 1 | 从 1 或 3 染色 |

所有边两端颜色不同，返回 true。

## Go 代码实现

```go
func isBipartite(graph [][]int) bool {
    n := len(graph)
    color := make([]int, n)
    for i := 0; i < n; i++ {
        if color[i] != 0 {
            continue
        }
        color[i] = 1
        queue := []int{i}
        for len(queue) > 0 {
            node := queue[0]
            queue = queue[1:]
            for _, neighbor := range graph[node] {
                if color[neighbor] == 0 {
                    color[neighbor] = -color[node]
                    queue = append(queue, neighbor)
                } else if color[neighbor] == color[node] {
                    return false
                }
            }
        }
    }
    return true
}
```

## 错误写法与反例

### 错误 1：只从 0 开始 BFS

图可能有多个连通分量。只检查一个分量会漏掉其它分量中的冲突。

### 错误 2：已染色邻居不检查冲突

仅跳过已染色节点会漏掉同色相邻的奇环。

### 错误 3：用 visited 代替 color

只知道访问过不够，还必须知道属于哪一组。

## 四问复盘

1. 这题为什么属于这个模式？
   因为判断二分图等价于判断图能否用两种颜色染色且相邻不同色。

2. 这个算法维护的不变量是什么？
   每条已检查边的两端颜色相反；未染色节点会在被发现时染成邻居反色。

3. 每一步为什么不会漏答案？
   外层遍历所有连通分量，BFS 检查每条边，任何同色冲突都会被发现。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是只检查一个连通分量，会把其它分量中的非二分结构漏掉。
