# 3. 无重复最长子串（Longest Substring Without Repeating Characters）

## 题目信息

- 题号：LeetCode 3
- 难度：中等
- 模式：滑动窗口 / HashMap
- 可视化：`doc/algorithm-visualizations/3.longest-substring-without-repeating-characters.html`

## 一句话思路

维护一个无重复窗口：右指针扩展，遇到窗口内重复字符时，左指针直接跳到重复字符上次位置的右侧。

## 为什么想到这个模式

题目问的是“最长子串”，子串必须连续；连续区间上的最长/最短问题，优先考虑滑动窗口。

这里窗口状态可以用 `last[ch]` O(1) 更新：只要知道字符最近一次出现位置，就能判断它是否还在当前窗口内。

## 状态定义 / 数据结构

- `left`：当前无重复窗口左边界。
- `right`：当前扫描到的位置。
- `last[ch]`：字符 `ch` 最近一次出现的下标。
- `ans`：目前见过的最长无重复窗口长度。
- 当前窗口：`s[left:right+1]`。

## 核心不变量

每次更新答案时，窗口 `s[left:right+1]` 内没有重复字符。

如果 `last[ch] >= left`，说明当前字符 `ch` 的旧位置还在窗口内，必须移动 `left`。如果 `last[ch] < left`，旧字符已经在窗口外，不影响当前窗口。

`left` 只能向右走，不能回退。

## 手推全过程

以 `s = "abcba"` 为例：

| right | char | last 情况 | left 变化 | 当前窗口 | ans |
|-------|------|-----------|-----------|----------|-----|
| 0 | a | 未出现 | 0 | `a` | 1 |
| 1 | b | 未出现 | 0 | `ab` | 2 |
| 2 | c | 未出现 | 0 | `abc` | 3 |
| 3 | b | `last[b]=1 >= left=0` | `left=2` | `cb` | 3 |
| 4 | a | `last[a]=0 < left=2` | 不变 | `cba` | 3 |

答案是 3，最长子串可以是 `"abc"` 或 `"cba"`。

## 错误写法与反例

错误写法：

```go
if pos, ok := last[ch]; ok {
    left = pos + 1
}
```

反例：`s = "abba"`

- 扫到第二个 `b` 时，`left` 从 0 跳到 2。
- 扫到最后一个 `a` 时，`last[a]=0`，旧 `a` 已经不在窗口内。
- 如果直接 `left = 0+1`，会把 `left` 从 2 拉回 1，窗口变成 `"bba"`，破坏无重复不变量。

正确写法：

```go
if pos, ok := last[ch]; ok && pos >= left {
    left = pos + 1
}
```

## Go 代码实现

```go
func lengthOfLongestSubstring(s string) int {
    last := make(map[byte]int)
    left := 0
    ans := 0

    for right := 0; right < len(s); right++ {
        ch := s[right]
        if pos, ok := last[ch]; ok && pos >= left {
            left = pos + 1
        }
        last[ch] = right
        ans = max(ans, right-left+1)
    }
    return ans
}
```

## 复杂度

| 指标 | 值 |
|------|-----|
| 时间 | O(n)，每个字符扫描一次 |
| 空间 | O(k)，k 为字符集大小 |

## 四问复盘

1. **为什么属于滑动窗口？** 子串连续，且窗口状态可随左右边界移动 O(1) 更新。
2. **维护的不变量是什么？** 当前窗口内没有重复字符，`left` 永不回退。
3. **为什么不会漏答案？** 每个 `right` 对应的最长合法窗口都会被检查，左侧被跳过的部分已经无法和当前字符组成合法窗口。
4. **最容易错的边界？** 旧字符不一定在窗口内，必须判断 `pos >= left`。
