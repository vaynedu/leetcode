# 105. 从前序与中序遍历序列构造二叉树

## 题目信息

- 题号：LeetCode 105
- 标题：Construct Binary Tree from Preorder and Inorder Traversal
- 难度：中等（Medium）
- 模式：树 / 递归构造

## 一句话思路

前序首元素是根；在中序中找到根，左边是左子树，右边是右子树，再递归构造。

## 为什么想到这个模式

前序遍历顺序是“根、左、右”，所以每段前序的第一个元素能确定当前子树根。中序遍历顺序是“左、根、右”，所以根的位置能划分左右子树的节点数量。

树构造题的核心是：每次确定根，再把数组切成左右子问题。

## 状态定义 / 数据结构

递归函数：

```text
buildTree(preorder, inorder) = 用这两段遍历结果构造同一棵子树
```

状态含义：

- `preorder[0]`：当前子树根。
- `idx`：根在 `inorder` 中的位置。
- 左子树节点数：`idx`。
- 右子树节点数：`len(inorder)-idx-1`。

## 核心不变量

每次递归传入的 `preorder` 和 `inorder` 都描述同一棵子树，且长度相同。

找到根后：

- `inorder[:idx]` 是左子树。
- `inorder[idx+1:]` 是右子树。
- `preorder[1:idx+1]` 是左子树前序。
- `preorder[idx+1:]` 是右子树前序。

## 手推全过程

输入：

```text
preorder = [3,9,20,15,7]
inorder  = [9,3,15,20,7]
```

| 子树 | 根 | 左子树 | 右子树 |
|------|----|--------|--------|
| 全树 | 3 | pre `[9]`, in `[9]` | pre `[20,15,7]`, in `[15,20,7]` |
| 右子树 | 20 | pre `[15]`, in `[15]` | pre `[7]`, in `[7]` |

构造结果：

```text
     3
    / \
   9  20
     /  \
    15   7
```

## Go 代码实现

```go
func buildTree(preorder []int, inorder []int) *TreeNode {
    if len(preorder) == 0 {
        return nil
    }
    root := &TreeNode{Val: preorder[0]}
    idx := 0
    for idx < len(inorder) && inorder[idx] != preorder[0] {
        idx++
    }
    root.Left = buildTree(preorder[1:idx+1], inorder[:idx])
    root.Right = buildTree(preorder[idx+1:], inorder[idx+1:])
    return root
}
```

## 错误写法与反例

### 错误 1：左子树前序切片少一个

```go
root.Left = buildTree(preorder[1:idx], inorder[:idx])
```

左子树节点数是 `idx`，前序应取 `preorder[1:idx+1]`。少一个会漏节点。

### 错误 2：忘记空数组终止

没有 `len(preorder) == 0` 的终止条件会在叶子继续递归时越界。

### 错误 3：假设有重复值

本题默认节点值不重复。若有重复值，只靠前序和中序无法用简单值定位唯一根。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n^2)，每层线性查找根；用 map 可优化到 O(n) |
| 空间 | O(h) 递归栈，切片引用不计复制 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为前序能确定根，中序能划分左右子树，天然递归构造。

2. 这个算法维护的不变量是什么？
   每次递归的前序片段和中序片段描述同一棵子树。

3. 每一步为什么不会漏答案？
   根节点被前序首元素唯一确定，中序左右两侧完整划分左右子树，所有节点都被分到唯一子问题。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是前序切片边界，`idx` 是左子树节点数，左前序必须取 `1:idx+1`。
