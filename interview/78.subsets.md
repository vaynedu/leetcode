# 78. 子集

## 题目信息

- 题号：LeetCode 78
- 标题：Subsets
- 难度：中等（Medium）
- 模式：回溯 / 子集

## 一句话思路

用 `start` 控制只能向后选择；每个递归节点对应一个合法子集，所以进入节点就收集 `path`。

## 为什么想到这个模式

题目要求返回所有子集，本质是对每个元素做“选或不选”的决策。回溯可以把每条决策路径表示成一个子集。

也可以理解为组合型枚举：

- 当前 `path` 是已经选择的元素。
- 下一层只能从 `start` 之后继续选，避免重复顺序。
- 和排列不同，子集不关心顺序，所以不需要 `used`。

## 状态定义 / 数据结构

- `path`：当前子集。
- `start`：下一次可选择的起始下标。
- `res`：所有子集。

递归含义：

```text
backtrack(start) = 在当前 path 基础上，只从 start..n-1 中继续选元素
```

## 核心不变量

任意时刻：

- `path` 中的下标严格递增。
- `start` 之前的元素已经决定过，不会再回头选择。
- 每个递归节点的 `path` 都是一个合法子集。

因此子集题的收集位置是在递归入口，而不是只在叶子节点。

## 手推全过程

以 `nums = [1,2,3]` 为例：

```text
[]                 收集
├── [1]            收集
│   ├── [1,2]      收集
│   │   └── [1,2,3] 收集
│   └── [1,3]      收集
├── [2]            收集
│   └── [2,3]      收集
└── [3]            收集
```

再加上 `[ ]`，共 `2^3 = 8` 个子集。

## Go 代码实现

```go
func subsets(nums []int) [][]int {
    res := [][]int{}
    path := []int{}

    var backtrack func(start int)
    backtrack = func(start int) {
        tmp := make([]int, len(path))
        copy(tmp, path)
        res = append(res, tmp)

        for i := start; i < len(nums); i++ {
            path = append(path, nums[i])
            backtrack(i + 1)
            path = path[:len(path)-1]
        }
    }

    backtrack(0)
    return res
}
```

## 错误写法与反例

### 错误 1：只在叶子节点收集

```go
if start == len(nums) {
    res = append(res, path)
}
```

这样只能得到部分路径，漏掉 `[1]`、`[1,2]` 这类非叶子子集。子集是“所有节点都是答案”。

### 错误 2：递归时传 `i` 而不是 `i+1`

```go
backtrack(i)
```

这会允许重复选择当前元素，甚至造成无限递归。子集每个下标最多选一次。

### 错误 3：保存答案不复制

和排列一样，`path` 会被回溯复用，保存时必须复制。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n * 2^n) |
| 空间 | O(n)，不计结果 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要枚举所有选择集合，每个元素只有选或不选，天然对应回溯决策树。

2. 这个算法维护的不变量是什么？
   `path` 中元素下标递增，`start` 之前的元素不会再被选择。

3. 每一步为什么不会漏答案？
   每个递归节点都被收集，覆盖了所有长度的子集；循环从 `start` 向后枚举所有下一个可选元素。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是只在叶子收集，会漏掉大量非完整路径对应的子集。
