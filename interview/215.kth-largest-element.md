# 215. 数组中的第 K 个最大元素

## 题目信息

- 题号：LeetCode 215
- 标题：Kth Largest Element in an Array
- 难度：中等（Medium）
- 模式：快速选择 / 堆

## 一句话思路

第 `k` 大等价于升序下标 `len(nums)-k`；用 QuickSelect 每次 partition 后只递归目标所在的一边。

## 为什么想到这个模式

题目只要第 K 大，不需要完整排序。完整排序是 O(n log n)，但 partition 能把 pivot 放到最终有序位置：

- pivot 左边都 `<= pivot`。
- pivot 右边都 `> pivot`。
- 如果 pivot 位置就是目标下标，答案确定。
- 否则只需要继续搜索一边。

这就是快速选择，平均 O(n)。

## 状态定义 / 数据结构

- `target = len(nums) - k`：第 K 大对应的升序排名。
- `left/right`：当前搜索区间。
- `pivotIndex`：partition 后 pivot 的最终位置。

`partition(nums, left, right)` 的含义：

```text
返回 p，使 nums[left..p-1] <= nums[p]，nums[p+1..right] > nums[p]
```

## 核心不变量

每轮 QuickSelect 都保持：目标下标 `target` 一定位于当前搜索区间 `[left, right]`。

partition 后：

- `pivotIndex == target`：pivot 就是答案。
- `pivotIndex < target`：目标在右侧，因为左侧元素排名都更小。
- `pivotIndex > target`：目标在左侧。

被排除的一侧不可能包含目标排名。

## 手推全过程

以 `nums = [3,2,1,5,6,4], k = 2` 为例：

```text
target = 6 - 2 = 4
```

目标是升序下标 4，也就是第 2 大。

| 步骤 | 区间 | pivot | pivotIndex | 判断 |
|------|------|-------|------------|------|
| 1 | `[3,2,1,5,6,4]` | 4 | 3 | `3 < 4`，去右边 |
| 2 | `[5,6]` | 6 | 5 | `5 > 4`，去左边 |
| 3 | `[5]` | - | 4 | 返回 5 |

答案是 `5`。

## Go 代码实现

```go
func findKthLargest(nums []int, k int) int {
    target := len(nums) - k
    return quickSelect(nums, 0, len(nums)-1, target)
}

func quickSelect(nums []int, left, right, target int) int {
    if left == right {
        return nums[left]
    }
    pivotIndex := partition(nums, left, right)
    if pivotIndex == target {
        return nums[pivotIndex]
    } else if pivotIndex < target {
        return quickSelect(nums, pivotIndex+1, right, target)
    }
    return quickSelect(nums, left, pivotIndex-1, target)
}
```

## 错误写法与反例

### 错误 1：target 写成 `k - 1`

```go
target := k - 1
```

这是第 K 小的下标，不是第 K 大。反例 `[3,2,1,5,6,4], k=2`，`k-1=1` 对应第 2 小 `2`，正确答案是 `5`。

### 错误 2：partition 后两边都递归

QuickSelect 的优势是只递归目标所在的一边。两边都递归会退化成接近快排，失去 O(n) 平均复杂度优势。

### 错误 3：忽略最坏情况

固定选择最右 pivot，在已经有序或大量重复的输入中可能退化到 O(n^2)。工程上可随机化 pivot，面试中至少要能说明。

## 复杂度

| 解法 | 时间 | 空间 |
|------|------|------|
| 快速选择平均 | O(n) | O(1) 原地，递归栈平均 O(log n) |
| 快速选择最坏 | O(n^2) | O(n) |
| 小根堆 | O(n log k) | O(k) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为只需要一个排名元素，不需要完整排序，partition 后可以只保留目标所在一边。

2. 这个算法维护的不变量是什么？
   目标升序下标 `target` 始终位于当前搜索区间内。

3. 每一步为什么不会漏答案？
   partition 后 pivot 的最终排名确定，目标排名只可能在一侧，另一侧可以整体排除。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是第 K 大转成升序下标：应是 `len(nums)-k`，写成 `k-1` 会变成第 K 小。
