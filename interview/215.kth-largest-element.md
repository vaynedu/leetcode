# 215. 数组第K大元素

## 题目
给定整数数组 `nums` 和整数 `k`，返回第 `k` 大的元素。要求时间复杂度优于 O(n log n)。

## 核心思想

**快速选择（Quick Select）** — 快排的变体，只递归一边

### 关键洞察
快排每次把 pivot 放到正确位置 `p`，数组左边的数都 ≤ pivot，右边的数都 ≥ pivot。

要找第 k 大（升序排名 `n-k`）：
- 如果 `p == n-k` → 找到答案
- 如果 `p < n-k` → 答案在右半边，递归右边
- 如果 `p > n-k` → 答案在左半边，递归左边

## 代码实现

```go
func findKthLargest(nums []int, k int) int {
    target := len(nums) - k // 升序排名目标位置
    return quickSelect(nums, 0, len(nums)-1, target)
}

func quickSelect(nums []int, left, right, target int) int {
    if left == right {
        return nums[left]
    }
    pivotIndex := partition(nums, left, right)
    if pivotIndex == target {
        return nums[pivotIndex]
    } else if pivotIndex < target {
        return quickSelect(nums, pivotIndex+1, right, target)
    } else {
        return quickSelect(nums, left, pivotIndex-1, target)
    }
}

func partition(nums []int, left, right int) int {
    pivot := nums[right] // 选最右元素为 pivot
    i := left
    for j := left; j < right; j++ {
        if nums[j] <= pivot {
            nums[i], nums[j] = nums[j], nums[i]
            i++
        }
    }
    nums[i], nums[right] = nums[right], nums[i]
    return i
}
```

## trace — nums=[3,2,1,5,6,4], k=2

目标：升序排名 6-2 = 4（第 4 小 = 第 2 大）

**Step 1:** partition([3,2,1,5,6,4], 0, 5)  
- pivot=4, 交换后位置=2  
- 左侧 [3,2,1] ≤ 4，右侧 [5,6] > 4  
- p=2, target=4 → `p < target` → 递归右边

**Step 2:** partition([5,6,4], 3, 5)  
- pivot=4, 位置=3  
- p=3, target=4 → `p < target` → 递归右边

**Step 3:** partition([6,4], 4, 5)  
- pivot=4, 位置=4  
- p=4 == target → **返回 nums[4] = 6** ✓

## 复杂度

| | 时间 | 空间 |
|--|------|------|
| 快速选择（平均） | O(n) | O(1) |
| 快速选择（最坏） | O(n²) | O(n) |
| 堆排序 | O(n log n) | O(n) |

### 为什么是 O(n)？
T(n) = T(n/2) + O(n) = n + n/2 + n/4 + ... = 2n = **O(n)**

## 两种解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|---------|
| 快速选择 | O(n) 平均 | O(1) | 通用首选 |
| 最小堆 | O(n log k) | O(k) | k 很小时 |

## 面试追问

**Q：最坏情况怎么优化？**
A：随机选择 pivot（三数取中/随机下标），让最坏情况概率极低。面试可以提但不一定要实现。

**Q：和 nth_element（STL）比？**
A：C++ `std::nth_element` 平均 O(n)，实现就是快速选择。Go 没有标准库，可以用 `sort.Inplace` 先排序 O(n log n)，但不是最优。

**Q：第 k 小怎么改？**
A：把 `target = len(nums) - k` 改成 `target = k-1`，partition 中 `<=` 改成 `<`（稳定版本）。本质一样，只是比较方向。
