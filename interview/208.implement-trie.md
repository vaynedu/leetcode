# 208. 实现 Trie（前缀树）

## 题目信息

- 题号：LeetCode 208
- 标题：Implement Trie (Prefix Tree)
- 难度：中等（Medium）
- 模式：Trie / 前缀树

## 一句话思路

用树上的一条路径表示一个字符串，节点保存子字符映射，`isEnd` 区分“前缀存在”和“完整单词存在”。

## 为什么想到这个模式

题目要求支持三个操作：

- `Insert(word)`：插入单词。
- `Search(word)`：查完整单词是否存在。
- `StartsWith(prefix)`：查前缀是否存在。

如果用普通数组或哈希集合，`Search` 很直接，但 `StartsWith` 需要扫描大量单词。Trie 的核心优势正是共享公共前缀，让前缀查询只和前缀长度有关。

看到这些信号时优先想到 Trie：

- 字符串集合。
- 高频前缀查询。
- 词典、自动补全、词根替换。
- 需要区分完整单词和前缀。

## 状态定义 / 数据结构

每个节点表示一个前缀状态：

```go
type TrieNode struct {
    children map[rune]*TrieNode
    isEnd    bool
}
```

含义：

- `children[ch]`：从当前前缀追加字符 `ch` 后到达的节点。
- `isEnd`：从根到当前节点的路径是否构成一个完整单词。
- `root`：空前缀，不代表任何字符。

公共辅助函数：

```text
searchPrefix(s) = 沿着 s 的字符逐层走，走不到返回 nil，走完返回末尾节点
```

## 核心不变量

从 `root` 到任意节点的路径，唯一对应一个前缀。

插入 `word` 后：

- `word` 的每个前缀路径都存在。
- 只有最后一个节点的 `isEnd` 被标记为 `true`。

查询时：

- `Search(word)` 必须同时满足路径存在且末尾节点 `isEnd == true`。
- `StartsWith(prefix)` 只需要路径存在，不要求 `isEnd`。

这个不变量保证了 `app` 和 `apple` 可以共享路径，但不会把 `app` 的前缀误判成完整单词，除非 `app` 被插入过。

## 手推全过程

依次执行：

```text
Insert("apple")
Search("apple")
Search("app")
StartsWith("app")
Insert("app")
Search("app")
```

插入 `"apple"` 后：

```text
root
 └─ a
    └─ p
       └─ p
          └─ l
             └─ e*
```

`*` 表示 `isEnd = true`。

查询过程：

| 操作 | 路径是否存在 | 末尾 isEnd | 结果 | 原因 |
|------|--------------|------------|------|------|
| Search("apple") | 是 | true | true | 完整单词存在 |
| Search("app") | 是 | false | false | 只是前缀，不是完整单词 |
| StartsWith("app") | 是 | 不需要 | true | 前缀路径存在 |
| Insert("app") | 复用已有路径 | 设置 app 节点 | - | 不需要新建前缀 |
| Search("app") | 是 | true | true | 现在是完整单词 |

## Go 代码实现

```go
type TrieNode struct {
    children map[rune]*TrieNode
    isEnd    bool
}

type Trie struct {
    root *TrieNode
}

func Constructor() Trie {
    return Trie{root: &TrieNode{children: map[rune]*TrieNode{}}}
}

func (t *Trie) Insert(word string) {
    node := t.root
    for _, ch := range word {
        if node.children[ch] == nil {
            node.children[ch] = &TrieNode{children: map[rune]*TrieNode{}}
        }
        node = node.children[ch]
    }
    node.isEnd = true
}

func (t *Trie) Search(word string) bool {
    node := t.searchPrefix(word)
    return node != nil && node.isEnd
}

func (t *Trie) StartsWith(prefix string) bool {
    return t.searchPrefix(prefix) != nil
}

func (t *Trie) searchPrefix(s string) *TrieNode {
    node := t.root
    for _, ch := range s {
        node = node.children[ch]
        if node == nil {
            return nil
        }
    }
    return node
}
```

## 错误写法与反例

### 错误 1：`Search` 只判断路径存在

```go
func (t *Trie) Search(word string) bool {
    return t.searchPrefix(word) != nil
}
```

反例：

```text
Insert("apple")
Search("app")
```

路径 `"app"` 存在，但 `"app"` 没有被插入过。正确结果是 `false`，错误写法会返回 `true`。

### 错误 2：插入时忘记设置 `isEnd`

```go
func (t *Trie) Insert(word string) {
    ...
    // 缺少 node.isEnd = true
}
```

反例：

```text
Insert("apple")
Search("apple")
```

路径存在，但末尾没有标记完整单词，`Search` 会错误返回 `false`。

### 错误 3：子节点 map 没初始化

```go
node.children[ch] = &TrieNode{}
```

后续继续向这个节点插入子字符时，`children` 是 nil map，写入会 panic。每个新节点都要初始化 `children`。

## 复杂度

设 `m` 是单词或前缀长度。

| 操作 | 时间 | 额外空间 |
|------|------|----------|
| Insert | O(m) | O(m)，最坏新建 m 个节点 |
| Search | O(m) | O(1) |
| StartsWith | O(m) | O(1) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为核心操作是字符串集合的前缀查询，Trie 能让查询只和字符串长度有关，并共享公共前缀。

2. 这个算法维护的不变量是什么？
   从根到节点的路径对应一个前缀；`isEnd` 只标记完整单词，不能和普通前缀混淆。

3. 每一步为什么不会漏答案？
   插入时逐字符建立完整路径，查询时逐字符沿同一路径走；任何缺失字符都说明该单词或前缀没有被插入过。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是 `Search` 忘记检查 `isEnd`，会把已存在的前缀误判成完整单词。
