# 113. 路径总和 II

## 题目信息

- 题号：LeetCode 113
- 标题：Path Sum II
- 难度：中等（Medium）
- 模式：二叉树 DFS / 回溯 / 根到叶路径

## 一句话思路

DFS 枚举所有根到叶路径，用 `path` 记录当前路径；到叶子时如果剩余目标等于叶子值，就复制当前路径加入答案。

## 为什么想到这个模式

题目要求返回所有满足条件的根到叶路径。和 112 一样，路径必须：

```text
从根开始，到叶子结束
```

但 112 只判断是否存在，113 要收集所有路径，所以递归过程中还需要维护当前路径 `path`。每走到一个节点就加入路径，分支结束后回到上一层继续试其它分支，这是典型回溯。

## 状态定义 / 数据结构

- `node`：当前节点。
- `target`：从当前节点到叶子还需要凑出的目标值。
- `path`：从根到当前节点之前的路径。
- `result`：所有合法根到叶路径。

递归含义：

```text
pathSumRec(node, target, path, result)
= 在当前 path 前缀下，从 node 开始寻找和为 target 的根到叶后缀
```

## 核心不变量

任意时刻：

- `path` 表示从根到当前节点的路径前缀。
- `target` 表示当前节点到叶子还需要凑出的路径和。
- 只有当前节点是叶子时，才可以把完整路径加入结果。
- 加入结果时必须复制 `path`，避免后续回溯修改影响已保存答案。

当前实现中 `path` 作为切片值传入子递归，每层 append 后形成当前层视角下的路径；即便如此，保存答案时仍应复制底层数组。

## 手推全过程

例子：

```text
      5
     / \
    4   8
   / \
  7   2

targetSum = 11
```

递归路径：

```text
5 -> 4 -> 7，路径和 16，失败
5 -> 4 -> 2，路径和 11，收集 [5,4,2]
5 -> 8，路径和 13，失败
```

答案：

```text
[[5,4,2]]
```

## Go 代码实现

```go
func PathSumII(root *TreeNode, targetSum int) [][]int {
    var result [][]int
    var path []int
    pathSumRec(root, targetSum, path, &result)
    return result
}

func pathSumRec(node *TreeNode, target int, path []int, result *[][]int) {
    if node == nil {
        return
    }
    path = append(path, node.Val)
    if node.Left == nil && node.Right == nil {
        if node.Val == target {
            tmp := make([]int, len(path))
            copy(tmp, path)
            *result = append(*result, tmp)
        }
        return
    }
    pathSumRec(node.Left, target-node.Val, path, result)
    pathSumRec(node.Right, target-node.Val, path, result)
}
```

## 错误写法与反例

### 错误 1：保存路径时不复制

```go
*result = append(*result, path)
```

如果后续递归复用并修改了同一个底层数组，已保存结果可能被污染。收集答案时应复制一份。

### 错误 2：非叶子节点就收集答案

反例：

```text
root = [1,2], target = 1
```

根节点值等于目标，但它不是叶子，不能收集 `[1]`。

### 错误 3：递归左右子树时忘记减去当前节点值

如果子树仍用原始 `target`，每层判断的目标含义就不一致，路径和会被重复计算或漏算。

## 与 112 / 437 的区别

| 题目 | 路径定义 | 返回值 |
|------|----------|--------|
| 112 | 根到叶 | 是否存在 |
| 113 | 根到叶 | 所有路径 |
| 437 | 任意节点到任意后代 | 路径条数 |

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n * h)，每个节点访问一次，收集答案时复制路径，单次最多 h |
| 空间 | O(h)，不计结果，递归栈和当前路径长度 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要枚举所有根到叶路径，并在路径满足目标和时收集完整路径。

2. 这个算法维护的不变量是什么？
   `path` 是当前根到节点的路径，`target` 是当前节点到叶子还需要凑出的和。

3. 每一步为什么不会漏答案？
   DFS 会访问所有根到叶路径；只有叶子节点进行收集，覆盖所有完整路径且不会收集半截路径。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是保存路径不复制，后续递归可能修改底层数组，导致结果里的路径被污染。
