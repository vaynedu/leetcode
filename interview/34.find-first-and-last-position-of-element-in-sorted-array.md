# 34. 在排序数组中查找元素的第一个和最后一个位置

## 题目信息

- 题号：LeetCode 34
- 难度：中等
- 模式：二分查找 / 边界查找
- 可视化：`doc/algorithm-visualizations/34.binary-search-boundary.html`

## 一句话思路

用两次 `lowerBound`：第一次找第一个 `>= target` 的位置，第二次找第一个 `> target` 的位置再减一。

## 为什么想到这个模式

数组有序，题目要求 O(log n)，并且要找的是 target 的左右边界，不是任意一个 target。

标准二分找到 target 后不能保证是第一个或最后一个，所以要转化为“边界位置”问题。

## 状态定义 / 数据结构

- `lowerBound(nums, x)`：返回第一个 `>= x` 的下标。
- 搜索区间使用左闭右开 `[left,right)`。
- `left`：当前可能答案的最左位置。
- `right`：当前搜索区间右边界，不包含。

右边界可以写成：

```text
right = lowerBound(target+1) - 1
```

也可以写成第一个 `> target` 的位置减一。

## 核心不变量

在 `lowerBound` 中，答案始终位于 `[left,right)`。

- 如果 `nums[mid] < target`，`mid` 及其左侧都不可能是第一个 `>= target`，所以 `left = mid + 1`。
- 否则 `mid` 可能是答案，必须保留，所以 `right = mid`。

循环结束时 `left == right`，这个位置就是第一个 `>= target` 的位置。

## 手推全过程

以 `nums=[5,7,7,8,8,10], target=8` 为例：

第一次：

```text
lowerBound(8) = 3
nums[3] == 8，所以左边界存在
```

第二次：

```text
lowerBound(9) = 5
右边界 = 5 - 1 = 4
```

答案：`[3,4]`。

不存在情况：`target=6`

```text
lowerBound(6) = 1
nums[1] = 7 != 6
返回 [-1,-1]
```

## 错误写法与反例

错误写法：找到 target 就返回。

```go
if nums[mid] == target {
    return mid
}
```

反例：`nums=[5,7,7,8,8,10], target=8`

一次二分可能返回 3，也可能返回 4，但题目要求同时返回最左和最右边界。

另一个错误：左边界搜索时写 `right = mid - 1`。

在左闭右开模板里，`mid` 可能就是第一个 `>= target`，不能丢掉，所以必须 `right = mid`。

## Go 代码实现

```go
func searchRange(nums []int, target int) []int {
    left := lowerBound(nums, target)
    if left == len(nums) || nums[left] != target {
        return []int{-1, -1}
    }

    right := lowerBound(nums, target+1) - 1
    return []int{left, right}
}

func lowerBound(nums []int, target int) int {
    left, right := 0, len(nums)
    for left < right {
        mid := left + (right-left)/2
        if nums[mid] < target {
            left = mid + 1
        } else {
            right = mid
        }
    }
    return left
}
```

## 复杂度

| 指标 | 值 |
|------|-----|
| 时间 | O(log n)，两次二分 |
| 空间 | O(1) |

## 四问复盘

1. **为什么属于二分？** 数组有序，且要在 O(log n) 内找边界。
2. **维护的不变量是什么？** `lowerBound` 的答案始终在 `[left,right)` 中。
3. **为什么不会漏答案？** 当 `nums[mid] >= target` 时保留 `mid`，因此不会跳过第一个满足条件的位置。
4. **最容易错的边界？** 左闭右开模板里 `right = mid`，不是 `mid - 1`。
