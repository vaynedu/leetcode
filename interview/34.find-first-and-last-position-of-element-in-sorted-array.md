# 34. 在排序数组中查找元素的第一个和最后一个位置

## 题目

给定一个按照升序排列的整数数组 `nums`，和一个目标值 `target`。找出给定目标值在数组中开始和结束的位置。如果数组中不存在目标值，返回 `[-1, -1]`。

**难度**：Medium
**标签**：数组 | 二分查找
**时间要求**：O(log n)

## 示例

```
输入：nums = [5,7,7,8,8,10], target = 8
输出：[3, 4]

输入：nums = [5,7,7,8,8,10], target = 6
输出：[-1, -1]
```

---

## 引导过程：像教练一样思考

### 第一步：看到 O(log n) 要求，想到什么？

O(log n) 几乎是二分查找的标志。看到这个要求，**第一反应应该是：能用二分吗？**

---

### 第二步：暴力解是什么，为什么不行？

**暴力解**：遍历数组，找所有 target 的位置 → O(n)，不满足要求。

---

### 第三步：一次二分能找到答案吗？

```
标准二分：while left <= right，找到 target 就返回
```

如果数组是 `[5,7,7,8,8,10]`，target = 8：
- 一次二分找到 mid = 8，返回索引 3 ✓

但如果数组是 `[5,7,7,8,8,10]`，target = 7：
- 一次二分可能找到索引 1 或 2，**无法同时知道第一个和最后一个在哪里**

**结论：一次二分不够，需要两次**

---

### 第四步：如何用两次二分解决？

#### 第一次：找左侧边界（第一个 >= target 的位置）

定义一个 `lowerBound(nums, target)` 函数，返回**第一个 >= target** 的索引。

以 `[5,7,7,8,8,10]`，target = 8 为例：
```
lowerBound(8) = 3   // 第一个 >= 8 的是索引 3 的 8 ✓
lowerBound(7) = 1   // 第一个 >= 7 的是索引 1 的 7 ✓
lowerBound(6) = 1   // 第一个 >= 6 的是索引 1 的 7
```

如果 `lowerBound(target)` 找到的值不是 target，说明不存在 → return [-1,-1]

#### 第二次：找右侧边界（最后一个 <= target 的位置）

**技巧**：找 target+1 的左侧边界，再减 1

为什么？`lowerBound(target+1)` 返回第一个 >= (target+1) 的位置，这个位置减 1 就是最后一个 <= target 的位置。

以 `[5,7,7,8,8,10]`，target = 8 为例：
```
lowerBound(9) = 5   // 第一个 >= 9 的是索引 5 的 10
right = 5 - 1 = 4    // 最后一个 <= 8 的位置是 4 ✓
```

---

### 第五步：lowerBound 怎么写？

**左闭右开区间** `[left, right)` 模板：

```go
func lowerBound(nums []int, target int) int {
    left, right := 0, len(nums)   // 注意 right = len(nums)
    for left < right {             // 注意是 < 不是 <=
        mid := left + (right-left)/2
        if nums[mid] < target {
            left = mid + 1         // target 在右边，收缩 left
        } else {
            right = mid            // target 在左边（含当前位置），收缩 right
        }
    }
    return left                    // left == right 时退出
}
```

**为什么用左闭右开？**
- 当 left == right 时表示"空区间"，自然结束
- 更容易处理边界（right 可以直接取 len(nums)）

---

### 第六步：完整代码

```go
func searchRange(nums []int, target int) []int {
    if len(nums) == 0 {
        return []int{-1, -1}
    }

    // 第一次：找左侧边界
    left := lowerBound(nums, target)
    if left == len(nums) || nums[left] != target {
        return []int{-1, -1}
    }

    // 第二次：找右侧边界 = lowerBound(target+1) - 1
    right := lowerBound(nums, target+1) - 1
    return []int{left, right}
}

func lowerBound(nums []int, target int) int {
    left, right := 0, len(nums)
    for left < right {
        mid := left + (right-left)/2
        if nums[mid] < target {
            left = mid + 1
        } else {
            right = mid
        }
    }
    return left
}
```

---

## 关键点总结

| 要点 | 说明 |
|------|------|
| 为什么两次二分 | 一次只能找到一个位置，无法同时确定左右边界 |
| 右侧边界技巧 | 对 target+1 做 lowerBound 再减 1 |
| lowerBound 模板 | 左闭右开 `[left, right)`，`left < right` 结束 |
| 边界检查 | left == len(nums) 或 nums[left] != target 说明不存在 |

## 复杂度

- **时间**：O(log n)，两次二分
- **空间**：O(1)
