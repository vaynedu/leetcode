# 226. 翻转二叉树

## 题目信息

- 题号：LeetCode 226
- 标题：Invert Binary Tree
- 难度：简单（Easy）
- 模式：二叉树 DFS / 递归改指针

## 一句话思路

对每个节点交换 `Left` 和 `Right`，再递归处理交换后的左右子树。

## 为什么想到这个模式

翻转二叉树的定义是：每个节点的左子树和右子树互换。

这不是只交换根节点，而是整棵树每个节点都要做同样操作。二叉树天然适合递归：

```text
翻转当前节点 = 交换左右子树 + 翻转左右子树内部
```

## 状态定义 / 数据结构

- `root`：当前要翻转的子树根节点。
- 返回值：翻转后的当前子树根节点，仍然是 `root`。

递归含义：

```text
invertTree(root) = 翻转以 root 为根的整棵子树，并返回 root
```

## 核心不变量

每次 `invertTree(node)` 返回后：

- 以 `node` 为根的子树已经完成镜像翻转。
- 原左子树出现在右侧，原右子树出现在左侧。
- 子树内部的每个节点也都完成了左右交换。

空节点返回 `nil`，作为递归终止条件。

## 手推全过程

例子：

```text
原树：
      4
     / \
    2   7
   / \ / \
  1  3 6  9

翻转后：
      4
     / \
    7   2
   / \ / \
  9  6 3  1
```

每个节点都执行一次左右交换。

## Go 代码实现

```go
func invertTree(root *TreeNode) *TreeNode {
    if root == nil {
        return nil
    }
    root.Left, root.Right = root.Right, root.Left
    invertTree(root.Left)
    invertTree(root.Right)
    return root
}
```

## 错误写法与反例

### 错误 1：只交换根节点

只交换根的左右子树，子树内部不会镜像。

反例：`[4,2,7,1,3,6,9]` 中，节点 `2` 内部也要从 `1,3` 变成 `3,1`。

### 错误 2：交换后只递归一边

左右两棵子树都必须继续翻转。漏掉一边会导致半棵树仍保持原结构。

### 错误 3：交换前递归和交换后递归混淆

交换前递归左右子树、再交换当前节点也可以；交换后递归当前左右子树也可以。关键是每个节点都被处理一次，且不要因为交换导致漏递归。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，每个节点交换一次 |
| 空间 | O(h)，递归栈高度，最坏 O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为每个节点都要执行相同的左右交换操作，二叉树子问题结构完全一致。

2. 这个算法维护的不变量是什么？
   `invertTree(node)` 返回后，`node` 子树已经整体镜像翻转。

3. 每一步为什么不会漏答案？
   每个非空节点都会被递归访问一次，并在访问时交换左右子树。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是只交换根或只递归一边，会导致局部翻转而不是整棵树镜像。
