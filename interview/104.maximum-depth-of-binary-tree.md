# 104. 二叉树最大深度

## 题目信息

- 题号：LeetCode 104
- 标题：Maximum Depth of Binary Tree
- 难度：简单（Easy）
- 模式：树 / DFS 递归

## 一句话思路

一棵树的最大深度等于左右子树最大深度的较大值加 1。

## 为什么想到这个模式

二叉树问题常见思路是把整棵树拆成“根节点 + 左子树 + 右子树”。最大深度是典型的子树返回值问题：父节点只需要知道左右子树各自的深度。

## 状态定义 / 数据结构

递归函数：

```text
maxDepth(root) = 以 root 为根的子树最大深度
```

返回值：

- `root == nil`：深度为 0。
- 非空节点：`max(leftDepth, rightDepth) + 1`。

## 核心不变量

每个递归调用都返回当前子树从根到最远叶子的节点数。

父节点不需要知道路径细节，只需要左右子树的最大深度，就能合并出当前子树答案。

## 手推全过程

树：

```text
     3
    / \
   9  20
     /  \
    15   7
```

| 节点 | 左深度 | 右深度 | 返回 |
|------|--------|--------|------|
| 9 | 0 | 0 | 1 |
| 15 | 0 | 0 | 1 |
| 7 | 0 | 0 | 1 |
| 20 | 1 | 1 | 2 |
| 3 | 1 | 2 | 3 |

答案是 `3`。

## Go 代码实现

```go
func maxDepth(root *TreeNode) int {
    if root == nil {
        return 0
    }
    left := maxDepth(root.Left)
    right := maxDepth(root.Right)
    if left > right {
        return left + 1
    }
    return right + 1
}
```

## 错误写法与反例

### 错误 1：空节点返回 1

```go
if root == nil {
    return 1
}
```

反例：空树应为 0；单节点树会被错误算成 2。

### 错误 2：返回左右深度之和

```go
return left + right + 1
```

这是节点数量相关逻辑，不是最大深度。最大深度只能选更深的一边。

### 错误 3：忘记加当前节点

```go
return max(left, right)
```

会少算当前根节点，单节点树会返回 0。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(h) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为答案由左右子树的返回值合并而来，是标准树递归。

2. 这个算法维护的不变量是什么？
   `maxDepth(node)` 始终返回以 `node` 为根的子树最大深度。

3. 每一步为什么不会漏答案？
   每个节点都递归访问一次，父节点取左右子树更深路径并加上自身。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是空节点深度，必须是 0；写错会整体深度偏大。
