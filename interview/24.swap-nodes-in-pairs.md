# 24. 两两交换链表中的节点

## 题目信息

- 题号：LeetCode 24
- 标题：Swap Nodes in Pairs
- 难度：中等（Medium）
- 模式：链表 / 指针重连 / 虚拟头节点

## 一句话思路

用 `prev` 指向已处理部分尾部，每轮取 `first=prev.Next`、`second=first.Next`，把这两个节点交换后让 `prev` 移到 `first`。

## 为什么想到这个模式

题目要求交换节点本身，而不是只交换节点值。链表题的关键是改指针。

每次只处理相邻两个节点：

```text
prev -> first -> second -> next
```

交换后变成：

```text
prev -> second -> first -> next
```

因为头两个节点也可能被交换，所以用 `dummy` 统一处理头节点变化。

## 状态定义 / 数据结构

- `dummy`：虚拟头，指向当前链表头。
- `prev`：已完成交换部分的尾节点。
- `first`：当前组第一个节点。
- `second`：当前组第二个节点。

循环条件：

```text
prev.Next != nil && prev.Next.Next != nil
```

只有剩余至少两个节点时才交换。

## 核心不变量

每轮循环开始时：

- `dummy.Next` 到 `prev` 之前的部分已经两两交换完成。
- `prev.Next` 是下一组待交换的第一个节点。
- 未处理部分仍保持原相对顺序。

每轮交换后，`prev` 移到 `first`，也就是交换后这一组的尾节点。

## 手推全过程

输入：

```text
1 -> 2 -> 3 -> 4
```

第一轮：

```text
prev=dummy, first=1, second=2, next=3
dummy -> 2 -> 1 -> 3 -> 4
prev 移到 1
```

第二轮：

```text
prev=1, first=3, second=4, next=nil
dummy -> 2 -> 1 -> 4 -> 3
```

答案：

```text
2 -> 1 -> 4 -> 3
```

## Go 代码实现

```go
func swapPairs(head *ListNode) *ListNode {
    dummy := &ListNode{}
    dummy.Next = head
    prev := dummy
    for prev.Next != nil && prev.Next.Next != nil {
        first := prev.Next
        second := first.Next
        first.Next = second.Next
        second.Next = first
        prev.Next = second
        prev = first
    }
    return dummy.Next
}
```

## 错误写法与反例

### 错误 1：不用 dummy，头节点交换后丢新头

反例：`[1,2]`

交换后新头是 `2`。没有 dummy 时，很容易仍返回旧头 `1`。

### 错误 2：指针重连顺序错误

必须先保存 `second`，并让 `first.Next` 接到 `second.Next`，否则可能断链或成环。

### 错误 3：交换后 `prev` 移错位置

交换后这一组尾节点是 `first`。如果 `prev` 移到 `second`，下一轮会重复处理或跳错节点。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要原地交换链表节点，需要维护前驱节点并重连相邻两个节点的指针。

2. 这个算法维护的不变量是什么？
   `prev` 之前的部分已经两两交换完成，`prev.Next` 是下一组待处理节点。

3. 每一步为什么不会漏答案？
   每轮处理恰好两个节点并把 `prev` 移到该组尾部，剩余节点继续按组处理；奇数尾节点自然保留。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是头两个节点交换后返回旧头，所以需要 dummy 保持新头可访问。
