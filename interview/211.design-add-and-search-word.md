# 211. 添加与搜索单词 - 数据结构设计

## 题目信息

- 题号：LeetCode 211
- 标题：Design Add and Search Words Data Structure
- 难度：中等（Medium）
- 模式：Trie / DFS 通配符

## 一句话思路

在 Trie 的精确匹配基础上，遇到 `'.'` 时遍历当前节点的所有子节点，递归匹配剩余字符串。

## 为什么想到这个模式

题目需要动态添加单词，并搜索普通字符串或含 `'.'` 的模式。普通前缀树能处理精确字符路径；`'.'` 表示任意单个字符，所以遇到它时不能走唯一边，只能尝试所有子节点。

选择 Trie 的信号：

- 字符串集合动态插入。
- 搜索按字符逐位匹配。
- 需要区分完整单词和前缀。
- 通配符只影响当前一位字符。

## 状态定义 / 数据结构

Trie 节点：

```go
type TrieNode211 struct {
    children map[rune]*TrieNode211
    isEnd    bool
}
```

搜索递归含义：

```text
searchHelper(word, node) = 从当前 node 出发，能否匹配剩余 word
```

返回 true 的条件：模式全部消费完，且当前节点 `isEnd == true`。

## 核心不变量

从根到任意节点的路径表示一个前缀。

搜索时：

- 普通字符：必须沿对应字符边继续走。
- `'.'`：尝试当前节点所有子节点，只要任意一条能匹配剩余模式即可。
- 模式结束：必须落在完整单词结尾。

`'.'` 只匹配一个字符，不是任意长度。

## 手推全过程

执行：

```text
addWord("bad")
addWord("dad")
addWord("mad")
search("pad")
search("bad")
search(".ad")
search("b..")
```

搜索结果：

| 模式 | 过程 | 结果 |
|------|------|------|
| `pad` | 根节点没有 `p` 分支 | false |
| `bad` | b -> a -> d，结尾 isEnd | true |
| `.ad` | `.` 尝试 b/d/m，b 分支能匹配 ad | true |
| `b..` | b 后两个 `.` 分别匹配 a/d | true |

## Go 代码实现

```go
func (w *WordDictionary) searchHelper(word string, node *TrieNode211) bool {
    for i, ch := range word {
        if ch == '.' {
            for _, child := range node.children {
                if w.searchHelper(word[i+1:], child) {
                    return true
                }
            }
            return false
        }
        if _, ok := node.children[ch]; !ok {
            return false
        }
        node = node.children[ch]
    }
    return node.isEnd
}
```

## 错误写法与反例

### 错误 1：把 `'.'` 当普通字符查

```go
node = node.children['.']
```

反例：添加 `"bad"` 后搜索 `".ad"` 应返回 true。Trie 中不会有 `'.'` 这条边，它是模式语义，不是字符。

### 错误 2：模式结束直接返回 true

```go
return true
```

反例：添加 `"bad"` 后搜索 `"ba"`。路径存在，但不是完整单词，应该返回 false。

### 错误 3：`'.'` 匹配任意长度

`'.'` 只能匹配一个字符。搜索 `"b."` 不能匹配 `"bad"`，因为长度不同。

## 复杂度

| 操作 | 时间 | 空间 |
|------|------|------|
| AddWord | O(m) | O(m) |
| Search | 无通配符 O(m)，大量通配符最坏 O(分支数^m) | O(m) 递归栈 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为需要维护字符串集合并按字符路径搜索，通配符只是在 Trie 搜索时引入 DFS 分支。

2. 这个算法维护的不变量是什么？
   从根到节点表示前缀；搜索函数始终表示“当前节点能否匹配剩余模式”。

3. 每一步为什么不会漏答案？
   普通字符走唯一可能路径；`'.'` 枚举所有子节点，覆盖所有可能单字符匹配。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是结尾不检查 `isEnd`，会把前缀误判成完整单词。
