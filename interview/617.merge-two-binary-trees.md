# 617. 合并二叉树

## 题目信息

- 题号：LeetCode 617
- 标题：Merge Two Binary Trees
- 难度：简单（Easy）
- 模式：二叉树 DFS / 递归合并

## 一句话思路

如果两个节点都存在，就把值相加并递归合并左右子树；如果某一边为空，直接返回另一边节点。

## 为什么想到这个模式

合并规则是按树的相同位置逐节点处理：

```text
两个节点都存在：值相加
只有一个节点存在：保留这个节点
两个都空：返回空
```

左右子树的合并规则和根节点完全一样，所以可以递归处理。

## 状态定义 / 数据结构

- `root1`：第一棵树当前节点，也是当前实现的原地合并目标。
- `root2`：第二棵树当前节点。
- 返回值：合并后的当前子树根节点。

递归含义：

```text
MergeTrees(root1, root2) = 合并这两个位置上的子树，并返回合并后的根
```

## 核心不变量

每次递归返回后：

- 如果两个输入节点都非空，返回的是值相加后的 `root1`。
- 如果其中一个为空，返回另一个非空节点。
- 返回子树已经满足合并规则。

当前实现会修改 `root1`，不是纯新建树。

## 手推全过程

例子：

```text
t1:
    1
   / \
  3   2
 /
5

t2:
    2
   / \
  1   3
```

合并：

```text
根：1 + 2 = 3
左：3 + 1 = 4
右：2 + 3 = 5
左左：5 + nil = 5
```

结果：

```text
    3
   / \
  4   5
 /
5
```

## Go 代码实现

```go
func MergeTrees(root1, root2 *TreeNode) *TreeNode {
    if root1 == nil {
        return root2
    }
    if root2 == nil {
        return root1
    }
    root1.Val += root2.Val
    root1.Left = MergeTrees(root1.Left, root2.Left)
    root1.Right = MergeTrees(root1.Right, root2.Right)
    return root1
}
```

## 错误写法与反例

### 错误 1：遇到单空直接返回 nil

如果一边为空，另一边整棵子树应该被保留。

反例：

```text
root1 = [1]
root2 = nil
```

正确结果是 `[1]`，不是空树。

### 错误 2：只合并根节点，不递归左右子树

合并是对所有对应位置生效，不能只处理根。

### 错误 3：误以为不会修改原树

当前实现把结果写回 `root1`。如果调用方还需要保留原 `root1`，应该新建节点而不是原地修改。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(k)，k 为两棵树重叠区域访问到的节点数，最坏 O(m+n) |
| 空间 | O(h)，递归栈高度，最坏 O(m+n) 的链式高度 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为两棵树同位置的合并规则一致，左右子树是相同子问题。

2. 这个算法维护的不变量是什么？
   `MergeTrees(a,b)` 返回的位置已经按题意合并；单空返回非空，双非空值相加后继续合并左右。

3. 每一步为什么不会漏答案？
   每个双非空位置都会递归处理左右子树；遇到单空位置直接返回完整非空子树，保留所有剩余节点。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是单空位置返回 nil，会丢掉另一棵树中本应保留的整段子树。
