# 79. 单词搜索

## 题目信息

- 题号：LeetCode 79
- 标题：Word Search
- 难度：中等（Medium）
- 模式：DFS / 回溯 / 网格搜索

## 一句话思路

枚举每个格子作为起点，用 DFS 按顺序匹配 `word[idx]`；当前路径上的格子不能重复使用，递归返回时恢复访问状态。

## 为什么想到这个模式

题目要判断网格里是否存在一条路径，路径由上下左右相邻格子组成，并且每个格子只能使用一次。

这类问题有三个明显信号：

- “从某个起点出发”。
- “上下左右相邻”。
- “同一条路径不能重复使用格子”。

所以它不是普通遍历，而是带路径状态的 DFS。因为访问标记只对当前路径有效，递归返回后必须撤销，这就是回溯。

## 状态定义 / 数据结构

- `i, j`：当前所在格子。
- `idx`：当前要匹配 `word[idx]`。
- `visited[i][j]`：当前路径是否已经使用过这个格子。
- `board`：字符网格。

递归含义：

```text
dfs(i, j, idx) = 从 board[i][j] 开始，能否匹配 word[idx:]
```

## 核心不变量

任意时刻：

- 当前递归路径已经匹配了 `word[0:idx]`。
- `visited == true` 的格子正好是当前路径上已经使用过的格子。
- 如果 `board[i][j] != word[idx]`，当前分支一定不可能成功。
- 离开当前格子前必须恢复 `visited[i][j] = false`，让其它起点或其它分支可以使用它。

## 手推全过程

以：

```text
board =
A B C E
S F C S
A D E E

word = "ABCCED"
```

一条成功路径：

```text
(0,0) A
  → (0,1) B
    → (0,2) C
      → (1,2) C
        → (2,2) E
          → (2,1) D
```

匹配顺序正好是 `A -> B -> C -> C -> E -> D`，所以返回 `true`。

反例 `word = "ABCB"`：

```text
A -> B -> C
```

最后还需要 `B`，但不能回到已经使用过的 `(0,1)`，所以返回 `false`。

## Go 代码实现

```go
func exist(board [][]byte, word string) bool {
    if len(board) == 0 || len(word) == 0 {
        return false
    }
    m, n := len(board), len(board[0])
    visited := make([][]bool, m)
    for i := range visited {
        visited[i] = make([]bool, n)
    }

    var dfs func(i, j, idx int) bool
    dfs = func(i, j, idx int) bool {
        if idx == len(word) {
            return true
        }
        if i < 0 || i >= m || j < 0 || j >= n {
            return false
        }
        if visited[i][j] || board[i][j] != word[idx] {
            return false
        }
        visited[i][j] = true
        defer func() { visited[i][j] = false }()
        return dfs(i+1, j, idx+1) || dfs(i-1, j, idx+1) ||
               dfs(i, j+1, idx+1) || dfs(i, j-1, idx+1)
    }

    for i := 0; i < m; i++ {
        for j := 0; j < n; j++ {
            if dfs(i, j, 0) {
                return true
            }
        }
    }
    return false
}
```

## 错误写法与反例

### 错误 1：忘记恢复 `visited`

```go
visited[i][j] = true
return dfs(...)
```

如果不恢复，某个失败分支使用过的格子会永久被标记，后续分支或其它起点无法再使用它，导致漏答案。

### 错误 2：不做 `visited` 判断

反例：`board=[["A","B"]]`，`word="ABA"`

如果允许重复使用格子，可能走 `A -> B -> A`，但题目禁止同一个格子在一条路径中重复使用。

### 错误 3：只从第一个字符相同的位置开始后就停止

网格中可能有多个起点字符相同。一个起点失败不代表其它起点失败，必须枚举所有格子作为潜在起点。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(M * N * 4 * 3^(L-1))，起点有 M*N 个，首步最多 4 个方向，之后不能回头所以约 3 个方向 |
| 空间 | O(M * N + L)，`visited` 加递归深度；若原地标记可降为 O(L) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要在网格中寻找一条满足顺序匹配的路径，每一步有上下左右多个选择，并且路径状态需要回溯。

2. 这个算法维护的不变量是什么？
   当前递归路径匹配了 `word[0:idx]`，`visited` 精确表示当前路径上已经使用的格子。

3. 每一步为什么不会漏答案？
   外层枚举所有起点，DFS 对每个合法格子尝试四个方向，只剪掉越界、重复访问和字符不匹配的必败分支。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是忘记恢复 `visited`，失败分支会污染其它分支，导致本来存在的路径被误判为不存在。
