# 62. 不同路径

## 题目信息

- 题号：LeetCode 62
- 标题：Unique Paths
- 难度：中等（Medium）
- 模式：动态规划 / 网格 DP

## 一句话思路

到达一个格子的路径数，只能来自上方或左方，所以 `dp[i][j] = dp[i-1][j] + dp[i][j-1]`。

## 为什么想到这个模式

题目问从左上角到右下角有多少条路径，机器人每次只能向右或向下走。到达 `(i, j)` 的最后一步只有两种可能：

- 从 `(i-1, j)` 向下走来。
- 从 `(i, j-1)` 向右走来。

也就是说，当前格子的答案依赖更小子问题，而且不同路径会重复共享中间格子的路径数。这是典型 DP 信号。

这题也可以用组合数学做，但面试中更建议先掌握 DP，因为它能自然扩展到有障碍物、最小路径和等网格题。

## 状态定义 / 数据结构

二维定义：

```text
dp[i][j] = 从左上角走到第 i 行第 j 列的路径数
```

转移：

```text
dp[i][j] = dp[i-1][j] + dp[i][j-1]
```

边界：

- 第一行只能一直向右走，路径数都是 `1`。
- 第一列只能一直向下走，路径数都是 `1`。

空间优化后：

```text
dp[j] = 当前行第 j 列的路径数
```

更新时：

```text
dp[j] = dp[j] + dp[j-1]
```

其中旧的 `dp[j]` 表示上方路径数，`dp[j-1]` 表示左方路径数。

## 核心不变量

使用一维数组时，每一行从左到右更新：

- 更新前的 `dp[j]` 是上一行同列的路径数。
- 更新后的 `dp[j-1]` 是当前行左侧的路径数。
- 所以 `dp[j] += dp[j-1]` 正好等价于 `上方 + 左方`。

第一行初始化为全 `1`，保证所有后续转移都有正确起点。

## 手推全过程

以 `m = 3, n = 7` 为例。

初始化第一行：

```text
[1, 1, 1, 1, 1, 1, 1]
```

处理第二行：

```text
j=1: [1, 2, 1, 1, 1, 1, 1]
j=2: [1, 2, 3, 1, 1, 1, 1]
j=3: [1, 2, 3, 4, 1, 1, 1]
j=4: [1, 2, 3, 4, 5, 1, 1]
j=5: [1, 2, 3, 4, 5, 6, 1]
j=6: [1, 2, 3, 4, 5, 6, 7]
```

处理第三行：

```text
j=1: [1, 3, 3, 4, 5, 6, 7]
j=2: [1, 3, 6, 4, 5, 6, 7]
j=3: [1, 3, 6,10, 5, 6, 7]
j=4: [1, 3, 6,10,15, 6, 7]
j=5: [1, 3, 6,10,15,21, 7]
j=6: [1, 3, 6,10,15,21,28]
```

答案是 `28`。

## Go 代码实现

```go
func uniquePaths(m int, n int) int {
    dp := make([]int, n)
    for j := 0; j < n; j++ {
        dp[j] = 1
    }

    for i := 1; i < m; i++ {
        for j := 1; j < n; j++ {
            dp[j] += dp[j-1]
        }
    }
    return dp[n-1]
}
```

## 错误写法与反例

### 错误 1：初始化为 0

```go
dp := make([]int, n)
```

如果不把第一行初始化为 `1`，所有转移都会从 `0` 开始，最终答案也是 `0`。

反例：`m = 3, n = 7`，正确答案是 `28`，错误初始化会得到 `0`。

### 错误 2：从 `j = 0` 开始转移

```go
for j := 0; j < n; j++ {
    dp[j] += dp[j-1]
}
```

`j = 0` 时没有左方格子，访问 `dp[-1]` 会越界。第一列路径数应该始终保持 `1`。

### 错误 3：一维数组从右往左更新

```go
for j := n - 2; j >= 0; j-- {
    dp[j] += dp[j+1]
}
```

这不再表示“上方 + 左方”，而是混入了右侧状态。网格路径的方向是向右/向下，因此压缩数组要从左到右更新。

|------|--------|
| 时间 | O(mn) |
| 空间 | O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为到达每个格子的路径数由上方和左方两个更小子问题组成，且子问题大量重叠。

2. 这个算法维护的不变量是什么？
   一维 `dp[j]` 更新前表示上一行同列，`dp[j-1]` 更新后表示当前行左侧，因此转移始终等价于 `上方 + 左方`。

3. 每一步为什么不会漏答案？
   所有合法路径到达 `(i,j)` 的最后一步只能来自上方或左方，这两类互不重叠且覆盖全部情况。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是第一行和第一列初始化。它们都只有一条路径，如果初始化错，后续所有状态都会被污染。
