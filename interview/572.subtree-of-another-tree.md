# 572. 另一棵树的子树

## 题目信息

- 题号：LeetCode 572
- 标题：Subtree of Another Tree
- 难度：简单（Easy）
- 模式：二叉树 DFS / 双重递归

## 一句话思路

遍历 `root` 的每个节点，把它当作候选子树根；如果候选子树和 `subRoot` 完全相同，就返回 `true`。

## 为什么想到这个模式

`subRoot` 是 `root` 的子树，意味着存在 `root` 中某个节点，使得：

```text
以该节点为根的整棵子树 == subRoot
```

所以需要两层判断：

- 外层：在 `root` 中枚举每个候选起点。
- 内层：判断两棵树是否结构和值都完全相同。

## 状态定义 / 数据结构

- `IsSubtree(root, subRoot)`：判断 `subRoot` 是否出现在 `root` 的某个子树位置。
- `isSameTree(a, b)`：判断两棵树是否完全相同。

`isSameTree` 的三个基本情况：

```text
双空：相同
单空：不同
都非空：值相同且左右子树相同
```

## 核心不变量

外层递归：

- 每个 `root` 节点都会被当作候选子树根检查一次。
- 当前节点不匹配时，继续检查左右子树。

内层递归：

- `isSameTree(a, b)` 返回 true 当且仅当 `a` 和 `b` 的结构和值完全一致。

## 手推全过程

例子：

```text
root:
    3
   / \
  4   5
 / \
1   2

subRoot:
  4
 / \
1   2
```

遍历候选根：

```text
root=3，与 subRoot 根值不同，继续
root=4，调用 isSameTree
  4 == 4
  左子树 1 == 1
  右子树 2 == 2
匹配成功
```

返回 `true`。

## Go 代码实现

```go
func IsSubtree(root, subRoot *TreeNode) bool {
    if root == nil {
        return false
    }
    if isSameTree(root, subRoot) {
        return true
    }
    return IsSubtree(root.Left, subRoot) || IsSubtree(root.Right, subRoot)
}

func isSameTree(a, b *TreeNode) bool {
    if a == nil && b == nil {
        return true
    }
    if a == nil || b == nil {
        return false
    }
    return a.Val == b.Val &&
        isSameTree(a.Left, b.Left) &&
        isSameTree(a.Right, b.Right)
}
```

## 错误写法与反例

### 错误 1：只判断根值相同

根值相同不代表整棵子树相同，左右结构和值都必须一致。

### 错误 2：只判断 `subRoot` 是不是一条路径

子树必须包含该候选节点下面的完整结构，不能只匹配其中一条路径。

反例：

```text
root candidate:
  4
 / \
1   2

subRoot:
  4
 /
1
```

如果候选节点右边多了 `2`，两棵树不完全相同。

### 错误 3：当前节点不匹配就直接返回 false

当前候选根失败，不代表左右子树里没有匹配位置。必须继续递归 `root.Left` 和 `root.Right`。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m * n)，m 为 root 节点数，n 为 subRoot 节点数，最坏每个候选点都比较一遍 |
| 空间 | O(h)，递归栈高度，最坏 O(m+n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为需要在一棵树中枚举候选根，并对每个候选根做一次两树相同判断。

2. 这个算法维护的不变量是什么？
   外层保证每个 root 节点都会被检查；内层保证结构和值完全相同才算匹配。

3. 每一步为什么不会漏答案？
   当前节点不匹配时继续搜索左右子树，因此 root 中所有可能的子树根都会被覆盖。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是把“部分路径匹配”当成“子树匹配”，会漏掉子树必须完整一致这个条件。
