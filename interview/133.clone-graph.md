# 133. 克隆图

## 题目信息

- 题号：LeetCode 133
- 标题：Clone Graph
- 难度：中等（Medium）
- 模式：图 / BFS / 哈希映射

## 一句话思路

用 map 记录“原节点 -> 克隆节点”，BFS 遍历原图；遇到邻居先克隆，再把克隆边接上。

## 为什么想到这个模式

图可能有环，不能简单递归复制，否则会无限循环。深拷贝需要保证：

- 每个原节点只克隆一次。
- 原图中同一个节点的多次引用，在克隆图里也指向同一个克隆节点。

所以必须用哈希表保存映射关系。

## 状态定义 / 数据结构

- `visited map[*Node]*Node`：原节点到克隆节点的映射。
- `queue`：待遍历的原图节点。

克隆边时：

```text
visited[cur].Neighbors append visited[neighbor]
```

## 核心不变量

任意时刻，`visited[x]` 是原节点 `x` 的唯一克隆节点。

BFS 遍历每条边时，如果邻居没克隆，先创建克隆节点并入队；无论是否新建，都把克隆邻居加入当前克隆节点的邻居列表。

## 手推全过程

原图：

```text
1 -- 2
|    |
4 -- 3
```

从 1 开始：

| 步骤 | 当前 | 新建克隆 | 接边 |
|------|------|----------|------|
| 1 | 1 | 2,4 | clone1 -> clone2, clone4 |
| 2 | 2 | 3 | clone2 -> clone1, clone3 |
| 3 | 4 | - | clone4 -> clone1, clone3 |
| 4 | 3 | - | clone3 -> clone2, clone4 |

map 保证 3、4 这类被多次遇到的节点不会重复克隆。

## Go 代码实现

```go
func cloneGraph(node *Node) *Node {
    if node == nil {
        return nil
    }
    visited := map[*Node]*Node{}
    queue := []*Node{node}
    visited[node] = &Node{Val: node.Val}
    for len(queue) > 0 {
        cur := queue[0]
        queue = queue[1:]
        for _, neighbor := range cur.Neighbors {
            if visited[neighbor] == nil {
                visited[neighbor] = &Node{Val: neighbor.Val}
                queue = append(queue, neighbor)
            }
            visited[cur].Neighbors = append(visited[cur].Neighbors, visited[neighbor])
        }
    }
    return visited[node]
}
```

## 错误写法与反例

### 错误 1：不用 map，遇到邻居就 new

同一个原节点会被克隆多次，克隆图结构不再共享同一个节点。

### 错误 2：只克隆节点，不克隆边

返回的图只有孤立节点，没有邻接关系。

### 错误 3：环图中递归不标记 visited

会在环里无限递归或重复构造。

## 四问复盘

1. 这题为什么属于这个模式？
   因为需要遍历图并处理环，必须用 visited 映射保证每个节点只克隆一次。

2. 这个算法维护的不变量是什么？
   `visited` 中每个原节点对应唯一克隆节点，克隆边始终连接这些唯一克隆节点。

3. 每一步为什么不会漏答案？
   BFS 会遍历所有可达节点和边；每条原边都会追加到对应克隆节点的邻居列表。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是重复 new 邻居，会让同一个原节点在克隆图里变成多个节点。
