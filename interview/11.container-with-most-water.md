# 11. 装水容器（Container With Most Water）

## 题目信息

- 题号：LeetCode 11
- 标题：Container With Most Water
- 难度：中等（Medium）
- 模式：双指针 / 两端收敛

## 一句话思路

面积由“宽度 × 短板高度”决定；宽度每次都会变小，所以只有移动短板，才可能换来更高的有效高度。

## 为什么想到这个模式

题目要从数组中选两个位置 `i < j`，最大化：

```text
area = min(height[i], height[j]) * (j - i)
```

暴力枚举所有 `(i, j)` 是 O(n^2)。这里的关键结构是：两个边界之间的宽度越大越有优势，而高度由较短的一边决定。先从最宽的左右两端开始，再逐步收缩，能把“宽度变小”这个必然损失转化为“尝试寻找更高短板”的过程。

这类题有明显的双指针信号：

- 答案由两个位置共同决定。
- 两端距离参与计算。
- 每一步可以证明排除一批不可能更优的组合。

## 状态定义 / 数据结构

- `left`：当前左边界。
- `right`：当前右边界。
- `best`：目前见过的最大面积。
- 当前面积：`min(height[left], height[right]) * (right - left)`。

不需要额外数据结构，空间 O(1)。

## 核心不变量

每一轮循环开始时，`best` 已经记录所有被检查过的边界组合中的最大面积。

更重要的不变量是排除逻辑：

- 如果 `height[left] <= height[right]`，那么所有以当前 `left` 为左边界、右边界在 `left+1...right` 范围内的组合，都不可能超过当前面积。
- 因为这些组合的宽度更小，高度最多还是 `height[left]`，面积不会更大。
- 所以移动 `left` 不会漏掉最优解。

右边较短时同理，移动 `right` 不会漏答案。

## 手推全过程

以 `height = [1,8,6,2,5,4,8,3,7]` 为例：

| left | right | 高度 | 宽度 | 面积 | 决策 |
|------|-------|------|------|------|------|
| 0 | 8 | min(1,7)=1 | 8 | 8 | 左边是短板，移动 left |
| 1 | 8 | min(8,7)=7 | 7 | 49 | 右边是短板，移动 right |
| 1 | 7 | min(8,3)=3 | 6 | 18 | 右边是短板，移动 right |
| 1 | 6 | min(8,8)=8 | 5 | 40 | 相等，移动任意一边 |
| 1 | 5 | min(8,4)=4 | 4 | 16 | 右边是短板，移动 right |
| 1 | 4 | min(8,5)=5 | 3 | 15 | 右边是短板，移动 right |
| 1 | 3 | min(8,2)=2 | 2 | 4 | 右边是短板，移动 right |
| 1 | 2 | min(8,6)=6 | 1 | 6 | 右边是短板，移动 right |

最大面积是 `49`，来自下标 `1` 和 `8`。

关键不是“遍历所有组合”，而是每次都安全排除一整批以短板为边界的组合。

## Go 代码实现

```go
func maxArea(height []int) int {
    left, right := 0, len(height)-1
    best := 0

    for left < right {
        width := right - left
        h := height[left]
        if height[right] < h {
            h = height[right]
        }
        area := width * h
        if area > best {
            best = area
        }

        if height[left] < height[right] {
            left++
        } else {
            right--
        }
    }
    return best
}
```

## 错误写法与反例

### 错误 1：移动长板

```go
if height[left] < height[right] {
    right--
} else {
    left++
}
```

反例：`[1,8,6,2,5,4,8,3,7]`

初始左右面积是 `8`，如果移动长板 `right`，左边短板仍是 `1`，宽度还变小，连续得到的面积只会被短板限制住，很容易错过 `8` 和 `7` 组成的面积 `49`。

### 错误 2：面积高度取 `max`

```go
area := max(height[left], height[right]) * (right - left)
```

容器能装多少水由短板决定。反例 `[1,100]`，真实面积是 `1`，错误写法会算成 `100`。

### 错误 3：循环写成 `left <= right`

```go
for left <= right {
    ...
}
```

当 `left == right` 时，宽度为 `0`，不是合法容器。虽然通常不会改变最大值，但会增加无意义状态，也容易和指针移动逻辑组合出越界风险。

## 四问复盘

1. 这题为什么属于这个模式？
   因为答案由左右两个位置共同决定，且可以从最大宽度开始两端收敛，每次根据短板安全排除一批组合。

2. 这个算法维护的不变量是什么？
   `best` 保存已检查组合的最大面积；被移动的短板对应的所有剩余组合都不可能比当前组合更优。

3. 每一步为什么不会漏答案？
   移动短板时，宽度只会变小，而高度上限是当前短板高度，所以以这个短板为边界的其它组合都不会产生更大面积。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是移动长板。移动长板无法突破当前短板限制，只会让宽度变小，可能漏掉真正的大面积组合。
