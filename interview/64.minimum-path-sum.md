# 64. 最小路径和

## 题目信息

- 题号：LeetCode 64
- 标题：Minimum Path Sum
- 难度：中等（Medium）
- 模式：动态规划 / 网格 DP

## 一句话思路

到达每个格子的最小路径和等于当前权重加上上方和左方的较小值。

## 为什么想到这个模式

路径只能向右或向下，所以到达 `(i,j)` 的最后一步只能来自：

- `(i-1,j)`
- `(i,j-1)`

这是典型网格 DP。题目要求最小和，所以转移取最小值。

## 状态定义 / 数据结构

可以原地修改 `grid`：

```text
grid[i][j] = 到达 (i,j) 的最小路径和
```

转移：

```text
grid[i][j] += min(grid[i-1][j], grid[i][j-1])
```

## 核心不变量

处理到某个格子时，它上方和左方已经变成各自的最小路径和。

第一行只能从左边来，第一列只能从上边来，所以要先初始化边界。

## 手推全过程

```text
grid =
1 3 1
1 5 1
4 2 1
```

初始化：

```text
第一行: 1 4 5
第一列: 1 2 6
```

内部转移：

| 位置 | 计算 | 值 |
|------|------|----|
| (1,1) | 5 + min(4,2) | 7 |
| (1,2) | 1 + min(5,7) | 6 |
| (2,1) | 2 + min(7,6) | 8 |
| (2,2) | 1 + min(6,8) | 7 |

答案是 `7`。

## Go 代码实现

```go
func minPathSum(grid [][]int) int {
    m, n := len(grid), len(grid[0])

    for j := 1; j < n; j++ {
        grid[0][j] += grid[0][j-1]
    }
    for i := 1; i < m; i++ {
        grid[i][0] += grid[i-1][0]
    }

    for i := 1; i < m; i++ {
        for j := 1; j < n; j++ {
            grid[i][j] += min(grid[i-1][j], grid[i][j-1])
        }
    }
    return grid[m-1][n-1]
}
```

## 错误写法与反例

### 错误 1：第一行/列不初始化

边界格子只有一个来源，若直接套通用转移会越界或用到错误值。

### 错误 2：取 max

本题是最小路径和，取 max 会变成最大路径和。

### 错误 3：不允许原地修改时仍修改输入

LeetCode 可以原地改；工程场景若输入需要保留，应使用额外 dp。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(mn) |
| 空间 | O(1)，原地修改 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为每个格子的最优值只依赖上方和左方两个已解决子问题。

2. 这个算法维护的不变量是什么？
   已处理格子的值已经变成从起点到该格子的最小路径和。

3. 每一步为什么不会漏答案？
   到达当前格的最后一步只能来自上或左，取二者较小值覆盖所有合法路径。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是第一行和第一列，它们只有一个来源，不能套通用转移。
