# 70. 爬楼梯

## 题目信息

- 题号：LeetCode 70
- 标题：Climbing Stairs
- 难度：简单（Easy）
- 模式：动态规划 / 斐波那契

## 一句话思路

到第 `i` 阶的最后一步只能来自 `i-1` 或 `i-2`，所以 `dp[i] = dp[i-1] + dp[i-2]`。

## 为什么想到这个模式

题目问“有多少种方法”，且当前台阶的方案数依赖前面台阶：

- 从第 `i-1` 阶走 1 步到 `i`。
- 从第 `i-2` 阶走 2 步到 `i`。

这两个来源互不重叠，覆盖所有可能最后一步，因此是最基础的一维 DP。

## 状态定义 / 数据结构

状态：

```text
dp[i] = 爬到第 i 阶的方法数
```

转移：

```text
dp[i] = dp[i-1] + dp[i-2]
```

初始化：

```text
dp[1] = 1
dp[2] = 2
```

空间优化后，只保留：

- `prev2`：`dp[i-2]`
- `prev1`：`dp[i-1]`

## 核心不变量

循环处理到 `i` 前：

- `prev2` 等于 `dp[i-2]`。
- `prev1` 等于 `dp[i-1]`。

计算 `curr = prev1 + prev2` 后，滚动更新为下一轮所需的两个状态。

## 手推全过程

以 `n = 5` 为例：

| i | 方案数 | 来源 |
|---|--------|------|
| 1 | 1 | `1` |
| 2 | 2 | `1+1`, `2` |
| 3 | 3 | `dp[2] + dp[1] = 2 + 1` |
| 4 | 5 | `dp[3] + dp[2] = 3 + 2` |
| 5 | 8 | `dp[4] + dp[3] = 5 + 3` |

答案是 `8`。

## Go 代码实现

```go
func climbStairs(n int) int {
    if n <= 2 {
        return n
    }
    prev2, prev1 := 1, 2
    for i := 3; i <= n; i++ {
        curr := prev1 + prev2
        prev2, prev1 = prev1, curr
    }
    return prev1
}
```

## 错误写法与反例

### 错误 1：把 `dp[2]` 初始化成 1

```go
dp[1], dp[2] := 1, 1
```

反例：`n = 2`，正确答案是 2：`1+1` 和 `2`。

### 错误 2：循环从 2 开始重复计算

```go
for i := 2; i <= n; i++ {
    curr := prev1 + prev2
}
```

如果 `prev2, prev1` 已经表示 `dp[1], dp[2]`，循环应从 3 开始。否则会把 `dp[2]` 覆盖成错误值。

### 错误 3：递归暴力不加缓存

暴力递归会重复计算大量子问题，时间接近 O(2^n)，大输入会超时。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为到第 `i` 阶的方法数由 `i-1` 和 `i-2` 两个更小状态组成，且子问题重复。

2. 这个算法维护的不变量是什么？
   循环中 `prev2` 和 `prev1` 分别保存当前要计算位置的前两个 DP 状态。

3. 每一步为什么不会漏答案？
   任意到第 `i` 阶的路径最后一步只能是 1 步或 2 步，这两类覆盖所有情况且互不重叠。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是 `dp[2] = 2`。写成 1 会从最早状态开始污染后续所有答案。
