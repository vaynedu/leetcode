# 32. 最长有效括号

## 题目
给你一个只含 `(` 和 `)` 的字符串，求最长有效括号子串的长度。

有效括号：要么是 `()`，要么是嵌套的有效括号 `(()xx)`。

## 核心思想

**栈底保存「最后一个未匹配的右括号下标」**

- 初始化：`stack = [-1]`（哨兵）
- 遇到 `(` → 入栈
- 遇到 `)` → 弹出栈顶（匹配左括号）
  - 弹出后栈空 → 用当前下标入栈（更新"最后未匹配位置"）
  - 弹出后栈非空 → `maxLen = max(maxLen, i - stack.top())`

### 为什么需要哨兵？

没有哨兵时，全部匹配的字符串最后一个右括号无法计算长度（没有参照物）。哨兵 `-1` 让长度计算统一为 `i - stack.top()`。

## 代码实现

```go
func longestValidParentheses(s string) int {
    if len(s) == 0 {
        return 0
    }
    maxLen := 0
    stack := []int{-1} // 哨兵

    for i := 0; i < len(s); i++ {
        if s[i] == '(' {
            stack = append(stack, i) // 存左括号下标
        } else {
            stack = append(stack[:len(stack)-1]) // 弹出栈顶
            if len(stack) == 0 {
                // 栈空：没有匹配的左括号，当前右括号成为"最后未匹配"
                stack = append(stack, i)
            } else {
                // 有效长度 = 当前下标 - 栈顶下标
                maxLen = max(maxLen, i-stack[len(stack)-1])
            }
        }
    }
    return maxLen
}
```

## trace — s = ")()())"

| Step | i | char | 栈状态 | maxLen |
|------|---|------|--------|--------|
| 1 | 0 | `)` | [-1,0] | 0 |
| 2 | 1 | `(` | [-1,0,1] | 0 |
| 3 | 2 | `)` | [-1,0] | 2 |
| 4 | 3 | `(` | [-1,0,3] | 2 |
| 5 | 4 | `)` | [-1,0] | 4 |
| 6 | 5 | `)` | [5] | 4 |

答案：4（子串 `()( )` 索引 1–4）

## 复杂度

| 指标 | 值 |
|------|-----|
| 时间 | O(n) — 每个字符最多入栈出栈各一次 |
| 空间 | O(n) — 最坏情况全是左括号 |

## 面试追问

**Q：有没有 O(1) 空间的解法？**
A：有，双向扫描。从左到右扫描时统计 `(` 和 `)` 数量，相等时更新长度；右括号多时重置。从右到左同理。适用于特殊场景，但栈解法更通用。

**Q：为什么哨兵是 -1 而不是 0？**
A：哨兵的作用是提供"参照下标"。第一个有效子串从下标 0 开始，长度 = i - (-1) = i + 1，正好等于第一个右括号出现时的索引。`-1` 让边界计算统一。
