# 32. 最长有效括号

## 题目信息

- 题号：LeetCode 32
- 标题：Longest Valid Parentheses
- 难度：困难（Hard）
- 模式：栈 / 下标哨兵

## 一句话思路

栈里存下标，栈底保存“最后一个未匹配右括号”的位置；遇到右括号后用当前下标减栈顶计算有效长度。

## 为什么想到这个模式

括号匹配天然有“最近未匹配”的结构，适合用栈。这里不只要判断是否合法，还要计算最长长度，所以栈里必须存下标。

关键是维护一个长度计算基准：

- 栈中保存未匹配的左括号下标。
- 当右括号无法匹配时，它成为新的非法边界。
- 栈底哨兵 `-1` 表示字符串开头前的非法边界。

## 状态定义 / 数据结构

- `stack`：下标栈。
- `stack[0]` 或栈底元素：当前有效段左侧的非法边界。
- `maxLen`：目前最长有效括号长度。

规则：

- 遇到 `(`：下标入栈。
- 遇到 `)`：先弹出一个下标尝试匹配。
- 弹出后栈空：当前 `)` 无法匹配，作为新的非法边界入栈。
- 弹出后栈非空：当前有效长度是 `i - stack.top()`。

## 核心不变量

任意时刻，栈顶到栈底表示当前还不能被确认合并的边界：

- 栈中除底部非法边界外，是尚未匹配的左括号下标。
- 栈底是当前连续有效区间左侧最近的非法位置。
- 当遇到可匹配的 `)` 后，`i - stack.top()` 就是以 `i` 结尾的最长有效括号长度。

哨兵 `-1` 让从下标 0 开始的有效串也能统一计算长度。

## 手推全过程

以 `s = ")()())"` 为例：

| i | char | 操作 | stack | maxLen |
|---|------|------|-------|--------|
| - | - | 初始化 | `[-1]` | 0 |
| 0 | `)` | 弹空，0 成为非法边界 | `[0]` | 0 |
| 1 | `(` | 入栈 | `[0,1]` | 0 |
| 2 | `)` | 匹配 1，长度 `2-0=2` | `[0]` | 2 |
| 3 | `(` | 入栈 | `[0,3]` | 2 |
| 4 | `)` | 匹配 3，长度 `4-0=4` | `[0]` | 4 |
| 5 | `)` | 弹空，5 成为非法边界 | `[5]` | 4 |

答案是 `4`。

## Go 代码实现

```go
func longestValidParentheses(s string) int {
    if len(s) == 0 {
        return 0
    }
    maxLen := 0
    stack := []int{-1}

    for i := 0; i < len(s); i++ {
        if s[i] == '(' {
            stack = append(stack, i)
        } else {
            stack = stack[:len(stack)-1]
            if len(stack) == 0 {
                stack = append(stack, i)
            } else {
                maxLen = max(maxLen, i-stack[len(stack)-1])
            }
        }
    }
    return maxLen
}
```

## 错误写法与反例

### 错误 1：没有哨兵

```go
stack := []int{}
```

反例：`"()"`。匹配后栈为空，没有基准下标计算长度。哨兵 `-1` 让长度变成 `1 - (-1) = 2`。

### 错误 2：栈空时不更新非法边界

```go
if len(stack) == 0 {
    continue
}
```

反例：`")()"`。第一个 `)` 应作为非法边界 0，否则后面 `"()"` 的长度计算会错。

### 错误 3：栈里存字符而不是下标

只存字符可以判断匹配，但无法计算连续有效子串长度。最长长度需要边界下标。

## 四问复盘

1. 这题为什么属于这个模式？
   因为括号匹配依赖最近未匹配的左括号，同时最长长度需要下标边界。

2. 这个算法维护的不变量是什么？
   栈底是最近非法边界，栈中其它元素是尚未匹配的左括号下标。

3. 每一步为什么不会漏答案？
   每个 `)` 都会尝试结算以当前位置结尾的最长有效段，所有可能结尾都会被检查。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是哨兵 `-1` 和栈空后更新非法边界。少了它们会算错从开头开始或非法右括号之后的有效长度。
