# 437. 路径总和 III

## 题目信息

- 题号：LeetCode 437
- 标题：Path Sum III
- 难度：中等（Medium）
- 模式：二叉树 DFS / 前缀和 / 回溯

## 一句话思路

在根到当前节点的路径上维护前缀和计数；当前前缀和为 `currSum` 时，之前出现过多少个 `currSum - target`，就有多少条以当前节点结尾的合法路径。

## 为什么想到这个模式

题目要求路径：

- 可以从任意节点开始。
- 可以在任意后代节点结束。
- 方向必须向下。

如果暴力以每个节点为起点再向下 DFS，最坏会到 O(n^2)。因为路径仍然都在“根到当前节点”的一条祖先链上，可以借用数组前缀和思想：

```text
某段路径和 = 当前前缀和 - 之前某个前缀和
```

要让这段路径和等于 `target`：

```text
之前前缀和 = currSum - target
```

所以用 map 记录当前祖先链上每个前缀和出现次数。

## 状态定义 / 数据结构

- `currSum`：从根走到当前节点的前缀和。
- `prefix[sum]`：当前递归路径上，前缀和 `sum` 出现过多少次。
- `prefix[0] = 1`：允许从根节点开始的路径被统计。

递归含义：

```text
helper(node, currSum, target, prefix)
= 在当前祖先前缀和状态下，统计以 node 子树中节点作为终点的合法路径数
```

## 核心不变量

进入某个节点时：

- `prefix` 只包含从根到当前节点父节点这条路径上的前缀和。
- `currSum` 是父节点为止的路径和。

处理当前节点后：

- 新的 `currSum += node.Val`。
- `prefix[currSum-target]` 就是以当前节点为终点的合法路径数量。
- 递归左右子树前，把当前 `currSum` 加入 `prefix`。
- 离开当前节点前，必须执行 `prefix[currSum]--`，撤销当前节点对兄弟分支的影响。

## 手推全过程

标准样例：

```text
        10
       /  \
      5   -3
     / \    \
    3   2    11
   / \   \
  3  -2   1

target = 8
```

合法路径有 3 条：

```text
5 -> 3
5 -> 2 -> 1
-3 -> 11
```

以路径 `10 -> 5 -> 3` 为例：

```text
到 10：currSum=10，查 prefix[2]，没有
到 5： currSum=15，查 prefix[7]，没有
到 3： currSum=18，查 prefix[10]，有 1 次
```

`prefix[10]` 对应祖先节点 `10` 后的位置，所以 `18 - 10 = 8`，路径就是 `5 -> 3`。

## Go 代码实现

```go
func PathSumIII(root *TreeNode, target int) int {
    prefix := map[int]int{0: 1}
    return pathSumIIIHelper(root, 0, target, prefix)
}

func pathSumIIIHelper(node *TreeNode, currSum, target int, prefix map[int]int) int {
    if node == nil {
        return 0
    }
    currSum += node.Val
    count := prefix[currSum-target]
    prefix[currSum]++

    count += pathSumIIIHelper(node.Left, currSum, target, prefix)
    count += pathSumIIIHelper(node.Right, currSum, target, prefix)

    prefix[currSum]--
    return count
}
```

## 错误写法与反例

### 错误 1：忘记初始化 `prefix[0] = 1`

如果没有这个初始化，从根节点开始的路径无法被统计。

反例：

```text
root = [8], target = 8
```

当前前缀和是 8，需要查 `prefix[0]`。如果 `prefix[0]` 不存在，会错误返回 0。

### 错误 2：忘记回溯 `prefix[currSum]--`

`prefix` 只应该表示当前祖先链。离开节点后不撤销，会把左子树路径污染到右子树，统计出不存在的跨分支路径。

### 错误 3：只统计从根开始的路径

题目允许路径从任意节点开始。只做普通根路径累加会漏掉 `5 -> 3`、`5 -> 2 -> 1`、`-3 -> 11` 这类不一定从根开始的路径。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，每个节点访问一次 |
| 空间 | O(h)，前缀和 map 和递归栈都跟当前路径高度相关，最坏 O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为任意向下路径都可以表示为两段根路径前缀和的差，适合在树 DFS 中维护前缀和计数。

2. 这个算法维护的不变量是什么？
   `prefix` 始终只记录当前根到父节点路径上的前缀和次数，`currSum` 是当前路径前缀和。

3. 每一步为什么不会漏答案？
   每个节点都会作为路径终点统计一次，`prefix[currSum-target]` 覆盖了所有可能的祖先起点。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是忘记回溯 `prefix[currSum]--`，会把兄弟分支拼成不存在的路径，导致答案偏大。
