# 98. 验证二叉搜索树

## 题目信息

- 题号：LeetCode 98
- 标题：Validate Binary Search Tree
- 难度：中等（Medium）
- 模式：二叉树 DFS / 中序遍历 / BST

## 一句话思路

BST 的中序遍历必须得到严格递增序列；遍历时记录前一个访问值，如果当前值小于等于前驱，就不是合法 BST。

## 为什么想到这个模式

二叉搜索树的定义不是“左孩子 < 当前节点 < 右孩子”这么局部，而是：

```text
左子树所有节点 < 当前节点 < 右子树所有节点
```

这种全局约束可以用两种方式验证：

- 递归传上下界：每个节点必须落在 `(low, high)` 内。
- 中序遍历：合法 BST 的中序序列严格递增。

当前实现采用中序遍历，因为它直接把 BST 的结构约束转成序列有序性。

## 状态定义 / 数据结构

- `node`：当前遍历到的节点。
- `prev`：中序遍历中，当前节点的前一个节点值。
- `inorder(node)`：判断以 `node` 为根的子树，中序遍历是否仍保持严格递增。

遍历顺序：

```text
左子树 → 当前节点 → 右子树
```

## 核心不变量

任意时刻：

- 已经访问过的节点，按中序顺序形成严格递增序列。
- `prev` 始终是当前节点在中序序列里的前驱值。
- 如果 `prev >= node.Val`，说明序列不再严格递增，BST 立即失效。

注意必须是严格递增，重复值也不合法。

## 手推全过程

合法例子：`[2,1,3]`

```text
    2
   / \
  1   3
```

中序遍历：

```text
1 -> 2 -> 3
```

严格递增，所以返回 `true`。

非法例子：`[5,1,4,null,null,3,6]`

```text
    5
   / \
  1   4
     / \
    3   6
```

中序遍历：

```text
1 -> 5 -> 3 -> 4 -> 6
```

访问到 `3` 时，前驱是 `5`，`5 >= 3`，所以返回 `false`。

## Go 代码实现

```go
func isValidBST(root *TreeNode) bool {
    var prev *int
    var inorder func(node *TreeNode) bool
    inorder = func(node *TreeNode) bool {
        if node == nil {
            return true
        }
        if !inorder(node.Left) {
            return false
        }
        if prev != nil && *prev >= node.Val {
            return false
        }
        prev = &node.Val
        return inorder(node.Right)
    }
    return inorder(root)
}
```

## 错误写法与反例

### 错误 1：只比较直接左右孩子

```go
if root.Left != nil && root.Left.Val >= root.Val { return false }
if root.Right != nil && root.Right.Val <= root.Val { return false }
```

反例：`[5,1,6,null,null,4,7]`

节点 `4` 是 `5` 的右子树节点，必须大于 `5`，但它只和父节点 `6` 比较时会被误判为合法。

### 错误 2：允许相等

```go
if prev != nil && *prev > node.Val { return false }
```

BST 题目要求严格小于和严格大于，重复值不合法。应该用 `>=` 判断失败。

### 错误 3：用局部 min/max 但不向下传递

只看当前节点和左右孩子，无法表达“整棵左子树都小于根、整棵右子树都大于根”的全局约束。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，每个节点访问一次 |
| 空间 | O(h)，递归栈高度，最坏 O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为 BST 的全局有序约束可以通过中序遍历转成严格递增序列判断。

2. 这个算法维护的不变量是什么？
   已访问的中序序列严格递增，`prev` 是当前节点的中序前驱。

3. 每一步为什么不会漏答案？
   中序遍历会按 BST 应有的升序访问所有节点，只要任何位置不递增，就能立即发现违反 BST 的节点。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是只比较直接孩子，会漏掉右子树里小于根、左子树里大于根的深层非法节点。
