# 63. 不同路径 II

## 题目信息

- 题号：LeetCode 63
- 标题：Unique Paths II
- 难度：中等（Medium）
- 模式：动态规划 / 网格 DP

## 一句话思路

和 62 一样用一维 `dp[j]` 表示当前格路径数；遇到障碍时该格不可达，直接置 0。

## 为什么想到这个模式

路径仍然只能从上方或左方来，所以基础转移还是：

```text
dp[i][j] = dp[i-1][j] + dp[i][j-1]
```

障碍物只是让某个格子路径数变成 0，并阻断后续经过它的路径。

## 状态定义 / 数据结构

- `dp[j]`：当前行第 `j` 列的路径数。
- 更新前的 `dp[j]`：上方路径数。
- 更新后的 `dp[j-1]`：左方路径数。

障碍处理：

```text
obstacleGrid[i][j] == 1 => dp[j] = 0
```

## 核心不变量

处理到 `(i,j)` 时，`dp[j]` 最终表示到达当前格的路径数。

- 若当前格是障碍，路径数必须为 0。
- 若不是障碍，路径数来自上方旧 `dp[j]` 和左方新 `dp[j-1]`。

第一行也用同一逻辑初始化：遇到障碍后，右侧只能从左边来，因此会继续保持 0。

## 手推全过程

```text
obstacleGrid =
0 0 0
0 1 0
0 0 0
```

| 行 | dp |
|----|----|
| 初始化第 0 行 | `[1,1,1]` |
| 第 1 行遇障碍 | `[1,0,1]` |
| 第 2 行 | `[1,1,2]` |

答案是 `2`。

## Go 代码实现

```go
func uniquePathsWithObstacles(obstacleGrid [][]int) int {
    m := len(obstacleGrid)
    n := len(obstacleGrid[0])
    dp := make([]int, n)

    for j := 0; j < n; j++ {
        if obstacleGrid[0][j] == 1 {
            dp[j] = 0
        } else if j == 0 {
            dp[j] = 1
        } else {
            dp[j] = dp[j-1]
        }
    }

    for i := 1; i < m; i++ {
        for j := 0; j < n; j++ {
            if obstacleGrid[i][j] == 1 {
                dp[j] = 0
            } else if j > 0 {
                dp[j] += dp[j-1]
            }
        }
    }
    return dp[n-1]
}
```

## 错误写法与反例

### 错误 1：障碍处不置 0

路径会错误穿过障碍，答案偏大。

### 错误 2：第一行遇障碍后仍初始化为 1

反例：`[[0,1,0]]`，障碍右侧不可达，答案应为 0。

### 错误 3：第一列没有特殊处理

`j=0` 没有左方，只能保留来自上方的 `dp[0]`，遇障碍则清零。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(mn) |
| 空间 | O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为到达每个格子的路径数仍由上方和左方两个子问题组成，只是障碍格路径数为 0。

2. 这个算法维护的不变量是什么？
   `dp[j]` 在处理后表示当前行第 `j` 列的路径数。

3. 每一步为什么不会漏答案？
   所有合法路径到当前格的最后一步只能来自上方或左方，障碍格被置 0 后自然阻断经过它的路径。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是第一行/第一列障碍，处理错会让障碍后方格子错误可达。
