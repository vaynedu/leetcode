# 76. 最小覆盖子串（Minimum Window Substring）

## 题目信息
- 题号：LeetCode 76
- 标题：Minimum Window Substring
- 难度：困难（Hard）

---

## 一句话思路
**滑动窗口 + 双 HashMap**：右指针扩展加入窗口，左指针收缩寻找最小覆盖，need/window + valid 计数判断是否覆盖。

---

## 核心思路

### 暴力解
枚举所有子串起点和终点，检查是否覆盖 t → O(N²×M)

### 滑动窗口优化（O(N)）
- R 右移扩展窗口，加入字符
- 当窗口覆盖 t 时（valid == len(need)），收缩 L 寻找最小
- 全程只遍历一次

### 关键机制
```
need map：t 中每个字符需要的数量
window map：窗口中每个字符的数量
valid：满足条件的字符种类数
```

---

## 关键点/考点
```
时间：O(N)，R 最多 N 次，L 也最多 N 次
空间：O(字符集大小)，最多 128（ASCII）

追问：
- 为什么 valid 用种类数？→ 避免每次遍历整个 map
- 如何处理 t 有重复字符？→ need['A']=2 需要两个 A
- 大小写敏感？→ 是，'A' 和 'a' 不同
```

---

## 易错点
```
1. valid 条件：window[c] >= need[c]，不是简单 >=
2. 收缩时判断：收缩后 window[c] < need[c]，valid--
3. 更新答案时机：valid == len(need) 时
4. 结果记录：找到后继续收缩，寻找更短窗口
```

---

## 记忆口诀
```
右扩左缩滑动窗，
need window 双映射。
valid 计数判覆盖，
等于 len 就收缩。
窗口变小更新答案，
直至不能缩为止。
```

---

## Go 代码实现

```go
func minWindow(s string, t string) string {
    need := make(map[rune]int)
    window := make(map[rune]int)

    // 初始化 need
    for _, c := range t {
        need[c]++
    }

    L, R := 0, 0
    valid := 0
    minLen := len(s) + 1
    minStr := ""

    for R < len(s) {
        c := rune(s[R])
        if _, ok := need[c]; ok {
            window[c]++
            if window[c] >= need[c] {
                valid++
            }
        }
        R++

        // 收缩窗口
        for valid == len(need) {
            // 更新最小窗口
            if R-L < minLen {
                minLen = R - L
                minStr = s[L:R]
            }

            d := rune(s[L])
            if _, ok := need[d]; ok {
                if window[d] >= need[d] {
                    valid--
                }
                window[d]--
            }
            L++
        }
    }

    return minStr
}
```

## 测试用例

### 正常情况
- `s="ADOBECODEBANC", t="ABC"` → "BANC"
- `s="a", t="a"` → "a"
- `s="a", t="aa"` → ""（不存在）

### 边界情况
- `s="", t="A"` → ""（s 为空）
- `s="A", t=""` → ""（t 为空）

### 极端情况
- `s="A"×10000 + "B", t="AB"` → "AB"
- `s="aaaaaaaaaaa...", t="aaa"` → 最短 3 个 a
