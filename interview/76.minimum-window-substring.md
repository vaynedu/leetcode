# 76. 最小覆盖子串（Minimum Window Substring）

## 题目信息

- 题号：LeetCode 76
- 难度：困难
- 模式：滑动窗口 / 双 HashMap
- 可视化：`doc/algorithm-visualizations/76.minimum-window-substring.html`

## 一句话思路

右指针扩张直到窗口覆盖 `t`，然后左指针尽量收缩，收缩过程中记录最短覆盖窗口。

## 为什么想到这个模式

题目要求 `s` 中包含 `t` 所有字符的最短子串。子串是连续区间，且窗口加入/移除一个字符后可以更新计数，所以优先使用滑动窗口。

和第 3 题不同，这里不是维护“无重复”，而是维护“是否覆盖目标计数”。

## 状态定义 / 数据结构

- `need[c]`：目标串 `t` 中字符 `c` 需要的次数。
- `window[c]`：当前窗口中字符 `c` 的次数。
- `valid`：有多少种字符已经满足 `window[c] == need[c]`。
- `[left,right)`：当前窗口，右边界开区间。
- `start, minLen`：当前最短覆盖窗口。

## 核心不变量

当 `valid == len(need)` 时，当前窗口已经覆盖 `t`。

收缩左边界时，只要移除某个关键字符后 `window[d] < need[d]`，窗口就不再覆盖 `t`，必须停止收缩并继续右扩。

`valid` 统计的是“满足要求的字符种类数”，不是字符总数。

## 手推全过程

以 `s = "ADOBECODEBANC", t = "ABC"` 为例：

| 阶段 | 窗口 | 状态 | 动作 |
|------|------|------|------|
| 右扩到 C | `ADOBEC` | A/B/C 都满足 | 记录候选，开始收缩 |
| 左移去 A | `DOBEC` | A 不满足 | 停止收缩，继续右扩 |
| 右扩到 A | `DOBECODEBA` | A/B/C 满足 | 再次收缩 |
| 收缩到 C | `CODEBA` | 覆盖仍成立 | 继续收缩 |
| 去 C | `ODEBA` | C 不满足 | 停止 |
| 右扩到 C | `ODEBANC` | 覆盖成立 | 收缩得到 `BANC` |

最短窗口是 `"BANC"`。

## 错误写法与反例

错误写法：

```go
if window[c] >= need[c] {
    valid++
}
```

反例：`s = "AAAB", t = "AAB"`

`need[A]=2`。当窗口里第 3 个 `A` 进入时，`window[A] >= need[A]` 仍成立，但 `A` 这个字符种类已经被统计过一次，不能重复 `valid++`。

正确写法：

```go
if window[c] == need[c] {
    valid++
}
```

收缩时也要对称：

```go
if window[d] == need[d] {
    valid--
}
window[d]--
```

## Go 代码实现

```go
func minWindow(s string, t string) string {
    need := map[byte]int{}
    window := map[byte]int{}
    for i := 0; i < len(t); i++ {
        need[t[i]]++
    }

    left, right := 0, 0
    valid := 0
    start, minLen := 0, len(s)+1

    for right < len(s) {
        c := s[right]
        right++
        if _, ok := need[c]; ok {
            window[c]++
            if window[c] == need[c] {
                valid++
            }
        }

        for valid == len(need) {
            if right-left < minLen {
                start = left
                minLen = right - left
            }

            d := s[left]
            left++
            if _, ok := need[d]; ok {
                if window[d] == need[d] {
                    valid--
                }
                window[d]--
            }
        }
    }

    if minLen == len(s)+1 {
        return ""
    }
    return s[start : start+minLen]
}
```

## 复杂度

| 指标 | 值 |
|------|-----|
| 时间 | O(n + m)，左右指针都只前进 |
| 空间 | O(k)，k 为字符集大小 |

## 四问复盘

1. **为什么属于滑动窗口？** 目标是连续子串，窗口计数可随右扩左缩维护。
2. **维护的不变量是什么？** `valid == len(need)` 等价于窗口覆盖 `t`。
3. **为什么不会漏答案？** 每次窗口覆盖时都尽量收缩，得到以当前右边界结尾的最短覆盖窗口。
4. **最容易错的边界？** `valid` 只能在 `window[c] == need[c]` 的临界点增减，不能用 `>=` 重复统计。
