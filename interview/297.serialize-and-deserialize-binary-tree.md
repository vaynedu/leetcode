# 297. 二叉树的序列化与反序列化

## 题目信息

- 题号：LeetCode 297
- 标题：Serialize and Deserialize Binary Tree
- 难度：困难（Hard）
- 模式：二叉树 / 前序遍历 / 递归编码

## 一句话思路

前序遍历输出“根、左、右”，空节点用 `#` 占位；反序列化时按同样顺序递归消费 token。

## 为什么想到这个模式

二叉树只靠节点值序列无法唯一还原，必须记录空节点位置。前序遍历配合空节点标记后，每个节点的左右子树边界都能由递归自然确定。

编码规则：

```text
nil -> "#"
node -> node.Val + "," + serialize(left) + "," + serialize(right)
```

这样序列化和反序列化共享同一棵递归结构。

## 状态定义 / 数据结构

序列化：

- 输入：当前节点 `root`。
- 输出：当前子树的前序字符串。

反序列化：

- `nodes`：按逗号切分后的 token。
- `idx`：当前消费到的 token 位置。
- `build()`：从 `nodes[idx]` 开始构建一棵子树，并把 `idx` 向后推进。

## 核心不变量

反序列化时，`idx` 始终指向当前要构建子树的根 token。

- token 是 `#`：当前子树为空，消费一个 token 后返回 nil。
- token 是数字：创建节点，然后递归构建左子树，再递归构建右子树。

因为序列化时也按“根、左、右”输出，反序列化的消费顺序和生成顺序完全一致。

## 手推全过程

树：

```text
    1
   / \
  2   3
     / \
    4   5
```

前序带空节点序列：

```text
1,2,#,#,3,4,#,#,5,#,#
```

反序列化消费过程：

```text
1 -> 建根
  2 -> 建左子树
    # -> 2.left = nil
    # -> 2.right = nil
  3 -> 建右子树
    4 -> 3.left
      # #
    5 -> 3.right
      # #
```

每个 `#` 都是结构信息，不能省略。

## Go 代码实现

```go
func (c *Codec) serialize(root *TreeNode) string {
    if root == nil {
        return "#"
    }
    left := c.serialize(root.Left)
    right := c.serialize(root.Right)
    return strconv.Itoa(root.Val) + "," + left + "," + right
}

func (c *Codec) deserialize(data string) *TreeNode {
    if data == "" {
        return nil
    }
    nodes := strings.Split(data, ",")
    idx := 0
    var build func() *TreeNode
    build = func() *TreeNode {
        if idx >= len(nodes) {
            return nil
        }
        val := nodes[idx]
        idx++
        if val == "#" {
            return nil
        }
        v, _ := strconv.Atoi(val)
        node := &TreeNode{Val: v}
        node.Left = build()
        node.Right = build()
        return node
    }
    return build()
}
```

## 错误写法与反例

### 错误 1：不记录空节点

```text
1,2,3,4,5
```

没有 `#` 时无法区分不同结构。例如一条左链和一条右链可能有相同的前序值序列。

### 错误 2：反序列化时 `idx` 不共享

如果每次递归都复制或重置 `idx`，左右子树会重复消费同一段 token，树结构错误。`idx` 必须随递归全局推进。

### 错误 3：序列化和反序列化顺序不一致

序列化用前序，反序列化也必须按前序消费。若一边前序一边层序，无法正确还原。

## 四问复盘

1. 这题为什么属于这个模式？
   因为二叉树递归结构天然适合用遍历编码，前序加空节点标记能唯一表示树。

2. 这个算法维护的不变量是什么？
   反序列化时 `idx` 始终指向当前子树根 token，构建完该子树后正好移动到下一个兄弟/祖先位置。

3. 每一步为什么不会漏答案？
   每个真实节点和空节点都被序列化成 token，反序列化按同样顺序逐个消费，结构信息完整保留。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是省略空节点 `#`。没有空节点标记，很多不同形状的树会得到相同序列，无法唯一还原。
