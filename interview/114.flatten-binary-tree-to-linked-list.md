# 114. 二叉树展开为链表

## 题目信息

- 题号：LeetCode 114
- 标题：Flatten Binary Tree to Linked List
- 难度：中等（Medium）
- 模式：二叉树 DFS / 原地改指针

## 一句话思路

先递归把左右子树各自展开，再把左链表接到根的右边，原右链表接到新右链表末尾，最后把 `Left` 置空。

## 为什么想到这个模式

题目要求按前序遍历顺序展开：

```text
根 -> 左子树前序 -> 右子树前序
```

如果左右子树已经分别被展开成链表，那么当前节点只需要做一次指针拼接：

```text
root.Right = flattened(left)
left tail.Right = flattened(original right)
root.Left = nil
```

因为当前节点的操作依赖左右子树已经处理好，所以当前实现是递归处理子树后再拼接。

## 状态定义 / 数据结构

- `root.Left`：原左子树，展开后应成为 `root.Right` 的第一段。
- `root.Right`：原右子树，必须先保存，避免被覆盖后丢失。
- `cur`：左子树展开链表的尾节点，用来连接原右子树。

## 核心不变量

每次 `flatten(node)` 返回后：

- 以 `node` 为根的子树已经展开成只走 `Right` 的链表。
- 链表顺序等于该子树的前序遍历顺序。
- 该子树所有节点的 `Left` 都应为 `nil`。

拼接当前节点时，要保持：

```text
node -> 原左子树前序链表 -> 原右子树前序链表
```

## 手推全过程

例子：

```text
      1
     / \
    2   5
   / \   \
  3   4   6
```

左右子树先展开：

```text
左子树：2 -> 3 -> 4
右子树：5 -> 6
```

在根节点 `1` 拼接：

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6
```

所有节点 `Left` 都置空。

## Go 代码实现

```go
func flatten(root *TreeNode) {
    if root == nil {
        return
    }
    flatten(root.Left)
    flatten(root.Right)
    if root.Left == nil {
        return
    }
    right := root.Right
    root.Right = root.Left
    root.Left = nil
    cur := root.Right
    for cur.Right != nil {
        cur = cur.Right
    }
    cur.Right = right
}
```

## 错误写法与反例

### 错误 1：覆盖 `root.Right` 前不保存原右子树

```go
root.Right = root.Left
```

如果没有先保存 `right := root.Right`，原右子树会丢失。

反例：`[1,2,5]` 应展开为 `1 -> 2 -> 5`，错误写法可能只剩 `1 -> 2`。

### 错误 2：忘记把 `root.Left` 置空

题目要求展开成单向链表，所有节点都应通过 `Right` 串起来。如果左指针不清空，结构仍然不是合法链表。

### 错误 3：把原右子树接到 `root.Right` 而不是左链表尾部

原右子树必须排在完整左子树前序链表之后，否则会破坏前序顺序。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | 当前实现最坏 O(n^2)，因为每个节点可能向右扫描尾部 |
| 空间 | O(h)，递归栈高度，最坏 O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为目标顺序是前序遍历，但实现需要先处理好左右子树再做原地指针拼接。

2. 这个算法维护的不变量是什么？
   `flatten(node)` 返回后，`node` 子树被展开为前序顺序的右链表，且左指针为空。

3. 每一步为什么不会漏答案？
   每个节点都递归展开左右子树，再按“根、左、右”的前序顺序拼接，所有节点都会保留在右链表中。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是覆盖右指针前不保存原右子树，会直接丢掉整段原右子树。
