# 207. 课程表

## 题目信息

- 题号：LeetCode 207
- 标题：Course Schedule
- 难度：中等（Medium）
- 模式：图 / 拓扑排序

## 一句话思路

把课程依赖建成有向图；不断学习入度为 0 的课程，如果最终能处理所有课程，则无环。

## 为什么想到这个模式

`prerequisites[i] = [a, b]` 表示要学 `a` 必须先学 `b`，这就是有向依赖关系 `b -> a`。

能否完成所有课程，等价于这个有向图是否有环。拓扑排序正是判断依赖图能否线性排序的标准方法。

## 状态定义 / 数据结构

- `graph[b]`：学完 `b` 后可以解锁的课程列表。
- `inDegree[a]`：课程 `a` 还有多少前置依赖未满足。
- `queue`：当前所有入度为 0、可以学习的课程。
- `visited`：已经学习的课程数量。

## 核心不变量

队列中的课程始终是当前没有未满足前置依赖的课程。

每弹出一门课程：

- 表示这门课已学完。
- 它指向的后续课程入度减一。
- 某个后续课程入度降为 0，就加入队列。

如果有环，环内课程入度永远无法全部变成 0。

## 手推全过程

`numCourses = 4, prerequisites = [[1,0],[2,0],[3,1],[3,2]]`

边：

```text
0 -> 1
0 -> 2
1 -> 3
2 -> 3
```

| 步骤 | 队列 | 处理 | 入度变化 |
|------|------|------|----------|
| 初始 | `[0]` | - | `0:0,1:1,2:1,3:2` |
| 1 | `[0]` | 学 0 | 1 和 2 入度变 0 |
| 2 | `[1,2]` | 学 1 | 3 入度变 1 |
| 3 | `[2]` | 学 2 | 3 入度变 0 |
| 4 | `[3]` | 学 3 | 完成 |

`visited == 4`，返回 true。

## Go 代码实现

```go
func canFinish(numCourses int, prerequisites [][]int) bool {
    graph := make([][]int, numCourses)
    inDegree := make([]int, numCourses)
    for _, p := range prerequisites {
        a, b := p[0], p[1]
        graph[b] = append(graph[b], a)
        inDegree[a]++
    }

    queue := []int{}
    for i := 0; i < numCourses; i++ {
        if inDegree[i] == 0 {
            queue = append(queue, i)
        }
    }

    visited := 0
    for len(queue) > 0 {
        course := queue[0]
        queue = queue[1:]
        visited++
        for _, next := range graph[course] {
            inDegree[next]--
            if inDegree[next] == 0 {
                queue = append(queue, next)
            }
        }
    }
    return visited == numCourses
}
```

## 错误写法与反例

### 错误 1：边方向建反

`[a,b]` 表示 `b -> a`，不是 `a -> b`。建反会得到错误的入度和顺序。

### 错误 2：只要队列非空就返回 true

有些无环部分可以先学，但剩余课程可能在环里。必须检查 `visited == numCourses`。

### 错误 3：入度降到 0 后不入队

这样后续可学习课程不会被处理，`visited` 偏小，误判有环。

## 四问复盘

1. 这题为什么属于这个模式？
   因为课程依赖是有向图，能否学完等价于图是否存在环。

2. 这个算法维护的不变量是什么？
   队列中始终是当前入度为 0、没有未满足依赖的课程。

3. 每一步为什么不会漏答案？
   每学一门课都会释放它指向的所有后续课程，所有可被解锁的课程都会在入度归零时入队。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是边方向 `[a,b]`，必须建 `b -> a`；建反会破坏入度语义。
