# 154. 寻找旋转排序数组中的最小值 II

## 题目信息

- 题号：LeetCode 154
- 标题：Find Minimum in Rotated Sorted Array II
- 难度：困难（Hard）
- 模式：二分查找 / 旋转数组 / 重复元素

## 一句话思路

和 153 一样比较 `nums[mid]` 与 `nums[right]`；相等时无法判断最小值在哪边，只能安全收缩 `right--`。

## 为什么想到这个模式

数组仍然来自升序数组旋转，最小值仍是旋转断点。但本题允许重复元素，导致 `nums[mid] == nums[right]` 时无法判断右半是否有序，也无法直接排除一半。

例如：

```text
[1,1,1,0,1]  mid 和 right 都是 1，最小值在左侧
[1,0,1,1,1]  mid 和 right 都是 1，最小值在左侧/中间不同位置
```

因此仍用旋转数组二分，但重复时只能缩小一个边界，最坏 O(n)。

## 状态定义 / 数据结构

- `left`：候选区间左边界。
- `right`：候选区间右边界。
- `mid`：中点。
- 候选区间：闭区间 `[left, right]`。

目标：最小值始终在 `[left, right]` 内。

## 核心不变量

每轮循环后，最小值仍然留在 `[left, right]`。

比较规则：

- `nums[mid] > nums[right]`：最小值一定在 `mid+1...right`。
- `nums[mid] < nums[right]`：最小值在 `left...mid`，保留 `mid`。
- `nums[mid] == nums[right]`：无法判断，但可以丢弃 `right`。

为什么相等时能 `right--`？

- 如果 `nums[right]` 不是最小值，丢掉没问题。
- 如果 `nums[right]` 是最小值，因为 `nums[mid] == nums[right]`，区间内还有一个相同值 `nums[mid]`，不会丢掉最小值这个数值。

## 手推全过程

以 `nums = [2,2,2,0,1]` 为例：

| left | mid | right | 值 | 判断 | 决策 |
|------|-----|-------|----|------|------|
| 0 | 2 | 4 | 2,2,1 | `2 > 1` | 最小值在右半，`left = 3` |
| 3 | 3 | 4 | 0,0,1 | `0 < 1` | 最小值在左半含 mid，`right = 3` |

返回 `nums[3] = 0`。

重复退化例子：`[1,1,1,1,1]`。

每次 `nums[mid] == nums[right]`，只能 `right--`，最终收缩到一个位置，时间退化为 O(n)。

## Go 代码实现

```go
func FindMinII(nums []int) int {
    if len(nums) == 0 {
        return 0
    }
    left, right := 0, len(nums)-1
    for left < right {
        mid := left + (right-left)/2
        if nums[mid] > nums[right] {
            left = mid + 1
        } else if nums[mid] < nums[right] {
            right = mid
        } else {
            right--
        }
    }
    return nums[left]
}
```

## 错误写法与反例

### 错误 1：相等时直接 `right = mid`

```go
if nums[mid] == nums[right] {
    right = mid
}
```

反例：`[1,1,1,0,1]`。

`mid` 和 `right` 相等不代表最小值在左半含 `mid` 的有序判断成立。直接跳到 `mid` 可能排除包含最小值的右侧区间。

### 错误 2：相等时直接 `left = mid + 1`

```go
if nums[mid] == nums[right] {
    left = mid + 1
}
```

反例：`[1,0,1,1,1]`。

最小值可能在 `mid` 左侧，直接移动 `left` 会把答案排除。

### 错误 3：`nums[mid] < nums[right]` 时排除 `mid`

```go
right = mid - 1
```

和 153 一样，`mid` 可能就是最小值，不能排除。

| 空间 | O(1) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为旋转数组的最小值是断点，通常能通过 `mid` 和 `right` 的关系二分定位；重复值只是在相等时削弱了排除能力。

2. 这个算法维护的不变量是什么？
   最小值始终保留在 `[left, right]` 候选区间内。

3. 每一步为什么不会漏答案？
   大于/小于时能确定最小值所在半区；相等时只丢弃一个与 `mid` 等值的右端点，不会丢掉最小值这个数值。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是 `nums[mid] == nums[right]` 的处理。直接跳半区没有依据，可能把真正最小值排除；只能 `right--` 保守收缩。
