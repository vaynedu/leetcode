# 154. 寻找旋转排序数组中的最小值 II

## 题目

假设一个长度为 n 的升序数组在某个位置被旋转了。找出数组中的最小元素。数组可能包含重复元素。

## 难度

Hard

## 标签

数组 | 二分查找

## 示例

```
输入：[1,3,5]
输出：1

输入：[2,2,2,0,1]
输出：0
```

## 核心思路

与153题相同思路，但增加重复处理：

**标准判断：**
```
if nums[mid] > nums[right]  → 最小值在右半边，left = mid + 1
else if nums[mid] < nums[right] → 最小值在左半边（含mid），right = mid
```

**新增：重复时无法判断**
```
else nums[mid] == nums[right] → 无法判断，收缩 right--
```

## 关键点

- `right--` 而非 `left++`，因为最小值不会在 right 位置（right 本身可能是最小值，但 left 才是更小的那侧）
- 重复元素越多越接近 O(n)

## 复杂度

- 时间：O(n) 最坏
- 空间：O(1)
