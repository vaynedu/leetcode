# 120. 三角形最小路径和

## 题目信息

- 题号：LeetCode 120
- 标题：Triangle
- 难度：中等（Medium）
- 模式：动态规划 / 自底向上

## 一句话思路

从底向上压缩 DP：`dp[j] = min(dp[j], dp[j+1]) + triangle[i][j]`，最终 `dp[0]` 是答案。

## 为什么想到这个模式

每个位置只能走到下一行相邻两个位置。若从底向上看，当前位置的最小路径和只依赖下一行的两个相邻状态。

从底向上可以避免处理顶部到边界的特殊情况，也能用一维数组。

## 状态定义 / 数据结构

- `dp[j]`：处理到当前行时，从当前位置 `j` 走到底部的最小路径和。
- 初始化：`dp` 全 0，长度 `n+1`，让 `dp[j+1]` 不越界。

转移：

```text
dp[j] = min(dp[j], dp[j+1]) + triangle[i][j]
```

## 核心不变量

处理完第 `i` 行后，`dp[j]` 表示从 `triangle[i][j]` 到底部的最小路径和。

下一轮处理 `i-1` 行时，正好可以使用它下方两个位置 `dp[j]` 和 `dp[j+1]`。

## 手推全过程

```text
triangle =
[2]
[3,4]
[6,5,7]
[4,1,8,3]
```

自底向上：

| 行 | dp |
|----|----|
| `[4,1,8,3]` | `[4,1,8,3]` |
| `[6,5,7]` | `[7,6,10]` |
| `[3,4]` | `[9,10]` |
| `[2]` | `[11]` |

答案是 `11`。

## Go 代码实现

```go
func minimumTotal(triangle [][]int) int {
    n := len(triangle)
    dp := make([]int, n+1)

    for i := n - 1; i >= 0; i-- {
        for j := 0; j <= i; j++ {
            dp[j] = min(dp[j], dp[j+1]) + triangle[i][j]
        }
    }
    return dp[0]
}
```

## 错误写法与反例

### 错误 1：从上往下一维正序覆盖

从上往下时，一个状态可能被当前行新值污染，需要倒序或额外数组。自底向上避免这个问题。

### 错误 2：dp 长度只开 n

转移用到 `dp[j+1]`，最后一列会越界。开 `n+1` 可以统一边界。

### 错误 3：取 max

题目要求最小路径和，取 max 会得到最大路径和。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n^2) |
| 空间 | O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为当前位置的最优值由下一行两个相邻位置的最优值决定。

2. 这个算法维护的不变量是什么？
   处理完第 `i` 行后，`dp[j]` 是从该行第 `j` 个位置到底部的最小路径和。

3. 每一步为什么不会漏答案？
   每个位置下一步只能走到 `j` 或 `j+1`，转移取这两个选择的最小值。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是一维数组覆盖方向和 `dp[j+1]` 越界，处理错会污染状态或 panic。
