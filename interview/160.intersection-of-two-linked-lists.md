# 160. 相交链表

## 题目信息

- 题号：LeetCode 160
- 标题：Intersection of Two Linked Lists
- 难度：简单（Easy）
- 模式：链表 / 双指针同步

## 一句话思路

两个指针分别从 A、B 出发，走到末尾后切换到另一条链表；走过相同总长度后会在交点或 nil 相遇。

## 为什么想到这个模式

两条链表长度可能不同，直接同步走会因为长度差错过交点。让两个指针交换起点后：

```text
pA 走过 A + B
pB 走过 B + A
```

总长度相同。如果有交点，它们会在交点相遇；如果没有交点，会同时到 nil。

## 状态定义 / 数据结构

- `p1`：先走 A，再走 B。
- `p2`：先走 B，再走 A。

不修改链表，不使用额外集合。

## 核心不变量

每轮循环后，`p1` 和 `p2` 都前进一步。最多走 `len(A)+len(B)` 步：

- 有交点：两者抵消长度差后在同一节点相遇。
- 无交点：两者最终同时为 nil。

比较的是节点指针，不是节点值。

## 手推全过程

```text
A: a1 -> a2 \
             c1 -> c2
B: b1 ------/
```

路径：

```text
p1: a1 -> a2 -> c1 -> c2 -> B头 b1 -> c1
p2: b1 -> c1 -> c2 -> A头 a1 -> a2 -> c1
```

两者在 `c1` 相遇。

## Go 代码实现

```go
func getIntersectionNode(headA, headB *ListNode) *ListNode {
    if headA == nil || headB == nil {
        return nil
    }
    p1, p2 := headA, headB
    for p1 != p2 {
        if p1 == nil {
            p1 = headB
        } else {
            p1 = p1.Next
        }
        if p2 == nil {
            p2 = headA
        } else {
            p2 = p2.Next
        }
    }
    return p1
}
```

## 错误写法与反例

### 错误 1：比较节点值

两条链表可能有相同值但不是同一节点。相交定义是共享节点地址，必须比较指针。

### 错误 2：走到 nil 后不切换链表

长度不同会导致两个指针永远错位。切换起点是为了抵消长度差。

### 错误 3：修改链表结构

本题不需要修改链表。临时连接成环也能做，但更容易破坏输入。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m+n) |
| 空间 | O(1) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为两条链表长度不同，需要用双指针交换起点来同步总路程。

2. 这个算法维护的不变量是什么？
   两个指针每轮走一步，最终都走过 A+B 的总长度。

3. 每一步为什么不会漏答案？
   如果存在公共尾部，抵消长度差后两个指针会同时进入公共部分并在第一个公共节点相遇。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是比较值而不是节点地址，会把值相同但不共享节点的链表误判为相交。
