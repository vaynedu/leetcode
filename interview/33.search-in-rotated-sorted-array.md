# 33. 搜索旋转排序数组

## 题目

整数数组 `nums` 按升序排列，预先在某个未知的位置旋转了。给定旋转后的数组和一个目标值 `target`，如果存在返回索引，否则返回 -1。

**难度**：Medium
**标签**：数组 | 二分查找
**时间要求**：O(log n)

## 示例

```
输入：nums = [4,5,6,7,0,1,2], target = 0
输出：4

输入：nums = [4,5,6,7,0,1,2], target = 3
输出：-1
```

---

## 引导过程：像教练一样思考

### 第一步：理解旋转数组长什么样

原数组 `[1,2,3,4,5,6,7]` 旋转后，可能变成：
```
[4,5,6,7,0,1,2]   // 旋转点在 7 和 0 之间
```

特征：**存在一个断点，左边所有数 > 右边所有数**。断点就是最小值的位置。

---

### 第二步：看到 O(log n)，想到什么算法？

O(log n) → 二分查找。

但这是"旋转后的有序数组"，**标准二分不行了**，需要变种。

---

### 第三步：标准二分在旋转数组上为什么会失败？

标准二分的前提是：**整个数组是有序的**。

旋转数组 `[4,5,6,7,0,1,2]` 不是整体有序，但有一个重要性质：

> **任意一个 mid，至少有一半是有序的！**

无论 mid 落在哪，总有一半区间是连续的升序。

---

### 第四步：怎么判断哪一半有序？

以 `[4,5,6,7,0,1,2]`，mid = 6 为例：

```
[left=0, mid=3, right=6]
nums[left] = 4, nums[mid] = 7, nums[right] = 2
```

**判断规则：**
```
if nums[left] <= nums[mid]:
    左半边 [left..mid] 是升序的
else:
    右半边 [mid..right] 是升序的
```

- `nums[0]=4 <= nums[3]=7` → 左半边 `[4,5,6,7]` 是升序的

---

### 第五步：确定有序的半边后，target 在哪？

**如果左半边有序：**
```
[4,5,6,7,0,1,2]
 左半边有序: 4 <= 5 <= 6 <= 7
```

判断 target 是否在左半边有序区间内：
```
if nums[left] <= target < nums[mid]:
    答案在左半边 → right = mid - 1
else:
    答案在右半边 → left = mid + 1
```

**如果右半边有序：**
```
[4,5,6,7,0,1,2]
 右半边有序: 0 <= 1 <= 2
```

判断 target 是否在右半边有序区间内：
```
if nums[mid] < target <= nums[right]:
    答案在右半边 → left = mid + 1
else:
    答案在左半边 → right = mid - 1
```

---

### 第六步：完整代码

```go
func search(nums []int, target int) int {
    if len(nums) == 0 {
        return -1
    }
    left, right := 0, len(nums)-1

    for left <= right {
        mid := left + (right-left)/2

        if nums[mid] == target {
            return mid
        }

        // 左半边 [left..mid] 是有序的
        if nums[left] <= nums[mid] {
            // target 在左半边有序区间内
            if nums[left] <= target && target < nums[mid] {
                right = mid - 1
            } else {
                left = mid + 1
            }
        } else { // 右半边 [mid..right] 是有序的
            // target 在右半边有序区间内
            if nums[mid] < target && target <= nums[right] {
                left = mid + 1
            } else {
                right = mid - 1
            }
        }
    }
    return -1
}
```

---

### 第七步：以 target=0 和 target=6 为例走一遍

**target = 0，找索引 4**

```
初始：[4,5,6,7,0,1,2]
left=0, right=6

第1轮：mid=3, nums[3]=7
  nums[0]=4 <= nums[3]=7 → 左半边有序
  nums[0]=4 <= target=0 < nums[3]=7？不满足
  → 答案在右半边，left = 4

第2轮：mid=4, nums[4]=0
  nums[4]=0 == target → 返回 4 ✓
```

**target = 6，找索引 2**

```
初始：[4,5,6,7,0,1,2]
left=0, right=6

第1轮：mid=3, nums[3]=7
  nums[0]=4 <= nums[3]=7 → 左半边有序
  nums[0]=4 <= target=6 < nums[3]=7？满足
  → 答案在左半边，right = 2

第2轮：mid=1, nums[1]=5
  nums[0]=4 <= nums[1]=5 → 左半边有序
  nums[0]=4 <= target=6 < nums[1]=5？不满足
  → 答案在右半边，left = 2

第3轮：mid=2, nums[2]=6
  nums[2]=6 == target → 返回 2 ✓
```

## 关键点总结

| 要点 | 说明 |
|------|------|
| 旋转数组的核心性质 | 任意 mid，至少有一半是有序的 |
| 如何判断哪半有序 | `nums[left] <= nums[mid]` → 左半边有序 |
| 判断后怎么缩区间 | target 在有序区间内则收缩对应边界，否则收缩另一侧 |
| 为什么用 `<=` | 考虑 left == mid 的情况，保证正确性 |

## 复杂度

- **时间**：O(log n)
- **空间**：O(1)

## 变种

**81题（搜索旋转数组 II）**：数组中可能有重复元素。
- 当 `nums[left] == nums[mid] == nums[right]` 时无法判断哪半有序
- 解决：`left++` 跳过，收缩左边界
- 最坏时间 O(n)
