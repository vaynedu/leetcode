# 33. 搜索旋转排序数组

## 题目信息

- 题号：LeetCode 33
- 难度：中等
- 模式：二分查找 / 旋转数组
- 可视化：`doc/algorithm-visualizations/33.rotated-array-search.html`

## 一句话思路

旋转数组整体无序，但每次二分时至少有一半有序；先判断哪半有序，再判断 target 是否落在有序半边。

## 为什么想到这个模式

题目要求 O(log n)，这是二分查找的强信号。

虽然数组被旋转后不是整体有序，但它仍然保留局部有序性：对任意 `mid`，`[left..mid]` 和 `[mid..right]` 至少有一边是升序的。这个性质足够每轮排除一半。

## 状态定义 / 数据结构

- `left, right`：闭区间 `[left,right]`，答案如果存在一定在这个区间里。
- `mid`：当前检查位置。
- `nums[left] <= nums[mid]`：左半边有序。
- 否则：右半边有序。

## 核心不变量

每次循环开始时，如果 target 存在，它一定在 `[left,right]` 中。

每轮根据“哪半有序 + target 是否落在有序区间内”排除不可能的一半，剩下区间仍然保留 target。

## 手推全过程

以 `nums=[4,5,6,7,0,1,2], target=0` 为例：

| left | mid | right | 判断 | 动作 |
|------|-----|-------|------|------|
| 0 | 3 | 6 | `[4,5,6,7]` 左半有序，0 不在 `[4,7)` | `left=4` |
| 4 | 5 | 6 | `[0,1]` 左半有序，0 在 `[0,1)` | `right=4` |
| 4 | 4 | 4 | `nums[mid]=0` | 返回 4 |

## 错误写法与反例

错误写法：只按标准二分比较 `nums[mid]` 和 `target`。

```go
if nums[mid] < target {
    left = mid + 1
} else {
    right = mid - 1
}
```

反例：`nums=[4,5,6,7,0,1,2], target=0`

`mid=3` 时 `nums[mid]=7 > 0`，标准二分会去左边 `[4,5,6]`，直接把真实答案所在的右半边排除。

旋转数组必须先判断哪一半有序。

## Go 代码实现

```go
func search(nums []int, target int) int {
    left, right := 0, len(nums)-1

    for left <= right {
        mid := left + (right-left)/2
        if nums[mid] == target {
            return mid
        }

        if nums[left] <= nums[mid] {
            if nums[left] <= target && target < nums[mid] {
                right = mid - 1
            } else {
                left = mid + 1
            }
        } else {
            if nums[mid] < target && target <= nums[right] {
                left = mid + 1
            } else {
                right = mid - 1
            }
        }
    }
    return -1
}
```

## 复杂度

| 指标 | 值 |
|------|-----|
| 时间 | O(log n) |
| 空间 | O(1) |

## 四问复盘

1. **为什么属于二分？** 题目要求 O(log n)，且旋转数组每轮至少有一半有序，可排除一半。
2. **维护的不变量是什么？** target 如果存在，一定还在 `[left,right]` 内。
3. **为什么不会漏答案？** 只有在证明 target 不在某个有序区间时，才排除那一半。
4. **最容易错的边界？** 左半有序判断用 `nums[left] <= nums[mid]`，并且 target 区间要写成半开：`nums[left] <= target && target < nums[mid]`。
