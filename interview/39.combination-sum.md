# 39. 组合总和

## 题目信息

- 题号：LeetCode 39
- 难度：中等
- 模式：回溯 / 组合枚举
- 可视化：`doc/algorithm-visualizations/39.combination-sum.html`

## 一句话思路

从 `start` 开始枚举候选数，选择当前数后递归仍传 `i`，表示当前数可以重复使用。

## 为什么想到这个模式

题目要求返回所有和为 `target` 的组合。看到“所有组合”，优先想到回溯。

这里组合不关心顺序，所以 `[2,2,3]` 和 `[2,3,2]` 是同一个答案。需要用 `start` 限制后续只能从当前位置及其右侧选择，避免排列重复。

## 状态定义 / 数据结构

- `path`：当前组合。
- `sum` 或 `remain`：当前和或剩余目标。
- `start`：当前层允许选择的最小下标。
- `res`：所有合法组合。

推荐用 `remain`，语义更直接：选一个数就减掉它，`remain == 0` 收集答案。

## 核心不变量

- `path` 中的候选下标非递减，因此不会出现同一组合的不同排列。
- `remain` 表示还需要凑出的目标值。
- 递归调用 `backtrack(i, remain-candidates[i])` 表示当前元素可以重复选。
- 如果排序后 `candidates[i] > remain`，后面更大的数也不可能使用，可以 break。

## 手推全过程

以 `candidates=[2,3,6,7], target=7` 为例：

```text
path=[] remain=7
├── 选 2 → path=[2] remain=5
│   ├── 选 2 → path=[2,2] remain=3
│   │   ├── 选 2 → remain=1
│   │   └── 选 3 → remain=0 收集 [2,2,3]
│   └── 选 3 → path=[2,3] remain=2，后续都太大
├── 选 3 → path=[3] remain=4
├── 选 6 → path=[6] remain=1
└── 选 7 → remain=0 收集 [7]
```

答案：`[[2,2,3],[7]]`。

## 错误写法与反例

错误写法：递归时传 `i+1`。

```go
backtrack(i+1, remain-candidates[i])
```

反例：`candidates=[2,3,6,7], target=7`

这样每个数只能选一次，会漏掉 `[2,2,3]`。

另一个错误：递归时总从 0 开始。

```go
backtrack(0, remain-candidates[i])
```

这会产生 `[2,3,2]`、`[3,2,2]` 等重复排列。

## Go 代码实现

```go
func combinationSum(candidates []int, target int) [][]int {
    sort.Ints(candidates)
    res := [][]int{}
    path := []int{}

    var backtrack func(start int, remain int)
    backtrack = func(start int, remain int) {
        if remain == 0 {
            cur := append([]int(nil), path...)
            res = append(res, cur)
            return
        }

        for i := start; i < len(candidates); i++ {
            if candidates[i] > remain {
                break
            }
            path = append(path, candidates[i])
            backtrack(i, remain-candidates[i])
            path = path[:len(path)-1]
        }
    }

    backtrack(0, target)
    return res
}
```

## 复杂度

| 指标 | 值 |
|------|-----|
| 时间 | 指数级，取决于分支数和 target/min(candidates) |
| 空间 | O(target/min)，递归深度 |

## 四问复盘

1. **为什么属于回溯？** 要枚举所有组合，每一层选择一个候选数。
2. **维护的不变量是什么？** `path` 下标非递减，`remain` 是剩余目标。
3. **为什么不会漏答案？** 当前数可重复选时递归传 `i`，所有合法重复组合都会被覆盖。
4. **最容易错的边界？** `i` 与 `i+1` 的区别：本题可重复选，所以传 `i`。
