# 416. 分割等和子集

## 题目信息

- 题号：LeetCode 416
- 标题：Partition Equal Subset Sum
- 难度：中等（Medium）
- 模式：动态规划 / 0/1 背包

## 一句话思路

把“分成两个和相等的子集”转化为“能否从数组中选一些数凑出 `sum/2`”，每个数只能用一次，所以是 0/1 背包。

## 为什么想到这个模式

如果数组能分成两个等和子集，那么总和一定是偶数，且其中一个子集的和一定是：

```text
target = sum / 2
```

于是问题变成：

```text
能不能从 nums 中选若干元素，使它们的和等于 target？
```

这是典型子集和问题。每个元素只能选或不选一次，所以是 0/1 背包，而不是完全背包。

## 状态定义 / 数据结构

- `dp[s]`：当前已经处理过的数字中，是否能凑出和 `s`。
- `dp[0] = true`：什么都不选可以凑出 0。
- `target = sum / 2`：目标背包容量。

转移：

```text
dp[s] = dp[s] || dp[s-v]
```

含义是：如果之前能凑出 `s-v`，现在选上 `v` 就能凑出 `s`。

## 核心不变量

处理每个数字 `v` 时：

- `dp[s]` 表示只使用已经遍历过的数字，能否凑出 `s`。
- 倒序遍历 `s`，保证当前数字 `v` 在这一轮最多被使用一次。
- `dp[0]` 始终为 `true`，作为所有子集和的起点。

如果某一轮后 `dp[target] == true`，说明已经找到一个和为 `target` 的子集。

## 手推全过程

以 `nums = [1,5,11,5]` 为例：

```text
sum = 22
target = 11
```

初始化：

```text
dp[0] = true
```

处理 `1`：

```text
可达：0, 1
```

处理第一个 `5`：

```text
可达：0, 1, 5, 6
```

处理 `11`：

```text
可达：0, 1, 5, 6, 11
```

此时 `dp[11] = true`，说明可以选 `[11]` 作为一个子集，剩余 `[1,5,5]` 的和也是 11。

## Go 代码实现

```go
func canPartition(nums []int) bool {
    sum := 0
    for _, v := range nums {
        sum += v
    }
    if sum%2 != 0 {
        return false
    }
    target := sum / 2
    dp := make([]bool, target+1)
    dp[0] = true
    for _, v := range nums {
        for i := target; i >= v; i-- {
            dp[i] = dp[i] || dp[i-v]
        }
    }
    return dp[target]
}
```

## 错误写法与反例

### 错误 1：金额正序遍历

```go
for s := v; s <= target; s++ {
    dp[s] = dp[s] || dp[s-v]
}
```

这是完全背包写法，会让同一个数字在一轮里被重复使用。

反例：`nums=[1,2,5]`

`sum=8,target=4`，正确答案是 `false`。如果正序遍历，处理数字 `1` 时可能推出 `dp[2]、dp[3]、dp[4]`，相当于把同一个 `1` 用了多次。

### 错误 2：忽略总和奇偶性

如果 `sum` 是奇数，不可能拆成两个整数等和子集，应直接返回 `false`。继续做 DP 只会让目标含义变错。

### 错误 3：把问题理解成连续子数组

子集不要求连续，也不要求保持原数组相邻关系。用滑动窗口或前缀和会漏掉非连续选择。

## 与 322 的区别

| 对比项 | 322. 零钱兑换 | 416. 分割等和子集 |
|--------|---------------|-------------------|
| 背包类型 | 完全背包 | 0/1 背包 |
| 元素使用 | 每种硬币可重复使用 | 每个数组元素只能用一次 |
| 遍历方向 | 金额正序 | 金额倒序 |
| dp 含义 | 最少硬币数 | 是否可达 |

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n * target)，`target = sum/2` |
| 空间 | O(target) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为等和分割等价于从数组中选一部分凑出 `sum/2`，每个元素只能选一次，是 0/1 背包。

2. 这个算法维护的不变量是什么？
   `dp[s]` 表示只使用已处理元素时，是否能凑出和 `s`；倒序遍历保证当前元素只用一次。

3. 每一步为什么不会漏答案？
   每个元素都通过 `dp[s] || dp[s-v]` 表达“不选/选”两种可能，所有子集选择都会被覆盖。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是把金额写成正序遍历，会把 0/1 背包误写成完全背包，导致同一个数被重复使用而误判为可分。
