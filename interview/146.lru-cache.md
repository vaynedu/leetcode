# 146. LRU 缓存

## 题目

请你设计并实现一个满足 LRU (最近最少使用) 缓存约束的数据结构。

```
实现 LRUCache 类：
- LRUCache(int capacity) 用正整数容量初始化 LRU 缓存
- int get(int key) 如果 key 存在，返回值；否则返回 -1
- void put(int key, int value) 如果 key 存在，更新值；否则插入。如果容量满，淘汰最久未使用的

实例：
LRUCache cache = new LRUCache(2);
cache.put(1, 1);
cache.put(2, 2);
cache.get(1);      // 返回 1
cache.put(3, 3);   // 淘汰 key=2（最久未使用）
cache.get(2);      // 返回 -1（已被淘汰）
cache.put(4, 4);   // 淘汰 key=1（最久未使用）
cache.get(1);      // 返回 -1（已被淘汰）
cache.get(3);      // 返回 3
cache.get(4);      // 返回 4
```

## 难度

**频率**：🔥🔥🔥🔥🔥（面试高频 Top5，HashMap + LinkedList 经典结合）

## 解题思路

### 核心数据结构

```
1. HashMap：key -> Node，O(1) 查找
2. 双向链表：按访问顺序排列，最近使用的在表头，最久未使用的在表尾
```

### 关键操作

```
get(key)：
1. 通过 HashMap 找到节点
2. 将节点移到表头（表示最近使用）
3. 返回值

put(key, value)：
1. 如果 key 存在：更新值，移到表头
2. 如果 key 不存在：
   - 如果容量满：删除表尾节点（最久未使用）
   - 创建新节点，加入表头
3. 如果 key 不存在且容量不满：直接加入表头
```

### 图解

```
双向链表结构：
head <-> [最近使用] <-> [...] <-> [最久未使用] <-> tail

淘汰时删除 tail 前一个节点
```

## 代码实现

```go
package interview

// LRUNode 双向链表节点
type LRUNode struct {
	Key  int
	Val  int
	Prev *LRUNode
	Next *LRUNode
}

// LRUCache O(1) get/put - HashMap + 双向链表
type LRUCache struct {
	capacity int
	cache    map[int]*LRUNode // key -> Node
	head     *LRUNode         // 表头（最近使用）
	tail     *LRUNode         // 表尾（最久未使用）
}

func NewLRUCache(capacity int) *LRUCache {
	h := &LRUNode{}
	t := &LRUNode{}
	h.Next = t
	t.Prev = h
	return &LRUCache{
		capacity: capacity,
		cache:    make(map[int]*LRUNode),
		head:     h,
		tail:     t,
	}
}

func (c *LRUCache) Get(key int) int {
	if node, ok := c.cache[key]; ok {
		// 移到表头（最近使用）
		c.moveToHead(node)
		return node.Val
	}
	return -1
}

func (c *LRUCache) Put(key int, value int) {
	if node, ok := c.cache[key]; ok {
		// 更新值，移到表头
		node.Val = value
		c.moveToHead(node)
	} else {
		// 新建节点
		node := &LRUNode{Key: key, Val: value}
		c.cache[key] = node
		c.addToHead(node)

		// 容量超限，删除最久未使用的
		if len(c.cache) > c.capacity {
			removed := c.removeTail()
			delete(c.cache, removed.Key)
		}
	}
}

// addToHead 将节点加入到表头
func (c *LRUCache) addToHead(node *LRUNode) {
	node.Prev = c.head
	node.Next = c.head.Next
	c.head.Next.Prev = node
	c.head.Next = node
}

// removeNode 从双向链表中删除节点
func (c *LRUCache) removeNode(node *LRUNode) {
	node.Prev.Next = node.Next
	node.Next.Prev = node.Prev
}

// moveToHead 将节点移到表头
func (c *LRUCache) moveToHead(node *LRUNode) {
	c.removeNode(node)
	c.addToHead(node)
}

// removeTail 删除表尾节点（最久未使用）
func (c *LRUCache) removeTail() *LRUNode {
	node := c.tail.Prev
	c.removeNode(node)
	return node
}
```

## 面试要点

```
1. 为什么用双向链表而不是单向？
   → 删除节点需要 O(1)，单向链表删除需要找前驱

2. 为什么 HashMap 的 value 存整个 Node 而不是只存 Value？
   → 需要 O(1) 找到节点才能移动到表头

3. get 和 put 都是 O(1)：
   - HashMap 查找 O(1)
   - 链表插入/删除 O(1)

4. 容量为 1 的边界情况
```

```go
func TestLRUCache(t *testing.T) {
	fmt.Println("=============================================")
	fmt.Println("       146. LRU 缓存")
	fmt.Println("=============================================")
	fmt.Println()

	cache := NewLRUCache(2)

	cache.Put(1, 1)
	cache.Put(2, 2)

	if cache.Get(1) != 1 {
		t.Errorf("Get(1) should be 1, got %d", cache.Get(1))
	}

	cache.Put(3, 3) // 淘汰 key=2

	if cache.Get(2) != -1 {
		t.Errorf("Get(2) should be -1, got %d", cache.Get(2))
	}

	cache.Put(4, 4) // 淘汰 key=1

	if cache.Get(1) != -1 {
		t.Errorf("Get(1) should be -1, got %d", cache.Get(1))
	}
	if cache.Get(3) != 3 {
		t.Errorf("Get(3) should be 3, got %d", cache.Get(3))
	}
	if cache.Get(4) != 4 {
		t.Errorf("Get(4) should be 4, got %d", cache.Get(4))
	}

	fmt.Println("✅ LRUCache - HashMap + 双向链表 - PASS")
}
```
