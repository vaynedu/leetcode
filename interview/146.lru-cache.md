# 146. LRU 缓存

## 题目
设计一个满足以下约束的缓存结构：
- `get(key)` — 如果 key 存在返回 value，否则返回 -1
- `put(key, value)` — 如果 key 存在更新值；如果不存在插入；如果缓存满，淘汰最久未使用的 key

所有操作必须 O(1)。

## 核心数据结构

**HashMap + 双向链表**

```
HashMap<key, Node*>  ←→  双向链表（头部=最近使用，尾部=最久未用）
```

- HashMap 提供 O(1) 查找
- 双向链表维护使用顺序，O(1) 移动节点

## 关键操作

### Get
1. HashMap 查找 key → O(1)
2. 找到后把 Node 移到链表头部 → O(1)
3. 未找到返回 -1

### Put
1. key 存在：更新 value，移到头部
2. key 不存在：
   - 缓存满：删除链表尾部节点（最久未用），从 HashMap 移除 → O(1)
   - 缓存未满：新建 Node，插入头部，HashMap 添加映射

### 为什么需要双向链表？
删除单向链表中的任意节点需要先找到它的前驱（O(n)）。双向链表直接通过 `node.prev` 获取前驱，删除操作 O(1)。

## 代码实现

```go
type Node struct {
    key, val   int
    prev, next *Node
}

type LRUCache struct {
    capacity int
    head     *Node // 虚拟头（最近使用）
    tail     *Node // 虚拟尾（最久未用）
    cache    map[int]*Node
}

func Constructor146(capacity int) LRUCache {
    h := &Node{} // 虚拟头
    t := &Node{} // 虚拟尾
    h.next = t
    t.prev = h
    return LRUCache{capacity: capacity, head: h, tail: t, cache: make(map[int]*Node)}
}

func (c *LRUCache) Get(key int) int {
    if node, ok := c.cache[key]; ok {
        c.moveToHead(node)
        return node.val
    }
    return -1
}

func (c *LRUCache) Put(key int, value int) {
    if node, ok := c.cache[key]; ok {
        node.val = value
        c.moveToHead(node)
    } else {
        node := &Node{key: key, val: value}
        c.cache[key] = node
        c.addToHead(node)
        if len(c.cache) > c.capacity {
            removed := c.tail.prev
            c.removeNode(removed)
            delete(c.cache, removed.key)
        }
    }
}
```

## 面试追问

**Q：LinkedHashMap 怎么做？**
A：Java 有现成的 `LinkedHashMap`，继承 `HashMap` 并维护插入顺序。可以重写 `removeEldestEntry` 实现 LRU，代码简洁但面试可能要求手写双向链表。

**Q：LFU 和 LRU 的区别？**
A：LFU（Least Frequently Used）按使用频率淘汰，适合访问模式稳定的场景。实现更复杂，需要维护 `count` 频率和对应的 key 集合。

**Q：虚拟头尾节点的作用？**
A：避免边界判断。没有虚拟节点时，链表空或只有一个节点时插入/删除需要特殊处理。虚拟头尾把这些边界情况统一成普通操作。
