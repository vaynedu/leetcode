# 40. 组合总和 II

## 题目信息

- 题号：LeetCode 40
- 标题：Combination Sum II
- 难度：中等（Medium）
- 模式：回溯 / 组合枚举 / 同层去重

## 一句话思路

先排序，再用 `start` 控制每个元素最多使用一次；同一递归层遇到相同值只保留第一个分支，避免重复组合。

## 为什么想到这个模式

题目要求返回所有和为 `target` 的组合，这是组合枚举问题，优先想到回溯。

这题比 39 多两个约束：

- `candidates` 有重复元素。
- 每个元素只能使用一次。

所以需要同时解决两个问题：

- 用 `backtrack(i+1, ...)` 保证同一个下标不会重复使用。
- 排序后用 `i > start && candidates[i] == candidates[i-1]` 做同层去重。

## 状态定义 / 数据结构

- `path`：当前组合。
- `sum`：当前组合和。
- `start`：当前层允许选择的起始下标。
- `res`：所有合法组合。

递归含义：

```text
backtrack(start, sum) = 已经选出 path，从 start 开始继续选择后续元素
```

## 核心不变量

任意时刻：

- `path` 中的下标严格递增，因此每个元素最多使用一次。
- `sum` 等于 `path` 中所有元素之和。
- 同一层相同数值只展开一次，避免生成重复组合。
- 排序后如果 `sum + candidates[i] > target`，后续更大的数也不用尝试。

## 手推全过程

以 `candidates = [10,1,2,7,6,1,5]`，`target = 8` 为例。

排序后：

```text
[1,1,2,5,6,7,10]
```

关键分支：

```text
start=0, path=[]
├── i=0 选第一个 1
│   ├── start=1, path=[1]
│   ├── i=1 选第二个 1 → [1,1,6]
│   ├── i=2 选 2 → [1,2,5]
│   └── i=5 选 7 → [1,7]
├── i=1 是同层重复的 1，跳过
└── i=2 选 2
    └── i=4 选 6 → [2,6]
```

答案：

```text
[[1,1,6],[1,2,5],[1,7],[2,6]]
```

## Go 代码实现

```go
func combinationSum2(candidates []int, target int) [][]int {
    sort.Ints(candidates)
    res := [][]int{}
    path := []int{}

    var backtrack func(pos, sum int)
    backtrack = func(pos, sum int) {
        if sum == target {
            tmp := make([]int, len(path))
            copy(tmp, path)
            res = append(res, tmp)
            return
        }
        for i := pos; i < len(candidates); i++ {
            if i > pos && candidates[i] == candidates[i-1] {
                continue
            }
            if sum+candidates[i] > target {
                break
            }
            path = append(path, candidates[i])
            backtrack(i+1, sum+candidates[i])
            path = path[:len(path)-1]
        }
    }

    backtrack(0, 0)
    return res
}
```

## 错误写法与反例

### 错误 1：去重条件写成 `i > 0`

```go
if i > 0 && candidates[i] == candidates[i-1] {
    continue
}
```

这会把不同层的重复值也跳过，导致漏答案。

反例：`candidates=[1,1,6], target=8`

正确答案需要 `[1,1,6]`，第二个 `1` 在下一层可以被使用，不能因为它和前一个值相同就全局跳过。

### 错误 2：递归时传 `i`

```go
backtrack(i, sum+candidates[i])
```

这会让同一个下标重复使用，变成 39 题的规则。

反例：`candidates=[1,7], target=2`

错误写法可能构造出 `[1,1]`，但数组里只有一个 `1`，不合法。

### 错误 3：没有排序就同层去重

同层去重依赖相同值相邻。如果不排序，`candidates[i] == candidates[i-1]` 不能识别所有重复值，会产生重复组合。

## 与 39 的区别

| 对比项 | 39. 组合总和 | 40. 组合总和 II |
|--------|--------------|-----------------|
| candidates | 通常无重复 | 可能有重复 |
| 元素使用 | 可重复选 | 每个下标只能选一次 |
| 递归参数 | `backtrack(i, ...)` | `backtrack(i+1, ...)` |
| 去重重点 | 防止排列重复 | 同层相同值只展开一次 |

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(2^n * n)，最坏枚举子集，复制答案需要路径长度 |
| 空间 | O(n)，不计结果，递归深度最多 n |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要枚举所有满足和为 `target` 的组合，每一层决定从剩余候选中选哪个数。

2. 这个算法维护的不变量是什么？
   `path` 使用的下标严格递增，`sum` 始终等于 `path` 之和，同层重复值只展开一次。

3. 每一步为什么不会漏答案？
   对每个位置都枚举选或不选的可能；排序后只跳过同层重复分支，不跳过下一层合法使用的重复值。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是 `i > start` 写成 `i > 0`，会把跨层重复值也剪掉，漏掉 `[1,1,6]` 这类答案。
