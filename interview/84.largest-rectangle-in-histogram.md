# 84. 柱状图中最大的矩形

## 题目信息

- 题号：LeetCode 84
- 标题：Largest Rectangle in Histogram
- 难度：困难（Hard）
- 模式：单调栈

## 一句话思路

维护单调递增下标栈；当遇到更矮柱子时，弹出的柱子以当前下标为右边界、弹出后栈顶为左边界计算面积。

## 为什么想到这个模式

每根柱子如果作为矩形高度，需要找到左右两侧第一个比它矮的位置。单调递增栈正好能在遇到更矮柱子时确定被弹柱子的右边界。

末尾加高度 0 的哨兵，强制清算栈中剩余柱子。

## 状态定义 / 数据结构

- `stack`：保存下标，且对应高度单调递增。
- `i`：当前扫描位置，也是被弹柱子的右侧第一个更矮位置。
- 弹出 `top` 后：
  - 高度：`heights[top]`
  - 右边界：`i - 1`
  - 左边界：弹出后栈顶 `stack[len-1] + 1`，若栈空则为 0。

## 核心不变量

栈内下标对应高度递增。栈中柱子还没有遇到右侧第一个更矮柱子，所以面积暂时不能结算。

当当前高度 `h` 小于等于栈顶高度时，栈顶柱子的右边界被确定，可以弹出并计算最大面积。

## 手推全过程

`heights = [2,1,5,6,2,3]`，追加哨兵 0。

关键弹栈：

| i | h | 弹出高度 | 宽度 | 面积 |
|---|---|----------|------|------|
| 1 | 1 | 2 | 1 | 2 |
| 4 | 2 | 6 | 1 | 6 |
| 4 | 2 | 5 | 2 | 10 |
| 6 | 0 | 3 | 1 | 3 |
| 6 | 0 | 2 | 4 | 8 |
| 6 | 0 | 1 | 6 | 6 |

最大面积是 `10`。

## Go 代码实现

```go
func largestRectangleArea(heights []int) int {
    heights = append(heights, 0)
    stack := []int{}
    maxArea := 0
    for i, h := range heights {
        for len(stack) > 0 && heights[stack[len(stack)-1]] >= h {
            height := heights[stack[len(stack)-1]]
            stack = stack[:len(stack)-1]
            width := i
            if len(stack) > 0 {
                width = i - stack[len(stack)-1] - 1
            }
            maxArea = max(maxArea, height*width)
        }
        stack = append(stack, i)
    }
    return maxArea
}
```

## 错误写法与反例

### 错误 1：不加哨兵

递增数组 `[1,2,3]` 中，扫描结束时栈里还有柱子未结算，会漏掉答案。

### 错误 2：宽度少减 1

弹出后栈顶是左侧第一个更矮柱子，不能包含它，所以宽度是 `i - stackTop - 1`。

### 错误 3：栈里存高度不存下标

面积需要宽度，宽度依赖下标边界，只存高度无法计算。

## 四问复盘

1. 这题为什么属于这个模式？
   因为每根柱子需要找左右第一个更矮边界，单调栈正好维护未确定右边界的柱子。

2. 这个算法维护的不变量是什么？
   栈内高度单调递增，栈中柱子还没遇到右侧更矮柱子。

3. 每一步为什么不会漏答案？
   每根柱子在被弹出时左右边界都已确定，并且只结算一次。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是宽度公式和末尾哨兵；错了会漏算递增尾部或多算边界。
