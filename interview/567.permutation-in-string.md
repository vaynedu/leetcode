# 567. 字符串的排列（Permutation in String）

## 题目信息
- 题号：LeetCode 567
- 标题：Permutation in String
- 难度：中等（Medium）

---

## 一句话思路
**变长窗口 + 双 HashMap**：判断 s2 中是否存在一段长度为 len(s1) 的窗口，其字符计数与 s1 完全相同。

---

## 核心思路

### 方法：固定长度窗口
```
1. 统计 s1 的字符计数 need
2. 用窗口遍历 s2，窗口长度 = len(s1)
3. 用 window 统计窗口内字符计数
4. 每当 valid == len(need)，找到一个排列
```

### 关键：如何判断两个 map 相等？
```
每次滑动时：
- 加入 s2[R]
- 移除 s2[R-len(s1)]
- 用 valid 计数是否完全匹配
```

---

## 关键点/考点
```
时间：O(N)，N = len(s2)
空间：O(1)，字符集固定（26个字母）

追问：
- 为什么用固定窗口？→ s1 长度固定
- valid 如何工作？→ 每当 window[c] == need[c] 时 valid++
- 什么时候立即返回 true？→ valid == len(need) 时
```

---

## 易错点
```
1. s1 和 s2 的字符可能超出 26 字母
2. valid++ 只在 == need[c] 时加，不是 >=
3. 移除字符时如果 window[d] == need[d]，valid--（变成不满足了）
```

---

## 记忆口诀
```
固定长度窗口滑，
need window 双映射。
等于时来 valid 加，
移除时看是否减少。
等于 len 就返回，
排列找到不用管。
```

---

## Go 代码实现

```go
func checkInclusion(s1 string, s2 string) bool {
    need := make(map[rune]int)
    window := make(map[rune]int)

    for _, c := range s1 {
        need[c]++
    }

    L, R := 0, 0
    valid := 0
    needLen := len(need)

    for R < len(s2) {
        c := rune(s2[R])
        if _, ok := need[c]; ok {
            window[c]++
            if window[c] == need[c] {
                valid++
            }
        }
        R++

        // 窗口长度等于 s1 时，收缩并检查
        if R-L == len(s1) {
            if valid == needLen {
                return true
            }
            d := rune(s2[L])
            if _, ok := need[d]; ok {
                if window[d] == need[d] {
                    valid--
                }
                window[d]--
            }
            L++
        }
    }

    return false
}
```

---

## 测试用例

### 正常情况
- `s1="ab", s2="eidbaooo"` → true（"ba" 在 "eidbaooo" 中）
- `s1="ab", s2="eidboaooo"` → false

### 边界情况
- `s1=""` → true（空串是任何串的排列）
- `s2=""` → false（s2 为空）

### 极端情况
- `s1="a", s2="a"` → true
- `s1="abc", s2="def"` → false
