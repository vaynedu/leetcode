# 322. 零钱兑换

## 题目信息

- 题号：LeetCode 322
- 标题：Coin Change
- 难度：中等（Medium）
- 模式：动态规划 / 完全背包

## 一句话思路

定义 `dp[x]` 为凑出金额 `x` 的最少硬币数，枚举硬币并从小到大更新金额，让每种硬币可以被重复使用。

## 为什么想到这个模式

题目问“凑成 amount 的最少硬币数”，有明显的最优子结构：

```text
凑成 x 的最后一枚硬币如果是 coin，
那么前面必须用最少硬币凑成 x - coin。
```

所以：

```text
dp[x] = min(dp[x], dp[x-coin] + 1)
```

这不是可靠的贪心题。比如 `coins=[1,3,4], amount=6`：

- 贪心先选 4，再选 1，再选 1，共 3 枚。
- 最优是 3 + 3，共 2 枚。

因此需要 DP 枚举所有可行转移。

## 状态定义 / 数据结构

- `dp[x]`：凑成金额 `x` 所需的最少硬币数。
- `dp[0] = 0`：凑成 0 元需要 0 枚硬币。
- `INF`：表示当前金额暂时无法凑出。

转移：

```text
for coin in coins:
    for x from coin to amount:
        dp[x] = min(dp[x], dp[x-coin] + 1)
```

金额从小到大遍历，是因为每个硬币可以重复使用；更新 `dp[x]` 时，`dp[x-coin]` 可能已经在当前硬币这一轮被更新过。

## 核心不变量

处理过程中：

- `dp[x]` 始终表示当前已考虑硬币集合下，凑出金额 `x` 的最少硬币数。
- `dp[0]` 始终为 0，是所有转移的起点。
- 只有当 `dp[x-coin]` 可达时，才能用它更新 `dp[x]`。
- 遍历完所有硬币后，`dp[amount]` 就是全局最优解；如果仍是 `INF`，说明无解。

## 手推全过程

以 `coins = [1,2,5]`，`amount = 11` 为例：

初始化：

```text
dp[0] = 0
dp[1..11] = INF
```

处理硬币 1：

```text
dp[1]=1, dp[2]=2, dp[3]=3, ..., dp[11]=11
```

处理硬币 2：

```text
dp[2]=1   // 2
dp[3]=2   // 1+2
dp[4]=2   // 2+2
...
dp[10]=5
dp[11]=6
```

处理硬币 5：

```text
dp[5]=1    // 5
dp[10]=2   // 5+5
dp[11]=3   // 5+5+1
```

答案是 `3`。

## Go 代码实现

```go
func coinChange(coins []int, amount int) int {
    dp := make([]int, amount+1)
    for i := range dp {
        dp[i] = math.MaxInt
    }
    dp[0] = 0
    for _, c := range coins {
        for i := c; i <= amount; i++ {
            if dp[i-c] != math.MaxInt {
                dp[i] = min(dp[i], dp[i-c]+1)
            }
        }
    }
    if dp[amount] == math.MaxInt {
        return -1
    }
    return dp[amount]
}
```

## 错误写法与反例

### 错误 1：用贪心选最大硬币

反例：`coins=[1,3,4], amount=6`

贪心得到 `4+1+1`，共 3 枚；最优是 `3+3`，共 2 枚。

### 错误 2：金额从大到小遍历

```go
for x := amount; x >= coin; x-- { ... }
```

这是 0/1 背包写法，会限制每枚硬币只能用一次。

反例：`coins=[2], amount=4`

正确答案是 2 枚 `2`。倒序遍历会漏掉重复使用同一面额。

### 错误 3：不判断 `dp[x-coin]` 是否可达

如果直接做 `dp[x-coin] + 1`，会把不可达状态当成可达状态；使用极大值时还可能溢出。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(amount * len(coins)) |
| 空间 | O(amount) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为最少硬币数可以由更小金额的最优解转移而来，且每种硬币可以重复使用，是完全背包。

2. 这个算法维护的不变量是什么？
   `dp[x]` 始终是当前已考虑硬币下，凑出金额 `x` 的最少硬币数；不可达状态保持为 `INF`。

3. 每一步为什么不会漏答案？
   每个硬币都会尝试更新所有不小于它的金额，金额正序遍历允许同一硬币重复参与后续状态。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是把金额写成倒序遍历，会把完全背包误写成 0/1 背包，漏掉重复使用同一硬币的答案。
