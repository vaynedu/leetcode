# 47. 全排列 II

## 题目信息

- 题号：LeetCode 47
- 标题：Permutations II
- 难度：中等（Medium）
- 模式：回溯 / 排列去重

## 一句话思路

先排序，再用 `used` 枚举排列；同一层遇到相同值时只让第一个未使用的相同值开启分支。

## 为什么想到这个模式

题目仍然要求所有排列，所以主体是回溯。但数组中有重复数字，如果把相同值的不同下标都当成独立分支，会产生重复排列。

排序后，相同值相邻，才能用同层剪枝判断：

```go
i > 0 && nums[i] == nums[i-1] && !used[i-1]
```

含义：当前层中，前一个相同值还没被使用，说明它已经代表这一层的相同值分支，当前值应跳过。

## 状态定义 / 数据结构

- `nums`：排序后的数组。
- `path`：当前排列前缀。
- `used[i]`：下标 `i` 是否已经在当前路径里。
- `res`：所有不重复排列。

递归含义：

```text
backtrack() = 在当前 path 前缀下，继续填下一个位置，并跳过同层重复分支
```

## 核心不变量

任意时刻：

- `path` 中每个下标最多使用一次。
- 同一递归层，相同数值只会选择最靠前的那个可用下标开启分支。
- 如果 `nums[i] == nums[i-1]` 且 `used[i-1] == false`，选择 `nums[i]` 会产生和选择 `nums[i-1]` 等价的同层分支，所以跳过。

注意：`!used[i-1]` 表示“前一个相同值不在当前路径”，也就是当前判断发生在同一层。

## 手推全过程

以 `nums = [1,1,2]` 为例，排序后记作 `[1a,1b,2]`。

```text
[]
├── 选 1a
│   ├── 选 1b → 选 2 → [1,1,2]
│   └── 选 2  → 选 1b → [1,2,1]
├── 选 1b
│   └── 跳过：1a 未使用，1b 是同层重复分支
└── 选 2
    └── 选 1a → 选 1b → [2,1,1]
```

答案是：

```text
[[1,1,2], [1,2,1], [2,1,1]]
```

## Go 代码实现

```go
func permuteUnique(nums []int) [][]int {
    sort.Ints(nums)
    res := [][]int{}
    path := []int{}
    used := make([]bool, len(nums))

    var backtrack func()
    backtrack = func() {
        if len(path) == len(nums) {
            tmp := make([]int, len(path))
            copy(tmp, path)
            res = append(res, tmp)
            return
        }
        for i := 0; i < len(nums); i++ {
            if used[i] || (i > 0 && nums[i] == nums[i-1] && !used[i-1]) {
                continue
            }
            used[i] = true
            path = append(path, nums[i])
            backtrack()
            path = path[:len(path)-1]
            used[i] = false
        }
    }

    backtrack()
    return res
}
```

## 错误写法与反例

### 错误 1：不排序直接去重

```go
if nums[i] == nums[i-1] { continue }
```

反例：`[1,2,1]`。相同值不相邻，无法判断重复分支，必须先排序。

### 错误 2：剪枝条件写成 `used[i-1]`

```go
if i > 0 && nums[i] == nums[i-1] && used[i-1] {
    continue
}
```

这会跳过合法的递归深层选择。比如 `[1a,1b,2]` 中，路径已有 `1a` 时，下一层应该允许选择 `1b` 形成 `[1,1,2]`。

### 错误 3：只对 `i` 去重，不保留 `used`

排列中同一个值可以来自不同下标，但每个下标只能用一次。没有 `used` 就无法区分“当前路径已用过”和“同层重复分支”。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n * n!)，重复元素会剪掉部分分支 |
| 空间 | O(n)，不计结果 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要枚举所有排列，同时需要对重复数字造成的等价分支做剪枝。

2. 这个算法维护的不变量是什么？
   `used` 标记当前路径已使用下标；同一层相同值只允许最靠前的可用下标开启分支。

3. 每一步为什么不会漏答案？
   剪掉的只是同层相同值产生的等价分支，保留的第一个分支已经覆盖相同数值选择的全部结果。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是 `!used[i-1]`。写反会把深层合法重复值也剪掉，导致答案缺失。
