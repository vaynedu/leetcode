# 96. 不同的二叉搜索树

## 题目信息

- 题号：LeetCode 96
- 标题：Unique Binary Search Trees
- 难度：中等（Medium）
- 模式：动态规划 / 卡特兰数

## 一句话思路

枚举每个值作为根，左子树数量乘右子树数量，再把所有根的选择累加。

## 为什么想到这个模式

BST 的结构由根节点决定。选择 `j` 作为根时：

- 左子树只能由 `1..j-1` 组成，有 `j-1` 个节点。
- 右子树只能由 `j+1..i` 组成，有 `i-j` 个节点。
- 左右子树可以自由组合，所以数量相乘。

只关心节点个数，不关心具体值区间，因此可以用一维 DP。

## 状态定义 / 数据结构

状态：

```text
dp[i] = 由 i 个连续有序值能组成的不同 BST 数量
```

转移：

```text
dp[i] = Σ dp[j-1] * dp[i-j]，j 从 1 到 i
```

初始化：

```text
dp[0] = 1
dp[1] = 1
```

`dp[0] = 1` 表示空树是一种合法结构，这样叶子节点左右空树才能正确相乘。

## 核心不变量

计算 `dp[i]` 时，所有更小规模 `dp[0..i-1]` 已经计算完成。

枚举根 `j` 时：

- 左子树结构数是 `dp[j-1]`。
- 右子树结构数是 `dp[i-j]`。
- 当前根对应的结构数是二者乘积。

所有根节点选择互斥，累加不会重复。

## 手推全过程

以 `n = 3` 为例：

```text
dp[0] = 1
dp[1] = 1
```

计算 `dp[2]`：

```text
root=1: dp[0] * dp[1] = 1
root=2: dp[1] * dp[0] = 1
dp[2] = 2
```

计算 `dp[3]`：

```text
root=1: dp[0] * dp[2] = 2
root=2: dp[1] * dp[1] = 1
root=3: dp[2] * dp[0] = 2
dp[3] = 5
```

答案是 `5`。

## Go 代码实现

```go
func numTrees(n int) int {
    if n <= 1 {
        return 1
    }
    dp := make([]int, n+1)
    dp[0] = 1
    dp[1] = 1

    for i := 2; i <= n; i++ {
        for j := 1; j <= i; j++ {
            dp[i] += dp[j-1] * dp[i-j]
        }
    }
    return dp[n]
}
```

## 错误写法与反例

### 错误 1：`dp[0]` 初始化为 0

```go
dp[0] = 0
```

反例：`n = 1`。根节点左右都是空树，如果空树数量为 0，乘积会变成 0，答案错误。

### 错误 2：左右子树数量按值而不是节点数算

`dp` 的维度是节点数量，不是具体取值。对于连续有序值，结构数量只和数量有关。

### 错误 3：根枚举少算端点

```go
for j := 2; j < i; j++ { ... }
```

这样会漏掉根为最小值或最大值的情况。比如 `n=2` 的两棵树都来自端点根。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n^2) |
| 空间 | O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为规模为 `i` 的 BST 数量可以由更小规模的左右子树数量组合出来，且子问题重复。

2. 这个算法维护的不变量是什么？
   计算 `dp[i]` 时，所有小于 `i` 的结构数量都已知；每个根选择的左右子树数量相乘。

3. 每一步为什么不会漏答案？
   每棵 BST 都有唯一根节点，枚举所有根就覆盖所有结构；不同根的集合互不重叠。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是 `dp[0] = 1`。空树不是没有方案，而是一种可组合的空结构。
