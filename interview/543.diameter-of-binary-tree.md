# 543. 二叉树的直径

## 题目信息

- 题号：LeetCode 543
- 标题：Diameter of Binary Tree
- 难度：简单（Easy）
- 模式：二叉树 DFS / 后序遍历

## 一句话思路

对每个节点计算左右子树高度，经过该节点的最长路径边数是 `leftHeight + rightHeight`，用全局变量维护最大值；DFS 返回当前子树高度。

## 为什么想到这个模式

题目问任意两个节点之间的最长路径，这条路径不一定经过根节点，但一定会经过某个“拐点”节点：

```text
左侧最深节点 → 拐点 → 右侧最深节点
```

如果以某个节点作为拐点，那么经过它的路径长度就是：

```text
左子树高度 + 右子树高度
```

要知道一个节点的左右高度，必须先算完左右子树，所以这是后序遍历。

## 状态定义 / 数据结构

- `dfs(node)`：返回以 `node` 为根的子树高度，按节点数计。
- `left`：左子树高度。
- `right`：右子树高度。
- `diameter`：目前见过的最大直径，按边数计。

注意：

```text
高度按节点数：空树 0，叶子节点 1
直径按边数：left + right
```

## 核心不变量

任意时刻：

- `dfs(node)` 返回从 `node` 向下走到最深叶子的节点数。
- 在处理完 `node` 的左右子树后，`left + right` 是所有经过 `node` 的路径里最长的一条。
- `diameter` 始终是已经处理过的节点中，最大的 `left + right`。

## 手推全过程

以 `[1,2,3,4,5]` 为例：

```text
      1
     / \
    2   3
   / \
  4   5
```

后序计算：

```text
节点 4：left=0,right=0, diameter=max(0,0), 返回高度 1
节点 5：left=0,right=0, diameter=max(0,0), 返回高度 1
节点 2：left=1,right=1, diameter=max(0,2), 返回高度 2
节点 3：left=0,right=0, diameter=max(2,0), 返回高度 1
节点 1：left=2,right=1, diameter=max(2,3), 返回高度 3
```

答案是 `3`，对应路径 `4 -> 2 -> 1 -> 3` 或 `5 -> 2 -> 1 -> 3`，有 3 条边。

## Go 代码实现

```go
func diameterOfBinaryTree(root *TreeNode) int {
    diameter := 0
    var dfs func(node *TreeNode) int
    dfs = func(node *TreeNode) int {
        if node == nil {
            return 0
        }
        left := dfs(node.Left)
        right := dfs(node.Right)
        diameter = max(diameter, left+right)
        return max(left, right) + 1
    }
    dfs(root)
    return diameter
}
```

## 错误写法与反例

### 错误 1：只计算经过根节点的路径

直径不一定经过根。

反例：

```text
        1
       /
      2
     / \
    3   4
   /     \
  5       6
```

最长路径可能在根的左子树内部，不能只看根节点的左右高度。

### 错误 2：把高度和直径单位混淆

当前实现中高度按节点数，直径按边数。叶子高度是 1，但叶子自己的直径是 0。

如果把返回高度和更新直径都加 1，容易把边数答案算大。

### 错误 3：DFS 返回直径而不是高度

父节点需要的是子树高度，用来拼出经过父节点的路径。如果 DFS 返回的是子树直径，父节点无法知道向下延伸的最长单边高度。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，每个节点访问一次 |
| 空间 | O(h)，递归栈高度，最坏 O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为每个节点的答案依赖左右子树高度，必须先处理子树再处理当前节点，是后序 DFS。

2. 这个算法维护的不变量是什么？
   `dfs` 返回当前子树高度，`diameter` 保存所有已处理节点的最大 `left + right`。

3. 每一步为什么不会漏答案？
   任意最长路径都有一个最高拐点，算法会在处理那个拐点时用左右高度计算一次这条路径。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是混淆节点数高度和边数直径，会导致单节点返回 1 而不是 0，整体答案偏大。
