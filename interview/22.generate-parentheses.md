# 22. 括号生成

## 题目信息

- 题号：LeetCode 22
- 难度：中等
- 模式：回溯 / 合法前缀剪枝
- 可视化：`doc/algorithm-visualizations/22.generate-parentheses.html`

## 一句话思路

按位置生成括号：左括号数量小于 `n` 时可以放左括号；右括号数量小于左括号数量时才可以放右括号。

## 为什么想到这个模式

题目要求生成所有合法组合，而不是求一个最优值。看到“所有组合/所有路径/所有方案”，优先想到回溯。

括号生成的关键不是最后再检查是否合法，而是在生成过程中始终保持“前缀合法”。

## 状态定义 / 数据结构

- `path`：当前已经生成的括号前缀。
- `left`：已经放入的左括号数量。
- `right`：已经放入的右括号数量。
- `res`：所有合法结果。

递归层数就是当前生成位置，最大深度为 `2*n`。

## 核心不变量

任意时刻：

- `left <= n`
- `right <= left`
- `path` 始终是一个合法前缀

`right <= left` 的含义是：任何前缀里，右括号数量不能超过左括号数量，否则已经无法变成合法括号串。

## 手推全过程

以 `n=3` 为例：

```text
path=""
├── "("
│   ├── "(("
│   │   ├── "(((" → "((()))"
│   │   └── "(()" → "(()())", "(())()"
│   └── "()"
│       ├── "()((" → "()(())"
│       └── "()()" → "()()()"
```

有效结果共 5 个：

```text
((()))
(()())
(())()
()(())
()()()
```

## 错误写法与反例

错误写法：只要右括号数量小于 `n` 就放右括号。

```go
if right < n {
    path = append(path, ')')
    backtrack(left, right+1)
}
```

反例：`n=2`

这样会生成 `")("` 或 `")())"` 这类前缀非法的路径。非法前缀一旦出现，后面再加括号也无法修复。

正确条件：

```go
if right < left {
    // 才能放右括号
}
```

## Go 代码实现

```go
func generateParenthesis(n int) []string {
    res := []string{}
    path := make([]byte, 0, 2*n)

    var backtrack func(left, right int)
    backtrack = func(left, right int) {
        if left == n && right == n {
            res = append(res, string(path))
            return
        }

        if left < n {
            path = append(path, '(')
            backtrack(left+1, right)
            path = path[:len(path)-1]
        }

        if right < left {
            path = append(path, ')')
            backtrack(left, right+1)
            path = path[:len(path)-1]
        }
    }

    backtrack(0, 0)
    return res
}
```

## 复杂度

| 指标 | 值 |
|------|-----|
| 时间 | O(Cn · n)，Cn 为第 n 个卡特兰数 |
| 空间 | O(n)，递归深度和 path 长度 |

## 四问复盘

1. **为什么属于回溯？** 要枚举所有合法括号组合，每一层决定当前位置放什么。
2. **维护的不变量是什么？** 当前 `path` 始终是合法前缀，`right <= left <= n`。
3. **为什么不会漏答案？** 每个合法括号串都能由“合法前缀逐步扩展”得到，回溯枚举了所有合法扩展。
4. **最容易错的边界？** 右括号条件是 `right < left`，不是 `right < n`。
