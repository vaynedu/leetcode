# 21. 合并两个有序链表

## 题目信息

- 题号：LeetCode 21
- 标题：Merge Two Sorted Lists
- 难度：简单（Easy）
- 模式：链表 / 双指针 / 虚拟头节点

## 一句话思路

用两个指针分别指向两个链表当前节点，每次把较小节点接到结果链表尾部；某个链表耗尽后直接接上另一个链表剩余部分。

## 为什么想到这个模式

两个输入链表都已经升序，因此每一步只需要比较两个当前头节点：

```text
较小者一定是合并后链表的下一个节点
```

这和归并排序里的 merge 步骤完全一样。链表拼接容易在头节点处写出特殊分支，所以用 `dummy` 统一处理。

## 状态定义 / 数据结构

- `l1`：第一个链表当前未合并节点。
- `l2`：第二个链表当前未合并节点。
- `dummy`：结果链表虚拟头。
- `cur`：结果链表尾节点。

## 核心不变量

任意循环开始时：

- `dummy.Next ... cur` 是已经合并好的有序前缀。
- `l1` 和 `l2` 分别指向还没有合并的剩余链表头。
- 下一次接入的节点一定是 `l1` 和 `l2` 中值较小的那个。

循环结束后，最多只剩一个链表非空，直接接到 `cur.Next` 即可，因为它本身已经有序。

## 手推全过程

`l1 = [1,2,4]`，`l2 = [1,3,4]`

| 步骤 | l1 当前 | l2 当前 | 接入 | result |
|------|---------|---------|------|--------|
| 1 | 1 | 1 | l1 的 1 | 1 |
| 2 | 2 | 1 | l2 的 1 | 1->1 |
| 3 | 2 | 3 | 2 | 1->1->2 |
| 4 | 4 | 3 | 3 | 1->1->2->3 |
| 5 | 4 | 4 | l1 的 4 | 1->1->2->3->4 |
| 结束 | nil | 4 | 接上 l2 | 1->1->2->3->4->4 |

## Go 代码实现

```go
func mergeTwoLists(l1 *ListNode, l2 *ListNode) *ListNode {
    dummy := &ListNode{}
    cur := dummy
    for l1 != nil && l2 != nil {
        if l1.Val <= l2.Val {
            cur.Next = l1
            l1 = l1.Next
        } else {
            cur.Next = l2
            l2 = l2.Next
        }
        cur = cur.Next
    }
    if l1 != nil {
        cur.Next = l1
    }
    if l2 != nil {
        cur.Next = l2
    }
    return dummy.Next
}
```

## 错误写法与反例

### 错误 1：忘记移动 `cur`

```go
cur.Next = l1
l1 = l1.Next
// 忘记 cur = cur.Next
```

结果链表尾指针不前进，后续会覆盖 `cur.Next`，丢节点。

### 错误 2：循环结束后忘记接剩余链表

反例：`l1=[1,2,4]`，`l2=[1]`

主循环结束后 `l1` 还有 `[2,4]`，必须直接接上。

### 错误 3：不用 dummy 导致头节点分支复杂

不用 dummy 也能做，但要单独处理结果链表为空的第一步，更容易漏空链表边界。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m+n) |
| 空间 | O(1)，原地复用节点 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为两个链表都已排序，可以用双指针每次选择较小头节点，像归并排序一样合并。

2. 这个算法维护的不变量是什么？
   结果链表前缀始终有序，`cur` 指向前缀尾部，`l1/l2` 指向未合并部分。

3. 每一步为什么不会漏答案？
   每轮恰好接入一个当前最小节点，循环后把剩余有序链表整体接上。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是忘记接剩余链表，会丢掉较长链表尾部节点。
