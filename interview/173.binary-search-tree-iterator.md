# 173. 二叉搜索树迭代器

## 题目信息

- 题号：LeetCode 173
- 标题：Binary Search Tree Iterator
- 难度：中等（Medium）
- 模式：BST / 中序遍历 / 栈

## 一句话思路

BST 的中序遍历是升序序列；用栈惰性模拟中序遍历，每次 `Next()` 返回当前最小的未访问节点。

## 为什么想到这个模式

迭代器要求：

- `Next()` 返回下一个最小值。
- `HasNext()` 判断是否还有元素。

BST 的中序遍历天然升序：

```text
左 -> 根 -> 右
```

如果一次性把中序结果全部存入数组，`Next()` 很简单，但空间是 O(n)。题目通常希望用 O(h) 空间，所以用栈按需展开。

## 状态定义 / 数据结构

- `node`：当前还没有一路向左展开的子树根。
- `stack`：中序遍历路径上的候选节点，栈顶是下一次可能访问的最小节点。

`Next()` 做两件事：

```text
1. 从 node 一路向左压栈，找到当前最小未访问节点。
2. 弹出栈顶作为答案，然后转向它的右子树。
```

## 核心不变量

任意时刻：

- `stack` 保存了尚未访问的祖先节点。
- 对栈顶节点来说，它的左侧路径已经被压入或访问完，栈顶是当前最小候选。
- `node` 指向下一段还需要展开左边界的子树。

每个节点最多入栈一次、出栈一次，所以 `Next()` 均摊 O(1)。

## 手推全过程

BST：

```text
      7
     / \
    3   15
       /  \
      9   20
```

中序结果：

```text
3 -> 7 -> 9 -> 15 -> 20
```

过程：

```text
Next: 压 7,3；弹 3，node=nil
Next: 弹 7，node=15
Next: 压 15,9；弹 9，node=nil
Next: 弹 15，node=20
Next: 压 20；弹 20
```

## Go 代码实现

```go
type BSTIterator struct {
    stack []*TreeNode
    node  *TreeNode
}

func NewBSTIterator(root *TreeNode) BSTIterator {
    return BSTIterator{node: root}
}

func (this *BSTIterator) Next() int {
    for this.node != nil {
        this.stack = append(this.stack, this.node)
        this.node = this.node.Left
    }
    this.node = this.stack[len(this.stack)-1]
    this.stack = this.stack[:len(this.stack)-1]
    val := this.node.Val
    this.node = this.node.Right
    return val
}

func (this *BSTIterator) HasNext() bool {
    return this.node != nil || len(this.stack) > 0
}
```

## 错误写法与反例

### 错误 1：Next 后不转向右子树

访问某个节点后，下一批候选来自它的右子树左边界。如果忘记 `node = node.Right`，会漏掉右子树。

### 错误 2：HasNext 只判断 stack

当前实现中 `node` 可能还没被展开进 stack。`HasNext()` 必须判断：

```go
this.node != nil || len(this.stack) > 0
```

### 错误 3：把所有节点预先展开

也能通过功能测试，但空间变成 O(n)，没有利用 BST 高度空间的优势。

## 复杂度

| 操作 | 时间复杂度 |
|------|------------|
| Next | 均摊 O(1)，单次最坏 O(h) |
| HasNext | O(1) |

空间复杂度：O(h)。

## 四问复盘

1. 这题为什么属于这个模式？
   因为 BST 的中序遍历就是升序序列，迭代器要按升序逐个输出。

2. 这个算法维护的不变量是什么？
   `stack` 和 `node` 共同表示尚未访问的中序状态，下一次弹出的节点是当前最小未访问节点。

3. 每一步为什么不会漏答案？
   每个节点都会沿左边界入栈一次，弹出访问后再展开右子树，完整模拟中序遍历。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是 `HasNext` 忘记判断 `node != nil`，初始化后 stack 为空但仍有整棵树未展开，会误判没有下一个。
