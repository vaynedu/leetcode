# 257. 二叉树的所有路径

## 题目信息

- 题号：LeetCode 257
- 标题：Binary Tree Paths
- 难度：简单（Easy）
- 模式：二叉树 DFS / 路径回溯

## 一句话思路

DFS 维护从根到当前节点的 `path`；到叶子节点时，把完整路径拼成 `"1->2->5"` 格式加入结果。

## 为什么想到这个模式

题目要求返回所有：

```text
根节点 -> 叶子节点
```

的路径。每条路径都由递归过程中的祖先节点组成，所以 DFS 时带上当前路径即可。

和 112 / 113 类似，关键判断点仍然是叶子节点：只有走到叶子，路径才完整。

## 状态定义 / 数据结构

- `node`：当前节点。
- `path`：从根到当前节点之前的路径值。
- `result`：所有格式化后的根到叶路径字符串。

递归含义：

```text
pathsRec(node, path, result)
= 在已有 path 前缀下，收集从 node 出发的所有根到叶路径
```

## 核心不变量

任意时刻：

- `path` 加上当前节点后，表示从根到当前节点的完整前缀。
- 只有 `node.Left == nil && node.Right == nil` 时，才把路径转成字符串。
- 非叶子节点继续递归左右子树，不能提前收集。

当前实现用切片值传递 `path`，递归返回时上一层路径长度自然恢复。

## 手推全过程

例子：

```text
    1
   / \
  2   3
   \
    5
```

DFS 路径：

```text
1 -> 2 -> 5  收集 "1->2->5"
1 -> 3       收集 "1->3"
```

答案：

```text
["1->2->5", "1->3"]
```

## Go 代码实现

```go
func BinaryTreePaths(root *TreeNode) []string {
    var result []string
    var path []int
    pathsRec(root, path, &result)
    return result
}

func pathsRec(node *TreeNode, path []int, result *[]string) {
    if node == nil {
        return
    }
    path = append(path, node.Val)
    if node.Left == nil && node.Right == nil {
        b := ""
        for i, v := range path {
            if i > 0 {
                b += "->"
            }
            b += itoa(v)
        }
        *result = append(*result, b)
        return
    }
    pathsRec(node.Left, path, result)
    pathsRec(node.Right, path, result)
}
```

## 错误写法与反例

### 错误 1：非叶子节点也收集路径

反例：

```text
1
 \
  2
```

只能返回 `"1->2"`，不能返回 `"1"`，因为 `1` 不是叶子。

### 错误 2：路径字符串拼接时漏掉负数或多位数

节点值不一定是个位数。用 `byte('0' + v)` 只能处理 0 到 9，遇到 `12` 或 `-1` 会错。应该用通用整数转字符串逻辑。

### 错误 3：共享 path 但不回溯

如果用同一个切片原地 append，递归返回时必须恢复长度。当前实现通过切片值传递避免显式回溯，但保存路径时仍要注意不要复用可变状态。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n * h)，每个叶子路径拼接最多 h 个节点 |
| 空间 | O(h)，不计结果，递归栈和当前路径长度 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要枚举所有根到叶路径，DFS 正好维护从根到当前节点的路径前缀。

2. 这个算法维护的不变量是什么？
   `path` 始终表示当前根到节点的路径前缀，只有叶子节点才形成完整答案。

3. 每一步为什么不会漏答案？
   DFS 会遍历所有节点，每条根到叶路径都会在对应叶子节点被收集一次。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是非叶子提前收集或字符串转换只支持个位数，会产生半截路径或错误路径字符串。
