# 2. 两数相加

## 题目信息

- 题号：LeetCode 2
- 标题：Add Two Numbers
- 难度：中等（Medium）
- 模式：链表 / 模拟 / 进位

## 一句话思路

链表按低位到高位存储，直接从头节点开始逐位相加；每一位生成 `sum % 10`，进位保存为 `sum / 10`。

## 为什么想到这个模式

题目说数字是逆序存储：

```text
[2,4,3] 表示 342
[5,6,4] 表示 465
```

链表头正好是个位，所以不需要反转链表，可以像手算加法一样从低位往高位加。

每一步只需要关注：

- 当前 `l1` 位。
- 当前 `l2` 位。
- 上一位带来的 `carry`。

## 状态定义 / 数据结构

- `dummy`：虚拟头节点，简化结果链表头部处理。
- `cur`：结果链表当前尾节点。
- `carry`：上一位产生的进位。
- `l1/l2`：当前待处理的两个链表节点。

循环条件：

```text
l1 != nil || l2 != nil || carry > 0
```

只要还有节点或还有进位，就必须继续生成结果节点。

## 核心不变量

任意循环开始时：

- `dummy.Next` 到 `cur` 是已经计算好的低位结果。
- `carry` 是上一位加法产生的进位，只可能是 0 或 1。
- `l1` 和 `l2` 指向下一位待相加节点；空链表视为当前位为 0。

每轮循环后，结果链表多一个正确数字节点。

## 手推全过程

`l1 = [2,4,3]`，`l2 = [5,6,4]`

```text
342 + 465 = 807
```

链表低位在前：

| 位 | l1 | l2 | carry in | sum | 写入 | carry out |
|----|----|----|----------|-----|------|-----------|
| 个位 | 2 | 5 | 0 | 7 | 7 | 0 |
| 十位 | 4 | 6 | 0 | 10 | 0 | 1 |
| 百位 | 3 | 4 | 1 | 8 | 8 | 0 |

结果：

```text
7 -> 0 -> 8
```

## Go 代码实现

```go
func addTwoNumbers(l1 *ListNode, l2 *ListNode) *ListNode {
    dummy := &ListNode{}
    cur := dummy
    carry := 0
    for l1 != nil || l2 != nil || carry > 0 {
        sum := carry
        if l1 != nil {
            sum += l1.Val
            l1 = l1.Next
        }
        if l2 != nil {
            sum += l2.Val
            l2 = l2.Next
        }
        carry = sum / 10
        cur.Next = &ListNode{Val: sum % 10}
        cur = cur.Next
    }
    return dummy.Next
}
```

## 错误写法与反例

### 错误 1：循环条件漏掉 `carry > 0`

```go
for l1 != nil || l2 != nil { ... }
```

反例：`[5] + [5]`

最后还有进位 1，正确结果是 `[0,1]`。漏掉 `carry` 会只返回 `[0]`。

### 错误 2：不同长度链表直接停止

反例：`[9,9,9] + [1]`

第二、三位仍要继续处理，短链表空位按 0 计算。

### 错误 3：把链表当作高位在前

本题是低位在前，不能从尾部或按字符串高位加法思路处理。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(max(m,n)) |
| 空间 | O(max(m,n))，结果链表长度最多多一位 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为链表低位在前，天然适合从头节点开始模拟逐位加法。

2. 这个算法维护的不变量是什么？
   结果链表已经保存处理过的低位，`carry` 精确表示上一位产生的进位。

3. 每一步为什么不会漏答案？
   循环条件覆盖了两个链表剩余节点和最终进位，短链表空位按 0 处理。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是漏掉最后的进位，例如 `[5]+[5]` 会从 `[0,1]` 错成 `[0]`。
