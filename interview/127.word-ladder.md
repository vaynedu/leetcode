# 127. 单词接龙

## 题目信息

- 题号：LeetCode 127
- 标题：Word Ladder
- 难度：困难（Hard）
- 模式：图 / BFS 最短路

## 一句话思路

把每个单词看成节点，能一位变换到另一个单词就有边；BFS 按层扩展，第一次到达 `endWord` 就是最短长度。

## 为什么想到这个模式

题目问最短转换序列长度，每次转换一步且每步代价相同。这是无权图最短路，BFS 是标准解法。

为了避免显式建图，可以枚举当前单词每个位置替换成 `a..z`，若新词在字典中，就作为邻居。

## 状态定义 / 数据结构

- `wordSet`：尚未访问的合法单词集合。
- `queue`：当前 BFS 层的单词。
- `step`：从 `beginWord` 到当前层的转换长度。

访问过的单词从 `wordSet` 删除，避免重复入队。

## 核心不变量

BFS 第 `step` 层队列中的所有单词，都是从 `beginWord` 出发用 `step` 个单词长度到达的。

第一次生成 `endWord` 时，返回 `step + 1`，一定是最短序列长度。

## 手推全过程

`begin = "hit", end = "cog"`

```text
wordList = ["hot","dot","dog","lot","log","cog"]
```

| step | 队列 |
|------|------|
| 1 | `hit` |
| 2 | `hot` |
| 3 | `dot`, `lot` |
| 4 | `dog`, `log` |
| 5 | `cog` |

答案是 `5`。

## Go 代码实现

```go
func ladderLength(beginWord string, endWord string, wordList []string) int {
    wordSet := map[string]bool{}
    for _, w := range wordList {
        wordSet[w] = true
    }
    if !wordSet[endWord] {
        return 0
    }
    delete(wordSet, beginWord)

    queue := []string{beginWord}
    step := 1
    for len(queue) > 0 {
        size := len(queue)
        for i := 0; i < size; i++ {
            word := queue[0]
            queue = queue[1:]
            chars := []byte(word)
            for j := 0; j < len(chars); j++ {
                original := chars[j]
                for c := 'a'; c <= 'z'; c++ {
                    if byte(c) == original {
                        continue
                    }
                    chars[j] = byte(c)
                    newWord := string(chars)
                    if newWord == endWord {
                        return step + 1
                    }
                    if wordSet[newWord] {
                        delete(wordSet, newWord)
                        queue = append(queue, newWord)
                    }
                }
                chars[j] = original
            }
        }
        step++
    }
    return 0
}
```

## 错误写法与反例

### 错误 1：不按层统计 step

每弹出一个单词就 `step++` 会把同一层不同单词算成不同距离。必须用 `size` 固定当前层。

### 错误 2：访问过不删除

会重复入队，甚至在相互可达单词之间反复循环，影响性能和正确性。

### 错误 3：`endWord` 不在字典仍继续搜索

题目要求中间和结束单词都必须在 wordList 中，`endWord` 不存在时直接返回 0。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(N * L * 26 * L)，生成字符串有 O(L) 成本 |
| 空间 | O(N * L) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为每次变换是等权一步，要求最短路径，适合 BFS。

2. 这个算法维护的不变量是什么？
   队列中同一层的单词都对应相同转换长度 `step`。

3. 每一步为什么不会漏答案？
   每个位置都尝试 26 个字符，覆盖所有一步可达邻居；BFS 按层保证首次到达最短。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是 step 递增时机，不按层递增会把路径长度算错。
