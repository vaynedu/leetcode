# 212. 单词搜索 II

## 题目信息

- 题号：LeetCode 212
- 标题：Word Search II
- 难度：困难（Hard）
- 模式：Trie / 网格 DFS 回溯

## 一句话思路

把所有单词放入 Trie，再从网格每个格子出发 DFS；路径不在 Trie 中就立刻剪枝，遇到 `isEnd` 收集单词。

## 为什么想到这个模式

如果对每个单词单独在网格里 DFS，重复搜索会很多。多个单词共享前缀，Trie 可以把这些前缀合并：

- DFS 当前路径如果不是任何单词前缀，立刻停止。
- DFS 到某个 Trie 节点 `isEnd`，说明找到一个完整单词。

这是“词典前缀匹配 + 网格路径搜索”的组合题。

## 状态定义 / 数据结构

Trie 节点：

```go
type TrieNode212 struct {
    children map[rune]*TrieNode212
    isEnd    bool
    word     string
}
```

DFS 状态：

- `(i, j)`：当前网格位置。
- `node`：当前路径已经匹配到的 Trie 节点。
- `board` 中的 `'#'`：当前路径已访问标记。
- `result`：找到的单词。

递归含义：

```text
dfs(i, j, node) = 从 board[i][j] 继续匹配 node 的下一层 Trie 路径
```

## 核心不变量

DFS 任意时刻：

- 网格路径上的字符等于 Trie 从根到当前节点的前缀。
- 当前路径中每个格子最多使用一次。
- 如果当前字符不在 `node.children` 中，说明没有任何单词以该路径为前缀，可以剪枝。

找到单词后把 `next.isEnd = false`，表示这个单词已经收集过，防止从其它路径重复加入结果。

## 手推全过程

示例：

```text
board =
o a a n
e t a e
i h k r
i f l v

words = ["oath","pea","eat","rain"]
```

Trie 包含四个单词。DFS 从 `(0,0)=o` 开始：

```text
o -> a -> t -> h
```

路径在 Trie 中到达 `"oath"` 的 `isEnd` 节点，收集 `"oath"`。

从 `(1,3)=e` 开始：

```text
e -> a -> t
```

收集 `"eat"`。

`"pea"` 和 `"rain"` 的前缀路径在网格中无法连通，因此不会被收集。

## Go 代码实现

```go
func FindWords(board [][]byte, words []string) []string {
    trie := NewTrie212()
    for _, w := range words {
        trie.Insert(w)
    }
    m, n := len(board), len(board[0])
    var result []string

    var dfs func(i, j int, node *TrieNode212)
    dfs = func(i, j int, node *TrieNode212) {
        ch := rune(board[i][j])
        next, ok := node.children[ch]
        if !ok {
            return
        }

        if next.isEnd {
            result = append(result, next.word)
            next.isEnd = false
        }

        board[i][j] = '#'
        dirs := [4][2]int{{-1, 0}, {1, 0}, {0, -1}, {0, 1}}
        for _, d := range dirs {
            ni, nj := i+d[0], j+d[1]
            if ni >= 0 && ni < m && nj >= 0 && nj < n && board[ni][nj] != '#' {
                dfs(ni, nj, next)
            }
        }
        board[i][j] = byte(ch)
    }

    for i := 0; i < m; i++ {
        for j := 0; j < n; j++ {
            dfs(i, j, trie.root)
        }
    }
    return result
}
```

## 错误写法与反例

### 错误 1：不恢复访问标记

```go
board[i][j] = '#'
dfs(...)
// 缺少 board[i][j] = original
```

这会影响从其它起点开始的搜索，导致合法单词被漏掉。

### 错误 2：找到单词后不去重

```go
if next.isEnd {
    result = append(result, next.word)
}
```

如果同一个单词能从多条路径找到，会重复加入。可以用 set，也可以像当前实现一样 `next.isEnd = false`。

### 错误 3：不用 Trie，对每个 word 单独 DFS

功能上可行，但会重复搜索大量共享前缀，容易超时。Trie 的价值在于前缀剪枝。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | 最坏 O(m * n * 4^L)，Trie 剪枝通常显著减少搜索 |
| 空间 | O(W * L + L)，Trie 空间加递归栈 |

`W` 是单词数，`L` 是最长单词长度。

## 四问复盘

1. 这题为什么属于这个模式？
   因为要在网格路径中匹配多个单词，Trie 能合并前缀，DFS 能枚举相邻路径。

2. 这个算法维护的不变量是什么？
   当前 DFS 路径对应 Trie 中的一个前缀；路径中的格子不会重复使用。

3. 每一步为什么不会漏答案？
   从每个格子都启动 DFS，四方向覆盖所有合法路径；Trie 剪枝只剪掉不可能成为任何单词前缀的路径。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是访问标记恢复。忘记恢复会污染其它搜索路径，导致漏答案。
