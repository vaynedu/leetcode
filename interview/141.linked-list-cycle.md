# 141. 环形链表

## 题目信息

- 题号：LeetCode 141
- 标题：Linked List Cycle
- 难度：简单（Easy）
- 模式：链表 / 快慢指针

## 一句话思路

慢指针每次走一步，快指针每次走两步；如果有环，快指针一定会在环内追上慢指针。

## 为什么想到这个模式

链表不能随机访问，也不能知道长度。判断是否有环，关键是利用两个不同速度的指针：

- 无环：快指针会先到达 nil。
- 有环：快慢指针进入环后，相对距离每轮减少 1，最终相遇。

## 状态定义 / 数据结构

- `slow`：每轮走一步。
- `fast`：每轮走两步。

不需要额外集合，空间 O(1)。

## 核心不变量

如果链表有环，快慢指针都进入环后，它们在环上的相对距离每轮变化 1。

环长有限，所以快指针必然追上慢指针。若没有环，`fast` 或 `fast.Next` 会先变成 nil。

## 手推全过程

链表：

```text
3 -> 2 -> 0 -> -4
     ^         |
     |_________|
```

快慢指针：

| 轮次 | slow | fast |
|------|------|------|
| 初始 | 3 | 3 |
| 1 | 2 | 0 |
| 2 | 0 | 2 |
| 3 | -4 | -4 |

相遇，说明有环。

## Go 代码实现

```go
func hasCycle(head *ListNode) bool {
    if head == nil {
        return false
    }
    slow := head
    fast := head
    for fast.Next != nil && fast.Next.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
        if slow == fast {
            return true
        }
    }
    return false
}
```

## 错误写法与反例

### 错误 1：循环条件只判断 `fast != nil`

```go
for fast != nil {
    fast = fast.Next.Next
}
```

当 `fast.Next == nil` 时会空指针。必须保证 `fast.Next` 也非空。

### 错误 2：比较节点值而不是节点指针

链表中可能有重复值。判断相遇必须比较节点地址：`slow == fast`。

### 错误 3：fast 每次也只走一步

速度相同不会追上，无法利用相对速度判断环。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为链表环检测可以用不同速度的两个指针在环内相遇来判断。

2. 这个算法维护的不变量是什么？
   `fast` 比 `slow` 每轮多走一步；若有环，二者在环内相对距离持续变化。

3. 每一步为什么不会漏答案？
   无环时 fast 必到 nil；有环时快慢都会进入环，有限环内必然相遇。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是循环条件，没检查 `fast.Next` 会在短链表或无环尾部 panic。
