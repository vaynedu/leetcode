# 20. 有效括号（Valid Parentheses）

## 题目信息

- 题号：LeetCode 20
- 标题：Valid Parentheses
- 难度：简单（Easy）
- 模式：栈 / 最近匹配

## 一句话思路

左括号入栈，右括号必须匹配最近的左括号；遍历结束后栈为空才有效。

## 为什么想到这个模式

括号匹配有“后打开的括号要先关闭”的特性，这是典型后进先出结构。栈顶始终表示最近还没有被匹配的左括号。

## 状态定义 / 数据结构

- `stack`：保存尚未匹配的左括号。
- `pairs`：右括号到左括号的映射，如 `')' -> '('`。

遍历字符：

- 左括号：入栈。
- 右括号：检查栈顶是否匹配。

## 核心不变量

遍历到任意位置时，`stack` 中保存的是当前前缀里还没匹配的左括号，并且顺序从底到顶对应它们出现的先后。

遇到右括号时，只有栈顶左括号能和它匹配；如果栈空或类型不对，整个字符串立即无效。

## 手推全过程

以 `s = "{[]}"` 为例：

| char | 操作 | stack |
|------|------|-------|
| `{` | 入栈 | `{` |
| `[` | 入栈 | `{ [` |
| `]` | 匹配 `[`，出栈 | `{` |
| `}` | 匹配 `{`，出栈 | 空 |

最终栈空，返回 true。

反例 `([)]`：

遇到 `)` 时栈顶是 `[`，不匹配，返回 false。

## Go 代码实现

```go
func isValid(s string) bool {
    stack := []rune{}
    pairs := map[rune]rune{')': '(', ']': '[', '}': '{'}

    for _, c := range s {
        if c == '(' || c == '[' || c == '{' {
            stack = append(stack, c)
        } else {
            if len(stack) == 0 || stack[len(stack)-1] != pairs[c] {
                return false
            }
            stack = stack[:len(stack)-1]
        }
    }
    return len(stack) == 0
}
```

## 错误写法与反例

### 错误 1：遍历完不检查栈空

反例：`"([]"`。遍历过程中没有类型冲突，但还剩左括号未匹配，应返回 false。

### 错误 2：右括号来时不检查栈空

反例：`"]"`。直接访问栈顶会越界。

### 错误 3：只统计数量

反例：`"([)]"`。括号数量都对，但嵌套顺序错，必须用栈检查最近匹配。

## 四问复盘

1. 这题为什么属于这个模式？
   因为括号匹配要求最近打开的左括号最先被关闭，符合栈的后进先出。

2. 这个算法维护的不变量是什么？
   栈中保存当前前缀尚未匹配的左括号，栈顶是下一个右括号唯一可能匹配的对象。

3. 每一步为什么不会漏答案？
   每个字符都被处理一次；任意不匹配会立即发现，所有左括号最终都必须出栈。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是遍历结束忘记检查栈空，会把缺少右括号的字符串误判为有效。
