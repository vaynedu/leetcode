# 17. 电话号码的字母组合

## 题目信息

- 题号：LeetCode 17
- 标题：Letter Combinations of a Phone Number
- 难度：中等（Medium）
- 模式：回溯 / 多叉决策树

## 一句话思路

每个数字对应一组字母，每一层决定当前位置选哪个字母；当路径长度等于 `digits` 长度时，得到一个完整组合。

## 为什么想到这个模式

题目要求返回所有可能的字母组合，不是求最优值，也不是判断是否存在，而是完整枚举。

对 `digits = "23"`：

```text
第 0 位数字 2：选 a / b / c
第 1 位数字 3：选 d / e / f
```

每个位置都有多个选择，选择一个字母后进入下一位，返回后继续尝试同层其它字母。这就是典型回溯。

和排列题的区别：

- 本题每一层只看当前数字对应的字母集合。
- 每层选择范围由 `digits[idx]` 决定，不需要 `used`。
- 路径天然从左到右构造，不需要 `start` 去重。

## 状态定义 / 数据结构

- `letters`：数字到字母集合的映射。
- `path`：当前已经选出的字母前缀，长度固定为 `len(digits)`。
- `idx`：当前正在处理 `digits[idx]`。
- `res`：所有完整组合。

递归含义：

```text
backtrack(idx) = 已经确定前 idx 个位置，继续决定第 idx 个位置
```

## 核心不变量

任意时刻：

- `path[0:idx]` 是已经确定的合法前缀。
- `path[k]` 一定来自 `digits[k]` 对应的字母集合。
- `idx` 表示下一个待填写位置。

当 `idx == len(digits)` 时，所有位置都已确定，`string(path)` 就是一个完整组合。

## 手推全过程

以 `digits = "23"` 为例：

```text
[]
├── a
│   ├── ad
│   ├── ae
│   └── af
├── b
│   ├── bd
│   ├── be
│   └── bf
└── c
    ├── cd
    ├── ce
    └── cf
```

每条路径长度都是 2，共 `3 * 3 = 9` 个组合。

## Go 代码实现

```go
func letterCombinations(digits string) []string {
    if len(digits) == 0 {
        return nil
    }
    letters := map[byte]string{
        '2': "abc", '3': "def", '4': "ghi",
        '5': "jkl", '6': "mno", '7': "pqrs",
        '8': "tuv", '9': "wxyz",
    }
    res := []string{}
    path := make([]byte, len(digits))

    var backtrack func(idx int)
    backtrack = func(idx int) {
        if idx == len(digits) {
            res = append(res, string(path))
            return
        }
        for _, c := range letters[digits[idx]] {
            path[idx] = byte(c)
            backtrack(idx + 1)
        }
    }

    backtrack(0)
    return res
}
```

## 错误写法与反例

### 错误 1：空字符串返回 `[]string{""}`

```go
if len(digits) == 0 {
    return []string{""}
}
```

题目语义是“没有输入数字就没有组合”，通常应返回空结果。当前实现返回 `nil`，和空切片在长度上都表示 0 个结果。

### 错误 2：把所有数字的字母都放在同一层选

```go
for _, lettersForDigit := range letters {
    for _, c := range lettersForDigit { ... }
}
```

这样会打乱位置约束，可能把第 0 位选成数字 3 的字母，生成不合法组合。

### 错误 3：递归终止条件写成 `idx == len(digits)-1`

如果在最后一位之前就收集答案，会得到长度不足的字符串；如果同时又继续递归，还可能重复收集。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(4^n * n)，每个位置最多 4 个字母，生成字符串需要 O(n) |
| 空间 | O(n)，不计结果，递归深度和路径长度都是 n |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要枚举所有字母组合，每个数字是一层决策，每层从对应字母集合中选择一个字母。

2. 这个算法维护的不变量是什么？
   `path[0:idx]` 始终是前 `idx` 个数字对应的合法字母前缀。

3. 每一步为什么不会漏答案？
   第 `idx` 层完整遍历 `digits[idx]` 对应的所有字母，递归树覆盖每个位置选择的笛卡尔积。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是空输入和终止条件；空输入不应产生一个空字符串组合，终止条件错会导致结果长度不对或漏掉最后一位。
