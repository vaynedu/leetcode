# 19. 删除链表的倒数第 N 个节点

## 题目信息

- 题号：LeetCode 19
- 标题：Remove Nth Node From End of List
- 难度：中等（Medium）
- 模式：链表 / 快慢指针 / dummy

## 一句话思路

用 dummy 统一删除头节点；让 fast 先走 `n+1` 步，再让 fast 和 slow 同步走，slow 停在待删除节点前一个位置。

## 为什么想到这个模式

要删除倒数第 N 个节点，如果先求长度需要两次遍历。快慢指针可以维护固定间距：

```text
fast 和 slow 相隔 n+1 步
```

当 fast 到达 nil 时，slow 正好在待删除节点的前驱。

## 状态定义 / 数据结构

- `dummy`：虚拟头节点，`dummy.Next = head`。
- `fast`：先走的指针。
- `slow`：最终停在待删除节点前驱。

为什么需要 `n+1` 步：因为删除需要改 `slow.Next`，所以 slow 必须停在待删节点前一个。

## 核心不变量

fast 先走完后，`fast` 与 `slow` 之间始终相隔 `n+1` 个指针步。

同步移动直到 `fast == nil`，此时 slow 后面正好还有 n 个节点，`slow.Next` 就是倒数第 n 个节点。

dummy 保证删除头节点时也有前驱。

## 手推全过程

`head = [1,2,3,4,5], n = 2`

```text
dummy -> 1 -> 2 -> 3 -> 4 -> 5
```

| 步骤 | fast | slow |
|------|------|------|
| 初始 | dummy | dummy |
| fast 先走 3 步 | 3 | dummy |
| 同步 1 | 4 | 1 |
| 同步 2 | 5 | 2 |
| 同步 3 | nil | 3 |

`slow.Next` 是 4，删除后得到 `1 -> 2 -> 3 -> 5`。

## Go 代码实现

```go
func removeNthFromEnd(head *ListNode, n int) *ListNode {
    dummy := &ListNode{}
    dummy.Next = head
    fast := dummy
    slow := dummy
    for i := 0; i <= n; i++ {
        if fast == nil {
            return head
        }
        fast = fast.Next
    }
    for fast != nil {
        slow = slow.Next
        fast = fast.Next
    }
    slow.Next = slow.Next.Next
    return dummy.Next
}
```

## 错误写法与反例

### 错误 1：fast 只先走 n 步

如果 fast 只走 n 步，最后 slow 会停在待删除节点本身，而不是前驱，无法直接删除 `slow`。

### 错误 2：不用 dummy

反例：`head=[1], n=1` 或删除头节点。没有 dummy 时需要额外处理 head 被删除的情况。

### 错误 3：返回原始 head

```go
return head
```

如果删除的是头节点，新的头是 `dummy.Next`，返回原始 head 会错误。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(L) |
| 空间 | O(1) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为倒数位置可以用快慢指针固定间距在一次遍历中定位。

2. 这个算法维护的不变量是什么？
   fast 和 slow 保持 `n+1` 步间距，fast 到 nil 时 slow 在待删除节点前驱。

3. 每一步为什么不会漏答案？
   同步移动保持间距不变，fast 到尾部的时刻唯一确定倒数第 n 个节点的前驱。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是删除头节点。没有 dummy 或返回 head，会返回已经被删除的旧头。
