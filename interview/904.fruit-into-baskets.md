# 904. 水果成篮（Fruit Into Baskets）

## 题目信息

- 题号：LeetCode 904
- 标题：Fruit Into Baskets
- 难度：中等（Medium）
- 模式：滑动窗口 / 至多 K 种

## 一句话思路

把题目翻译成“最长连续子数组，最多包含 2 种不同数字”，用窗口计数并在种类超过 2 时收缩。

## 为什么想到这个模式

题目要求从某棵树开始，向右连续摘水果，且篮子最多装两种水果。也就是说，我们要找：

```text
最长连续区间，区间内不同水果种类数 <= 2
```

这是典型的变长滑动窗口：

- 右指针扩张，加入新水果。
- 如果种类数超过 2，左指针收缩。
- 每次窗口恢复合法后更新最大长度。

选择这个模式的信号：

- 连续子数组。
- “最多 K 种”约束。
- 目标是最长长度。

## 状态定义 / 数据结构

- `window`：当前窗口中每种水果的数量。
- `L`：窗口左边界。
- `R`：窗口右边界。
- `maxLen`：目前见过的最长合法窗口长度。

窗口合法条件：

```text
len(window) <= 2
```

## 核心不变量

每次更新 `maxLen` 前，窗口 `[L, R]` 一定满足 `len(window) <= 2`。

当 `len(window) > 2` 时，持续移动 `L`：

- 移除 `fruits[L]` 的一次计数。
- 如果计数变成 0，必须从 map 中删除该水果种类。
- 直到窗口重新只包含两种水果。

`L` 只向右移动，每个元素最多进入窗口一次、离开窗口一次。

## 手推全过程

以 `fruits = [1,2,3,2,2]` 为例：

| R | 加入 | window | 是否收缩 | 合法窗口 | maxLen |
|---|------|--------|----------|----------|--------|
| 0 | 1 | `{1:1}` | 否 | `[1]` | 1 |
| 1 | 2 | `{1:1,2:1}` | 否 | `[1,2]` | 2 |
| 2 | 3 | `{1:1,2:1,3:1}` | 是，移除 1 | `[2,3]` | 2 |
| 3 | 2 | `{2:2,3:1}` | 否 | `[2,3,2]` | 3 |
| 4 | 2 | `{2:3,3:1}` | 否 | `[2,3,2,2]` | 4 |

答案是 `4`。

## Go 代码实现

```go
func totalFruit(fruits []int) int {
    window := make(map[int]int)
    L := 0
    maxLen := 0

    for R := 0; R < len(fruits); R++ {
        window[fruits[R]]++

        for len(window) > 2 {
            window[fruits[L]]--
            if window[fruits[L]] == 0 {
                delete(window, fruits[L])
            }
            L++
        }

        if R-L+1 > maxLen {
            maxLen = R - L + 1
        }
    }

    return maxLen
}
```

## 错误写法与反例

### 错误 1：收缩一次就停止

```go
if len(window) > 2 {
    removeLeftOnce()
}
```

反例：`[1,1,1,2,3]`。

加入 `3` 后，只移除一个 `1`，窗口仍然包含 `1/2/3` 三种水果。必须 `for len(window) > 2` 持续收缩。

### 错误 2：计数归 0 不删除 key

```go
window[fruits[L]]--
L++
```

如果不 `delete`，`len(window)` 仍然包含数量为 0 的水果，窗口会被误判为超过两种，导致过度收缩。

### 错误 3：收缩前更新答案

```go
window[fruits[R]]++
maxLen = max(maxLen, R-L+1)
for len(window) > 2 { ... }
```

这样会把非法窗口长度计入答案。反例 `[1,2,3]`，正确答案是 `2`，错误写法可能记录 `3`。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1)，窗口最多保留 3 种左右的计数；一般写作 O(k) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要求最长连续区间，且窗口内最多允许两种元素，是标准变长滑动窗口。

2. 这个算法维护的不变量是什么？
   更新答案前窗口一定合法，即 `len(window) <= 2`；`window` 中只保留当前窗口内计数大于 0 的水果种类。

3. 每一步为什么不会漏答案？
   右指针枚举每个结尾，左指针只在窗口非法时收缩到刚好合法，因此每个以 `R` 结尾的最长合法窗口都会被检查。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是计数归 0 后不删除 key，会让 `len(window)` 错误变大，导致窗口被过度收缩。
