# 155. 最小栈

## 题目信息

- 题号：LeetCode 155
- 标题：Min Stack
- 难度：中等（Medium）
- 模式：栈 / 辅助栈 / 数据结构设计

## 一句话思路

用一个普通栈保存数据，再用一个同步长度的最小栈保存“每个深度下的当前最小值”，这样 `GetMin` 直接读最小栈栈顶。

## 为什么想到这个模式

普通栈的 `Push/Pop/Top` 都是 O(1)，但如果每次 `GetMin` 都扫描整个栈，就会是 O(n)。

要让 `GetMin` 也是 O(1)，就必须在每次入栈时把“当前最小值”也记录下来。

所以维护两个栈：

- `stack[i]`：第 i 层真实数据。
- `minStack[i]`：当栈高度为 i+1 时的最小值。

## 状态定义 / 数据结构

- `stack`：正常数据栈。
- `minStack`：与 `stack` 同步增长/缩短，栈顶是当前全局最小值。

Push 规则：

```text
minStack 为空：val 入 minStack
否则：min(val, minStack.top) 入 minStack
```

Pop 规则：

```text
stack 和 minStack 同时弹出
```

## 核心不变量

任意时刻：

- `len(stack) == len(minStack)`。
- `minStack[len(minStack)-1]` 是当前 `stack` 中所有元素的最小值。
- 每一层的 `minStack[i]` 都对应 `stack[0:i]` 的最小值。

因此 `GetMin()` 可以 O(1) 返回 `minStack` 栈顶。

## 手推全过程

操作序列：

```text
Push(-2)
Push(0)
Push(-3)
GetMin()
Pop()
Top()
GetMin()
```

状态变化：

| 操作 | stack | minStack | 当前最小 |
|------|-------|----------|----------|
| Push(-2) | [-2] | [-2] | -2 |
| Push(0) | [-2,0] | [-2,-2] | -2 |
| Push(-3) | [-2,0,-3] | [-2,-2,-3] | -3 |
| Pop() | [-2,0] | [-2,-2] | -2 |

## Go 代码实现

```go
type MinStack struct {
    stack    []int
    minStack []int
}

func NewMinStack() *MinStack {
    return &MinStack{
        stack:    []int{},
        minStack: []int{},
    }
}

func (this *MinStack) Push(val int) {
    this.stack = append(this.stack, val)
    if len(this.minStack) == 0 || val <= this.minStack[len(this.minStack)-1] {
        this.minStack = append(this.minStack, val)
    } else {
        this.minStack = append(this.minStack, this.minStack[len(this.minStack)-1])
    }
}

func (this *MinStack) Pop() {
    if len(this.stack) == 0 {
        return
    }
    this.stack = this.stack[:len(this.stack)-1]
    this.minStack = this.minStack[:len(this.minStack)-1]
}

func (this *MinStack) Top() int {
    return this.stack[len(this.stack)-1]
}

func (this *MinStack) GetMin() int {
    return this.minStack[len(this.minStack)-1]
}
```

## 错误写法与反例

### 错误 1：最小栈只在遇到更小值时入栈，但 Pop 不同步

如果 `minStack` 不和 `stack` 同步长度，就必须在 Pop 时判断被弹出的值是否等于当前最小值。当前实现选择同步复制最小值，逻辑更简单。

### 错误 2：重复最小值只保存一次

反例：

```text
Push(2), Push(1), Push(1), Pop()
```

弹出一个 `1` 后，当前最小值仍然是 `1`。如果重复最小值没被正确记录，可能错误恢复成 `2`。

### 错误 3：GetMin 时扫描 stack

能得到正确答案，但时间复杂度是 O(n)，不满足题目要求。

## 复杂度

| 操作 | 时间复杂度 |
|------|------------|
| Push | O(1) |
| Pop | O(1) |
| Top | O(1) |
| GetMin | O(1) |

空间复杂度：O(n)。

## 四问复盘

1. 这题为什么属于这个模式？
   因为需要在栈操作 O(1) 的同时查询最小值，必须额外维护当前最小值状态。

2. 这个算法维护的不变量是什么？
   `minStack` 与 `stack` 等长，且每一层记录对应栈前缀的最小值。

3. 每一步为什么不会漏答案？
   每次 Push 都记录新深度下的最小值，每次 Pop 同步删除该深度状态，栈顶始终对应当前栈。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是重复最小值，弹出一个最小值后仍可能还有相同最小值，不能过早恢复到更大值。
