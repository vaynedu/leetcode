# 142. 环形链表 II

## 题目信息

- 题号：LeetCode 142
- 标题：Linked List Cycle II
- 难度：中等（Medium）
- 模式：链表 / 快慢指针 / 环入口

## 一句话思路

先用快慢指针在环内相遇；再让一个指针从头出发、一个从相遇点出发，同速前进，相遇处就是环入口。

## 为什么想到这个模式

141 只能判断是否有环。本题要找入口，需要利用 Floyd 判圈的数学性质。

设：

- 头到入口距离为 `a`。
- 入口到相遇点距离为 `b`。
- 相遇点再回入口距离为 `c`。

快指针走了慢指针两倍路程，推导可得：从头到入口的距离，等于从相遇点继续走到入口的距离加若干圈。

## 状态定义 / 数据结构

- `slow/fast`：第一阶段检测相遇。
- `ptr`：第二阶段从头出发。
- `slow`：第二阶段从相遇点出发。

返回入口节点；无环返回 nil。

## 核心不变量

第一阶段：

- 有环则快慢必在环内相遇。
- 无环则 fast 到达 nil。

第二阶段：

- `ptr` 从 head 出发，`slow` 从相遇点出发。
- 两者每次走一步。
- 它们第一次相遇的位置就是环入口。

## 手推全过程

链表：

```text
3 -> 2 -> 0 -> -4
     ^         |
     |_________|
```

第一阶段相遇在环内某点。第二阶段：

| 步骤 | ptr 从 head | slow 从相遇点 |
|------|-------------|---------------|
| 初始 | 3 | 相遇点 |
| 1 | 2 | 2 |

两者在节点 2 相遇，2 是入口。

## Go 代码实现

```go
func detectCycle(head *ListNode) *ListNode {
    if head == nil {
        return nil
    }
    slow := head
    fast := head
    for fast.Next != nil && fast.Next.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
        if slow == fast {
            ptr := head
            for ptr != slow {
                ptr = ptr.Next
                slow = slow.Next
            }
            return ptr
        }
    }
    return nil
}
```

## 错误写法与反例

### 错误 1：相遇点直接当入口返回

相遇点不一定是入口。入口到相遇点可能还有距离，必须第二阶段同步走。

### 错误 2：第二阶段 fast 继续走两步

第二阶段必须两个指针同速走一步，否则数学关系不成立。

### 错误 3：循环条件漏判 `fast.Next`

和 141 一样，`fast.Next.Next` 前必须保证 `fast.Next` 非空。

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为链表环入口可以通过 Floyd 快慢指针相遇后的距离关系定位。

2. 这个算法维护的不变量是什么？
   第一阶段快慢速度差保证有环必相遇；第二阶段两个指针到入口的剩余距离相同。

3. 每一步为什么不会漏答案？
   有环时相遇点一定在环内；从 head 和相遇点同步走会在入口第一次相遇。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是把第一次相遇点当入口，会在入口不等于相遇点时返回错误节点。
