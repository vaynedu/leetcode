# 23. 合并 K 个排序链表

## 题目信息

- 题号：LeetCode 23
- 标题：Merge k Sorted Lists
- 难度：困难（Hard）
- 模式：堆 / 多路归并

## 一句话思路

把每条链表的当前头节点放入小根堆，每次弹出全局最小节点接到结果后，再把它的后继节点入堆。

## 为什么想到这个模式

每条链表内部已经升序。要合并 K 条有序链表，本质是 K 路归并：

- 当前答案的下一个节点，一定是 K 条链表当前头节点中的最小者。
- 取出某条链表的头节点后，这条链表的下一个节点才成为新的候选。

这正是优先队列适用场景：动态维护一组候选中的最小值。

## 状态定义 / 数据结构

- `MinHeap`：保存每条链表的当前候选节点。
- 堆元素：`node` 和 `heapIdx`。
- `dummy/curr`：结果链表的虚拟头和当前尾节点。

堆中最多有 K 个元素，因为每条链表最多放一个当前候选。

## 核心不变量

任意时刻：

- 堆中保存所有还未合并链表的当前最小候选。
- 堆顶是全局下一个应该接入结果链表的节点。
- 某个节点被弹出后，只有它的后继节点可能成为该链表新的候选。

结果链表始终保持升序，因为每次接入的都是所有候选中的最小节点。

## 手推全过程

输入：

```text
[
  1 -> 4 -> 5,
  1 -> 3 -> 4,
  2 -> 6
]
```

| 步骤 | 弹出 | 后继入堆 | 结果 |
|------|------|----------|------|
| 1 | 1(链1) | 4(链1) | 1 |
| 2 | 1(链2) | 3(链2) | 1 -> 1 |
| 3 | 2(链3) | 6(链3) | 1 -> 1 -> 2 |
| 4 | 3(链2) | 4(链2) | 1 -> 1 -> 2 -> 3 |
| 5 | 4(链1) | 5(链1) | 1 -> 1 -> 2 -> 3 -> 4 |
| 6 | 4(链2) | nil | 1 -> 1 -> 2 -> 3 -> 4 -> 4 |
| 7 | 5(链1) | nil | 1 -> 1 -> 2 -> 3 -> 4 -> 4 -> 5 |
| 8 | 6(链3) | nil | 1 -> 1 -> 2 -> 3 -> 4 -> 4 -> 5 -> 6 |

## Go 代码实现

```go
func mergeKLists(lists []*ListNode) *ListNode {
    h := &MinHeap{}
    heap.Init(h)

    for i, node := range lists {
        if node != nil {
            heap.Push(h, &MinHeapItem{node: node, heapIdx: i})
        }
    }

    dummy := &ListNode{}
    curr := dummy

    for h.Len() > 0 {
        item := heap.Pop(h).(*MinHeapItem)
        curr.Next = item.node
        curr = curr.Next
        if item.node.Next != nil {
            heap.Push(h, &MinHeapItem{node: item.node.Next, heapIdx: item.heapIdx})
        }
    }

    return dummy.Next
}
```

## 错误写法与反例

### 错误 1：一次性把所有节点入堆

功能上可行，但空间会从 O(k) 变成 O(n)。多路归并只需要每条链表保留一个当前候选。

### 错误 2：弹出节点后忘记把后继入堆

```go
item := heap.Pop(h).(*MinHeapItem)
curr.Next = item.node
```

反例：`1 -> 4 -> 5` 会只合并 `1`，后面的 `4,5` 丢失。

### 错误 3：没有跳过 nil 头节点

把 nil 放入堆会在比较 `node.Val` 时 panic。初始化时必须跳过 nil 链表。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n log k)，n 是总节点数 |
| 空间 | O(k) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为每一步只需要从 K 条链表的当前候选中取最小值，是典型多路归并。

2. 这个算法维护的不变量是什么？
   堆里每条未结束链表最多保存一个当前候选，堆顶始终是全局最小候选。

3. 每一步为什么不会漏答案？
   某条链表的后继只有在当前节点被弹出后才可能参与竞争，弹出后立即入堆，候选集合始终完整。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是弹出后忘记加入后继节点，会导致整条链表后续节点被漏掉。
