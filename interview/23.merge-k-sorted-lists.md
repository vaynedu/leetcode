# 23. 合并K个排序链表

## 题目
给你一个链表数组，每个链表都是升序的。将它们合并成一个升序链表返回。

## 核心思想

**最小堆（小顶堆）**

把所有链表的头节点加入最小堆，每次弹出最小的，接入结果链表，并将其后继入堆。

```
初始堆：所有链表头节点（跳过 nil）
弹出最小 → 接入结果 → 后继入堆
重复直到堆空
```

## 代码实现

```go
type ListNode struct {
    Val  int
    Next *ListNode
}

type MinHeapItem struct {
    node   *ListNode
    heapIdx int // 链表下标，值相等时区分
}

type MinHeap []*MinHeapItem

func (h MinHeap) Len() int           { return len(h) }
func (h MinHeap) Less(i, j int) bool {
    if h[i].node.Val != h[j].node.Val {
        return h[i].node.Val < h[j].node.Val
    }
    return h[i].heapIdx < h[j].heapIdx
}
func (h *MinHeap) Swap(i, j int)     { (*h)[i], (*h)[j] = (*h)[j], (*h)[i] }
func (h *MinHeap) Push(x interface{}) { *h = append(*h, x.(*MinHeapItem)) }
func (h *MinHeap) Pop() interface{} {
    n := len(*h)
    item := (*h)[n-1]
    *h = (*h)[:n-1]
    return item
}

func mergeKLists(lists []*ListNode) *ListNode {
    h := &MinHeap{}
    heap.Init(h)

    // 初始化：所有链表头入堆
    for i, node := range lists {
        if node != nil {
            heap.Push(h, &MinHeapItem{node: node, heapIdx: i})
        }
    }

    dummy := &ListNode{}
    curr := dummy

    for h.Len() > 0 {
        item := heap.Pop(h).(*MinHeapItem)
        curr.Next = item.node
        curr = curr.Next
        if item.node.Next != nil {
            heap.Push(h, &MinHeapItem{node: item.node.Next, heapIdx: item.heapIdx})
        }
    }

    return dummy.Next
}
```

## trace — [[1,4,5], [1,3,4], [2,6]]

**Step 1:** Pop 1₁ → result: [1] → 4₁ 入堆  
**Step 2:** Pop 1₂ → result: [1,1] → 3₂ 入堆  
**Step 3:** Pop 2₃ → result: [1,1,2] → 6₃ 入堆  
**Step 4:** Pop 3₂ → result: [1,1,2,3] → 4₂ 入堆  
**Step 5:** Pop 4₁ → result: [1,1,2,3,4] → 5₁ 入堆  
**Step 6:** Pop 4₂ → result: [1,1,2,3,4,4] → nil 不入堆  
**Step 7:** Pop 5₁ → result: [1,1,2,3,4,4,5] → nil  
**Step 8:** Pop 6₃ → result: [1,1,2,3,4,4,5,6] → 完成

## 复杂度

| 指标 | 值 |
|------|-----|
| 时间 | O(n log k) — n=总节点数，k=链表数 |
| 空间 | O(k) — 堆中最多 k 个元素 |

## 面试追问

**Q：为什么不用分治归并？**
A：可以。分治是两两合并，复杂度同样是 O(n log k)，但代码更简洁（类似归并排序）。堆的好处是逻辑直观，适合理解。

**Q：heapIdx 的作用？**
A：Go 的 `container/heap` 用 `Less(i,j)` 比较两个元素。当两个链表节点值相等时（Val 相同），需要用额外下标区分，保证排序稳定。否则相等的节点顺序不确定，可能导致结果不稳定。

**Q：k=1 和 k=n 的边界情况？**
A：k=1 时直接返回那条链表，不需要堆。k=0 或所有链表都 nil 时返回 nil。代码中 `if node != nil` 已经处理了空链表的情况。

**Q：和顺序合并相比？**
A：顺序合并两两合并，每次合并 O(n)，共合并 k-1 次 → O(nk)。堆优化到 O(n log k)，当 k 很大时差异明显。
