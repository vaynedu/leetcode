# 90. 子集 II

## 题目信息

- 题号：LeetCode 90
- 标题：Subsets II
- 难度：中等（Medium）
- 模式：回溯 / 子集去重

## 一句话思路

先排序，仍然每个节点都收集答案；同一层遇到相同值时跳过后面的重复分支。

## 为什么想到这个模式

题目要求所有不重复子集，主体仍是子集回溯。区别是 `nums` 中有重复数字，如果不剪枝，会生成重复子集。

排序后相同值相邻，同层去重条件是：

```go
if i > start && nums[i] == nums[i-1] {
    continue
}
```

`i > start` 表示只剪同一层的重复分支，不剪递归深层的合法重复选择。

## 状态定义 / 数据结构

- `nums`：排序后的数组。
- `path`：当前子集。
- `start`：下一层可选起点。
- `res`：所有不重复子集。

递归含义：

```text
backtrack(start) = 在当前 path 下，从 start..n-1 中选择后续元素，并跳过同层重复
```

## 核心不变量

任意时刻：

- `path` 下标递增。
- 每个递归节点都代表一个合法子集。
- 同一层相同值只开启第一个分支，后续相同值分支被跳过。

同一值在不同层可以被选择，例如 `[2,2]` 是合法子集；不能用全局去重把它剪掉。

## 手推全过程

以 `nums = [1,2,2]` 为例：

```text
[]                    收集
├── [1]               收集
│   └── [1,2a]        收集
│       └── [1,2a,2b] 收集
│   └── 2b 同层重复，跳过
├── [2a]              收集
│   └── [2a,2b]       收集
└── 2b 同层重复，跳过
```

答案：

```text
[], [1], [1,2], [1,2,2], [2], [2,2]
```

## Go 代码实现

```go
func subsetsWithDup(nums []int) [][]int {
    sort.Ints(nums)
    res := [][]int{}
    path := []int{}

    var backtrack func(start int)
    backtrack = func(start int) {
        tmp := make([]int, len(path))
        copy(tmp, path)
        res = append(res, tmp)

        for i := start; i < len(nums); i++ {
            if i > start && nums[i] == nums[i-1] {
                continue
            }
            path = append(path, nums[i])
            backtrack(i + 1)
            path = path[:len(path)-1]
        }
    }

    backtrack(0)
    return res
}
```

## 错误写法与反例

### 错误 1：去重条件写成 `i > 0`

```go
if i > 0 && nums[i] == nums[i-1] {
    continue
}
```

反例：`[2,2]`。

这会把第二个 `2` 在深层也跳过，导致漏掉合法子集 `[2,2]`。必须是 `i > start`，只剪同层。

### 错误 2：不排序就去重

相同值不相邻时无法判断重复分支。必须先 `sort.Ints(nums)`。

### 错误 3：只在叶子节点收集

子集题每个节点都是答案，叶子收集会漏掉短子集。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n * 2^n)，重复值会剪掉部分分支 |
| 空间 | O(n)，不计结果 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要枚举所有子集，同时需要对重复数字造成的同层等价分支做剪枝。

2. 这个算法维护的不变量是什么？
   `path` 下标递增；同一层相同值只选择第一个分支，递归深层允许选择重复值。

3. 每一步为什么不会漏答案？
   被剪掉的是同层重复值产生的等价子集，保留的第一个分支已经覆盖它们。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是 `i > start`。写成 `i > 0` 会剪掉不同层的合法重复选择，漏掉 `[2,2]`。
