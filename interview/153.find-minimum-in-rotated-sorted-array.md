# 153. 寻找旋转排序数组中的最小值

## 题目信息

- 题号：LeetCode 153
- 标题：Find Minimum in Rotated Sorted Array
- 难度：中等（Medium）
- 模式：二分查找 / 旋转数组

## 一句话思路

比较 `nums[mid]` 和 `nums[right]`：如果 `nums[mid] > nums[right]`，最小值在右半边；否则最小值在左半边含 `mid`。

## 为什么想到这个模式

数组原本升序，旋转后变成两段升序数组，最小值就是旋转断点。题目没有重复元素，所以每次比较 `mid` 和 `right` 都能判断最小值在哪一侧。

选择二分的信号：

- 数组由有序数组旋转而来。
- 要找断点或最小值。
- 可以通过 `nums[mid]` 与右端点大小关系排除一半。

## 状态定义 / 数据结构

- `left`：候选区间左边界。
- `right`：候选区间右边界。
- `mid`：中点。
- 候选区间：闭区间 `[left, right]`。

目标：最小值始终在 `[left, right]` 内。

## 核心不变量

循环过程中，`[left, right]` 始终包含最小值。

比较规则：

- 如果 `nums[mid] > nums[right]`，说明 `mid` 落在左侧较大段，最小值一定在 `mid+1...right`。
- 否则 `nums[mid] < nums[right]`，说明 `mid...right` 是有序段，最小值在 `left...mid`，可能就是 `mid`。

因为没有重复元素，不需要处理 `nums[mid] == nums[right]`。

## 手推全过程

以 `nums = [4,5,6,7,0,1,2]` 为例：

| left | mid | right | 值 | 判断 | 决策 |
|------|-----|-------|----|------|------|
| 0 | 3 | 6 | 4,7,2 | `7 > 2` | 最小值在右半，`left = 4` |
| 4 | 5 | 6 | 0,1,2 | `1 < 2` | 最小值在左半含 mid，`right = 5` |
| 4 | 4 | 5 | 0,0,1 | `0 < 1` | 最小值在左半含 mid，`right = 4` |

`left == right == 4`，返回 `nums[4] = 0`。

## Go 代码实现

```go
func findMin153(nums []int) int {
    if len(nums) == 0 {
        return 0
    }
    left, right := 0, len(nums)-1
    for left < right {
        mid := left + (right-left)/2
        if nums[mid] > nums[right] {
            left = mid + 1
        } else {
            right = mid
        }
    }
    return nums[left]
}
```

## 错误写法与反例

### 错误 1：`right = mid - 1`

```go
if nums[mid] < nums[right] {
    right = mid - 1
}
```

反例：`[3,4,5,1,2]`。

当 `nums[mid] < nums[right]` 时，`mid` 自己可能就是最小值，不能排除 `mid`，应写 `right = mid`。

### 错误 2：比较 `nums[mid]` 和 `nums[left]`

```go
if nums[mid] > nums[left] {
    left = mid + 1
}
```

反例：`[1,2,3,4]`。

数组没有旋转时最小值在左端，用左端比较很容易把左侧最小值排除。和 `right` 比较更稳定。

### 错误 3：循环写成 `left <= right`

```go
for left <= right {
    ...
}
```

本题是收缩到一个候选点，循环条件应是 `left < right`。如果写成 `<=`，在 `left == right` 时仍会继续更新，容易死循环或越界。

## 四问复盘

1. 这题为什么属于这个模式？
   因为旋转数组由两段有序区间组成，最小值是断点，可以通过和右端点比较二分定位。

2. 这个算法维护的不变量是什么？
   最小值始终保留在闭区间 `[left, right]` 中。

3. 每一步为什么不会漏答案？
   `nums[mid] > nums[right]` 时最小值必在右侧；否则 `mid` 可能是最小值，所以保留 `mid` 并收缩右边界。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是写成 `right = mid - 1`，会在 `mid` 就是最小值时把答案排除。
