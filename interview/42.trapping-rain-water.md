# 42. 接雨水

## 题目信息

- 题号：LeetCode 42
- 标题：Trapping Rain Water
- 难度：困难（Hard）
- 模式：双指针 / 边界最大值

## 一句话思路

每个位置能接水取决于左右两侧最高墙的较小值；双指针每次处理较低的一侧，因为另一侧已有足够边界。

## 为什么想到这个模式

位置 `i` 的水量是：

```text
min(leftMax[i], rightMax[i]) - height[i]
```

暴力或预处理数组都可以做。双指针优化的关键是：如果 `height[left] < height[right]`，左侧当前位置的右边界至少有 `height[right]`，因此可以安全用 `leftMax` 结算左侧。

## 状态定义 / 数据结构

- `left/right`：当前待处理区间两端。
- `leftMax`：从左到当前为止的最高墙。
- `rightMax`：从右到当前为止的最高墙。
- `water`：累计雨水量。

## 核心不变量

`leftMax` 和 `rightMax` 分别是已扫描区域的最高边界。

每轮处理较低侧：

- 若 `height[left] < height[right]`，左侧位置的水量只受 `leftMax` 限制。
- 否则处理右侧，水量只受 `rightMax` 限制。

被处理的位置以后不需要再回头。

## 手推全过程

`height = [0,1,0,2,1,0,1,3,2,1,2,1]`

几个关键位置：

| 位置 | 高度 | 左最高 | 右侧有墙 | 水量 |
|------|------|--------|----------|------|
| 2 | 0 | 1 | 右侧有 2/3 | 1 |
| 4 | 1 | 2 | 右侧有 3 | 1 |
| 5 | 0 | 2 | 右侧有 3 | 2 |
| 6 | 1 | 2 | 右侧有 3 | 1 |
| 9 | 1 | 左侧有 3 | rightMax=2 | 1 |

总水量是 `6`。

## Go 代码实现

```go
func trap(height []int) int {
    if len(height) == 0 {
        return 0
    }
    left, right := 0, len(height)-1
    leftMax, rightMax := 0, 0
    water := 0
    for left < right {
        if height[left] < height[right] {
            if height[left] >= leftMax {
                leftMax = height[left]
            } else {
                water += leftMax - height[left]
            }
            left++
        } else {
            if height[right] >= rightMax {
                rightMax = height[right]
            } else {
                water += rightMax - height[right]
            }
            right--
        }
    }
    return water
}
```

## 错误写法与反例

### 错误 1：从高的一侧处理

高侧无法确定另一边是否有足够边界，可能提前计算错误。双指针必须处理较低侧。

### 错误 2：用当前高度直接相减

水量不是 `height[right] - height[left]`，而是由两侧历史最高墙决定。

### 错误 3：负水量也累加

当当前柱子高于历史最高墙时，应该更新 max，而不是累加负值。

## 四问复盘

1. 这题为什么属于这个模式？
   因为水量由左右边界共同决定，双指针可从两端向中间安全结算较低侧。

2. 这个算法维护的不变量是什么？
   `leftMax/rightMax` 分别保存已扫描两侧最高墙，处理较低侧时另一侧已提供足够边界。

3. 每一步为什么不会漏答案？
   每个位置只会被左或右指针处理一次，处理时它的限制边界已经确定。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是处理高侧或用当前边界代替历史最高墙，会导致水量少算或多算。
