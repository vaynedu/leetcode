# 46. 全排列

## 题目信息

- 题号：LeetCode 46
- 标题：Permutations
- 难度：中等（Medium）
- 模式：回溯 / 排列

## 一句话思路

每一层决定当前位置放哪个还没用过的数字，用 `used` 标记已选择元素，路径长度等于 `n` 时收集答案。

## 为什么想到这个模式

题目要求返回所有可能的排列，本质是在枚举所有决策路径：

```text
第 0 位选谁 → 第 1 位选谁 → ... → 第 n-1 位选谁
```

每一步都有多个选择，选择后进入下一层，返回时撤销选择继续试其它分支。这是典型回溯。

排列和组合的区别：

- 组合/子集通常用 `start` 控制只能向后选。
- 排列每一层都可以从所有数字里选，只是不能重复使用同一个下标。

## 状态定义 / 数据结构

- `path`：当前已经构造出的排列前缀。
- `used[i]`：`nums[i]` 是否已经放入当前 `path`。
- `res`：所有完整排列。

递归含义：

```text
backtrack() = 在当前 path 前缀下，继续填下一个位置
```

## 核心不变量

任意时刻：

- `path` 中的元素都来自 `nums`，且每个下标最多使用一次。
- `used[i] == true` 当且仅当 `nums[i]` 已经在当前 `path` 中。
- `len(path)` 表示当前正在填第几个位置。

当 `len(path) == len(nums)` 时，当前路径就是一个完整排列，必须复制后加入结果。

## 手推全过程

以 `nums = [1,2,3]` 为例：

```text
[]
├── [1]
│   ├── [1,2]
│   │   └── [1,2,3]
│   └── [1,3]
│       └── [1,3,2]
├── [2]
│   ├── [2,1]
│   │   └── [2,1,3]
│   └── [2,3]
│       └── [2,3,1]
└── [3]
    ├── [3,1]
    │   └── [3,1,2]
    └── [3,2]
        └── [3,2,1]
```

共 `3! = 6` 个排列。

## Go 代码实现

```go
func permute(nums []int) [][]int {
    res := [][]int{}
    path := []int{}
    used := make([]bool, len(nums))

    var backtrack func()
    backtrack = func() {
        if len(path) == len(nums) {
            tmp := make([]int, len(path))
            copy(tmp, path)
            res = append(res, tmp)
            return
        }
        for i := 0; i < len(nums); i++ {
            if used[i] {
                continue
            }
            used[i] = true
            path = append(path, nums[i])
            backtrack()
            path = path[:len(path)-1]
            used[i] = false
        }
    }

    backtrack()
    return res
}
```

## 错误写法与反例

### 错误 1：忘记撤销选择

```go
path = append(path, nums[i])
used[i] = true
backtrack()
```

如果不执行 `path = path[:len(path)-1]` 和 `used[i] = false`，后续分支会继承前一个分支状态，无法枚举其它排列。

### 错误 2：保存答案时不复制

```go
res = append(res, path)
```

`path` 底层数组会在回溯中被复用，直接保存可能导致结果被后续修改污染。应复制一份。

### 错误 3：使用 `start` 限制选择

```go
backtrack(start int)
for i := start; i < len(nums); i++ { ... }
```

这会变成组合/子集枚举，无法生成 `[2,1,3]` 这种顺序不同的排列。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n * n!) |
| 空间 | O(n)，不计结果 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要枚举所有排列，每个位置都有多个选择，选择后递归，返回后撤销选择。

2. 这个算法维护的不变量是什么？
   `path` 是当前排列前缀，`used` 精确标记哪些下标已被放入 `path`。

3. 每一步为什么不会漏答案？
   每一层都尝试所有未使用元素作为当前位置，递归树覆盖所有 `n!` 种下标顺序。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是保存答案不复制或撤销选择不完整，会导致结果被污染或分支缺失。
