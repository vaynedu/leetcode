# 102. 二叉树层序遍历

## 题目信息

- 题号：LeetCode 102
- 标题：Binary Tree Level Order Traversal
- 难度：中等（Medium）
- 模式：树 / BFS / 队列

## 一句话思路

用队列做 BFS；每轮先记录当前队列长度 `size`，只处理这 `size` 个节点，正好得到一层。

## 为什么想到这个模式

题目要求“逐层、从左到右”访问二叉树。层序遍历天然是 BFS：先访问离根最近的一层，再访问下一层。

DFS 也能带深度收集，但 BFS 更直接，因为队列的先进先出顺序正好保证同一层节点按从左到右处理。

## 状态定义 / 数据结构

- `queue`：待访问节点队列。
- `size`：当前层开始时队列中的节点数。
- `level`：当前层的值列表。
- `result`：所有层的结果。

关键是 `size := len(queue)` 必须在处理当前层之前记录，后续入队的子节点属于下一层，不能影响当前层循环。

## 核心不变量

每轮外层循环开始时，队列中保存的正好是当前层的所有节点。

处理这一层时：

- 出队 `size` 次，只消费当前层节点。
- 每个节点的左右子节点按顺序入队，成为下一层。
- 当前层处理完后，队列中正好是下一层节点。

## 手推全过程

树：

```text
     3
    / \
   9  20
     /  \
    15   7
```

| 轮次 | 开始队列 | size | 当前层 | 结束队列 |
|------|----------|------|--------|----------|
| 1 | `[3]` | 1 | `[3]` | `[9,20]` |
| 2 | `[9,20]` | 2 | `[9,20]` | `[15,7]` |
| 3 | `[15,7]` | 2 | `[15,7]` | `[]` |

答案：`[[3],[9,20],[15,7]]`。

## Go 代码实现

```go
func levelOrder(root *TreeNode) [][]int {
    if root == nil {
        return nil
    }

    result := [][]int{}
    queue := []*TreeNode{root}

    for len(queue) > 0 {
        size := len(queue)
        level := []int{}
        for i := 0; i < size; i++ {
            node := queue[0]
            queue = queue[1:]
            level = append(level, node.Val)
            if node.Left != nil {
                queue = append(queue, node.Left)
            }
            if node.Right != nil {
                queue = append(queue, node.Right)
            }
        }
        result = append(result, level)
    }
    return result
}
```

## 错误写法与反例

### 错误 1：循环条件直接用动态 `len(queue)`

```go
for i := 0; i < len(queue); i++ { ... }
```

处理当前层时会把刚入队的下一层节点也一起处理，导致层级混在一起。必须先固定 `size`。

### 错误 2：用栈代替队列

栈是后进先出，会变成 DFS 顺序，不能保证按层从左到右。

### 错误 3：空树返回 `[[]]`

空树没有任何层，通常应返回 `nil` 或空切片；不能构造一个空层。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(w)，w 是最大层宽，最坏 O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要求按层访问，BFS 队列天然按距离根节点的层次推进。

2. 这个算法维护的不变量是什么？
   每轮开始时队列中正好是当前层节点，处理完后队列中正好是下一层节点。

3. 每一步为什么不会漏答案？
   每个非空节点入队一次、出队一次；出队时记录值，并把左右子节点加入下一层。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是不固定当前层 `size`，会把下一层节点提前消费，导致结果分层错误。
