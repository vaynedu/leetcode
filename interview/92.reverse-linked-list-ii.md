# 92. 反转链表 II

## 题目信息

- 题号：LeetCode 92
- 标题：Reverse Linked List II
- 难度：中等（Medium）
- 模式：链表 / 局部反转 / 头插法

## 一句话思路

先让 `prev` 指向 `left` 前一个节点，再用头插法把区间内 `cur.Next` 逐个移动到 `prev.Next`，重复 `right-left` 次。

## 为什么想到这个模式

题目只反转链表中间一段 `[left, right]`，区间外节点顺序不变。

链表局部反转有两个关键连接点：

```text
left 前一个节点 prev
反转区间的尾节点 cur，也就是原 left 节点
```

用头插法可以在一次遍历中完成反转，不需要额外数组。

## 状态定义 / 数据结构

- `dummy`：虚拟头，处理 `left == 1` 时头节点变化。
- `prev`：反转区间前一个节点，即第 `left-1` 个节点。
- `cur`：反转区间当前尾节点，初始为第 `left` 个节点。
- `next`：每轮从 `cur` 后面剪下、插到 `prev` 后面的节点。

每轮操作：

```text
next = cur.Next
cur.Next = next.Next
next.Next = prev.Next
prev.Next = next
```

## 核心不变量

每轮循环开始时：

- `prev.Next` 是当前已经反转好的区间头。
- `cur` 是当前已经反转区间的尾，也是原来的 `left` 节点。
- `cur.Next` 是下一个要被移动到区间头部的节点。

执行 `right-left` 次后，区间 `[left,right]` 完成反转。

## 手推全过程

输入：

```text
1 -> 2 -> 3 -> 4 -> 5
left=2, right=4
```

初始：

```text
prev=1, cur=2
1 -> 2 -> 3 -> 4 -> 5
```

第一轮，把 `3` 插到 `prev` 后：

```text
1 -> 3 -> 2 -> 4 -> 5
```

第二轮，把 `4` 插到 `prev` 后：

```text
1 -> 4 -> 3 -> 2 -> 5
```

答案：

```text
1 -> 4 -> 3 -> 2 -> 5
```

## Go 代码实现

```go
func reverseBetween(head *ListNode, left int, right int) *ListNode {
    dummy := &ListNode{}
    dummy.Next = head
    prev := dummy
    for i := 0; i < left-1; i++ {
        prev = prev.Next
    }
    cur := prev.Next
    for i := 0; i < right-left; i++ {
        next := cur.Next
        cur.Next = next.Next
        next.Next = prev.Next
        prev.Next = next
    }
    return dummy.Next
}
```

## 错误写法与反例

### 错误 1：不用 dummy 处理 `left == 1`

反例：`[1,2,3]，left=1,right=2`

反转后新头是 `2`。没有 dummy 时容易返回旧头 `1`。

### 错误 2：循环次数写成 `right-left+1`

头插法每次移动的是 `cur.Next`，区间有 `right-left+1` 个节点，但只需要移动后面的 `right-left` 个节点。

### 错误 3：移动 `cur` 指针

头插法中 `cur` 应始终是反转区间尾节点。如果每轮把 `cur = cur.Next`，会跳过节点或断链。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，最多遍历到 right |
| 空间 | O(1) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要原地反转链表的一段连续区间，头插法能稳定维护区间前驱和区间尾。

2. 这个算法维护的不变量是什么？
   `prev.Next` 是已反转区间头，`cur` 是已反转区间尾，`cur.Next` 是下一个待头插节点。

3. 每一步为什么不会漏答案？
   每轮把一个节点从 `cur` 后剪下插到区间头，执行 `right-left` 轮正好反转整个区间。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是 `left == 1`，如果没有 dummy，会丢失反转后的新头节点。
