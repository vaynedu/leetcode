# 112. 路径总和

## 题目信息

- 题号：LeetCode 112
- 标题：Path Sum
- 难度：简单（Easy）
- 模式：二叉树 DFS / 根到叶路径

## 一句话思路

从根节点向下递归，每经过一个节点就从目标值中减去节点值；只有到达叶子节点时，剩余目标正好等于叶子值才返回 `true`。

## 为什么想到这个模式

题目要求判断是否存在一条：

```text
根节点 → 叶子节点
```

的路径，使路径和等于 `targetSum`。

路径必须从根开始，并且必须到叶子结束，所以每一步只能向左子树或右子树继续试。这是典型二叉树 DFS。

## 状态定义 / 数据结构

- `root`：当前走到的节点。
- `targetSum`：从当前节点到某个叶子，还需要凑出的目标和。

递归含义：

```text
HasPathSum(node, remain)
= 从 node 出发，是否存在一条到叶子的路径，路径和等于 remain
```

## 核心不变量

任意递归调用中：

- `targetSum` 表示当前节点到叶子这段路径还需要凑出的和。
- 进入子节点时，目标值变为 `targetSum - root.Val`。
- 只有当前节点是叶子时，才能判断路径是否完整。

不能在非叶子节点提前因为当前累计和等于目标就返回 `true`。

## 手推全过程

例子：

```text
        5
       / \
      4   8
     /   / \
    11  13  4
   /  \      \
  7    2      1

targetSum = 22
```

递归过程：

```text
HasPathSum(5, 22)
→ HasPathSum(4, 17)
  → HasPathSum(11, 13)
    → HasPathSum(7, 2)  // 叶子，但 7 != 2，失败
    → HasPathSum(2, 2)  // 叶子，2 == 2，成功
```

路径是：

```text
5 -> 4 -> 11 -> 2
```

## Go 代码实现

```go
func HasPathSum(root *TreeNode, targetSum int) bool {
    if root == nil {
        return false
    }
    if root.Left == nil && root.Right == nil {
        return root.Val == targetSum
    }
    return HasPathSum(root.Left, targetSum-root.Val) ||
        HasPathSum(root.Right, targetSum-root.Val)
}
```

## 错误写法与反例

### 错误 1：非叶子节点就判断成功

```go
if root.Val == targetSum {
    return true
}
```

反例：

```text
root = [1,2], targetSum = 1
```

根节点值等于目标，但路径必须到叶子。正确答案是 `false`。

### 错误 2：空树 target 为 0 返回 true

题目要求路径必须存在。空树没有根到叶路径，所以 `root == nil` 时应返回 `false`。

### 错误 3：把 112 和 437 混淆

112 必须从根到叶；437 可以从任意节点开始、任意后代结束。两题的路径定义不同，不能套同一套判断。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，最坏访问所有节点 |
| 空间 | O(h)，递归栈高度，最坏 O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为要在二叉树中寻找根到叶子的路径，每一步递归选择左子树或右子树。

2. 这个算法维护的不变量是什么？
   `targetSum` 始终表示当前节点到某个叶子还需要凑出的路径和。

3. 每一步为什么不会漏答案？
   每个非空节点都会继续尝试左右子树，所有根到叶路径都会被检查一次。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是在非叶子节点提前返回成功，会把未结束的路径误判为合法根到叶路径。
