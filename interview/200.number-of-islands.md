# 200. 岛屿数量

## 题目信息

- 题号：LeetCode 200
- 标题：Number of Islands
- 难度：中等（Medium）
- 模式：图 / 网格 DFS

## 一句话思路

遍历网格，每遇到一个未访问的陆地 `'1'` 就计数加一，并用 DFS 把整座岛标记为已访问。

## 为什么想到这个模式

网格中的每个陆地格子可以看成图节点，上下左右相邻的陆地之间有边。一个岛就是一个连通块。题目要求岛屿数量，本质是数连通块数量。

DFS/BFS 都可以；当前实现用 DFS 原地把访问过的陆地改成 `'0'`。

## 状态定义 / 数据结构

- `grid`：输入网格，同时作为 visited 标记。
- `count`：已发现岛屿数量。
- `dfs(i,j)`：从 `(i,j)` 出发，把同一个岛的所有陆地标记为水。
- `dirs`：四个方向。

## 核心不变量

遍历过程中：

- 已经被 DFS 访问过的陆地都会被改成 `'0'`。
- 每次外层循环遇到 `'1'`，它一定属于一座之前没统计过的新岛。
- 一次 DFS 会消除这座岛的所有连通陆地，避免重复计数。

## 手推全过程

网格：

```text
1 1 0
1 0 0
0 0 1
```

遍历过程：

| 位置 | 值 | 动作 | count |
|------|----|------|-------|
| (0,0) | 1 | 发现新岛，DFS 标记 (0,0)(0,1)(1,0) | 1 |
| 后续左上连通块 | 0 | 已访问，跳过 | 1 |
| (2,2) | 1 | 发现新岛，DFS 标记 (2,2) | 2 |

答案是 `2`。

## Go 代码实现

```go
func numIslands(grid [][]byte) int {
    if len(grid) == 0 || len(grid[0]) == 0 {
        return 0
    }
    m, n := len(grid), len(grid[0])
    count := 0
    dirs := [][]int{{-1, 0}, {1, 0}, {0, -1}, {0, 1}}

    var dfs func(i, j int)
    dfs = func(i, j int) {
        if i < 0 || i >= m || j < 0 || j >= n || grid[i][j] == '0' {
            return
        }
        grid[i][j] = '0'
        for _, d := range dirs {
            dfs(i+d[0], j+d[1])
        }
    }

    for i := 0; i < m; i++ {
        for j := 0; j < n; j++ {
            if grid[i][j] == '1' {
                count++
                dfs(i, j)
            }
        }
    }
    return count
}
```

## 错误写法与反例

### 错误 1：DFS 不标记访问

如果不把访问过的 `'1'` 改成 `'0'`，相邻格子会互相递归回来，导致无限递归或重复计数。

### 错误 2：把对角线也算相邻

题目只允许上下左右。反例：

```text
1 0
0 1
```

正确答案是 2，不是 1。

### 错误 3：发现 `'1'` 后只计数不淹岛

同一座岛的每个格子都会被外层循环重复计数，答案偏大。

| 时间 | O(mn) |
| 空间 | O(mn) 最坏递归栈 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为岛屿就是网格图中的连通块，统计岛屿数量就是统计连通块数量。

2. 这个算法维护的不变量是什么？
   已访问陆地会被标记为水；外层遇到的每个 `'1'` 都是一座新岛的起点。

3. 每一步为什么不会漏答案？
   外层遍历覆盖所有格子，DFS 从任一陆地能覆盖同一连通块的全部陆地。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是访问标记和方向定义。不标记会重复访问；加入对角线会把不同岛错误合并。
