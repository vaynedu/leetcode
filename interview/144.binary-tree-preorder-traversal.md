# 144. 二叉树前序遍历

## 题目信息

- 题号：LeetCode 144
- 标题：Binary Tree Preorder Traversal
- 难度：简单（Easy）
- 模式：二叉树 DFS / 栈

## 一句话思路

前序遍历顺序是 `根 -> 左 -> 右`；递归版按这个顺序访问，迭代版用栈时要先压右节点再压左节点。

## 为什么想到这个模式

题目直接要求二叉树前序遍历。前序的定义就是：

```text
先访问当前节点，再访问左子树，最后访问右子树
```

递归天然匹配树结构；迭代版则用显式栈模拟递归调用顺序。

## 状态定义 / 数据结构

- `result`：已经访问过的节点值。
- 递归版 `preorderRec(node)`：访问以 `node` 为根的子树。
- 迭代版 `stack`：保存待访问节点，栈顶是下一个要访问的节点。

## 核心不变量

递归版：

- 每个非空节点先加入 `result`，再递归左子树和右子树。

迭代版：

- 栈顶永远是下一个应该按前序访问的节点。
- 因为栈是后进先出，必须先压右子节点，再压左子节点，才能保证左子树先被访问。

## 手推全过程

例子：

```text
      1
     / \
    2   3
   / \
  4   5
```

前序遍历：

```text
1 -> 2 -> 4 -> 5 -> 3
```

迭代栈过程：

```text
pop 1，push 3，push 2
pop 2，push 5，push 4
pop 4
pop 5
pop 3
```

## Go 代码实现

```go
func Preorder144(root *TreeNode) []int {
    var result []int
    preorderRec(root, &result)
    return result
}

func preorderRec(node *TreeNode, result *[]int) {
    if node == nil {
        return
    }
    *result = append(*result, node.Val)
    preorderRec(node.Left, result)
    preorderRec(node.Right, result)
}

func Preorder144Iter(root *TreeNode) []int {
    var result []int
    if root == nil {
        return result
    }
    stack := []*TreeNode{root}
    for len(stack) > 0 {
        node := stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        result = append(result, node.Val)
        if node.Right != nil {
            stack = append(stack, node.Right)
        }
        if node.Left != nil {
            stack = append(stack, node.Left)
        }
    }
    return result
}
```

## 错误写法与反例

### 错误 1：迭代时先压左再压右

栈是后进先出。先压左再压右，会导致右子树先访问，变成 `根 -> 右 -> 左`。

### 错误 2：把访问根节点放在递归左子树之后

那就变成中序遍历，不是前序遍历。

### 错误 3：空树返回 nil/空切片混淆

两者长度都是 0，通常测试应检查内容长度为 0，而不是依赖 nil 细节。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，每个节点访问一次 |
| 空间 | O(h) 递归栈或显式栈，最坏 O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为题目要求二叉树前序遍历，本质是按“根、左、右”访问整棵树。

2. 这个算法维护的不变量是什么？
   `result` 中节点顺序始终符合已访问部分的前序顺序；迭代栈顶是下一个要访问节点。

3. 每一步为什么不会漏答案？
   每个非空节点都会被访问一次，并继续把左右子树纳入递归或栈中。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是迭代版压栈顺序，先压左会导致右子树先出栈，遍历顺序错误。
