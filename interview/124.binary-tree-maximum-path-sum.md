# 124. 二叉树中的最大路径和

## 题目信息

- 题号：LeetCode 124
- 标题：Binary Tree Maximum Path Sum
- 难度：困难（Hard）
- 模式：树 / 后序 DFS / 全局最优

## 一句话思路

DFS 返回“向父节点延伸的最大贡献”，同时用 `left + root + right` 更新全局最大路径和。

## 为什么想到这个模式

路径可以从任意节点到任意节点，但在某个节点处最多只能“拐弯”一次：左子树路径 + 当前节点 + 右子树路径。

父节点只能接收一条向下延伸的路径，不能同时接收左右两边，否则会分叉。因此需要区分：

- 返回给父节点的贡献：只能选一边。
- 当前节点作为拐点的答案：可以同时用左右两边。

## 状态定义 / 数据结构

递归返回值：

```text
dfs(node) = 从 node 出发，向下延伸到某一侧的最大路径贡献
```

全局变量：

```text
maxSum = 已见过的任意路径最大和
```

节点处候选答案：

```text
node.Val + max(0, left) + max(0, right)
```

负贡献直接丢弃。

## 核心不变量

每个 `dfs(node)` 返回的是一条可以被父节点继续连接的单链路径贡献。

而 `maxSum` 记录的是任意形态路径的最大值，可以在当前节点处同时包含左右子树贡献。

这两个值不能混用：返回值不能同时带左右两边。

## 手推全过程

树：

```text
   -10
   /  \
  9   20
      / \
     15  7
```

| 节点 | left | right | 当前拐点路径 | 返回贡献 |
|------|------|-------|--------------|----------|
| 9 | 0 | 0 | 9 | 9 |
| 15 | 0 | 0 | 15 | 15 |
| 7 | 0 | 0 | 7 | 7 |
| 20 | 15 | 7 | 42 | 35 |
| -10 | 9 | 35 | 34 | 25 |

最大路径和是 `42`，路径为 `15 -> 20 -> 7`。

## Go 代码实现

```go
func maxPathSum(root *TreeNode) int {
    maxSum := -1 << 63
    var dfs func(node *TreeNode) int
    dfs = func(node *TreeNode) int {
        if node == nil {
            return 0
        }
        left := dfs(node.Left)
        right := dfs(node.Right)
        pathSum := node.Val + max(0, left) + max(0, right)
        maxSum = max(maxSum, pathSum)
        return node.Val + max(0, max(left, right))
    }
    dfs(root)
    return maxSum
}
```

## 错误写法与反例

### 错误 1：返回左右两边总和

```go
return node.Val + max(0, left) + max(0, right)
```

返回给父节点时不能带左右两边，否则路径会分叉，不再是一条合法路径。

### 错误 2：不丢弃负贡献

反例：`[1,-2,3]`。负子树只会让路径和变小，应使用 `max(0, child)`。

### 错误 3：`maxSum` 初始化为 0

反例：单节点 `[-3]`，正确答案是 `-3`。初始化为 0 会错误返回 0。

## 四问复盘

1. 这题为什么属于这个模式？
   因为每个节点需要先拿到左右子树贡献，再决定当前节点作为拐点的最优路径。

2. 这个算法维护的不变量是什么？
   `dfs` 返回可向父节点延伸的单边最大贡献，`maxSum` 保存全局任意路径最大和。

3. 每一步为什么不会漏答案？
   任意最大路径都有一个最高拐点，遍历到该节点时会用左右贡献更新一次全局答案。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是全负数。`maxSum` 必须初始化为极小值，否则会把空路径 0 当成答案。
