# 145. 二叉树后序遍历

## 题目信息

- 题号：LeetCode 145
- 标题：Binary Tree Postorder Traversal
- 难度：简单（Easy）
- 模式：二叉树 DFS / 栈 / 反转技巧

## 一句话思路

后序遍历顺序是 `左 -> 右 -> 根`；递归版按定义访问，迭代版可先得到 `根 -> 右 -> 左`，最后反转。

## 为什么想到这个模式

题目直接要求二叉树后序遍历。后序的定义是：

```text
先处理左子树，再处理右子树，最后处理当前节点
```

递归最直接。迭代时，如果先访问根，再让右子树先于左子树进入结果，可得到：

```text
根 -> 右 -> 左
```

反转后就是：

```text
左 -> 右 -> 根
```

## 状态定义 / 数据结构

- `result`：遍历结果。
- 递归版 `postorderRec(node)`：收集以 `node` 为根的后序序列。
- 迭代版 `stack`：模拟遍历，先构造反向后序。

## 核心不变量

递归版：

- 当前节点只有在左右子树都处理完后才加入结果。

迭代版：

- 反转前的 `result` 是 `根 -> 右 -> 左` 的顺序。
- 最终反转后得到真正的 `左 -> 右 -> 根`。

## 手推全过程

例子：

```text
      1
     / \
    2   3
   / \
  4   5
```

后序遍历：

```text
4 -> 5 -> 2 -> 3 -> 1
```

迭代技巧：

```text
先得到：1 -> 3 -> 2 -> 5 -> 4
反转为：4 -> 5 -> 2 -> 3 -> 1
```

## Go 代码实现

```go
func Postorder145(root *TreeNode) []int {
    var result []int
    postorderRec(root, &result)
    return result
}

func postorderRec(node *TreeNode, result *[]int) {
    if node == nil {
        return
    }
    postorderRec(node.Left, result)
    postorderRec(node.Right, result)
    *result = append(*result, node.Val)
}

func Postorder145Iter(root *TreeNode) []int {
    var result []int
    if root == nil {
        return result
    }
    stack := []*TreeNode{root}
    for len(stack) > 0 {
        node := stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        result = append(result, node.Val)
        if node.Left != nil {
            stack = append(stack, node.Left)
        }
        if node.Right != nil {
            stack = append(stack, node.Right)
        }
    }
    for i, j := 0, len(result)-1; i < j; i, j = i+1, j-1 {
        result[i], result[j] = result[j], result[i]
    }
    return result
}
```

## 错误写法与反例

### 错误 1：递归时先加入根

如果先 `append(node.Val)`，就变成前序遍历，不是后序。

### 错误 2：迭代版忘记反转

当前迭代技巧得到的是 `根 -> 右 -> 左`，不反转会返回完全相反的顺序。

### 错误 3：迭代版压栈顺序写反

要得到反转前的 `根 -> 右 -> 左`，弹出根后应先压左、再压右，让右先出栈。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，每个节点访问一次，反转 O(n) |
| 空间 | O(h) 到 O(n)，递归栈或显式栈 |

## 四问复盘

1. 这题为什么属于这个模式？
   因为题目要求二叉树后序遍历，本质是按“左、右、根”访问整棵树。

2. 这个算法维护的不变量是什么？
   递归版保证根在左右子树之后加入；迭代版保证反转前序列是 `根、右、左`。

3. 每一步为什么不会漏答案？
   每个非空节点都会被访问一次，并且左右子树都会被递归或压栈处理。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是迭代版忘记反转或压栈顺序写反，会返回 `根右左` 或其它错误顺序。
