# 236. 二叉树的最近公共祖先

## 题目信息

- 题号：LeetCode 236
- 标题：Lowest Common Ancestor of a Binary Tree
- 难度：中等（Medium）
- 模式：二叉树 / 后序 DFS

## 一句话思路

后序遍历让左右子树先报告是否找到了 `p` 或 `q`；如果左右两边都找到了，当前节点就是最近公共祖先。

## 为什么想到这个模式

最近公共祖先取决于 `p` 和 `q` 在当前节点左右子树中的分布：

- 一个在左子树，一个在右子树：当前节点就是 LCA。
- 两个都在同一侧：LCA 在那一侧。
- 当前节点本身是 `p` 或 `q`：当前节点可能就是 LCA。

这些判断都依赖左右子树的返回结果，所以自然想到后序 DFS：先处理左右子树，再合并当前节点答案。

这类题的信号：

- 树结构。
- 答案依赖左右子树是否包含目标节点。
- 需要从子树向父节点返回信息。

## 状态定义 / 数据结构

递归函数定义：

```text
lowestCommonAncestor(root, p, q)
```

返回值含义：

- 返回 `nil`：当前子树没有找到 `p` 或 `q`。
- 返回 `p` 或 `q`：当前子树找到了其中一个目标节点。
- 返回某个祖先节点：当前子树已经找到了最近公共祖先。

这个返回值不是简单的布尔值，而是“找到的有效节点”。

## 核心不变量

对任意节点 `root`，递归返回值始终满足：

- 如果当前子树没有 `p/q`，返回 `nil`。
- 如果当前子树只包含一个目标，返回那个目标节点。
- 如果当前子树同时包含 `p` 和 `q`，返回这棵子树内的最近公共祖先。

后序合并逻辑：

- `left != nil && right != nil`：左右各找到一个，当前 `root` 是 LCA。
- 只有一边非空：说明目标都在这一边，向上返回这一边的结果。
- 两边都空：返回 `nil`。

## 手推全过程

树：

```text
        3
       / \
      5   1
     / \ / \
    6  2 0  8
      / \
     7   4
```

求 `p = 5, q = 1`：

| 当前节点 | 左返回 | 右返回 | 当前返回 | 原因 |
|----------|--------|--------|----------|------|
| 5 | - | - | 5 | 当前节点就是 p |
| 1 | - | - | 1 | 当前节点就是 q |
| 3 | 5 | 1 | 3 | 左右各找到一个目标 |

答案是 `3`。

求 `p = 5, q = 4`：

| 当前节点 | 左返回 | 右返回 | 当前返回 | 原因 |
|----------|--------|--------|----------|------|
| 4 | - | - | 4 | 当前节点就是 q |
| 2 | 4 | nil | 4 | 目标在一侧，向上返回 |
| 5 | - | - | 5 | 当前节点就是 p，直接返回 |

答案是 `5`，因为一个节点可以是自己的祖先。

## Go 代码实现

```go
func lowestCommonAncestor(root, p, q *TreeNode) *TreeNode {
    if root == nil || root == p || root == q {
        return root
    }

    left := lowestCommonAncestor(root.Left, p, q)
    right := lowestCommonAncestor(root.Right, p, q)

    if left != nil && right != nil {
        return root
    }
    if left != nil {
        return left
    }
    return right
}
```

## 错误写法与反例

### 错误 1：只用前序判断当前节点

```go
if root == p || root == q {
    return root
}
// 不检查左右子树合并
```

反例：`p = 5, q = 1`

如果没有后序合并，根节点 `3` 无法知道左右子树分别找到了目标，就不能返回真正的 LCA。

### 错误 2：左右都找到时返回左边

```go
if left != nil && right != nil {
    return left
}
```

反例：`p = 5, q = 1`

左右子树各找到一个目标时，最近公共祖先是当前节点 `3`，不是左边的 `5`。

### 错误 3：忽略“节点可以是自己的祖先”

```go
if root == p || root == q {
    return nil
}
```

反例：`p = 5, q = 4`

`5` 是 `4` 的祖先，正确答案是 `5`。如果遇到目标节点不返回自己，就会漏掉这种情况。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(h)，递归栈高度 |

`h` 是树高，最坏链表树为 O(n)，平衡树为 O(log n)。

## 四问复盘

1. 这题为什么属于这个模式？
   因为答案依赖左右子树是否包含 `p/q`，需要子树先返回信息，再由父节点合并判断。

2. 这个算法维护的不变量是什么？
   每个递归调用返回当前子树内找到的目标节点或 LCA；没有目标则返回 `nil`。

3. 每一步为什么不会漏答案？
   后序遍历覆盖每个节点；左右返回值完整表达当前子树中 `p/q` 的分布，左右都非空时立即捕获最近的分叉点。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是当前节点等于 `p` 或 `q` 时没有直接返回。这样会漏掉“一个节点是另一个节点祖先”的情况。
