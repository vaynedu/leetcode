# 53. 最大子数组和

## 题目信息

- 题号：LeetCode 53
- 难度：简单
- 模式：动态规划 / Kadane / 贪心
- 可视化：`doc/algorithm-visualizations/53.maximum-subarray.html`

## 一句话思路

维护“以当前位置结尾的最大连续子数组和”：如果前缀和为负，它只会拖累后面的数，直接从当前数重新开始。

## 为什么想到这个模式

题目要求连续子数组的最大和。连续子数组天然有“以 i 结尾”的状态：当前位置要么接在前一个子数组后面，要么自己重新开始。

这就是一维 DP：

```text
dp[i] = max(dp[i-1] + nums[i], nums[i])
```

因为 `dp[i]` 只依赖 `dp[i-1]`，可以压缩成一个变量 `curSum`。

## 状态定义 / 数据结构

- `curSum`：以当前下标 `i` 结尾的最大连续子数组和。
- `maxSum`：全局最大连续子数组和。

注意：数组至少包含一个元素，所以两个变量都应初始化为 `nums[0]`。

## 核心不变量

遍历到 `i` 时：

- `curSum` 一定是“必须以 `nums[i]` 结尾”的最大和。
- `maxSum` 一定是 `[0..i]` 范围内见过的最大连续子数组和。

如果 `curSum + nums[i] < nums[i]`，说明之前那段前缀对当前数有负贡献，应丢弃。

## 手推全过程

以 `nums = [-2,1,-3,4,-1,2,1,-5,4]` 为例：

| i | nums[i] | curSum | maxSum | 说明 |
|---|---------|--------|--------|------|
| 0 | -2 | -2 | -2 | 初始化 |
| 1 | 1 | `max(-2+1,1)=1` | 1 | 从 1 重新开始 |
| 2 | -3 | `max(1-3,-3)=-2` | 1 | 当前和变负 |
| 3 | 4 | `max(-2+4,4)=4` | 4 | 丢弃负前缀，从 4 开始 |
| 4 | -1 | `max(4-1,-1)=3` | 4 | 继续接上 |
| 5 | 2 | `max(3+2,2)=5` | 5 | 更新全局 |
| 6 | 1 | `max(5+1,1)=6` | 6 | 得到最优 `[4,-1,2,1]` |
| 7 | -5 | `max(6-5,-5)=1` | 6 | 全局保持 |
| 8 | 4 | `max(1+4,4)=5` | 6 | 全局保持 |

答案：6。

## 错误写法与反例

错误写法：

```go
curSum := 0
maxSum := 0
```

反例：`nums = [-3,-1,-2]`

正确答案是 `-1`。如果 `maxSum` 初始化为 0，会错误返回 0，但题目要求子数组至少包含一个元素。

正确写法：

```go
curSum := nums[0]
maxSum := nums[0]
```

## Go 代码实现

```go
func maxSubArray(nums []int) int {
    curSum := nums[0]
    maxSum := nums[0]

    for i := 1; i < len(nums); i++ {
        curSum = max(curSum+nums[i], nums[i])
        maxSum = max(maxSum, curSum)
    }
    return maxSum
}
```

## 复杂度

| 指标 | 值 |
|------|-----|
| 时间 | O(n) |
| 空间 | O(1) |

## 四问复盘

1. **为什么属于 DP？** 当前最优连续和依赖“以前一个元素结尾的最优连续和”。
2. **维护的不变量是什么？** `curSum` 必须以当前元素结尾，`maxSum` 是全局最优。
3. **为什么不会漏答案？** 每个连续子数组都有一个结尾位置，而该位置的最优值被 `curSum` 覆盖。
4. **最容易错的边界？** 全负数数组，不能把 `curSum/maxSum` 初始化为 0。
