# 230. 二叉搜索树中第 K 小的元素

## 题目信息

- 题号：LeetCode 230
- 标题：Kth Smallest Element in a BST
- 难度：中等（Medium）
- 模式：BST / 中序遍历

## 一句话思路

BST 的中序遍历是升序序列，因此第 `k` 次访问到的节点就是第 `k` 小元素。

## 为什么想到这个模式

题目给的是二叉搜索树，并且问“第 K 小”。BST 最重要的性质是：

```text
左子树 < 根 < 右子树
```

按中序遍历：

```text
左 → 根 → 右
```

访问顺序天然就是从小到大，所以不需要把所有节点排序，只要计数到第 `k` 个即可。

## 状态定义 / 数据结构

- `count`：中序遍历已经访问了多少个节点。
- `result`：第 `k` 次访问时记录的答案。
- `inorder(node)`：按升序访问以 `node` 为根的 BST。

递归访问顺序：

```text
inorder(left)
visit(root)
inorder(right)
```

## 核心不变量

任意时刻：

- 中序遍历已经访问过的节点值是升序排列。
- `count` 等于已经访问过的节点数量。
- 当 `count == k` 时，当前节点就是第 `k` 小元素。

找到答案后，结果不再需要更新。

## 手推全过程

以树 `[5,3,6,2,4,null,null,1]` 为例：

```text
        5
       / \
      3   6
     / \
    2   4
   /
  1
```

中序遍历：

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6
```

- `k = 1`，答案是 `1`。
- `k = 3`，答案是 `3`。
- `k = 5`，答案是 `5`。

## Go 代码实现

```go
func kthSmallest(root *TreeNode, k int) int {
    count := 0
    result := 0
    var inorder func(node *TreeNode)
    inorder = func(node *TreeNode) {
        if node == nil {
            return
        }
        inorder(node.Left)
        count++
        if count == k {
            result = node.Val
            return
        }
        inorder(node.Right)
    }
    inorder(root)
    return result
}
```

## 错误写法与反例

### 错误 1：把 `k` 当成 0-indexed

题目里的 `k` 是 1-indexed。第 1 小就是中序遍历访问到的第 1 个节点，不是数组下标 1。

### 错误 2：用普通二叉树遍历顺序

前序或层序都不能保证升序。

反例：

```text
    2
   / \
  1   3
```

层序是 `2,1,3`，第 1 个不是最小值；中序才是 `1,2,3`。

### 错误 3：找到答案后继续覆盖结果

如果没有把结果只记录在 `count == k` 的时刻，后续遍历可能覆盖掉答案。当前实现只在等于 `k` 时赋值，因此不会被后续节点覆盖。

## 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(h + k)，理想早停；当前实现找到后仍可能展开部分递归，最坏 O(n) |
| 空间 | O(h)，递归栈高度，最坏 O(n) |

## 四问复盘

1. 这题为什么属于这个模式？
   因为 BST 的中序遍历天然是升序序列，而第 K 小就是升序序列中的第 K 个元素。

2. 这个算法维护的不变量是什么？
   中序已访问节点保持升序，`count` 精确表示已经访问的节点数量。

3. 每一步为什么不会漏答案？
   中序遍历会按从小到大的顺序访问所有节点，计数到 `k` 时必然正好是第 `k` 小。

4. 哪个边界最容易错，错了会怎样？
   最容易错的是把 `k` 当成 0-indexed，会整体偏一位，尤其 `k=1` 时返回第二小或错误值。
