LeetCode 33 · Binary Search + Rotated Array

搜索旋转排序数组

数组整体不再有序，但每次二分至少有一半仍然有序。

核心洞察

题目：在旋转后的升序数组中查找 target，要求 O(log n)。
关键：先判断哪半边有序，再判断 target 是否落在这个有序半边里。

观察

至少一半有序

旋转点只会破坏一处顺序；任意 mid 两边至少有一边是升序。

选择

用有序半边做范围判断

只有有序半边能可靠判断 target 在不在里面。

复杂度

O(log n) 时间 O(1) 空间

每轮都排除一半，闭区间 [left,right] 不断收缩。

手推全过程

样例：nums=[4,5,6,7,0,1,2]，target=0

Step 1 left=0, mid=3, right=6
nums[left] <= nums[mid]，左半 [4,5,6,7] 有序 target=0 不在 [4,7) left = mid + 1 = 4
标准二分会看到 7 > 0 就往左走，但旋转数组里左边反而全是大数。
先判断有序半边，才能安全排除。
Step 2 left=4, mid=5, right=6
左半 [0,1] 有序 target=0 在 [0,1) right = mid - 1 = 4
这次 target 落在左半有序区间内，所以保留左半。
闭区间变成 [4,4]。
Step 3 left=4, mid=4, right=4
nums[mid] = 0 return 4
找到 target，下标为 4。

Go 代码

闭区间二分：先比较 nums[mid]，再判断有序半边。

func search(nums []int, target int) int {
    left, right := 0, len(nums)-1

    for left <= right {
        mid := left + (right-left)/2
        if nums[mid] == target {
            return mid
        }

        if nums[left] <= nums[mid] {
            // 左半边有序：[left..mid]
            if nums[left] <= target && target < nums[mid] {
                right = mid - 1
            } else {
                left = mid + 1
            }
        } else {
            // 右半边有序：[mid..right]
            if nums[mid] < target && target <= nums[right] {
                left = mid + 1
            } else {
                right = mid - 1
            }
        }
    }
    return -1
}

易错点

旋转数组不能直接套标准二分，必须先找有序半边。

错误：只比较 nums[mid] 和 target

if nums[mid] < target {
    left = mid + 1
} else {
    right = mid - 1
}

在 [4,5,6,7,0,1,2] 里，mid=7 大于 0，但答案在右边。

正确：先判断哪边有序

if nums[left] <= nums[mid] {
    // 左半有序
} else {
    // 右半有序
}

只有有序半边才能做 target 范围判断。

错误：区间边界包含 mid

if nums[left] <= target &&
   target <= nums[mid] {
    right = mid - 1
}

前面已经判断过 nums[mid] 不等于 target，再包含 mid 容易让条件表达混乱。

正确：半开判断更清晰

if nums[left] <= target &&
   target < nums[mid] {
    right = mid - 1
}

左半用 [nums[left], nums[mid])，右半用 (nums[mid], nums[right]]。

核心算法

旋转数组二分

每轮先判断左半还是右半有序，再决定保留哪一半。

正确性直觉

只在有序半边判断范围

有序半边能证明 target 是否在其中，因此可安全排除另一半。

易错关键点

不要套标准二分

旋转后 nums[mid] > target 不代表答案一定在左边。
