加油站

从哪个加油站出发能跑完一圈 — 贪心的精髓：totalSurplus < 0 则无解；否则从 0 开始，curSurplus < 0 时跳过之前的站点，从下一个重新出发

核心洞察

题目：gas = 每站加油量，cost = 到下一站油耗，问能否绕一圈以及从哪个站出发。
关键 1：如果 totalSurplus = Σ(gas[i] - cost[i]) < 0，则无解。
关键 2：如果从 0 跑到 i 时 curSurplus < 0，说明 0..i 都不可能作为起点，跳到 i+1 重新计。

全局判断

totalSurplus ≥ 0 是必要条件

总油量必须 ≥ 总油耗，否则跑不完一圈。

贪心选择

curSurplus < 0 → 换起点

从 0 跑到 i 都没油了，0..i 都不可能作为起点（油不够自己跑到 i）。

复杂度

O(n) 时间 O(1) 空间

一次遍历，只需两个变量：curSurplus 和 totalSurplus。

手推全过程

gas = [1,2,3,4,5]，cost = [3,4,5,1,2]

Step 1 totalSurplus = (1-3)+(2-4)+(3-5)+(4-1)+(5-2) = -5+5=0
totalSurplus = 0，有解
Step 2 从 0 出发
i=0: cur=0+(1-3)=-2 → cur < 0
从 0 出发失败，更新 start=1，cur=0
Step 3 从 1 出发
i=1: cur=0+(2-4)=-2 → cur < 0
从 1 出发也失败，更新 start=2，cur=0
Step 4 从 2 出发
i=2: cur=0+(3-5)=-2 → cur < 0
更新 start=3，cur=0
Step 5 从 3 出发
i=3: cur=0+(4-1)=3
i=4: cur=3+(5-2)=6 → 跑完一圈！
start=3 是正确答案

最终结果：从加油站 3 出发可以绕完一圈

Go 代码

O(n) 一次遍历，无需环形数组模拟

func canCompleteCircuit(gas, cost []int) int {
    totalSurplus := 0
    curSurplus := 0
    start := 0

    for i := 0; i < len(gas); i++ {
        diff := gas[i] - cost[i]
        totalSurplus += diff
        curSurplus += diff

        // 从 start 跑到 i 时油箱空了，start 及其之前都不可能
        if curSurplus < 0 {
            start = i + 1
            curSurplus = 0
        }
    }

    if totalSurplus < 0 {
        return -1
    }
    return start
}

易错点

totalSurplus < 0 的检查必须在最后返回前判断，不能漏掉

❌ 错误：跳过 totalSurplus 检查

// 只检查 curSurplus，不检查 totalSurplus
for i := 0; i < len(gas); i++ {
    curSurplus += gas[i] - cost[i]
    if curSurplus < 0 {
        start = i+1; curSurplus = 0
    }
}
return start // 如果 totalSurplus<0，可能返回错误索引

gas=[2], cost=[3] 时 totalSurplus=-1，但遍历完 start=0，可能错误返回 0。

✓ 正确：最后检查 totalSurplus

if totalSurplus < 0 {
    return -1 // 全局不够，无解
}
return start

totalSurplus ≥ 0 是有解的必要条件，两步都不可少。

核心

cur < 0 → 换起点

从起点跑到 i 时 curSurplus < 0，说明 0..i 都不能作为起点。

贪心证明

局部失败 → 全局排除

0..i 区间总油量不够自己跑完，排除后不影响最终答案的存在性。

关键细节

totalSurplus < 0 → 无解

必须最后检查全局剩余，全局不够则返回 -1。
