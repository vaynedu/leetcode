LeetCode 55 · Greedy

跳跃游戏

判断能否到达最后一个位置 — 贪心的精髓：维护「当前能到的最远位置」，遍历时持续扩展，遇到死胡同则失败

核心洞察

题目：nums[i] 表示从 i 能跳的最远距离，问能否到达最后一个位置。
关键：维护一个「最远可达位置」reach，遍历时不断更新 reach = max(reach, i+nums[i])。 如果在某位置 i > reach，说明前面没有任何点能跳到 i，死胡同。

观察

i > reach = 死胡同

如果当前索引超过了已知的最远可达范围，说明到达 i 的路径不存在。

贪心选择

持续扩展最远边界

每次更新 reach 为 max(reach, i+nums[i])，代表目前已知能跳到的最远位置。

复杂度

O(n) 时间 O(1) 空间

一次遍历，只用一个变量记录最远边界。

手推全过程

nums = [2,3,1,1,4]

Step 1 i=0 nums[0]=2 → reach=2
从位置 0 最多跳到索引 2
Step 2 i=1 reach=2 → 更新 reach=max(2,1+3)=4
reach = max(2, 4) = 4，已超过最后一个索引 4
可以到达！

最终结果：true（reach=4 ≥ len-1=4）

Go 代码

一次遍历，维护最远可达位置

func canJump(nums []int) bool {
    reach := 0

    for i, jump := range nums {
        // 当前索引超出已知最远范围 → 死胡同
        if i > reach {
            return false
        }
        // 扩展最远可达范围
        reach = max(reach, i+jump)
    }
    return true
}

易错点

容易写成 BFS 遍历所有路径，但其实是 O(n) 的贪心

❌ 错误：每次取最大跳跃距离

// nums=[3,2,1,0,4] 时会出错
pos := 0
for i := 0; i < len(nums); i++ {
    pos = max(pos, nums[i]) // 没有考虑当前位置
}
// pos=3，但实际从0跳不到3（3步跳不到4步的距离）

没有用 i+jump 更新，只取最大值会忽略路径可达性。[3,2,1,0,4] 的答案是 false。

✓ 正确：用 i+jump 更新

reach := 0
for i, jump := range nums {
    if i > reach { return false }
    reach = max(reach, i+jump)
}

必须用 i+jump 更新，因为每次跳跃都从当前索引出发，不是从原点。

核心

维护最远可达边界

reach 记录目前已知能跳到的最远位置，不断用 i+jump 扩展。

贪心证明

局部扩展 → 全局最优

能到达最远的点，一定是从之前某点可达的。不需要搜索所有路径。

关键细节

i > reach → 失败

当遍历到超出可达范围的索引，说明路径断裂，直接返回 false。
