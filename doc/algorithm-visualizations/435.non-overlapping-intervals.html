LeetCode 435 · Greedy

无重叠区间

移除最少区间使剩余区间不重叠 — 贪心的精髓：按右边界排序，优先保留结束最早的区间，给后面留更多空间

核心洞察

题目：给出若干区间，问至少移除多少个区间使剩余不重叠。
关键：按右边界升序排序后，遍历时比较相邻两个区间的左边界和上一个保留区间的右边界。 贪心策略：优先保留结束最早的区间，因为它给后面所有区间留了最大空间。

观察

按右端点排序

排序后，每个区间只需要和上一个保留区间比较。不重叠时加入，结果互不影响。

贪心选择

保留结束最早的

遇到重叠时，移除右端点更大的区间（即结束更晚的），因为它更可能和更多后续区间冲突。

复杂度

O(n log n)

瓶颈在排序。遍历本身 O(n)，只用一个变量跟踪上一个保留区间的右端点。

手推全过程

intervals = [[1,2],[2,3],[3,4],[1,3]]

Step 1 排序后: [1,2] [2,3] [3,4] [1,3]
按右端点升序：[1,2]→2, [2,3]→3, [3,4]→4, [1,3]→3
count=0, prev_end=1（第一个区间保留）
Step 2 处理 [2,3]
当前区间 [2,3]，上一个保留 [1,2]，prev_end=2
2 ≥ 2？是，重叠。count++ → 移除 [2,3]
prev_end 保持 2（不更新）
Step 3 处理 [3,4]
当前区间 [3,4]，上一个保留 [1,2]，prev_end=2
3 ≥ 2？否，不重叠，保留！count 保持 1
prev_end = 4（更新为当前区间右端点）
Step 4 处理 [1,3]
当前区间 [1,3]，上一个保留 [3,4]，prev_end=4
1 ≥ 4？否，不重叠，保留！count 保持 1
prev_end = 3（更新，但不影响最终结果）

最终结果：移除 1 个区间（[2,3]），剩余 [[1,2],[3,4],[1,3]]

Go 代码

按右端点排序，遍历时移除和上一保留区间重叠的区间

func eraseOverlapIntervals(intervals [][]int) int {
    if len(intervals) == 0 {
        return 0
    }

    // 1. 按右端点升序排序
    sort.Slice(intervals, func(i, j int) bool {
        return intervals[i][1] < intervals[j][1]
    })

    // 2. 第一个区间作为基准
    count := 0
    prevEnd := intervals[0][1]

    // 3. 遍历剩余区间
    for i := 1; i < len(intervals); i++ {
        cur := intervals[i]
        if cur[0] >= prevEnd {
            // 不重叠：保留，更新 prevEnd
            prevEnd = cur[1]
        } else {
            // 重叠：移除当前区间（贪心：保留结束更早的）
            count++
        }
    }
    return count
}

易错点

和 56 合并区间类似，容易在排序方式和比较符号上出错

❌ 错误：按左端点排序

// 按左端点排序：[[1,2],[1,3],[2,3],[3,4]]
sort.Slice(intervals, func(i,j int)bool{
    return intervals[i][0] < intervals[j][0]
})
// 遍历时 prevEnd=2, [1,3]重叠但左端点1不更新prevEnd，逻辑混乱

按左端点排序时，相邻区间可能跨多个重叠，贪心策略失效。必须按右端点排序。

✓ 正确：按右端点排序

sort.Slice(intervals, func(i,j int)bool{
    return intervals[i][1] < intervals[j][1]
})
// 排序后: [[1,2],[2,3],[1,3],[3,4]]
// 遍历: 保留[1,2]，移除[2,3]，保留[3,4]，保留[1,3] → 移除1个

按右端点排序后，每个区间只可能和上一个保留区间冲突，贪心选择才有意义。

核心

按右端点 + 贪心移除

排序后，遇到重叠就移除当前区间，保留结束最早的，给后面留更多空间。

贪心证明

局部的最优 → 全局最优

移除结束最晚的区间永远不比移除结束更早的差，因为它对后续区间的影响范围更大。

关键细节

按右端点排序，不是左端点

排序方式决定贪心策略是否有效。按左端点排序会让问题变得更复杂。
