LEETCODE 76 · MINIMUM WINDOW SUBSTRING

最小覆盖子串

右扩左缩 → valid 驱动收缩 → 维护全局最优

① 什么是变长窗口

变长窗口 = 窗口大小**不固定**，先扩张到满足条件，再尽可能收缩找最优。和固定窗口最大的区别是：valid 驱动左边界收缩。

固定窗口

窗口大小固定，滑动时左右指针同步移动

右移 → 左移 → 右移 → 左移

变长窗口（本题）

窗口大小变化，右扩到底，再逐步左缩找最优

R→→→→→→→ 然后 L↘ L↘ L↘

t = "ABC"  →  need = {A:1, B:1, C:1}

Aneed=1
Bneed=1
Cneed=1

核心机制

valid 驱动收缩

valid == len(need) 时才满足条件，此时尝试左缩。
左缩时若某个字符数量等于 need，valid--，停止收缩。

窗口内容

双 HashMap

need 记录 t 中每个字符需要的次数
window 记录窗口内每个字符出现的次数
只有 window[c] == need[c] 时 valid 才加一

时间复杂度

O(n)

R 最多移动 n 次，L 也最多移动 n 次。
每个字符最多被访问两次（一次R，一次L）。

Time · O(n)  ·  Space · O(1) · fixed alphabet (26+26)
