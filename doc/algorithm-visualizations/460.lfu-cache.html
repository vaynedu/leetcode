LFU 缓存

Least Frequently Used — 按使用频率淘汰的缓存，比 LRU 更精细

LRU vs LFU：核心区别

LRU（最近最少使用）：时间维度，淘汰最久未访问的。
LFU（最不经常使用）：频率维度，淘汰使用次数最少的。
LFU 更精准但实现更复杂，需要追踪每个 key 的访问频率。

LRU #146

淘汰：时间最久未访问
数据结构：HashMap + 双向链表
难点：O(1) 移动节点到头部

LFU #460（更难）

淘汰：频率最低 + 时间最久未访问
数据结构：HashMap + 频率双向链表
难点：O(1) 增频率、O(1) 找最差频率

LFU 的 tricky 点：同一个频率下，按 LRU 规则淘汰（时间最久）
例如 freq=1 有多个 key → 淘汰其中最久未访问的

核心数据结构

需要同时支持：O(1) get/put，O(1) 增频率，O(1) 找最低频率。
解法：HashMap + 频率桶（频率 → 双向链表）。

HashMap (key → Node*)

key 找到节点指针
Node 含: key, val, freq, prev, next
O(1) 定位

Freq Buckets []list

freq=1,2,3,... 各一个双向链表
同 freq 内按 LRU 顺序
minFreq 变量指向最低非空频率

minFreq 追踪器

每次访问递增 freq
当最低频率桶变空，minFreq++
淘汰时从 freq=minFreq 的桶尾删

频率桶结构示例

freq=1
key:a key:c
freq=2

freq=3
key:d

minFreq = 1，淘汰时从 freq=1 桶的尾部（最久未访问）淘汰

Go 代码模板

type Node struct {
    key, val int
    freq     int
    prev, next *Node
}

type LFUCache struct {
    capacity int
    minFreq  int
    // key → node 指针
    nodes map[int]*Node
    // freq → 双向链表（头插法，最新在头，最旧在尾）
    freqMap map[int]*list.List
    // list.Element → Node（用于 O(1) 定位节点）
    elemMap map[*list.Element]*Node
}

func Constructor(capacity int) LFUCache {
    return LFUCache{
        capacity: capacity,
        minFreq:  0,
        nodes:    make(map[int]*Node),
        freqMap:  make(map[int]*list.List),
        elemMap:  make(map[*list.Element]*Node),
    }
}

func (c *LFUCache) Get(key int) int {
    node, ok := c.nodes[key]
    if !ok {
        return -1
    }
    // 频率增加：从旧桶移到新桶
    c.incrementFreq(node)
    return node.val
}

func (c *LFUCache) Put(key, val int) {
    if c.capacity == 0 {
        return
    }
    if node, ok := c.nodes[key]; ok {
        node.val = val
        c.incrementFreq(node)
        return
    }
    // 新 key
    if len(c.nodes) == c.capacity {
        // 淘汰 minFreq 桶的最旧节点
        c.evict()
    }
    // 插入新节点，freq=1
    node := &Node{key: key, val: val, freq: 1}
    c.nodes[key] = node
    c.addToFreqList(node)
    c.minFreq = 1
}

// 频率+1：从原 freq 桶移除，加入 freq+1 桶
func (c *LFUCache) incrementFreq(node *Node) {
    oldFreq := node.freq
    newFreq := oldFreq + 1
    node.freq = newFreq

    // 从旧桶移除
    c.removeFromFreqList(node)
    // 更新 minFreq
    if oldFreq == c.minFreq && c.freqMap[oldFreq].Len() == 0 {
        c.minFreq = newFreq
    }
    // 加入新桶
    c.addToFreqList(node)
}

// 淘汰最低频率桶的最旧节点
func (c *LFUCache) evict() {
    lst := c.freqMap[c.minFreq]
    last := lst.Back() // 最旧
    node := c.elemMap[last]
    lst.Remove(last)
    delete(c.nodes, node.key)
    delete(c.elemMap, last)
}

// 加入 freq 桶（头插，最新）
func (c *LFUCache) addToFreqList(node *Node) {
    if c.freqMap[node.freq] == nil {
        c.freqMap[node.freq] = list.New()
    }
    elem := c.freqMap[node.freq].PushFront(node)
    c.elemMap[elem] = node
}

// 从 freq 桶移除
func (c *LFUCache) removeFromFreqList(node *Node) {
    // 需要遍历或额外存 elem，这里简化：实际需要存 elem 指针
    lst := c.freqMap[node.freq]
    for e := lst.Front(); e != nil; e = e.Next() {
        if e.Value.(*Node) == node {
            lst.Remove(e)
            delete(c.elemMap, e)
            break
        }
    }
}

面试核心问答

Q: LFU vs LRU 哪个更难实现？

LFU 更难。LRU 只需要一个双向链表，LFU 需要两层结构：频率桶+桶内LRU。频率变化时要迁移节点，还要维护最小频率。

Q: 为什么 freqMap 用 list 而不用 slice？

list 的 O(1) 插入/删除（PushFront/Remove）适合 LRU 规则（尾部淘汰）。slice 删除中间元素是 O(n)。

Q: 如何做到真正的 O(1) removeFromFreqList？

Node 需要存 elem *list.Element 指针，这样 remove 时直接 elem.Remove()，无需遍历桶。

核心

HashMap + 频率桶

两层结构：key 找节点，freq 找桶，桶内按 LRU 淘汰

关键

minFreq 追踪

记录当前最低非空频率，淘汰时直接定位，频率增加时更新

精华

Node 存 elem 指针

避免从桶中遍历查找节点，真正 O(1) 迁移
