买卖股票最佳时机 II

可以多次买卖，问最大利润 — 贪心的精髓：累加所有上升段，利润 = Σ(每个上升段的终点 - 起点)

核心洞察

题目：可以无限次买卖（先买后卖，不能卖空），求最大利润。
关键：把利润看作所有上升段的累加：每天如果价格比昨天高，就等于昨天买今天卖。 这个贪心成立因为没有交易次数限制，持有时间越长利润越少，所以只累加正收益。

观察

上升段 = 连续的正差值

prices[i] - prices[i-1] > 0 说明存在利润空间。

贪心选择

累加所有正差值

如果有上涨就持有（或买入），每天都计算和前一天的差值，正值就计入利润。

复杂度

O(n) 时间 O(1) 空间

一次遍历，只维护一个 profit 变量。

手推全过程

prices = [7,1,5,3,6,4]

Day 0→1 7 → 1，下降 6
diff = 1 - 7 = -6
diff < 0，不计入利润
Day 1→2 1 → 5，上涨 4
diff = 5 - 1 = +4
profit += 4，profit = 4
Day 2→3 5 → 3，下降 2
diff = 3 - 5 = -2
diff < 0，不计入利润
Day 3→4 3 → 6，上涨 3
diff = 6 - 3 = +3
profit += 3，profit = 7
Day 4→5 6 → 4，下降 2
diff = 4 - 6 = -2
diff < 0，不计入利润

最终利润 = 7（Day1买入→Day2卖出 +4，Day3买入→Day4卖出 +3）

Go 代码

累加所有正差值

func maxProfit(prices []int) int {
    profit := 0
    for i := 1; i < len(prices); i++ {
        diff := prices[i] - prices[i-1]
        if diff > 0 {
            profit += diff
        }
    }
    return profit
}

易错点

和 121 的区别：121 只能一次买卖，122 可以无限次，正差值累加是 122 独有的

❌ 错误：找最高点和最低点

// 122 中找全局最高/最低点不适用，因为有无限次交易
buy := min(prices)
sell := max(prices)
profit := sell - buy // 这只能做一次交易

对于 [1,5,3,6]，最低1最高6，但分段交易 1→5(+4) + 3→6(+3)=7 > 5。所以不能直接用最大-最小。

✓ 正确：累加所有正差值

for i := 1; i < len(prices); i++ {
    diff := prices[i] - prices[i-1]
    if diff > 0 { profit += diff }
}
// [1,5,3,6]: (5-1)+(6-3) = 4+3 = 7

累加正差值等价于在每个谷买入、峰卖出，但不需要显式找峰谷。

核心

累加正差值

遍历每天的差值，正值就加到利润里，等价于在每个局部谷买入、峰卖出。

贪心证明

无交易限制时成立

因为可以无限次交易，每天持有到明天再卖的收益不小于今天卖，所以上升段持有更优。

关键细节

累加 diff > 0，不是 >=

等于 0 时不计入，但也不亏损，正好处于持平状态。
