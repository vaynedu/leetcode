LeetCode 53 · Greedy + DP

最大子数组和

找一个连续子数组使其和最大 — Kadane 算法：遍历时如果当前和变成负数，就从下一个位置重新开始

核心洞察

题目：找连续子数组的最大和（至少包含一个元素）。
关键：Kadane 算法：如果当前「连续和」是负数，它对后续求和只会拖后腿，直接从下一个数重新开始。 这既是贪心（丢弃负和）也是 DP（最优子结构）。

观察

负连续和 = 有害无益

如果当前子数组和是负数，加上任何数只会更小，不如从新位置开始。

贪心选择

丢弃负和，从头再来

当 sum < 0 时，重置 sum = nums[i]，用新的起点继续。

复杂度

O(n) 时间 O(1) 空间

一次遍历，两个变量：当前连续和 sum，和全局最大 maxSum。

手推全过程

nums = [-2,1,-3,4,-1,2,1,-5,4]

i=0 num=-2 sum=-2 maxSum=-2
sum = -2，maxSum = -2
sum < 0，但只有一个数，保留
i=1 num=1 sum=2 maxSum=2
sum = max(-2+1, 1) = max(-1, 1) = 1 → 更新 sum=1
maxSum = max(-2, 1) = 1
i=2 num=-3 sum=-2 maxSum=2
sum = max(1+(-3), -3) = max(-2, -3) = -2
sum < 0，重置思路：下一轮如果再加负数会更小
i=3 num=4 sum=4 maxSum=4
sum = max(-2+4, 4) = max(2, 4) = 4
maxSum = max(2, 4) = 4，发现更好的子数组 [4]
i=4 num=-1 sum=3 maxSum=4
sum = max(4+(-1), -1) = max(3, -1) = 3
maxSum = max(4, 3) = 4，保持
i=5 num=2 sum=5 maxSum=5
sum = 3+2 = 5，maxSum = max(4, 5) = 5
i=6 num=1 sum=6 maxSum=6
sum = 5+1 = 6，maxSum = max(5, 6) = 6

最终结果：maxSum = 6，对应子数组 [4,-1,2,1]

Go 代码

Kadane 算法，O(n) 一次遍历

func maxSubArray(nums []int) int {
    curSum := nums[0] // 当前连续和
    maxSum := nums[0] // 全局最大和

    for i := 1; i < len(nums); i++ {
        // 如果当前和是负数，从当前数重新开始；否则累加
        curSum = max(curSum+nums[i], nums[i])
        // 更新全局最大和
        maxSum = max(maxSum, curSum)
    }
    return maxSum
}

易错点

curSum 和 maxSum 的初始化，以及 max 函数的使用

❌ 错误：curSum 初始化为 0

curSum := 0 // 错误：全负数时结果会是0
for i := 0; i < len(nums); i++ {
    curSum = max(curSum+nums[i], nums[i])

nums = [-3,-1,-2]，正确答案 -1，但如果 curSum=0，第一轮 curSum=max(0-3,-3)=-3，maxSum=max(0,-3)=0，结果错误。

✓ 正确：curSum 初始化为第一个元素

curSum := nums[0]
maxSum := nums[0]
// [-3,-1,-2]: curSum=-3,maxSum=-3 → -3,-1 → curSum=max(-4,-1)=-1,maxSum=-1 → -2 → maxSum=-1 ✓

初始化为第一个元素，保证至少包含一个元素，且全负数时正确。

核心

Kadane 算法

遍历时维护 curSum（当前连续和），当 curSum < 0 时重置为当前数。每次更新 maxSum。

正确性

负和必丢弃

包含负和的子数组不可能是最大和的前缀，丢弃它不影响最优解的存在性。

关键细节

curSum 初始化为第一个元素

不能初始化为 0，否则全负数数组会返回 0 而非最大负值。
