LeetCode 455 · Greedy

分发饼干

用尺寸最小的饼干满足胃口最小的孩子 — 贪心的精髓：优先满足最容易满足的需求，让更多孩子得到饼干

核心洞察

题目：每个孩子有胃口 g，每个饼干有尺寸 s，给孩子分配饼干使得尽可能多的孩子满意。
关键：将两数组排序后，用最小的饼干去喂胃口最小的孩子，能喂就喂，不能喂就换更大的饼干。 贪心策略：每次用当前可用的最小资源去满足最微小的需求。

观察

双排序 + 双指针

两个数组各排一次序，然后用两个指针 i, j 分别遍历孩子和饼干。

贪心选择

小饼干喂小胃口

小饼干喂不了当前孩子，就不可能喂任何后续孩子（胃口更大）。所以换下一个饼干。

复杂度

O(n log n)

排序是瓶颈 O(n log n)，双指针遍历 O(n)。

手推全过程

g = [1,2,3]，s = [1,1]，有多少孩子能满意？

初始 排序后: g=[1,2,3] s=[1,1]
g_i=0（指向胃口1），s_j=0（指向饼干1）
count=0
Step 1 g_i=1 s_j=1 → 1 ≥ 1，满意！
饼干1(1) ≥ 胃口1(1)，满足！count=1
两个指针都右移：g_i→1, s_j→1
Step 2 g_i=2 s_j=1 → 1 ≥ 2？不满足
饼干1(1) < 胃口2(2)，不满足
s_j 右移（饼干不够，换下一块）：s_j→2，超出范围
结束 s_j=2 超出，遍历结束
最多 1 个孩子满意

最终结果：1 个孩子得到饼干

Go 代码

双排序 + 双指针，贪心匹配

func findContentChildren(g, s []int) int {
    sort.Ints(g)
    sort.Ints(s)

    i, j := 0, 0
    for i < len(g) && j < len(s) {
        if s[j] >= g[i] {
            // 饼干能满足孩子，两指针右移
            i++
            j++
        } else {
            // 饼干太小，换下一块饼干（孩子指针不动）
            j++
        }
    }
    return i // i 就是满足的孩子数量
}

易错点

容易把指针移动逻辑搞反，导致结果偏大或偏小

❌ 错误：不排序直接遍历

// g=[3,1,2] s=[1,1]
for i,j := 0,0; iif s[j] >= g[i] { i++; j++ } // 直接比较，未排序
    else { j++ }
}
// 顺序是 3≥1?否→j++，然后 1≥1?是→count=1，答案错误

不排序时，用大饼干去喂小胃口孩子，导致后面的小饼干无法满足剩余孩子，结果偏小。

✓ 正确：先排序再双指针

sort.Ints(g) // 胃口排序: [1,2,3]
sort.Ints(s) // 饼干排序: [1,1]
// i=0(g=1), j=0(s=1): 1≥1 ✓ count=1
// i=1(g=2), j=1(s=1): 1≥2 ✗ j++ → j=2 结束
return i // = 1

排序后，贪心策略才能成立——每个饼干只和当前最"容易满足"的孩子比较。

核心

双排序 + 双指针

胃口和饼干各自排序后，用最小的饼干依次尝试满足胃口最小的孩子。

贪心证明

局部最优 → 全局最优

喂不了当前孩子的大饼干，喂后面的孩子也只能喂同样多，浪费了更大尺寸。

关键细节

饼干不够时只移动饼干指针

孩子指针不动，等下一块更大的饼干来满足。不能同时右移两个指针。
