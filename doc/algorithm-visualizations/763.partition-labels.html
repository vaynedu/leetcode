LeetCode 763 · Greedy

划分字母区间

记录每个字符最后出现位置，贪心地在段尾字符的最远位置"切断" — 切分点就是当前段内所有字符最后一次出现的最远位置

核心洞察

题目：把字符串划分为尽可能多的片段，每段内的字符只出现在该段中。
关键：遍历时维护当前段的"最远边界"（段内所有字符的最后位置），当遍历到该最远位置时切段。 这就是贪心：切点恰好是必须切的地方，不早切也不晚切。

观察

切分点 = 段内字符的最远最后位置

如果某字符的最后一次出现在 index 5，那包含 index 3 的段必须延伸到 index 5。

贪心选择

尽量晚切

不到万不得已不切。扩展段直到遇到当前段内所有字符的最远出现位置，才切。

复杂度

O(n) 时间 O(1) 空间

两遍扫描：第一遍记录最后位置（26字母数组），第二遍遍历切段。

手推全过程

s = "ababcbacadefecd"

Step 1 预处理：记录每个字符最后出现位置
a→8, b→5, c→9, d→13, e→14, f→12
start=0, end=0, 遍历时 end = max(end, last[char])
Step 2 i=0 'a' last[a]=8 → end=max(0,8)=8
还没到 end(8)，继续扩展
Step 3-7 i=1..7 扩展段
i=1 'b'→5, i=2 'a'→8, i=3 'b'→5, i=4 'c'→9, i=5 'b'→5, i=6 'a'→8, i=7 'c'→9
end = max(8,5,8,5,9,5,8,9) = 9
Step 8 i=8 'd' last[d]=13 → end=max(9,13)=13
还没到 end(13)，继续扩展
Step 9-13 i=9..13 继续扩展到 end=14
i=9 'a'→8, i=10 'd'→13, i=11 'e'→14, i=12 'f'→12, i=13 'e'→14
end = max(13,8,13,14,12,14) = 14
切段 i=14 到达 end=14 → 输出段 [0,8] 长度 9
第一段 "ababcbacadefecd"[0..8] = "ababcbacadefecd"? No! 实际 [0..8] = "ababcbacad"
start=9，继续
第二段 i=14 'e' last[e]=14 → end=14, 到达边界
输出 [9,14] 长度 6 → "efecd"

最终结果：[9, 6] → 划分 "ababcbacad" 和 "efecd"

Go 代码

两遍扫描：第一遍记录最后位置，第二遍遍历切段

func partitionLabels(s string) []int {
    // 1. 记录每个字符最后一次出现的位置
    last := [26]int{}
    for i, c := range s {
        last[c-'a'] = i
    }

    result := []int{}
    start, end := 0, 0

    for i, c := range s {
        // 扩展当前段的右边界
        if last[c-'a'] > end {
            end = last[c-'a']
        }
        // 到达当前段最远边界，切段
        if i == end {
            result = append(result, end-start+1)
            start = end + 1
        }
    }
    return result
}

易错点

end 的更新时机容易出错，需要确保 end 是段内所有字符的最后位置

❌ 错误：没有实时更新 end

// 错误：end 只在切段时更新，段内字符的最后位置可能被遗漏
for i, c := range s {
    end = last[c-'a'] // 无条件更新，会提前切段
    if i == end {
        result = append(result, end-start+1)
    }
}

没有 `if last[c] > end` 判断，end 可能被段内靠前字符的最后位置拉回，导致提前切段。

✓ 正确：只扩展不回缩

if last[c-'a'] > end {
    end = last[c-'a'] // 只扩展到更远位置，不回缩
}
if i == end {
    result = append(result, end-start+1)
}

`>` 而不是 `=`：end 只增不减，保证段被充分扩展后才切。

核心

最远最后位置切段

遍历时维护当前段的最远边界，抵达该边界时切段。切点恰好是必须切的位置。

贪心证明

尽量晚切

不提前切分，因为段内任何字符的最后位置如果超出当前边界，就必须延伸。

关键细节

end 只扩展不回缩

`if last[c] > end` 保证 end 只增不减，漏掉会提前切段。
