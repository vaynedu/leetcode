搜索旋转排序数组 II

旋转数组搜索升级版：可能有重复元素

核心洞察

题目：在旋转排序数组中搜索目标值，数组可能有重复元素。
与 #33 的区别：数组中有重复值时，nums[L] == nums[mid] == nums[R] 会导致无法判断哪半边有序。

nums = [2,5,6,0,0,1,2] target = 0

关键：遇到 nums[L] == nums[mid] == nums[R] 时，L++ 收缩左边界。
此时无法判断哪半边有序，但可以确定的是 L 位置的元素一定不是唯一解（因为有重复）。

如何判断哪半边有序？

情况 A：nums[L] != nums[mid]

可以明确判断哪边有序

nums[L] ≤ nums[mid] → 左半边 [L..mid] 有序

target ∈ [nums[L], nums[mid]) → 搜左

否则 → 右半边 [mid..R] 有序

target ∈ (nums[mid], nums[R]] → 搜右

情况 B：nums[L] == nums[mid] == nums[R]

无法判断，两边都可能有序

nums[L] == nums[mid] == nums[R] → L++，跳过这个不确定的位置

不能确定哪边有序时，收缩左边界是最安全的选择

Go 代码模板

func search(nums []int, target int) bool {
    L, R := 0, len(nums)-1

    for L <= R {
        mid := L + (R-L)/2
        if nums[mid] == target {
            return true
        }

        // 关键：无法判断哪边有序时，收缩左边界
        if nums[L] == nums[mid] && nums[mid] == nums[R] {
            L++
        } else if nums[L] <= nums[mid] {  // 左半边有序
            if nums[L] <= target && target < nums[mid] {
                R = mid - 1
            } else {
                L = mid + 1
            }
        } else {  // 右半边有序
            if nums[mid] < target && target <= nums[R] {
                L = mid + 1
            } else {
                R = mid - 1
            }
        }
    }
    return false
}

与 #33 的关键区别

#33 无重复

一定能判断哪边有序
时间复杂度 O(log n)

#81 有重复

nums[L]==nums[mid]==nums[R] 时 L++
时间复杂度 O(n) 最坏情况

核心

判断哪边有序

nums[L] ≤ nums[mid] → 左有序，否则右有序

关键

重复时 L++

无法判断时收缩左边界，跳过重复元素

与33区别

最坏 O(n)

重复元素多时退化为线性搜索
