核心洞察

题目：用两个栈（inStack + outStack）实现队列。
队列 FIFO，栈 LIFO——两个方向相反的数据结构互补。

Go 代码模板

type MyQueue struct {
    inStack  []int  // 入队列时压这里
    outStack []int  // 出队列时从这里弹
}

func Constructor() MyQueue {
    return MyQueue{
        inStack:  []int{},
        outStack: []int{},
    }
}

// 入队：O(1)
func (q *MyQueue) Push(x int) {
    q.inStack = append(q.inStack, x)
}

// 关键：将 inStack 反转到 outStack
func (q *MyQueue) shift() {
    for len(q.inStack) > 0 {
        q.outStack = append(q.outStack, q.inStack[len(q.inStack)-1])
        q.inStack = q.inStack[:len(q.inStack)-1]
    }
}

// 出队：均摊 O(1)
func (q *MyQueue) Pop() int {
    if len(q.outStack) == 0 {
        q.shift()
    }
    val := q.outStack[len(q.outStack)-1]
    q.outStack = q.outStack[:len(q.outStack)-1]
    return val
}

// 查看队首：均摊 O(1)
func (q *MyQueue) Peek() int {
    if len(q.outStack) == 0 {
        q.shift()
    }
    return q.outStack[len(q.outStack)-1]
}

func (q *MyQueue) Empty() bool {
    return len(q.inStack) == 0 && len(q.outStack) == 0
}

均摊 O(1) 分析

每次 pop/peek 时，如果 outStack 为空，会把 inStack 的 n 个元素全部倒入 outStack（O(n)）。
但这 n 个元素的 cost 已经被前面 n 次 push 分摊了，所以每次 pop 的均摊成本是 O(1)。

Amortized 含义

n 次 Push + n 次 Pop
= O(2n) → 每操作 O(1)

最坏情况

单独 peek/pop 时 outStack 空
= O(n)（因为要倒入）

核心

双栈反转顺序

inStack 接收，outStack 吐出，中间的 shift() 反转顺序

时机

outStack 空时倒入

只在 outStack 为空时触发一次倒入，平摊所有 cost
