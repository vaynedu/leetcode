LeetCode 236 · Tree DFS + Postorder

二叉树的最近公共祖先

后序递归：左右子树分别返回目标节点，若左右都非空，当前节点就是 LCA。

核心洞察

题目：给定二叉树中的两个节点 p 和 q，返回它们最深的公共祖先。
关键：递归返回值不是 bool，而是“这棵子树里找到的目标节点或 LCA”。

观察

返回值携带答案候选

空树返回 nil；遇到 p 或 q 直接返回该节点；子树里找到谁就把谁向上传。

判定

左右都非空，当前就是 LCA

如果 p 和 q 分别出现在当前节点左右两边，当前节点是它们第一次汇合的位置。

复杂度

O(n) 时间 O(h) 空间

每个节点最多访问一次；递归栈高度为树高，最坏链式树 O(n)。

手推全过程

样例：root=[3,5,1,6,2,0,8,null,null,7,4]，p=5，q=1

Step 1 dfs(5) returns 5
root == p return 5
递归进入左子树根节点 5，发现它就是 p。
直接返回 5，不需要继续向下找 6、2、7、4。
Step 2 dfs(1) returns 1
root == q return 1
递归进入右子树根节点 1，发现它就是 q。
右子树向上返回 1。
Step 3 root=3 left=5 right=1
left=5 right=1 return root=3
在根节点 3 汇总左右递归结果：左边找到了 5，右边找到了 1。
左右都非 nil，所以 3 是最近公共祖先。

Go 代码

后序遍历：先问左右子树，再根据左右返回值判断当前节点身份。

func LowestCommonAncestor(root, p, q *TreeNode) *TreeNode {
    if root == nil || root == p || root == q {
        return root
    }

    left := LowestCommonAncestor(root.Left, p, q)
    right := LowestCommonAncestor(root.Right, p, q)

    if left != nil && right != nil {
        return root
    }
    if left != nil {
        return left
    }
    return right
}

易错点

这题不能只处理“左右各找到一个”的情况，还要处理“一个节点是另一个节点祖先”的情况。

错误：遇到 p/q 还继续向下找

if root == nil {
    return nil
}
left := dfs(root.Left)
right := dfs(root.Right)
if left != nil && right != nil {
    return root
}

当 p=5、q=4 时，5 是 4 的祖先。若不在 root==p 时返回 5，容易漏掉“祖先本身就是 LCA”。

正确：命中 p/q 立即返回

if root == nil || root == p || root == q {
    return root
}

返回值表示“当前子树里找到的目标或 LCA”。命中 p/q 时，它可能就是最终 LCA。

核心算法

后序 DFS

先递归左右子树，再用左右返回值判断当前节点是否是 LCA。

正确性直觉

第一次汇合点

左右都找到目标时，当前节点就是两条搜索路径从下往上第一次汇合的位置。

易错关键点

命中 p/q 立即返回

因为 p 可能是 q 的祖先，目标节点本身也可能是最近公共祖先。
