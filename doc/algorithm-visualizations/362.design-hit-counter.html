LeetCode 362 · Design

设计命中率计数器

统计过去 5 分钟内的点击次数，每 hit(t) 记一次，每 getHitCount(t) 返回 t-300 到 t 之间的命中数

核心洞察

题目：设计一个计数器，hit(t) 在时间 t 记录一次点击，getHitCount(t) 返回 [t-300, t] 秒内的点击次数。
难点：数据量可能很大（每秒百万次），但时间窗口只有 300 秒。

方案 A（推荐）：环形数组

固定大小 300 数组，时间对 300 取模定位。
hit: arr[t%300]++，count: 遍历 300 个槽
O(300) = O(1)，且无内存泄漏

方案 B：Queue + 过期清理

每次 hit 入队，getHitCount 时弹出所有 < t-300 的元素
O(n) 最坏，但均摊尚可

环形数组原理

300 秒的窗口，只需要 300 个槽。index = t % 300 决定写入位置。
关键：记录每个槽最后一次写入的时间戳，过期则忽略。

数组结构: counts[300] + timestamps[300]

hit(100): index=100%300=100 → counts[100]++, timestamps[100]=100
hit(400): index=400%300=100 → counts[100]++, timestamps[100]更新为400
hit(700): index=700%300=100 → counts[100]++, timestamps[100]=700
getHitCount(700): 检查 timestamps[100]=700, 在窗口内 → 1次命中
（实际应叠加所有 300 个槽，下一页手推详述）

手推全过程

时间线：hit(1), hit(2), hit(3), hit(301), hit(302), getHitCount(303)

时间戳 1 2 3 ...301 ...302 index(303%300)=3
slot index 1 2 3 1 2 3
counts 1 1 覆盖!0 1 1 1
ts 1 2 303 301 302 303
Step 1 hit(1), hit(2), hit(3) → slots[1]=1, [2]=1, [3]=1，timestamps 分别为 1,2,3
Step 2 hit(301) → index=1，slot[1]++，ts[1]=301
Step 3 hit(302) → index=2，slot[2]++，ts[2]=302
Step 4 getHitCount(303) → 遍历 slots，保留 ts >= 3 的
slot[0]: ts=0 < 3 → 跳过
slot[1]: ts=301 ≥ 3 → 保留，count+=1
slot[2]: ts=302 ≥ 3 → 保留，count+=1
slot[3]: ts=303 ≥ 3 → 保留，count+=1
getHitCount(303) = 3

Go 代码模板（环形数组）

const WINDOW = 300

type HitCounter struct {
    counts     [WINDOW]int   // 每秒命中计数
    timestamps [WINDOW]int   // 每槽最后更新时间戳
}

func Constructor() HitCounter {
    return HitCounter{}
}

// hit: O(1)
func (h *HitCounter) Hit(timestamp int) {
    idx := timestamp % WINDOW
    // 如果时间戳变了（新的一秒），重置计数
    if h.timestamps[idx] != timestamp {
        h.counts[idx] = 1
        h.timestamps[idx] = timestamp
    } else {
        h.counts[idx]++
    }
}

// getHitCount: O(WINDOW) = O(300) = O(1)
func (h *HitCounter) GetHitCount(timestamp int) int {
    count := 0
    for i := 0; i < WINDOW; i++ {
        // 只统计窗口内的（timestamp - 300 < ts <= timestamp）
        if h.timestamps[i] > timestamp-WINDOW && h.timestamps[i] <= timestamp {
            count += h.counts[i]
        }
    }
    return count
}

面试亮点分析

亮点 1：环形数组复用

固定 300 槽，时间对 300 取模。
vs Queue 方案：无内存持续增长，
因为数组大小恒定。

亮点 2：时间戳去重

同一秒内多次 hit，只需增加计数，
不创建新槽。时间戳比对决定
是否 reset counts[idx]。

扩展 1：分布式版本

多机器：每个机器独立计数，
定时上报聚合。
用 Redis INCRBY 按时间桶聚合。

扩展 2：可变时间窗口

支持任意窗口大小 N：
数组大小改为 N，按 t%N 定位。
getHitCount 改为检查 ts > t-N。

核心

环形数组 + 时间戳

固定 300 槽，时间取模定位，过期检测决定是否计入

时间

hit O(1)，getHitCount O(300)

O(300) = O(1) 常数，环形数组无 GC 压力

面试

可变时间窗口扩展

改变 WINDOW 常量即可支持任意窗口大小
