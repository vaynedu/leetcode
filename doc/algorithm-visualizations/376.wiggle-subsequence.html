LeetCode 376 · Greedy

摆动序列

找最长摆动子序列 — 贪心的精髓：只数峰和谷，平坡全部跳过；峰谷计数法 O(n)

核心洞察

题目：找最长摆动子序列（相邻差值正负交替）。
关键：只需要统计「峰」和「谷」的数量，上升和下降段中间的点全部可以跳过。 等价于统计差值序列中符号变化的次数。

观察

中间点可以跳过

上升段中间的任何一个点，都不是谷也不是峰，对摆动没有贡献。

贪心选择

只数峰和谷

遍历时遇到方向变化就计数，方向不变则跳过。

复杂度

O(n) 时间 O(1) 空间

一次遍历，只需要 prevDiff 和 result 两个变量。

手推全过程

nums = [1,7,4,9,2,5]

Step 1 差值序列: +6, -3, +5, -7, +3
初始 result = 1（至少一个元素）
Step 2 遍历差值
+6 > 0 → prev=+6；-3 < 0 → 符号变！result=2，prev=-3
+5 > 0，符号变！result=3，prev=+5
-7 < 0，符号变！result=4，prev=-7
+3 > 0，符号变！result=5，prev=+3

最终结果：摆动长度 = 6（整个序列本身就是一个摆动序列）

Go 代码

峰谷计数法，O(n)

func wiggleMaxLength(nums []int) int {
    if len(nums) < 2 {
        return len(nums)
    }

    result := 1
    i := 1

    // 跳过开头的相等元素
    for i < len(nums) && nums[i] == nums[i-1] {
        i++
    }
    if i == len(nums) {
        return 1
    }

    prevDiff := nums[i] - nums[i-1]
    if prevDiff > 0 { result = 2 }

    for i++; i < len(nums); i++ {
        diff := nums[i] - nums[i-1]
        if (diff > 0 && prevDiff <= 0) || (diff < 0 && prevDiff >= 0) {
            result++
            prevDiff = diff
        }
    }
    return result
}

易错点

差值为 0 时不是摆动，需要跳过

❌ 错误：差值相等时也计数

// [1,2,2,2,1] 时会多数
prevDiff := nums[1] - nums[0]
for i := 2; i < len(nums); i++ {
    diff := nums[i] - nums[i-1]
    if diff * prevDiff < 0 { result++ } // diff=0 时 0*prevDiff=0，不触发

差值为 0 时不产生摆动，但如果 prevDiff 和 diff 都是 0，product=0 不会触发计数。

✓ 正确：跳过相等元素

// 先跳过开头的相等元素
for i < len(nums) && nums[i] == nums[i-1] { i++ }
// 遍历时 diff 和 prevDiff 同号则跳过，异号（且非零）则计数

严格判断符号变化 (diff>0 && prev<=0) || (diff<0 && prev>=0)，相等时 prevDiff 保持不变。
