买卖股票最佳时机

一次遍历：边跟踪历史最低价，边计算当前卖出的最大利润 — 贪心体现在"只关心到今天为止的最低价"

核心洞察

题目：在某天买入、某天卖出，求最大利润（只能先买后卖一次）。
关键：不必枚举所有买卖组合，只需在遍历时维护"到今天为止的最低价"， 每次用当前价格减去历史最低价，就是以今天卖出的最大利润。

观察

利润 = 卖出价 − 买入价

想最大化利润，等价于：对于每个卖出日，找到之前最便宜的买入日。

贪心选择

维护历史最低价

一次遍历中，minPrice 永远记录「之前出现过的最低价格」，每次比较当前利润更新 maxProfit。

复杂度

O(n) 时间 O(1) 空间

只遍历一次，两个变量 minPrice 和 maxProfit 是常量级空间。

手推全过程

prices = [7,1,5,3,6,4]，求最大利润

Step 1 i=0 price=7 minPrice=7 maxProfit=0
初始化：minPrice=7（第一个价格），maxProfit=0（还不确定能否赚钱）
以今天卖出利润 = 7-7 = 0，不更新 maxProfit
Step 2 i=1 price=1 → minPrice=1 maxProfit=0
遇到更低价：1 < 7 → minPrice = 1
以今天卖出利润 = 1-1 = 0，maxProfit 保持 0
Step 3 i=2 price=5 minPrice=1 maxProfit=4
5 ≥ minPrice(1)，minPrice 不变
以今天卖出利润 = 5-1 = 4 > maxProfit(0) → maxProfit=4！
找到了一个盈利方案：在第1天(价格1)买入，第2天(价格5)卖出
Step 4 i=3 price=3 minPrice=1 maxProfit=4
3 ≥ minPrice(1)，minPrice 不变
以今天卖出利润 = 3-1 = 2 < 4，不更新
Step 5 i=4 price=6 minPrice=1 maxProfit=5
以今天卖出利润 = 6-1 = 5 > 4 → maxProfit=5！
最佳方案更新：在第1天(价格1)买入，第4天(价格6)卖出
Step 6 i=5 price=4 minPrice=1 maxProfit=5
以今天卖出利润 = 4-1 = 3 < 5，不更新

最终结果：最大利润 = 5（第1天买入 price=1，第4天卖出 price=6）

Go 代码

一次遍历，维护最低价和最大利润

func maxProfit(prices []int) int {
    minPrice := int(1e9) // 初始化为极大值
    maxProfit := 0

    for _, price := range prices {
        // 更新历史最低价
        if price < minPrice {
            minPrice = price
        }
        // 以今天价格卖出的利润 = 今天价格 - 历史最低价
        profit := price - minPrice
        if profit > maxProfit {
            maxProfit = profit
        }
    }
    return maxProfit
}

易错点

这题看似简单，但初始化和比较顺序容易出错

❌ 错误：minPrice 初始化为 prices[0]

// 如果 prices 为空，数组越界
minPrice := prices[0]
for i := 1; i < len(prices); i++ { ... }

空数组时 panic。若不用空数组保护，最小值初始化为一个大数更安全。

✓ 正确：minPrice 初始化为 math.MaxInt

minPrice := int(1e9) // 或 math.MaxInt
for _, price := range prices {
    if price < minPrice { minPrice = price }
    maxProfit = max(maxProfit, price-minPrice)
}

先比价再更新 maxProfit 的顺序很重要——确保 minPrice 是「当前价格之前」的最低价。

另一个陷阱：先更新 maxProfit 还是先更新 minPrice？
如果先算 profit 再更新 minPrice，price-minPrice 会用刚更新的 minPrice（等于当前 price），利润为 0。 正确顺序是：先更新 minPrice，再计算 profit。

核心

一次遍历 + 两个变量

minPrice 维护历史最低买入价，maxProfit 记录遍历过程中出现的最大差值。

贪心证明

最优子结构

以第 i 天卖出的最优利润，只取决于 [0..i-1] 的最低价，与之后无关。

关键细节

先更新 minPrice，再算 profit

顺序错误会导致当前价格被当作最低价，profit 变成 0。
