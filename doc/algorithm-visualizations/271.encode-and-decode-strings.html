LeetCode 271 · Design

字符串编码与解码

将字符串列表编码成单个字符串，并能够解码回原列表

核心洞察

题目：设计 encode/decode 接口，["hello", "world"] → 编码 → 解码回原列表。
难点：字符串本身可能包含 #、数字等特殊字符，简单的拼接+分隔符方案会出错。
关键：用 "长度#内容" 格式，长度和内容用 "#" 分隔。

编码过程：

原始 ["hello", "world"]
↓ encode
编码 5#hello5#world

为什么不用分隔符？
如果字符串包含分隔符（如 "a#b"），简单拼接就无法唯一解析。
用 "长度#内容"：每个 segment 前缀了数字长度，解码时先读数字，再读对应长度字符，天然避开了内容中的特殊字符。

手推全过程

编码: ["lint", "code", "love", "you"]，解码: "4#lint4#code4#love3#you"

Encode 遍历每个字符串，拼成 "len#str"
"lint" → len=4 → "4#lint"
"code" → len=4 → "4#code"
"love" → len=4 → "4#love"
"you" → len=3 → "3#you"
拼接 → "4#lint4#code4#love3#you"
Decode 逐字符解析 "4#lint4#code4#love3#you"
指针 p=0，读数字: 4，遇到 '#'
取 str[0:4]="lint"，p 移到下一段开头
继续读数字: 4 → str[4:8]="code"
继续读数字: 4 → str[8:12]="love"
继续读数字: 3 → str[12:15]="you"
解码完成: ["lint", "code", "love", "you"]

必须避免的陷阱

陷阱 1：纯分隔符方案

用 "|" 分隔："hello|world" → "a|b" 编码成 "a|b" 无法区分。
即使选罕见分隔符，面试官会说 "字符串里就有 | 怎么办？"

陷阱 2：数字转字符

把长度转成单个字符（char(4)='E'）。问题：字符串长度可能 > 255（一个字节）。
超出范围就崩溃。

陷阱 3：JSON 序列化

虽然 JSON 能处理特殊字符，但过度工程化。
面试官想看到的是 "长度#内容" 的朴素编码思想。

Go 代码模板

type Codec struct{}

func (c *Codec) Encode(strs []string) string {
    var sb strings.Builder
    for _, s := range strs {
        sb.WriteString(strconv.Itoa(len(s)))
        sb.WriteByte('#')
        sb.WriteString(s)
    }
    return sb.String()
}

func (c *Codec) Decode(data string) []string {
    var result []string
    p := 0
    for p < len(data) {
        // 读数字（长度）
        start := p
        for data[p] != '#' {
            p++
        }
        // 此时 data[p] == '#'
        length, _ := strconv.Atoi(data[start:p])
        p++ // 跳过 '#'

        // 读对应长度的内容
        result = append(result, data[p:p+length])
        p += length
    }
    return result
}

扩展与面试追问

扩展：嵌套结构

encode(deepList) 时递归处理：
基础对象 → 直接 len#content
数组/对象 → 递归编码后整体作为字符串处理

扩展：长字符串分块

如果单个字符串 > 1MB，分块传输：
"chunk_5#hello" + "chunk_8#worldstuff"
解码时合并 chunk

面试加分点

说出 "长度#内容" 方案，并解释为什么比 JSON/分隔符更好（简单、无依赖、长度透明）。这是大厂系统设计的基础编码思想。

实际应用

TCP 粘包/拆包：用同样思路——
每条消息前加长度字段。
Redis 协议 RESP 也是这个思想。

核心

长度#内容 格式

每个 segment = 数字 + "#" + 原始内容，避免特殊字符冲突

解码

找 # 读长度，再读内容

指针遍历：遇到 # 停下来读长度，然后跳过对应长度字符

精华

TCP/Redis 协议同款

这是所有二进制协议的基础：长度前缀 + 变长内容
