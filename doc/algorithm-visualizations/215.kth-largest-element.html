215. 数组第K大元素

快速选择 Quick Select — 平均 O(n)，最坏 O(n²)

快速选择核心思想

类似快排的 partition，每次把 pivot 放到正确位置：

  • pivot 排好序后的下标 = p
  • 若 p == target → 找到答案
  • 若 p < target → 答案在右半边
  • 若 p > target → 答案在左半边

为什么比快排快？

快排两边都递归，快速选择只递归一边。
平均 T(n) = T(n/2) + O(n) = O(n)
只递归一边，剪掉了另一半

k = 2

pivot=4 最终下标 = 2 → 升序第 3 小，不是第 4 小 → 递归右半边

递归 trace（5 步）

① partition 范围 [0,5]，target=4，pivot=4 → p=2
p=2 < target=4 → 递归右半 [3,5]
② partition 范围 [3,5]（对应元素 [5,6,4]），target=4
pivot=4 → 交换后 p=3（只有 4 本身在 <= 区）
③ partition 范围 [4,5]（对应元素 [6,4]），target=4
pivot=4 → p=4（4 是这个子数组第一个）
④ check p=4 == target → 找到答案，返回 nums[4]=6
⑤ answer 快速选择返回 6，即第 2 大的元素 ✓

两种解法对比

快速选择（默认）
