LeetCode 406 · Greedy

根据身高重建队列

按身高降序排列后，按 k 值插入 — 贪心的精髓：高个子先排好序，低个子依次按 k 插入时自然满足所有约束

核心洞察

题目：people = [[身高,k], ...]，k 是前面不低于此人身高的个数，重建队列。
关键：按身高降序排列后，高个子已经就位，此时 k 值就是它在当前队列中的位置。 依次把每个人插入到下标 k 的位置，低个子不会影响高个子。

观察

高个子不受低个子影响

把高个子排好后，低个子插入时，高个子的相对顺序不变。

贪心选择

先处理最高的

每次插入一个人到位置 k，插入后队列仍然有序，因为所有人都是按身高从高到低处理的。

复杂度

O(n log n) 排序 + O(n²) 插入

排序是主要瓶颈。插入操作 O(n)，整体 O(n²)。用链表可优化到 O(n)。

手推全过程

people = [[7,0],[6,1],[7,0],[5,0],[5,2],[4,4]]

Step 1 排序后: [7,0],[7,0],[6,1],[5,0],[5,2],[4,4]
按身高降序，身高相同时按 k 升序

队列: [[7,0]]
Step 3 插入 [7,0] 到位置 0
队列: [[7,0],[7,0]]（位置 0 被占用，原来的往后移）
Step 4 插入 [6,1] 到位置 1
队列: [[7,0],[6,1],[7,0]]
Step 5 插入 [5,0] 到位置 0
队列: [[5,0],[7,0],[6,1],[7,0]]
Step 6 插入 [5,2] 到位置 2
队列: [[5,0],[5,2],[7,0],[6,1],[7,0]]
Step 7 插入 [4,4] 到位置 4
队列: [[5,0],[5,2],[7,0],[6,1],[4,4],[7,0]]

最终结果：[[5,0],[5,2],[7,0],[6,1],[4,4],[7,0]]

Go 代码

按身高降序 + k 升序排序，然后依次插入

func reconstructQueue(people [][]int) [][]int {
    // 1. 按身高降序，k 升序排序
    sort.Slice(people, func(i, j int) bool {
        if people[i][0] != people[j][0] {
            return people[i][0] > people[j][0] // 身高降序
        }
        return people[i][1] < people[j][1]   // k 升序
    })

    // 2. 依次插入到位置 k
    result := [][]int{}
    for _, p := range people {
        k := p[1]
        // 插入到下标 k 的位置（后面元素后移）
        result = append(result, []int{})
        copy(result[k+1:], result[k:])
        result[k] = p
    }
    return result
}

易错点

排序顺序是核心，必须先高后低，否则插入时会破坏高个子的约束

❌ 错误：按身高升序排列

sort.Slice(people, func(i,j int)bool{
    return people[i][0] < people[j][0] // 升序！
})
// 矮个子先处理，插入后被高个子打乱，无法恢复

升序时，先插入矮个子，高个子后来时无法正确插入，因为会破坏已有的身高顺序。

✓ 正确：按身高降序排列

// 排序后: [7,0],[7,0],[6,1],[5,0],[5,2],[4,4]
// 依次插入: [7,0]→pos0, [7,0]→pos0, [6,1]→pos1 ...
// 高个子先占位，低个子不会影响身高约束

降序保证了每次插入时，所有更高的人已经就位，k 值就是当前应该插入的位置。

核心

先高后低依次插入

按身高降序排序后，依次插入到位置 k，高个子的相对顺序不受低个子影响。

贪心证明

最高的人位置是确定的

最高的人 k 值就是它最终位置，因为没有人比它高。

关键细节

身高降序，相等时 k 升序

相等身高按 k 升序，保证插入位置不越界。
