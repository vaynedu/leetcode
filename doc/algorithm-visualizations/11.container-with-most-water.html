LeetCode 11 · Two Pointers

盛最多水的容器

从最大宽度开始向内收敛：面积由短板决定，所以每一步移动较短的那一边。

核心洞察

题目：选两个下标 i < j，最大化 min(height[i], height[j]) * (j - i)。
关键：宽度收缩不可避免，只有移动短板，才可能换来更高的有效高度。

观察

短板决定装水高度

两根柱子再远，水面高度也只能到较矮柱子的高度。

选择

每轮移动短板

移动长板只会让宽度变小，短板上限不变，不可能更优。

复杂度

O(n) 时间 O(1) 空间

左右指针只向中间移动，每个位置最多被访问一次。

手推全过程

样例：height = [1,8,6,2,5,4,8,3,7]

Step 1 left=0, right=8
width = 8 min(1, 7) = 1 area = 8 1 < 7，left++
当前最大面积更新为 8。
左边 1 是短板，以它为左边界的其它组合宽度更小，高度最多还是 1，可以排除。
Step 2 left=1, right=8
width = 7 min(8, 7) = 7 area = 49 8 > 7，right--
发现当前最优：下标 1 和 8，面积 7 * 7 = 49。
右边 7 是短板，继续尝试把右边界换成更高的柱子。
Step 3 left=1, right=7
width = 6 min(8, 3) = 3 area = 18 8 > 3，right--
面积小于 49，最大值不变。
右边 3 更短，保留左边 8，继续向内找更高右边界。
Step 4 left=1, right=6
width = 5 min(8, 8) = 8 area = 40 相等，移动任意一边
高度更高了，但宽度已经从 7 缩到 5，所以面积仍然小于 49。
相等时移动任意一边都可以；代码里用 else 移动右边。

Go 代码

双指针从两端向内收敛，每轮先计算面积，再移动较短边。

func maxArea(height []int) int {
    left, right := 0, len(height)-1
    best := 0

    for left < right {
        width := right - left
        h := height[left]
        if height[right] < h {
            h = height[right]
        }

        area := width * h
        if area > best {
            best = area
        }

        // 宽度必然变小，只移动短边才可能提高有效高度
        if height[left] < height[right] {
            left++
        } else {
            right--
        }
    }
    return best
}

易错点

这题的核心不是会写双指针，而是能解释为什么移动短板不会漏答案。

错误：移动长板

if height[left] < height[right] {
    right--
} else {
    left++
}

移动长板时，短板高度没有变高，宽度却变小，可能错过真正答案 49。

正确：移动短板

if height[left] < height[right] {
    left++
} else {
    right--
}

短板内移后，才有机会遇到更高的柱子，抵消宽度减少。

错误：高度取 max

area := max(height[left], height[right]) *
        (right - left)

容器水面会被较矮柱子限制，取高柱会把面积算大。

正确：高度取 min

h := min(height[left], height[right])
area := h * (right - left)

宽度来自两个下标距离，高度来自短板。

核心算法

双指针两端收敛

从最大宽度开始，每轮计算面积，再移动较短边。

正确性直觉

短板组合可整批排除

短板不变且宽度变小，面积不可能更大，所以能安全移动短板。

易错关键点

面积高度取 min

装水高度由较矮的柱子决定，不能用较高柱子计算。
