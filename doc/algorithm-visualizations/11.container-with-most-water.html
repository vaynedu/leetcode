② 核心思想：为什么移动短边？

面积由 宽度 和 短板高度 共同决定。宽度只会随着指针内移而减小，所以要想面积更大，必须让 短板变高。

如果移动长边 →

假设 height[left] < height[right]

宽度 = right − left（必定减小）

高度 ≤ height[left]（短板还是 left）

宽度↓ × 高度不变或↓
→ 面积不可能变大

如果移动短边 →

height[left] < height[right]

新 left 可能是更高的柱子

宽度减小，但高度可能增加

高度↑ × 宽度↓
→ 有可能找到更大面积

一句话：宽度已经注定要减小，只有让短板变高才有可能打破当前面积的上限。
所以：height[left] < height[right] → left++
否则：height[right] ≤ height[left] → right--

核心思想

夹逼

左右指针从两端向中间收缩，
宽度必定减小，所以要找更高的短板。

决策规则

移动短边

height[L] < height[R] → L++
height[R] ≤ height[L] → R--

时间复杂度

O(n)

每个指针最多移动 n 步，
不回头，所以是线性时间。

Time · O(n)  ·  Space · O(1)  ·  Two pointers, inward
