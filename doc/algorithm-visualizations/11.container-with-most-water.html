LEETCODE 11 · CONTAINER WITH MOST WATER

装水容器

双指针左右夹逼 · 短板决定高度 · 移动短边指针

核心洞察

每根柱子的高度 = height[i]，两柱子之间的水容量 = 宽度 × 短板高度。 双指针从两端向中间收敛，宽度必定减小，所以要找更高的短板来增大面积。

Go 代码

双指针夹逼，O(n) 时间，O(1) 空间

// LeetCode 11. Container With Most Water
func maxArea(height []int) int {
    left, right := 0, len(height)-1
    max := 0

    for left < right {
        // 短板决定高度
        h := min(height[left], height[right])
        w := right - left
        area := h * w

        if area > max {
            max = area
        }

        // 移动短边指针
        if height[left] < height[right] {
            left++
        } else {
            right--
        }
    }
    return max
}

易错点

双指针的决策逻辑和边界条件容易出错

❌ 错误：移动长边指针
// 错误：长边不动，短边内移 ← 对 // 但写成：长边内移 ← 错！ if height[left] > height[right] { right-- // ← 这样会漏掉最优解 }
如果 height[left] > height[right]，
说明 left 是长边，此时让 right--（移动短边）。
如果写成 left++（移动长边），面积只会更小。
✓ 正确：统一移动短边
if height[left] < height[right] { left++ // 短边内移 } else { right-- // 短边内移（包含相等的情况） }
不管谁长谁短，统一移动较短的那一边。
相等时移动任意一边都行，用 else 覆盖。
❌ 错误：循环条件写成 left <= right
for left <= right { // ← left==right 时会多算一次 ... }
left == right 时，单柱子没有宽度，面积为 0。
且 left == right 时再移动会重复计算。
用 left < right 确保不重叠。
✓ 正确：left < right（左闭右开逻辑）
for left < right { // ← 正确 ... }
双指针区间是 [left, right]，不重叠。
当 left == right 时已经相遇，无需再计算。
这是标准的左闭右开区间逻辑。
