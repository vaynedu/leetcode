LeetCode 45 · Greedy

跳跃游戏 II

求到达最后一个位置的最少跳数 — 贪心的精髓：在当前覆盖范围内选择能跳最远的下一步，步数最少

核心洞察

题目：nums[i] 表示从 i 能跳的最远距离，求到末尾的最少跳数。
关键：维护当前步能到的最远边界 curEnd，和这一步覆盖范围内的最远扩展 curFarthest。 当遍历到 curEnd 时，增加一步，用 curFarthest 更新 curEnd。

观察

BFS 等价于贪心

每一步的选择构成 BFS 的层，最少跳数 = BFS 的最小深度，但不需要构建 BFS 树。

贪心选择

每步选能跳最远的

在当前覆盖范围内，用 curFarthest 决定下一步跳到哪。

复杂度

O(n) 时间 O(1) 空间

一次遍历，三个变量：steps、curEnd、curFarthest。

手推全过程

nums = [2,3,1,1,4]

初始化 steps=0 curEnd=0 curFarthest=0
i=0 curFarthest=max(0,0+2)=2
i=0 在 curEnd=0 范围内，扩展 curFarthest=2
i == curEnd(0) → steps++, curEnd=curFarthest=2
i=1,2 curEnd=2
i=1: curFarthest=max(2,1+3)=4；i=2: curFarthest=max(4,2+1)=4
i==curEnd(2) → steps++, curEnd=curFarthest=4
i=3,4 curEnd=4 ≥ len-1，结束
curEnd=4 已覆盖末尾，steps=2

最终结果：最少 2 步（0→1→4 或 0→2→4）

Go 代码

贪心 BFS，O(n) 一次遍历

func jump(nums []int) int {
    steps := 0
    curEnd := 0
    curFarthest := 0

    for i := 0; i < len(nums)-1; i++ {
        // 扩展最远可达
        curFarthest = max(curFarthest, i+nums[i])

        // 到达当前步的边界，增加一步
        if i == curEnd {
            steps++
            curEnd = curFarthest
        }
    }
    return steps
}

易错点

curEnd 的更新时机容易出错，必须在 i == curEnd 时才增加步数

❌ 错误：没有 curEnd 边界判断

// 每次都选最大跳距，但跳跃次数可能不是最少的
reach := 0; steps := 0
for i := 0; i < len(nums)-1; {
    reach = max(reach, i+nums[i])
    i = reach // 跳到最远，不一定是当前步最优
    steps++
}

[2,3,1,1,4] 中从 0 直接跳到 2（reach=2）再加到 4 是 2 步，但可能跳过头。

✓ 正确：curEnd 边界 + curFarthest

curEnd := 0
curFarthest := 0
steps := 0
for i := 0; i < len(nums)-1; i++ {
    curFarthest = max(curFarthest, i+nums[i])
    if i == curEnd {
        steps++
        curEnd = curFarthest
    }
}

在 curEnd 边界内积累所有扩展，边界到了才跳，确保每跳都是当前能达到最远的 BFS 层。
