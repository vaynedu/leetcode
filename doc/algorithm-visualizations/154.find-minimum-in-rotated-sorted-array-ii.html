寻找旋转排序数组最小值 II

#153 的升级版：数组中可能包含重复元素

核心洞察

题目：找旋转排序数组中的最小值，数组可能有重复元素。
与 #153 的区别：遇到 nums[mid] == nums[R] 时无法判断最小值在左还是右。

处理方式：R-- 跳过右端的重复元素

nums[mid] == nums[R] 时，最小值一定不在 nums[R]（因为 nums[R] 和 nums[mid] 一样大），所以 R-- 跳过它。

nums[mid] > nums[R]

最小值在右半边
L = mid + 1

nums[mid] == nums[R]

无法判断
R-- 跳过

nums[mid] < nums[R]

最小值在左半边
R = mid

Go 代码模板

func findMin(nums []int) int {
    L, R := 0, len(nums)-1

    for L < R {
        mid := L + (R-L)/2

        if nums[mid] > nums[R] {
            L = mid + 1
        } else if nums[mid] < nums[R] {
            R = mid
        } else {
            // nums[mid] == nums[R]，无法判断哪边，R-- 跳过
            R--
        }
    }
    return nums[R]
}

与 #153 的关键区别

#153 无重复

nums[mid] != nums[R] 时才进入分支
严格二分，时间 O(log n)

#154 有重复

多一个分支：nums[mid]==nums[R] 时 R--
最坏 O(n)，但实际很快

核心

R-- 处理重复

nums[mid] == nums[R] 时，R-- 跳过无法判断的情况

判断

三分支

大于、小于、等于分别处理

效率

最坏 O(n)

大量重复时退化为线性，但平均仍很快
