LeetCode 56 · Greedy

合并区间

按起点排序，依次合并重叠区间 — 贪心的精髓在于"只关心当前区间和已合并结果的最后一个"

核心洞察

题目：输入若干区间，合并所有重叠区间后返回。
关键：先按起点排序，排序后每个区间只可能和"已合并结果的最后一个"产生重叠。 这就是贪心能成立的原因 — 不需要回头看更早的区间。

观察

排序后只需比较两个邻居

区间按起点排序后，区间 i 要么和区间 i-1 重叠，要么独立。不存在跨越两个以上区间的大重叠。

贪心选择

扩展 vs 新建二选一

当前区间起点 ≤ 上一个合并区间的终点 → 扩展；否则 → 新建区间。简单明了。

复杂度

O(n log n) 瓶颈在排序

一次遍历 O(n)，排序 O(n log n)。合并过程本身是 O(n) 的。

手推全过程

输入：[[1,3],[2,6],[8,10],[15,18]]，手推每一步在想什么

Step 1 排序后：[1,3] [2,6] [8,10] [15,18]
按起点从小到大排序，这是整个算法的前提。
merged = [[1,3]]，以第一个区间作为起始基准
Step 2 处理 [2,6]
当前区间 [2,6]，上一个合并结果 [1,3]
2 ≤ 3？是，重叠！merged[-1].end = max(3, 6) = 6
merged = [[1,6]]
Step 3 处理 [8,10]
当前区间 [8,10]，上一个合并结果 [1,6]
8 ≤ 6？否，不重叠，直接加入
merged = [[1,6], [8,10]]
Step 4 处理 [15,18]
当前区间 [15,18]，上一个合并结果 [8,10]
15 ≤ 10？否，不重叠，直接加入
merged = [[1,6], [8,10], [15,18]]

最终结果：[[1,6], [8,10], [15,18]]

Go 代码

核心：排序后一次遍历，比较当前起点和 merged 尾部终点

func merge(intervals [][]int) [][]int {
    // 1. 按起点升序排序
    sort.Slice(intervals, func(i, j int) bool {
        return intervals[i][0] < intervals[j][0]
    })

    // 2. 第一个区间作为基准
    merged := [][]int{intervals[0]}

    // 3. 遍历剩余区间
    for i := 1; i < len(intervals); i++ {
        cur := intervals[i]
        // 比较当前起点和 merged 尾部终点的关系
        if cur[0] <= merged[len(merged)-1][1] {
            // 重叠：扩展 merged 尾部区间的终点（取较大值）
            merged[len(merged)-1][1] = max(merged[len(merged)-1][1], cur[1])
        } else {
            // 不重叠：直接加入结果
            merged = append(merged, cur)
        }
    }
    return merged
}

易错点

合并条件的边界容易出错，用 ≤ 还是 < 决定了是否会把相邻端点错误合并

❌ 错误：写成 <

if cur[0] < merged[-1][1] {  // 漏掉等于的情况
    merged[-1][1] = max(...)

[1,2] 和 [2,3] 相邻不重叠，但如果用 <，2 < 2 为 false，正确区间被错误跳过，返回了 [[1,2]] 而不是 [[1,3]]。

✓ 正确：写成 <=

if cur[0] <= merged[-1][1] {
    merged[-1][1] = max(merged[-1][1], cur[1])

相等时 [2,3] 的 2 和 [1,2] 的 2 相邻，合并后是 [1,3]，这是正确行为。贪心就是靠这个 ≤ 把相邻区间正确合并。

另一个常见错误：排序方式。
如果按终点排序会怎样？[[1,10],[2,3]] 按终点排序变成 [2,3],[1,10]，遍历时 1≤3 会合并成 [1,10]，丢失了中间的空隙。必须按起点排序。

核心

排序 + 单次遍历

按起点排序后，每个区间只需和上一个合并结果比较，不回头看更早的。

贪心证明

最优子结构

排序后最优解的第一个区间就是排序后的第一个区间，之后的最优解由剩余区间的最优合并构成。

关键细节

用 ≤ 不是 <

相等时端点相邻，应该合并。漏掉等号会导致 [[1,2],[2,3]] 被错误处理成两个区间。
