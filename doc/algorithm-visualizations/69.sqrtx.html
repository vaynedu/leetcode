x 的平方根

在 [1, x] 范围内二分，找 floor(√x)

核心洞察

题目：计算 x 的平方根的整数部分（即 floor(√x)）。
关键：在 [0, x] 范围内二分，找到最大的 m 使得 m² ≤ x。

x = 16
目标
→
√x = 4
要找的是
4² = 16 ≤ 16 ✓
→
floor = 4
整数部分
5² = 25 > 16 ✗

为什么用 mid*mid 而不是 mid？
本质是找最大的 m 使得 m² ≤ x。在 [0, x] 范围内二分，mid*mid <= x 时答案在右半边。

手推全过程

x = 16，找 floor(√16)

Step 1 L=0 R=16 mid=8
mid=8
检查 8²
8²=64 > 16 ✗
mid=8, 8²=64 > 16 → R = mid - 1 = 7
答案在左半边 [0, 7]
Step 2 L=0 R=7 mid=3
mid=3
检查 3²
3²=9 < 16 ✓? 不...
mid=3, 3²=9 ≤ 16 ✓
ans=3, L = mid + 1 = 4，答案可能在右半边
Step 3 L=4 R=7 mid=5
mid=5
检查 5²
5²=25 > 16 ✗
mid=5, 5²=25 > 16 → R = mid - 1 = 4
Step 4 L=4 R=4 mid=4
mid=4
检查 4²
4²=16 ≤ 16 ✓
mid=4, 4²=16 ≤ 16 ✓
ans=4, L = mid + 1 = 5
Step 5 L=5 R=4 → 退出
L=5, R=4 → L > R，退出循环
ans=4 就是 floor(√16)

最终结果：ans = 4

Go 代码模板

用 ans 记录满足条件的最大 mid，防止溢出用 x/mid 而不是 mid*mid

func mySqrt(x int) int {
    L, R := 0, x
    ans := 0
    for L <= R {
        mid := L + (R-L)/2
        if mid*mid <= x {
            ans = mid    // mid 可能是答案，记录下来
            L = mid + 1  // 尝试更大的
        } else {
            R = mid - 1
        }
    }
    return ans
}

防溢出：mid*mid 可能超出 int 范围，改用 mid <= x/mid 更安全。

溢出陷阱

❌ mid*mid 可能溢出

x=INT_MAX 时，mid*mid 会溢出 int64

INT_MAX ≈ 2.1B，mid ≈ 46340 时
mid*mid ≈ 2.1B，刚好接近上限

✓ 用 x/mid 代替 mid*mid

mid <= x/mid 等价于 mid² ≤ x

x/mid 不可能溢出，因为 x 和 mid 都是 int

核心

找最大满足条件的值

ans 记录满足 mid² ≤ x 的最大 mid

关键

ans 单独记录

循环结束时 ans 就是答案，不是 L 或 R

防坑

x/mid 代替 mid*mid

避免整数溢出，Go 中 int 可能溢出
