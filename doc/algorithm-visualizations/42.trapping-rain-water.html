LeetCode 42 · Two Pointers + Boundary Max

接雨水

每个位置的水量由左右最高墙的较小值决定；双指针每次结算较低侧。

核心洞察

题目：给定柱子高度，计算下雨后能接多少水。
关键：位置 i 的水量 = min(左侧最高墙, 右侧最高墙) - height[i]，双指针用 leftMax/rightMax 在线结算。

观察

每格独立结算

水不是按区间一次算，而是每个位置看左右最高边界。

选择

处理较低侧

较低侧的另一边已经有足够边界，可以用当前侧历史最高墙结算。

复杂度

O(n) 时间 O(1) 空间

每个下标只被一个指针处理一次，只维护两个最高墙。

手推全过程

样例：height = [0,1,0,2,1,0,1,3,2,1,2,1]，最终水量为 6。

Step 1 left=0, right=11
height[left]=0 height[right]=1 处理 left leftMax=0
左侧更低，右边至少有高度 1 的边界，当前位置只需要看 leftMax。
height[0] 更新 leftMax，不接水。
Step 2 left=2, right=10
leftMax=1 height[2]=0 water += 1
下标 2 左边最高墙是 1，右边还有更高墙。
当前位置能接 1 - 0 = 1 格水。
Step 3 left=4, right=10
leftMax=2 height[4]=1 water += 1
遇到高度 2 后，左侧最高墙提高到 2。
下标 4 能接 2 - 1 = 1 格水。
Step 4 left=5, right=10
leftMax=2 height[5]=0 water += 2
这是样例里单格接水最多的位置。
累计水量来到 4。
Step 5 right=9
rightMax=2 height[9]=1 water += 1 total = 6
右侧更低时对称处理，用 rightMax 结算。
左右都只向内移动，最终总水量为 6。

Go 代码

双指针优化版：不需要预处理左右最大数组。

func trap(height []int) int {
    if len(height) == 0 {
        return 0
    }

    left, right := 0, len(height)-1
    leftMax, rightMax := 0, 0
    water := 0

    for left < right {
        if height[left] < height[right] {
            if height[left] >= leftMax {
                leftMax = height[left]
            } else {
                water += leftMax - height[left]
            }
            left++
        } else {
            if height[right] >= rightMax {
                rightMax = height[right]
            } else {
                water += rightMax - height[right]
            }
            right--
        }
    }
    return water
}

易错点

接雨水最常见的错误，是把“当前两端高度”误当成“历史最高边界”。

错误：用当前两端直接相减

water += height[right] - height[left]

水量取决于历史最高墙，不是当前两端柱子的差值。

正确：用 leftMax/rightMax

if height[left] < leftMax {
    water += leftMax - height[left]
} else {
    leftMax = height[left]
}

处理左侧时，当前位置由左侧历史最高墙限制。

错误：负水量也累加

water += leftMax - height[left]
leftMax = max(leftMax, height[left])

当当前柱子高于 leftMax 时，这会加负数，导致总量减少。

正确：先判断更新墙还是接水

if height[left] >= leftMax {
    leftMax = height[left]
} else {
    water += leftMax - height[left]
}

高柱子变成新边界，低洼处才产生水量。

核心算法

较低侧先结算

低侧的另一边已有足够边界，所以当前格水量可确定。

正确性直觉

维护历史最高墙

leftMax/rightMax 表示已扫描两侧最高墙，被处理位置无需回头。

易错关键点

不能加负水量

遇到更高柱子先更新最高墙，只有低于最高墙时才累加水。
