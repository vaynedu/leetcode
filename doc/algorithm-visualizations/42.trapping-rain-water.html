② 核心思想：双指针夹逼

每个位置的水高由左右两边的最高柱子决定。双指针从两端向中间逼近，哪边的高度小，就先处理哪边。

为什么先处理短边？

当前位置的水高 = min(maxLeft, maxRight) - height[i]

如果 maxLeft < maxRight，则水高由 maxLeft 决定。

此时左边的 maxLeft 已经是最终答案，
不需要再看右边了。

同理，maxRight < maxLeft 时，右边的 maxRight 已确定。

伪代码
left=0, right=n-1
maxL=0, maxR=0

while left < right:
  if maxL < maxR:
    水[left] = maxL - h[left]
    maxL = max(maxL, h[left])
    left++
  else:
    水[right] = maxR - h[right]
    maxR = max(maxR, h[right])
    right--
一句话：谁小谁先结算，因为它的 max 已经确定，不会再变了。

核心公式

每格独立计算

water[i] = min(maxLeft[i], maxRight[i]) − height[i]
（如果结果 < 0，则为 0）

双指针优化

谁小谁先结算

不记录所有 maxLeft/maxRight，
而是跟踪 maxL 和 maxR，
哪个小就先算哪边。

复杂度

O(n) / O(1)

时间 O(n)，只遍历一次。
空间 O(1)，无额外数组。

Time · O(n)  ·  Space · O(1)  ·  Two pointers
