LeetCode 239 · Sliding Window + Monotonic Deque

滑动窗口最大值

用单调递减队列保存候选下标：队首永远是当前窗口最大值。

核心洞察

题目：固定窗口大小 k，每滑动一步输出窗口最大值。
关键：deque 存下标，且对应值单调递减；新元素会淘汰所有更小且更早过期的队尾候选。

观察

更小且更老 = 永无机会

新元素更大，并且更晚过期，队尾小元素不可能再成为最大值。

不变量

队首最大，队内递减

deque 中下标在窗口内，对应值从队首到队尾单调递减。

复杂度

O(n) 时间 O(k) 空间

每个下标最多入队一次、出队一次，总操作线性。

手推全过程

样例：nums = [1,3,-1,-3,5,3,6,7]，k = 3

R=0 加入 1，窗口未形成
push index 0 deque = [0] → values [1]
窗口长度还不到 3，不输出答案。
R=1 加入 3，弹掉 1
pop 0，因为 1 < 3 push index 1 deque = [1] → values [3]
3 比 1 更大且更晚过期，1 永远不可能成为后续窗口最大值。
R=2 窗口 [0..2] 形成
push index 2 deque = [1,2] → values [3,-1] answer = nums[1] = 3
-1 比队尾 3 小，保留在后面，未来 3 出窗口后它可能被提拔。
R=4 加入 5，清空更小候选
pop 3, 2, 1 push index 4 answer = 5
新来的 5 把窗口里所有更小候选都淘汰，deque 变成 [4]。
R=6/7 6、7 依次成为新最大值
R=6 输出 6 R=7 输出 7
最终答案为 [3,3,5,5,6,7]。

Go 代码

deque 存下标，不存值；过期判断必须依赖下标。

func maxSlidingWindow(nums []int, k int) []int {
    if len(nums) == 0 || k == 0 {
        return []int{}
    }

    deque := []int{} // 存下标，保持对应值单调递减
    result := []int{}

    for right := 0; right < len(nums); right++ {
        for len(deque) > 0 && nums[deque[len(deque)-1]] < nums[right] {
            deque = deque[:len(deque)-1]
        }
        deque = append(deque, right)

        if deque[0] <= right-k {
            deque = deque[1:]
        }

        if right >= k-1 {
            result = append(result, nums[deque[0]])
        }
    }
    return result
}

易错点

这题的边界集中在“存下标”和“队尾要用 for 弹干净”。

错误：deque 存值

deque = append(deque, nums[right])

只存值无法判断队首是否滑出窗口，重复值时尤其容易错。

正确：deque 存下标

deque = append(deque, right)
if deque[0] <= right-k {
    deque = deque[1:]
}

下标能同时取值和判断过期。

错误：只弹一个队尾

if nums[deque[len(deque)-1]] < nums[right] {
    deque = deque[:len(deque)-1]
}

新元素可能比多个队尾都大，只弹一个会破坏单调递减。

正确：用 for 弹到单调

for len(deque) > 0 &&
    nums[deque[len(deque)-1]] < nums[right] {
    deque = deque[:len(deque)-1]
}

弹完后，队内值才继续保持递减。

Deque（index / value）

核心算法

单调递减队列

队列里保存候选最大值下标，队首就是当前窗口最大值。

正确性直觉

更小更老可淘汰

被新元素从队尾弹出的元素更小且更早过期，不会再赢。

易错关键点

必须存下标

下标用于判断窗口过期：deque[0] <= right-k。
