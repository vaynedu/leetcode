LeetCode 112 · Tree DFS

路径总和

DFS 判断是否存在一条根到叶路径，使路径和等于 targetSum。

核心洞察

题目：只需要判断是否存在，不需要收集路径。
关键：路径必须从根到叶；非叶子节点即使当前值等于 target，也不能提前返回 true。

观察

叶子才代表完整路径

根到中间节点只是路径前缀，只有走到左右孩子都为空的节点，路径才完成。

递归选择

target 逐层递减

进入子树时传 targetSum-root.Val，把“路径和”转成“剩余目标”。

复杂度

O(n) 时间 O(h) 空间

最坏访问所有节点；递归栈深度等于树高，链式树最坏 O(n)。

手推全过程

样例：root=[5,4,8,11,null,13,4,7,2,null,null,null,1]，targetSum=22

Step 1 path=[5,4,11,7] target=2 leaf=7
5→ 4→ 11→ 7
到达叶子 7 前，已经扣掉 5、4、11，所以剩余 target 是 2。
7 != 2，这条根到叶路径不满足。
Step 2 path=[5,4,11,2] target=2 leaf=2
5→ 4→ 11→ 2
回溯后尝试叶子 2，剩余 target 仍然是 2。
2 == 2，找到路径 [5,4,11,2]，立即返回 true。
Step 3 short-circuit result=true
因为 112 只问是否存在，左子树找到后，递归的 || 可以短路。
无需继续检查右子树 8 -> 13 或 8 -> 4 -> 1。

Go 代码

根到叶判断版：nil 返回 false；叶子处判断值是否等于剩余 target。

func HasPathSum(root *TreeNode, targetSum int) bool {
    if root == nil {
        return false
    }

    if root.Left == nil && root.Right == nil {
        return root.Val == targetSum
    }

    return HasPathSum(root.Left, targetSum-root.Val) ||
        HasPathSum(root.Right, targetSum-root.Val)
}

易错点

112 的关键是“根到叶”，不是“根到任意节点”。

错误：中间节点满足就返回 true

if root.Val == targetSum {
    return true
}

输入 root=[1,2], target=1 时，根节点 1 不是叶子，正确答案是 false。

正确：只在叶子节点判断

if root.Left == nil && root.Right == nil {
    return root.Val == targetSum
}

叶子节点才表示一条完整根到叶路径，非叶子必须继续向下递归。

核心算法

Tree DFS

递归维护剩余 target，向左右子树继续寻找根到叶路径。

正确性直觉

所有根到叶都会被检查

每个非叶节点都尝试左右子树，任一叶子命中即可返回 true。

易错关键点

只能在叶子判断

中间节点满足 target 也只是路径前缀，不是一条完整路径。
