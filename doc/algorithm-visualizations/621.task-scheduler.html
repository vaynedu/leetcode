LeetCode 621 · Greedy

任务调度器

给定任务频率和冷却时间 n，求完成所有任务的最短时间 — 贪心的精髓：时间片由出现最多的任务决定，其他任务填入间隔即可

核心洞察

题目：任务 A-Z，每个任务执行后需冷却 n 个时间单位，求完成所有任务的最短时间。
关键：总时间由出现最多的任务决定：有 maxCount 个 A，则至少需要 (maxCount-1)*(n+1) + maxCount 个时间单位。 其他任务（包括同频的）填入这些槽位，不够则追加。

观察

时间片 = 频率最高的任务决定

maxCount 个 A 被 n 间隔开，形成 (maxCount-1) 个完整槽，每个槽长 n+1。

贪心选择

其他任务填入间隔

用其他任务（或 idle）填满每个槽，槽满后多余的同类任务追加到末尾。

公式

len(formula) = max((maxCount-1)*(n+1), total)

理论槽位和实际任务数取最大值——实际任务更多时无需等待。

手推全过程

tasks = ['A','A','A','B','B','B']，n = 2

Step 1 统计频率
A=3, B=3，maxCount=3
桶的数量 = maxCount-1 = 2，每个桶容量 = n+1 = 3
Step 2 构建槽位
槽 1: [A, B, B]（填 A 及其冷却后第一个可用的 B）
槽 2: [A, B, B]
末尾追加: [A]（剩余的 A）
Step 3 计算结果
公式: (3-1)*(2+1) + 3 = 2*3+3 = 9
实际时间线: A B B A B B A = 7 < 9？实际是 A B idle A B idle A = 7？
等等……我们来用贪心模拟一下

最终结果：8 个时间单位（A B B A B B A B）

Go 代码

先统计频率，套公式计算

func leastInterval(tasks []byte, n int) int {
    // 1. 统计每个任务出现次数
    cnt := [26]int{}
    for _, t := range tasks {
        cnt[t-'A']++
    }

    // 2. 找到最大频率 maxCount
    maxCount := 0
    for _, c := range cnt {
        if c > maxCount { maxCount = c }
    }

    // 3. 计算有多少个最高频任务（决定槽数）
    maxFreqTasks := 0
    for _, c := range cnt {
        if c == maxCount { maxFreqTasks++ }
    }

    // 4. 公式
    partLen := (maxCount - 1) * (n + 1) + maxFreqTasks
    return max(partLen, len(tasks))
}

易错点

公式里的 + maxFreqTasks 不是 +1，而是 + 最高频任务的种类数

❌ 错误：末尾加 1

// AAAA, n=2: maxCount=4, 应该是 (4-1)*(2+1)+1=10
return max((maxCount-1)*(n+1)+1, len(tasks))
// 但如果是 AAAA BBB: maxCount=3, maxFreqTasks=2
// 用 +1 结果是 (3-1)*3+1=7，但实际是 A B _ A B _ A B = 8

当有多个最高频任务时，最后一排不只一个位置，需要加所有最高频任务的数量。

✓ 正确：末尾加 maxFreqTasks

// AAAA BBB, n=2: maxCount=3, maxFreqTasks=2
return max((3-1)*(2+1)+2, len(tasks))
// = max(8, 7) = 8

最后一排包含了所有最高频的任务，每个最高频任务占一个槽位。

核心

公式化简

总时间 = max((maxCount-1)*(n+1) + maxFreqTasks, len(tasks))。

贪心直觉

最高频任务决定槽位数

槽数 = maxCount-1，每个槽容量 = n+1，末尾特殊处理。

关键细节

末尾加的是 maxFreqTasks 不是 1

有多个最高频任务时，最后一排需要把它们都放进去。
