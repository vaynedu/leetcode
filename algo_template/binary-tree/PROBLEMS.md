## 如何判断二叉树题

- 输入是 `*TreeNode`，或者问题天然可以拆成左子树和右子树。
- 题目问遍历结果、路径、深度、祖先、子树属性、结构修改、BST 性质。
- 父节点答案可以由左右子树答案合并。

## 核心问题

写递归前先回答四个问题：

1. 当前节点要做什么？
2. 左右子树分别要返回什么？
3. 空节点返回什么？
4. 当前节点如何合并左右子树结果？

## 核心不变量

- DFS 返回值必须表达“当前子树能贡献给父节点的信息”。
- 前序适合自顶向下传状态，后序适合自底向上合并答案。
- 路径类问题中，进入节点时加入路径，离开节点时撤销路径。
- BFS 队列中始终保存下一批待处理节点；按层处理时先固定当前层 size。
- BST 题不能只比较当前节点和左右孩子，通常要维护上下界或利用中序有序性。

## 决策流程

```text
1. 判断问题属于遍历、路径、结构修改、BST、构造、LCA，还是返回值递归。
2. 明确应该使用前序、中序、后序、层序，还是自定义 dfs 返回值。
3. 写出 dfs(root) 的语义：它处理什么，返回什么。
4. 处理空节点 base case。
5. 递归左右子树。
6. 在当前节点合并左右结果，更新答案或返回给父节点。
7. 检查是否会漏节点、重复计算、丢子树、污染路径状态。
```

## 常用模板对照

```text
前序：根 左 右
  适合：先处理当前节点，再把状态传给子树。

中序：左 根 右
  适合：BST 有序输出、排名、第 K 小。

后序：左 右 根
  适合：左右子树先给答案，再由当前节点合并。

层序：队列 + 当前层 size
  适合：按层输出、最短层数、层级状态。

返回值递归：
  适合：父节点答案依赖子树贡献，例如高度、直径、最大路径和、LCA。
```

## 每题复习模板

复习一题时不要只看代码，按这个顺序过一遍：

1. 识别信号：为什么这是二叉树题，属于哪个分组？
2. 遍历选择：前序、中序、后序、层序，还是返回值递归？
3. 递归定义：`dfs(root)` 返回什么？
4. 空节点：base case 返回什么？
5. 合并逻辑：当前节点如何用左右子树结果得到答案？
6. 易错点：会不会漏节点、重复算、丢子树、污染路径？
7. 面试表达：能否 1 分钟讲清楚思路和复杂度？
