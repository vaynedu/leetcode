# 图 / BFS / DFS / 拓扑排序

## 识别信号

- 输入是节点和边、网格连通块、依赖关系、最短步数。
- 题目出现课程、先修、能否完成、顺序安排。
- 状态之间可以互相到达，需要 visited 防止重复。

## 核心不变量

- BFS：队列中保存当前边界，第一次到达某节点时步数最短。
- DFS：递归栈表示当前搜索路径，visited 防止重复进入。
- 拓扑排序：入度为 0 的节点表示当前没有未满足依赖。
- 染色：相邻节点颜色必须相反。

## 决策流程

```text
1. 建图：邻接表、入度表、网格方向。
2. 选择遍历：最短步数用 BFS，连通/路径可用 DFS。
3. 标记 visited 或颜色。
4. 拓扑排序先把入度 0 入队。
5. 弹出节点，处理邻居，更新入度/距离/颜色。
6. 根据访问数量、是否冲突、是否到达目标返回。
```

## 易错点

- BFS 入队时就应标记 visited，避免重复入队。
- 拓扑排序最后要检查处理节点数是否等于总节点数。
- 网格 DFS 要先判断越界和访问状态。

## 为什么不会漏答案

BFS 按层扩展，先访问到的路径步数最短；拓扑排序每次只移除无依赖节点，所有合法顺序都必须先处理这些节点之一。
