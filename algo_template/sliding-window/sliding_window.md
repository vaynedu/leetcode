# 滑动窗口（Sliding Window）

## 识别信号

- 题目要求连续子数组或连续子串。
- 目标是最长、最短、固定长度统计、满足条件的窗口数量。
- 窗口右边加入一个元素、左边移除一个元素后，状态可以 O(1) 或接近 O(1) 更新。

## 核心不变量

- 固定窗口：窗口长度始终等于 `k`。
- 变长窗口：每次更新答案时，窗口状态满足题目约束，或刚好是需要收缩前的临界状态。
- 左指针只能右移，不能回退；每个元素最多进窗口一次、出窗口一次。

## 决策流程

```text
1. 右指针扩张：加入 s[right] / nums[right]
2. 更新窗口状态：计数、sum、valid、zeroCount、lastPos
3. 判断是否需要收缩：重复、超出种类、valid 满足、窗口过大
4. 收缩左边界：移除 s[left] / nums[left]
5. 在正确时机更新答案：最长通常收缩后，最短通常满足条件时
```

## 为什么不会漏答案

窗口左右指针都单调右移。所有可能以 `right` 结尾的合法窗口，都会在收缩过程中被检查到；被左指针排除的前缀已经不可能组成更优或更合法的当前窗口。

## 一、核心思想

滑动窗口是一种**双指针**技巧，通过左右指针构成一个窗口，在数组/字符串上滑动：

```
[L, R)  半开半闭区间
 R →
←─────→
L
```

- R 不断右移**扩展窗口**
- 满足条件时 L 右移**收缩窗口**
- 遍历完所有情况，O(N) 时间

## 二、模板分类

### 类型1: 固定窗口（窗口大小 k）

```
先算第一个窗口 → 右移时减左加右 → O(1) 更新
```

适用场景：子数组最大平均数、元音计数等

```go
// 模板
sum := firstWindowSum
maxSum := sum
for R := k; R < N; R++ {
    sum += nums[R] - nums[R-k]
    maxSum = max(maxSum, sum)
}
```

**代表题目：**
| 题号 | 名称 | 核心 |
|------|------|------|
| 643 | 子数组最大平均数 | sum 统计 |
| 1456 | 定长子串元音最大数 | set 判元音 |

---

### 类型2: 变长窗口 - 双 HashMap + valid

```
need map：目标字符计数
window map：窗口字符计数
valid：已满足条件的字符种类数

扩展：window[c]++ → if window[c] == need[c] { valid++ }
收缩：if valid == len(need) → 满足条件
```

适用场景：最小覆盖子串、字符串排列、字母异位词

```go
// 模板
for R < N {
    add c
    for valid == len(need) {
        update answer
        remove s[L]
        L++
    }
}
```

**代表题目：**
| 题号 | 名称 | 区别 |
|------|------|------|
| 76 | 最小覆盖子串 | 记录最小窗口 |
| 567 | 字符串的排列 | 找到一个返回 |
| 438 | 找到所有字母异位词 | 记录所有位置 |

---

### 类型3: 变长窗口 - 条件收缩

```
用一个变量记录窗口状态（如重复次数、零的个数）
超过阈值时收缩
```

适用场景：无重复最长子串、水果成篮、最大连续1

```go
// 模板
for R < N {
    update window state
    while exceed threshold {
        remove s[L]
        L++
    }
    update answer
}
```

**代表题目：**
| 题号 | 名称 | 状态变量 |
|------|------|---------|
| 3 | 无重复最长子串 | last[pos] 记录位置 |
| 904 | 水果成篮 | len(window) 种类数 |
| 1004 | 最大连续1 | zeroCount 零的个数 |

---

### 类型4: 单调 deque

```
deque 存下标，保持单调递减
队首永远是当前窗口最大
```

适用场景：滑动窗口最大值

```go
// 模板
for R < N {
    // 1. 入队前踢掉比新元素小的（从队尾）
    for len(deque) > 0 && nums[deque[end]] < nums[R] {
        deque = deque[:len(deque)-1]
    }
    deque = append(deque, R)

    // 2. 踢掉滑出窗口的队首
    if deque[0] <= R-k {
        deque = deque[1:]
    }

    // 3. 队首是最大值
    if R >= k-1 {
        result = append(result, nums[deque[0]])
    }
}
```

**代表题目：**
| 题号 | 名称 |
|------|------|
| 239 | 滑动窗口最大值 |

---

### 类型5: 双指针（变长窗口特例）

适用场景：装水容器（面积取决于短板）

```go
// 模板
left, right := 0, N-1
for left < right {
    area = min(height[left], height[right]) * (right - left)
    if height[left] < height[right] {
        left++
    } else {
        right--
    }
}
```

**代表题目：**
| 题号 | 名称 |
|------|------|
| 11 | 装水容器 |

---

## 三、通用口诀

```
滑动窗口三步走
右扩左缩记心头
扩展更新状态
收缩判断条件
条件满足记录答案
直至遍历完所有
```

## 六、复杂度总结

| 类型 | 时间 | 空间 |
|------|------|------|
| 固定窗口 | O(N) | O(1) |
| 变长窗口 | O(N) | O(K) 或 O(1) |
| 单调 deque | O(N) | O(K) |
| 双指针 | O(N) | O(1) |

**每个元素最多入队出队各一次**，所以是 O(N)。
