# 算法模式总览

这个目录按“解题模式”组织，而不是按题号组织。目标是看到新题时能回答四个问题：

1. 这题为什么属于这个模式？
2. 这个算法维护的不变量是什么？
3. 每一步为什么不会漏答案？
4. 哪个边界最容易错，错了会怎样？

## 模式选择表

| 模式 | 识别信号 | 核心不变量 | 代表题 |
|------|----------|------------|--------|
| 滑动窗口 | 连续子数组/子串，最长/最短/计数 | 窗口状态可被右扩左缩维护 | 3, 76, 239, 438, 567 |
| 双指针 | 排序数组、两端收敛、成对关系 | 被排除区间不可能产生更优解 | 11, 15, 42, 141, 142 |
| 二分查找 | 有序、答案单调、可行性判定 | 答案始终保留在搜索区间内 | 33, 34, 35, 69, 74, 81 |
| 回溯/DFS | 所有组合/排列/路径/切分 | path 表示当前合法前缀，递归层表示决策位置 | 17, 22, 39, 46, 78 |
| Trie | 前缀匹配、字典、通配符、词根替换 | 从根到节点的路径等于一个前缀 | 208, 211, 212, 648 |
| 动态规划 | 最优值/计数，状态依赖过去 | dp 定义在转移前后始终一致 | 53, 62, 70, 96, 120, 139 |
| 树/DFS/BFS | 树结构、路径、祖先、层序 | 子树返回值能被父节点合并 | 102, 112, 113, 236, 437 |
| 队列/单调队列 | 层序遍历、最短步数、窗口最大值 | 队列保存下一层或当前候选集合 | 102, 200, 239 |
| 贪心 | 局部选择可证明不影响全局最优 | 每次选择后仍存在最优解 | 53, 55, 121, 455, 763 |

## 新题判断流程

```text
1. 是否要求所有方案/路径/排列？
   是 → 回溯/DFS

2. 是否是连续子数组/子串？
   是 → 固定长度看固定窗口；长度不固定看滑动窗口

3. 是否有序，或答案具有单调性？
   是 → 二分查找

4. 是否是最优值/计数，且当前答案依赖之前状态？
   是 → 动态规划

5. 是否是树或图结构？
   树 → DFS/BFS/后序合并
   图依赖关系 → 拓扑排序
   图最短步数 → BFS

6. 是否需要动态维护 Top K、最大值、最小值？
   Top K → 堆
   窗口最大值 → 单调队列
   最近未匹配/下一个更大 → 栈/单调栈
```

## 每个模式文档统一格式

每个 `algo_template/{pattern}/*.md` 应尽量包含：

```markdown
## 识别信号
什么题一眼想到它。

## 核心不变量
算法每一步维护什么条件。

## 决策流程
什么时候扩张、收缩、递归、排除、更新答案。

## 模板分类
该模式下的常见子类型。

## 代表题
从简单到困难的训练路径。

## 易错点
边界、更新时机、重复处理、初始化。

## 可视化入口
对应 `doc/algorithm-visualizations/*.html`。
```

## 单题笔记统一格式

每道高频题的 `interview/*.md` 建议逐步补齐：

```markdown
## 为什么想到这个模式

## 状态定义 / 数据结构

## 核心不变量

## 手推全过程

## 错误写法与反例

## 复杂度
```

不要一次性机械改完 100 题。优先补：

1. 高频题
2. 模式代表题
3. 容易错边界的题
4. 已有可视化图解的题

## 可视化策略

可视化只做高价值题：

- 模式代表题：3, 33, 39, 53, 76, 208, 239
- 状态变化复杂题：42, 146, 212, 297, 460
- 易错边界题：34, 81, 154, 567
